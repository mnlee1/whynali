# 🚀 바이브 코딩(Vibe Coding) 프로젝트 가이드라인

> **이 문서는 기획/개발 지식이 없는 웹디자이너·퍼블리셔가 AI와 협업하여 고품질 서비스를 만들기 위한 참조 가이드입니다.**

---

## 📋 1. 기획 원칙 (PRD 작성법)
AI에게 바로 코드를 요청하지 않고, 반드시 아래 프레임워크에 따라 기획(PRD)을 먼저 완성한다.

### 5W1H 프레임워크
* **Who (대상):** 이 서비스를 사용할 최종 사용자는 누구인가?
* **Why (목적):** 이 서비스가 해결하려는 핵심 문제는 무엇인가?
* **What (기능):** 반드시 포함되어야 할 핵심 기능 3가지는 무엇인가?
* **Where/When (환경):** 웹/모바일 중 어디서, 어떤 상황에 사용되는가?
* **How (방식):** 사용자는 어떤 흐름(Flow)으로 서비스를 이용하는가?

### 맥락(Context) 설정
* AI에게 나의 배경(웹디자이너/퍼블리셔)을 알리고, 전문적인 시맨틱 마크업과 유지보수가 쉬운 CSS 구조를 강조한다.

---

## 🛠️ 2. 단계별 개발 전략 (Standard Workflow)
한 번에 완성을 시도하지 않고, 아래의 4단계 사이클을 반복한다.

1.  **[1단계] 뼈대(Skeleton) 구축:** 기능을 넣지 않고 HTML 구조와 섹션별 고유 ID만 먼저 잡는다.
2.  **[2단계] 기능(Logic) 구현:** 각 섹션별로 필요한 인터랙션과 데이터를 연결한다.
3.  **[3단계] 디자인(Style) 고도화:** 구체적인 레퍼런스(예: 구글 스타일, 특정 테마 등)를 제시하여 UI 퀄리티를 높인다.
4.  **[4단계] 최종 점검(Review):** PRD의 요구사항이 모두 반영되었는지 AI에게 자가 검토를 시킨다.

---

## 🔍 3. 유지보수 및 에러 대응 프로토콜
개발 지식이 없어도 당황하지 않고 아래와 같이 대응한다.

* **코드 수정 방식:** 섹션별로 이름을 붙여 관리하므로, "전체 수정"이 아닌 "Header 섹션의 폰트 크기만 수정해줘"와 같이 좁은 범위를 타격한다.
* **에러 해결:** 에러 메시지가 뜨면 직접 읽으려 하지 말고 **[에러 메시지 복사]** 또는 **[화면 캡처]**를 통해 AI에게 "이 에러를 분석하고 해결책을 반영해줘"라고 요청한다.
* **디자인 피드백:** "예쁘게 해줘" 대신 "모던한 다크모드 스타일로, 가독성을 위해 자간을 0.05em 늘려줘"와 같이 퍼블리셔의 언어를 섞어 지시한다.

## 🚩 5. AI를 향한 명령 (Initial Prompt)
새 프로젝트 시작 시 AI에게 전달할 문구:
> "너는 나의 바이브 코딩 파트너야. 나는 웹디자이너이자 퍼블리셔이고, 기획/개발 지식보다는 이 가이드라인(vibe-guide.md)에 따라 단계별로 프로젝트를 완성하는 것을 선호해. 준비됐으면 첫 번째 단계인 PRD 작성부터 시작하자."
