## 📋 프로젝트 개요

**왜난리(WhyNali)** - 한국 이슈를 한눈에 파악하고 여론을 확인하는 이슈 내비게이션 서비스

### 핵심 콘셉트
- 뉴스/커뮤니티에 분산된 이슈 정보를 한곳에 모아 시계열로 제공
- 발단→전개→파생→진정 타임라인 구조
- 사용자 참여(감정 표현, 댓글, 투표)를 통한 여론 확인

## 🎯 주요 기능

### 1. 이슈 관리 시스템
- **자동 수집**: 네이버 뉴스(30분마다), 커뮤니티(3분마다)
- **자동 생성**: 수집 데이터 기반 이슈 후보 자동 생성
- **AI 카테고리 분류**: Groq AI로 5개 카테고리 자동 분류 (사회/정치/연예/스포츠/기술)
- **AI 중복 체크**: 4단계 검증 (제목 → 키워드 → 반대어/숫자 → AI 정밀 비교)
- **승인 프로세스**: 관리자 승인/반려 시스템
- **화력 분석**: 실시간 화력 지수 계산 및 등급 표시
- **상태 관리**: 점화 → 논란중 → 종결 자동 전환
- **카테고리**: 연예, 스포츠, 정치, 사회, 기술

### 2. 타임라인 시스템
- 발단 → 전개 → 파생 → 진정 단계 표시
- 출처 링크 연결
- 시간순 정렬
- 자동/수동 타임라인 포인트 추가

### 3. 투표 시스템 ⭐ (현재 브랜치: feature/vote-system)

**구현 상태**: ✅ 완료 (일부 DB 마이그레이션 대기)

#### 핵심 기능
- ✅ 이슈당 복수 투표 가능 (시점별 여론 변화 추적)
- ✅ 2-6개 선택지 지원
- ✅ 실시간 득표 비율 그래프
- ✅ 이슈 상태별 투표 구분 (점화/논란중/종결)
- ✅ 로그인 사용자 투표 참여/취소
- ✅ 비로그인 사용자 참여 유도 UI

#### 투표 생성 방식
1. **직접 생성**: 관리자가 제목 및 선택지 입력
2. **AI 생성**: 이슈 메타데이터 기반 자동 생성 (`lib/ai/vote-generator.ts`)

#### 자동 종료 시스템 🚀
- ✅ **날짜 기준**: 특정 날짜/시각 도달 시 자동 마감
- ✅ **참여자 수 기준**: 목표 참여자 수 달성 시 자동 마감
- ✅ **이슈 연동**: 이슈 종결 시 3일 유예 후 자동 마감 (활동 있으면 +1일 연장)
- ✅ **크론잡**: GitHub Actions 매 시간마다 자동 실행

**자동 종료 파일**:
- `lib/vote-auto-closer.ts`: 자동 마감 유틸리티
- `app/api/cron/auto-end-votes/route.ts`: 크론잡 엔드포인트
- `.github/workflows/cron-auto-end-votes.yml`: GitHub Actions 워크플로우

#### UI 개선사항
- ✅ 비로그인 사용자: 강화된 CTA (전체 참여자 수 강조)
- ✅ 자동 종료 정보 실시간 표시 (남은 시간, 진행률 바)
- ✅ 진행중/마감 투표 분리 표시
- ✅ 이슈 상태별 과거 투표 그룹화

#### 주요 API 엔드포인트
- `GET /api/votes?issue_id={id}`: 투표 목록 조회
- `POST /api/votes/{id}/vote`: 투표 참여
- `DELETE /api/votes/{id}/vote`: 투표 취소
- `POST /api/admin/votes`: 투표 생성
- `POST /api/admin/votes/generate`: AI 투표 생성
- `POST /api/admin/votes/{id}/approve`: 투표 승인
- `POST /api/admin/votes/{id}/close`: 수동 종료
- `GET /api/cron/auto-end-votes`: 자동 종료 크론잡

#### 주의사항 ⚠️
**DB 마이그레이션 필요**:
- `auto_end_date`, `auto_end_participants` 컬럼 추가 필요
- 현재 코드에서 일부 주석 처리됨 (마이그레이션 후 해제)
- 마이그레이션 가이드: `docs/99_투표_자동종료_마이그레이션.md`

### 4. 커뮤니티 (토론 주제)
- AI 생성 토론 주제 (Groq 기반)
- 토론 주제별 댓글
- 이슈 연결
- 관리자 승인/반려

### 5. AI 시스템 (Groq 기반 - 무료, 항상 활성화)

**모델**: Groq Llama 3.1 8B Instant

**핵심 기능**:
1. **카테고리 자동 분류**: 키워드 기반 분류의 신뢰도가 낮을 때 AI로 재분류
2. **중복 이슈 체크**: 4단계 검증 (제목 일치 → 공통 키워드 → 반대어/숫자 필터 → AI 정밀 비교, 신뢰도 80% 이상)
3. **커뮤니티 급증 검증**: 진짜 이슈인지 AI 판단하여 밈, 드립, 장난 글 자동 필터링 (신뢰도 70% 이상)
4. **뉴스-이슈 연결 검증**: 복합 키워드 필터링 후 AI로 관련도 검증 (배치 처리로 5건씩, 관련도 70% 이상만 연결)
5. **커뮤니티 글 매칭**: 이슈와 커뮤니티 글 자동 연결 (제목 기반 키워드 매칭 + AI 검증)
6. **토론 주제 생성**: 이슈별 토론 주제 자동 생성 (관리자 승인 필수)
7. **투표 생성**: 이슈별 투표 주제 및 선택지 자동 생성 (관리자 승인 필수)

**특징**:
- 다중 키 순환 시스템 (Supabase 기반 키 차단 상태 공유)
- Rate Limit 자동 처리 (429 에러 시 다음 키로 자동 전환)
- 법적 안전성: 제목·메타데이터만 사용, 본문·요약문 미사용

**관련 파일**:
- `lib/ai/groq-provider.ts`: Groq AI 프로바이더 (다중 키 순환)
- `lib/candidate/category-classifier.ts`: 카테고리 분류
- `lib/candidate/duplicate-checker.ts`: 중복 체크
- `lib/candidate/community-burst-detector.ts`: 커뮤니티 급증 검증
- `lib/linker/ai-news-validator.ts`: 뉴스 검증
- `lib/linker/issue-community-linker.ts`: 커뮤니티 매칭
- `lib/ai/discussion-generator.ts`: 토론 주제 생성
- `lib/ai/vote-generator.ts`: 투표 생성

### 6. 사용자 참여

## 🗄 데이터베이스 스키마

### 핵심 테이블

#### 1. issues (이슈)
```sql
- id: UUID (PK)
- title: TEXT (제목)
- description: TEXT (설명)
- status: TEXT (점화|논란중|종결)
- category: TEXT (연예|스포츠|정치|사회|기술)
- heat_index: NUMERIC (화력 지수)
- approval_status: TEXT (대기|승인|반려|병합됨)
- merged_into_id: UUID (병합된 대상 이슈 ID, FK to issues)
- approved_at: TIMESTAMPTZ
- created_at, updated_at: TIMESTAMPTZ
```

**병합됨 상태 처리**:
- 중복 이슈 발견 시 `approval_status = '병합됨'` + `merged_into_id` 설정
- 일반 사용자 API에서는 `merged_into_id IS NULL` 필터로 제외
- 관리자 페이지에서는 필터로 선택 가능 (기본값: 제외)
- 병합된 이슈 페이지 접근 시 자동으로 원본 이슈로 리다이렉트

#### 2. timeline_points (타임라인)
```sql
- id: UUID (PK)
- issue_id: UUID (FK → issues)
- occurred_at: TIMESTAMPTZ (발생 시각)
- source_url: TEXT (출처 링크)
- stage: TEXT (발단|전개|파생|진정)
- created_at: TIMESTAMPTZ
```

#### 3. votes (투표)
```sql
- id: UUID (PK)
- issue_id: UUID (FK → issues)
- title: TEXT (투표 제목)
- phase: TEXT (대기|진행중|마감)
- issue_status_snapshot: TEXT (투표 생성 시 이슈 상태)
- started_at: TIMESTAMPTZ (시작 시각)
- ended_at: TIMESTAMPTZ (종료 시각)
- auto_end_date: TIMESTAMPTZ (자동 종료 날짜) ⚠️ 마이그레이션 필요
- auto_end_participants: INTEGER (목표 참여자 수) ⚠️ 마이그레이션 필요
- approval_status: TEXT (대기|승인|반려)
- created_at: TIMESTAMPTZ
```

#### 4. vote_choices (투표 선택지)
```sql
- id: UUID (PK)
- vote_id: UUID (FK → votes)
- label: TEXT (선택지 텍스트)
- count: INTEGER (득표 수)
```

#### 5. user_votes (사용자 투표 기록)
```sql
- id: UUID (PK)
- vote_id: UUID (FK → votes)
- vote_choice_id: UUID (FK → vote_choices)
- user_id: UUID (FK → users)
- created_at: TIMESTAMPTZ
- UNIQUE(vote_id, user_id) (중복 투표 방지)
```

#### 6. discussion_topics (토론 주제)
```sql
- id: UUID (PK)
- issue_id: UUID (FK → issues)
- body: TEXT (토론 주제 본문)
- is_ai_generated: BOOLEAN
- approval_status: TEXT (대기|승인|반려)
- approved_at: TIMESTAMPTZ
- created_at: TIMESTAMPTZ
```

#### 7. comments (댓글)
```sql
- id: UUID (PK)
- issue_id: UUID (FK → issues, nullable)
- discussion_topic_id: UUID (FK → discussion_topics, nullable)
- user_id: UUID (FK → users)
- body: TEXT
- like_count, dislike_count: INTEGER
- visibility: TEXT (public|pending_review|deleted)
- parent_id: UUID (FK → comments, 대댓글)
- created_at, updated_at: TIMESTAMPTZ
```

#### 8. reactions (감정 표현)
```sql
- id: UUID (PK)
- issue_id: UUID (FK → issues)
- user_id: UUID (FK → users)
- type: TEXT (좋아요|싫어요|화나요|팝콘각|응원|애도|사이다)
- created_at: TIMESTAMPTZ
- UNIQUE(issue_id, user_id)
```

#### 9. news_data (뉴스 수집)
```sql
- id: UUID (PK)
- title: TEXT
- link: TEXT
- source: TEXT
- published_at: TIMESTAMPTZ
- category: TEXT
- issue_id: UUID (FK → issues, nullable)
- created_at: TIMESTAMPTZ
```

#### 10. community_data (커뮤니티 수집)
```sql
- id: UUID (PK)
- title: TEXT
- url: TEXT
- view_count, comment_count: INTEGER
- written_at: TIMESTAMPTZ
- source_site: TEXT
- issue_id: UUID (FK → issues, nullable)
- created_at: TIMESTAMPTZ
```

#### 11. safety_rules (세이프티봇)
```sql
- id: UUID (PK)
- kind: TEXT
- value: TEXT
- created_at: TIMESTAMPTZ
```

#### 12. admin_logs (관리자 로그)
```sql
- id: UUID (PK)
- action: TEXT
- target_type: TEXT
- target_id: UUID
- admin_id: TEXT
- details: TEXT
- created_at: TIMESTAMPTZ
```

#### 13. ai_key_status (AI 키 상태 관리)
```sql
- provider: TEXT (PK)
- key_hash: TEXT (PK)
- is_blocked: BOOLEAN (차단 여부)
- blocked_until: TIMESTAMPTZ (차단 해제 시각)
- fail_count: INT (실패 횟수)
- updated_at: TIMESTAMPTZ
- PRIMARY KEY (provider, key_hash)
```

**용도**: Supabase 기반 AI 키 차단 상태 공유 (서버리스 인스턴스 간)
- Rate Limit 발생 시 키를 차단 상태로 저장
- 모든 서버리스 인스턴스가 실시간으로 차단 상태 확인
- blocked_until 지나면 자동 복구

## 🔄 자동화 시스템 (Cron Jobs)

### DB 마이그레이션 대기 중
투표 자동 종료 기능을 완전히 활성화하려면 다음 마이그레이션 실행 필요:

```sql
-- 투표 자동 종료 옵션 필드 추가
ALTER TABLE votes ADD COLUMN IF NOT EXISTS auto_end_date TIMESTAMP WITH TIME ZONE;
ALTER TABLE votes ADD COLUMN IF NOT EXISTS auto_end_participants INTEGER;

-- 인덱스 추가
CREATE INDEX IF NOT EXISTS idx_votes_auto_end_date ON votes(auto_end_date) WHERE phase = '진행중';
```

마이그레이션 후:
1. `app/api/admin/votes/route.ts` 주석 해제
2. `components/issue/VoteSection.tsx` 주석 해제
3. `app/admin/votes/page.tsx` 주석 해제
4. `types/index.ts` 주석 해제
