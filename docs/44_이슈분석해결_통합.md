# 이슈 분석 및 해결 통합

본 문서는 제니 이슈 미등록, 그루핑 조건 완화, 중복 이슈 원인, 옥택연 점화 지속 문제를 통합한 문서입니다.

## 1. 제니 이슈 미등록 원인 분석

### 1.1 발견된 사실

**AI 재검증 로그:**
```
? [AI 검증] "사생활 존중 좀" 제니, 파리 거리서 발 묶였다" 
  vs "제니 "매우 스트레스" 결국 폭발했다…'사인 요구' 거절 현장 포착"
  
✓ [병합 추천] 신뢰도 90%
  이유: 제니의 사생활과 관련된 같은 사건
```

**결과:**
- AI가 두 제니 뉴스를 같은 이슈로 정확히 판단 (90% 신뢰도)
- 그룹 병합은 성공
- 그러나 이슈로 등록되지 않음

### 1.2 미등록 원인

**이슈 등록 조건 미충족 (가장 가능성 높음)**

현재 등록 조건:
```
필수 조건:
1. 뉴스 5건 이상
2. 고유 출처 2곳 이상
3. 커뮤니티 반응 1건 이상  ← 핵심!
4. 화력 15점 이상
```

**추정:**
- 제니 관련 뉴스는 병합되어 그룹 형성
- 하지만 커뮤니티(더쿠/네이트판) 반응이 없음
- → 조건 3번 미충족으로 이슈 등록 실패

**뉴스 건수 부족 (가능성 있음)**
- 각 그룹이 1-2건의 뉴스만 보유
- 병합 후에도 총 2-4건으로 5건 미만
- → 조건 1번 미충족

### 1.3 해결 방안

**Option 1: 커뮤니티 필수 조건 완화 (권장)**
```bash
# 현재
CANDIDATE_REQUIRE_COMMUNITY=true  # 필수

# 변경
CANDIDATE_REQUIRE_COMMUNITY=false  # 선택
CANDIDATE_MIN_UNIQUE_SOURCES=3     # 출처 3곳 이상 → 2곳
```

**Option 2: 뉴스 건수 조건 완화**
```bash
CANDIDATE_ALERT_THRESHOLD=3        # 5 → 3
CANDIDATE_MIN_HEAT_TO_REGISTER=20  # 30 → 20
```

**Option 3: 연예 카테고리 예외 처리**
```typescript
if (category === '연예') {
    // 커뮤니티 필수 조건 제외
    // 뉴스 3건 + 화력 20점 이상만으로 등록
}
```

## 2. 그루핑 조건 완화 구현

### 2.1 문제점

**CORE_KEYWORDS 특혜 문제:**
- "민희진", "옥택연" 등 핵심 인물만 1개 키워드로 그루핑
- 기타 인물은 3개 이상 필요
- → 불평등, 누락 발생

### 2.2 개선 내용

**그루핑 조건 변경:**

Before (CORE_KEYWORDS 특혜):
```typescript
if (
    commonCount >= 3 ||                              // 3개 이상
    (commonCount >= 2 && sameCategory) ||            // 2개 + 같은 카테고리
    (commonCount >= 1 && hasCommonPerson)            // 1개 + 핵심 인물 ← 특혜!
)
```

After (모든 인물 동등):
```typescript
if (
    commonCount >= 2 ||                              // 2개 이상
    (commonCount >= 1 && sameCategory)               // 1개 + 같은 카테고리
)
```

**GROUPING_STOPWORDS 대폭 강화:**

추가된 범용어 (30개+):
```typescript
// 연예 범용어

'출연', '등장', '활동', '복귀', '컴백', '데뷔', '은퇴',
'화제', '관심', '주목', '인기', '열풍', '돌풍',

// 엔터테인먼트 용어
'드라마', '영화', '예능', '쇼', '공연', '무대', '콘서트',
'MV', '뮤비', '앨범', '노래', '음악', '작품', '촬영',
```

### 2.3 mergeRelatedGroups 개선

Before:
```typescript
if (similarity >= 0.4 || hasCommonCorePerson) {
    // 병합
}
```

After:
```typescript
if (
    similarity >= 0.3 ||                             // 유사도 0.3 이상
    (similarity >= 0.2 && sameCategory) ||           // 같은 카테고리 + 0.2
    hasCommonCorePerson                              // 참고용

    // 병합
}
```

## 3. 중복 이슈 원인 분석

### 3.1 제목 완전 중복 이슈 (7건)

**같은 제목으로 여러 개 생성된 케이스:**

1. **민희진 하이브** (2개)
   - 동일 시각(2026. 2. 26. PM 1:55:30) 생성
   - 화력: 0점, 20점

2. **차은우 방지법** (3개!)
   - 거의 동일 시각(2026. 3. 4. PM 2:41~2:42) 생성
   - 화력: 0점, 3점, 0점

3. **DL이앤씨** (3개!)
   - 거의 동일 시각(2026. 3. 4. AM 10:24~10:25) 생성
   - 화력: 0점, 10점, 0점

### 3.2 이하늬 쪼개진 이슈 (8개!)

**같은 날(2026.3.9) 시간대별로 8개 생성:**
```
PM 1:30 - "곰탕집?…64억 건물에 세금 60억 추징" (5개 뉴스)
PM 1:58 - "차은우 장어집, 이하늬 곰탕집" (5개 뉴스)
PM 2:18 - "기획사 분점이 곰탕집?" (1개 뉴스)
PM 4:05 - "곰탕집 논란.."임대차 관계, 투자목적 NO" 해명" (2개 뉴스)
PM 4:35 - "이하늬 기획사 맞다"면서 답변 회피" (1개 뉴스)
PM 5:03 - "곰탕집설…소속사 측 "입장 정리 중"" (1개 뉴스)
PM 5:55 - "곰탕집?…소속사 "임대차 계약일뿐"" (2개 뉴스)
PM 7:03 - "곰탕집?…관계자 "입장 없어"" (5개 뉴스)
```

**모두 같은 "이하늬 곰탕집 논란" 사건**

### 3.3 원인 분석

**원인 1: 동시 처리로 인한 제목 완전 중복**
- Cron이 동시에 여러 후보를 처리
- 중복 체크가 같은 트랜잭션 내에서 실행되지 않음
- Race condition 발생

**원인 2: 시간대별로 뉴스가 다르게 표현되어 쪼개짐**
- 이하늬 케이스: 시간대별로 다른 표현
- 공통 키워드: "이하늬" (1개만)
- "곰탕집", "기획사", "분점" 등은 일부만 공통

### 3.4 해결 방안

**Step 1: 이슈 생성 전 중복 체크 (긴급)**
```typescript
// 제목 정확 일치 체크
const existing = await supabase
    .from('issues')
    .select('id')
    .eq('title', title)
    .single()

if (existing) {
    console.log(`[중복 방지] 이미 존재: ${title}`)
    return null
}
```

**Step 2: 그루핑 조건 완화 (완료)**
- 공통 키워드 1개 + 같은 카테고리 → 그루핑
- GROUPING_STOPWORDS 대폭 강화

**Step 3: AI 중복 그룹 체크 (완료)**
- 같은 카테고리 + 24시간 내 그룹 AI 비교
- 신뢰도 80% 이상 → 자동 병합

## 4. 옥택연 점화 지속 원인 분석

### 4.1 현상
- 옥택연 이슈가 "점화" 상태로 고정
- 화력 69점, 등록 후 48시간 경과
- "논란중"으로 전환되지 않음

### 4.2 원인

**상태 전환 조건:**
```
점화 → 논란중:
(등록 후 6시간 경과) AND (화력 40점 이상)
```

**문제:**
- `registered_at` 컬럼이 NULL
- 상태 전환 로직에서 `registered_at`을 기준으로 시간 계산
- → NULL이므로 6시간 경과 판단 불가

### 4.3 해결 방안

**Step 1: 기존 이슈 수정 (긴급)**
```sql
UPDATE issues 
SET registered_at = created_at 
WHERE registered_at IS NULL;
```

**Step 2: 이슈 생성 시 registered_at 설정**
```typescript
await supabase.from('issues').insert({
    title,
    created_at: now,
    registered_at: now,  // ← 추가
    // ...
})
```

**Step 3: 상태 전환 로직 보강**
```typescript
// registered_at이 NULL이면 created_at 사용
const registeredAt = issue.registered_at || issue.created_at
```
