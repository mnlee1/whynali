## 정리 기준

트랙 A 이슈는 다음 조건을 **모두** 만족해야 합니다:

1. source_track = 'track_a' (또는 'manual')
2. 화력 15점 이상
3. 뉴스 1건 이상 연결
4. **커뮤니티 글 1건 이상 연결** (필수)
5. 타임라인 1개 이상

## 분석

### 공통 문제점

두 이슈 모두 **커뮤니티 글이 0건**이었습니다. 이는:

1. 트랙 A 프로세스를 정상적으로 거치지 않았거나
2. 수동 생성 이슈이거나
3. 레거시 프로세스(트랙 A 이전)로 생성된 이슈

트랙 A는 커뮤니티 급증 감지에서 시작하므로, 커뮤니티 글 연결은 **필수 조건**입니다.

### 화력은 충분했지만

- 이슈 1: 화력 24점 (뉴스 26건)
- 이슈 2: 화력 24점 (뉴스 24건)

두 이슈 모두 뉴스만으로 화력 24점을 기록했지만, 커뮤니티 반응이 없어 트랙 A 기준을 충족하지 못했습니다.

## 트랙 A 프로세스 강화

관리자 페이지에 다음 내용이 명시되어 있습니다:

### 필수 조건
- **1단계:** 커뮤니티 급증 감지 (10분간 10건 이상)
- **5단계:** AI 필터링으로 관련 커뮤니티 글 선별
  - **관련 커뮤니티 글 0건이면 이슈 생성 건너뛰기**
- **7단계:** 커뮤니티 글 연결 (이미 필터링됨)

### 자동 삭제 조건
- 화력 15점 미만
- 뉴스 0건
- **커뮤니티 0건**
- 타임라인 생성 실패

## 향후 조치

1. 트랙 A 크론이 정상 작동하는지 모니터링
2. 수동 생성 이슈도 커뮤니티 연결 권장
3. 주기적으로 `npx tsx scripts/check_all_issues_status.ts` 실행하여 레거시 이슈 확인

## 실행 스크립트

정리에 사용된 스크립트:
- `scripts/check_all_issues_status.ts` - 전체 이슈 상태 확인
- `scripts/cleanup_legacy_issues.ts` - 레거시 이슈 자동 정리
- `scripts/check_invalid_track_a_issues.ts` - 트랙 A 이슈 검증
