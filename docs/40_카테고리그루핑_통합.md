# 카테고리 그루핑 시스템 통합

본 문서는 화력 저장, AI 카테고리 분류, 기존 이슈 재분류를 통합한 문서입니다.

## 1. 등록 시 화력 저장

### 1.1 목적
이슈 등록 당시의 화력 점수를 저장하여 관리자가 시간 경과 후에도 등록 기준을 명확히 이해할 수 있도록 개선

### 1.2 문제점
- 이슈 등록 시: 화력 18점 (등록 조건 충족)
- 2시간 후: 화력 12점 표시
- 관리자 오해: "15점 미만인데 왜 등록되어 있지?"

### 1.3 해결 방안
등록 시점 화력을 별도 저장하고 "18점 → 12점" 형태로 표시

### 1.4 SQL 마이그레이션

```sql
-- issues 테이블에 등록 시점 화력 컬럼 추가
ALTER TABLE issues 
ADD COLUMN IF NOT EXISTS created_heat_index INTEGER;

-- 컬럼 설명
COMMENT ON COLUMN issues.created_heat_index IS '이슈 등록 당시의 화력 점수 (등록 기준 확인용)';

-- 기존 데이터 마이그레이션 (현재 화력을 등록 시 화력으로 복사)
UPDATE issues 
SET created_heat_index = heat_index 
WHERE created_heat_index IS NULL AND heat_index IS NOT NULL;
```

### 1.5 영향 범위
- `types/issue.ts` - Interface에 `created_heat_index` 추가
- `lib/candidate/issue-candidate.ts` - 이슈 생성 시 화력 저장
- `app/admin/issues/page.tsx` - UI에 "등록 시 → 현재" 표시

## 2. AI 카테고리 분류 시스템

### 2.1 문제점

**케이스 1: 스포츠마케팅 오분류**
```
제목: '2026 나주방문의 해' 스포츠마케팅으로 체류형 관광 전환 박차
키워드 분류: 스포츠 (잘못됨)
실제 카테고리: 사회 (지역 관광 정책)
```

**케이스 2: 연예인 가짜뉴스 오분류**
```
제목: SNS에 퍼진 '쿠엔틴 타란티노 이란 공격 사망설' 진실은?
키워드 분류: 스포츠? (잘못됨)
실제 카테고리: 사회 (가짜뉴스)
```

**핵심 문제:** 단순 키워드 매칭으로는 맥락을 이해할 수 없음

### 2.2 해결 방안: 하이브리드 분류

**전략:**
```
1차: 키워드 기반 분류 (빠름)
   ↓
2차: 신뢰도 체크
   ↓ (낮음)
3차: AI 재분류 (정확함)
   ↓
4차: 최종 카테고리 결정
```

**AI 재분류 조건:**
다음 중 하나라도 해당하면 AI 재분류:
1. 키워드 점수 < 3 - 매칭된 키워드가 너무 적음
2. 다수결과 키워드 불일치 - 네이버 카테고리와 키워드 결과가 다름
3. 맥락 규칙 없고 점수 낮음 - 특별한 규칙 없이 점수만 낮음

**비용 최적화:**
- 신뢰도 높을 때: 키워드만 사용 (무료, 빠름)
- 신뢰도 낮을 때: AI 사용 (Groq 무료)
- 예상 AI 호출: 하루 20-30회 (전체의 30-40%)

### 2.3 환경 변수

```bash
ENABLE_AI_CATEGORY=true           # AI 카테고리 분류 활성화
GROQ_API_KEY=your-groq-api-key    # 이미 설정되어 있음
```

### 2.4 테스트

```bash
# .env.local 설정 후
npx tsx scripts/test-category-classifier.ts
```

**예상 결과:**
```
케이스 1: 스포츠마케팅
  키워드: 스포츠 → AI: 사회 ✅
  신뢰도: 95%
  이유: 지역 관광 정책 관련 내용

케이스 2: 연예인 가짜뉴스
  키워드: 스포츠? → AI: 사회 ✅
  신뢰도: 98%
  이유: 가짜뉴스 관련 내용
```

## 3. 기존 이슈 재분류 가이드

### 3.1 사용 방법

**Dry Run (테스트 모드):**
```bash
npx tsx scripts/reclassify-issues.ts --dry-run
```

**스포츠 카테고리만 재분류:**
```bash
# Dry Run
npx tsx scripts/reclassify-issues.ts --category 스포츠 --dry-run

# 실제 적용
npx tsx scripts/reclassify-issues.ts --category 스포츠
```

**최근 100개 이슈 재분류:**
```bash
npx tsx scripts/reclassify-issues.ts
```

**전체 이슈 재분류 (주의!):**
```bash
# Dry Run 먼저 확인
npx tsx scripts/reclassify-issues.ts --all --dry-run

# 실제 적용
npx tsx scripts/reclassify-issues.ts --all
```

### 3.2 옵션

| 옵션 | 설명 | 예시 |
|------|------|------|
| `--dry-run` | 테스트 모드 (실제 업데이트 안 함) | `--dry-run` |
| `--category` | 특정 카테고리만 | `--category 스포츠` |
| `--all` | 전체 이슈 | `--all` |
| `--limit` | 개수 제한 (기본 100) | `--limit 50` |

### 3.3 예시 출력

```
=== 이슈 재분류 시작 ===

모드: 실제 업데이트
대상: 스포츠 카테고리
제한: 100개

총 23개 이슈 발견

[1/23] "김태희, '한남더힐' 127억에 매각…85억 시세차익"
  현재 카테고리: 스포츠
  AI 분류: 연예 (신뢰도: 95%)
  이유: 연예인의 부동산 거래는 연예 카테고리
  ✅ 변경: 스포츠 → 연예

[2/23] "손흥민 2골, 토트넘 승리 견인"
  현재 카테고리: 스포츠
  AI 분류: 스포츠 (신뢰도: 98%)
  이유: 축구 경기 관련
  ⏭️  동일: 변경 불필요

=== 재분류 완료 ===

총 처리: 23개
변경됨: 8개
스킵됨: 14개
에러: 1개
```
