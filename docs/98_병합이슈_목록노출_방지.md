# 병합된 이슈 목록 노출 방지 수정 완료

## 문제 상황
'병합됨' 상태의 이슈가 일반 사용자 이슈 목록에 계속 표시되는 문제

## 원인 분석
1. DB 데이터 정합성 점검 결과: **정상** (병합된 이슈 1건, 모두 정상 처리됨)
2. API 필터링: `approval_status = '승인'` 조건으로 이미 '병합됨' 제외 중
3. 추가 안전장치: `merged_into_id IS NULL` 필터도 적용 중

## 수정 내용

### 1. API 필터링 강화
모든 이슈 조회 API에서 병합된 이슈 명시적 제외:

#### `/api/issues/route.ts`
```typescript
.eq('approval_status', '승인')
.eq('visibility_status', 'visible')
.is('merged_into_id', null)  // 병합된 이슈 이중 체크
```

#### `/api/search/route.ts`
```typescript
.eq('approval_status', '승인')
.eq('visibility_status', 'visible')
.is('merged_into_id', null)  // 병합된 이슈 이중 체크
```

### 2. 관리자 페이지 개선

#### 관리자 API 로직 수정 (`/api/admin/issues/route.ts`)
```typescript
.not('approval_status', 'is', null)
.neq('approval_status', '병합됨')  // 병합된 이슈 항상 제외
```
- 관리자도 병합된 이슈를 볼 필요 없음
- 모든 데이터가 primary 이슈에 통합되었으므로 필터 불필요

#### UI 개선
1. **승인 상태 배지**: '병합됨' 상태에 보라색 배지 추가
   ```typescript
   if (issue.approval_status === '병합됨') {
       return {
           label: '병합됨',
           className: 'bg-purple-100 text-purple-700 border-purple-200'
       }
   }
   ```

2. **액션 버튼**: 병합된 이슈는 "병합된 이슈 보기" 버튼 표시
   ```tsx
   {issue.approval_status === '병합됨' && issue.merged_into_id && (
       <a href={`/issue/${issue.merged_into_id}`} target="_blank">
           병합된 이슈 보기
       </a>
   )}
   ```

3. **미리보기 드로어**: 병합된 이슈에서 "병합된 이슈 보기" 버튼 추가

### 3. 점검 스크립트 생성
`scripts/check-consistency.mjs` - 병합 데이터 정합성 점검

점검 항목:
1. `approval_status='병합됨'` + `merged_into_id=NULL`
2. `merged_into_id` 존재 + `approval_status != '병합됨'`
3. `approval_status='승인'` + `merged_into_id` 존재 (목록 노출 위험)

실행:
```bash
node scripts/check-consistency.mjs
```

### 4. 문서 업데이트
`.claude/프로젝트_전체_분석.md`에 병합 상태 처리 정책 추가

## 결과
- 일반 사용자 목록: 병합된 이슈 완전 제외
- 관리자 페이지: 병합된 이슈 완전 제외 (필터도 제거)
- 병합된 이슈 접근 시: 원본 이슈로 자동 리다이렉트
- 데이터 정합성: 점검 스크립트로 지속 모니터링 가능

### 병합된 이슈를 왜 아예 안 보여주나?
- 병합된 이슈의 모든 데이터(뉴스, 댓글, 투표 등)가 primary 이슈로 이동됨
- primary 이슈에서 모든 정보를 확인할 수 있음
- 병합된 이슈는 빈 껍데기이므로 볼 필요 없음
- DB에는 이력 추적을 위해 보관되지만 UI에서는 완전히 숨김
