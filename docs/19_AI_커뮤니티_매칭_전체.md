# AI 커뮤니티 매칭 구현 전체 히스토리

날짜: 2026-02-26

## 1. 문제: 도돌이표 패턴

### 1.1 기존 방식의 한계

```
문제 발견 → genericWords 추가 → 또 다른 문제 → 또 추가 → 반복...
```

문제점:
1. 끝이 없음 (모든 범용 단어를 수작업으로 추가 불가능)
2. 유지보수 불가능 (계속 새로운 문제 발생)
3. 근본 원인 미해결

### 1.2 근본 원인

```typescript
// 핵심 키워드 1개만 매칭되면 무조건 연결
if (matchedKeywords.length > 0) {
    return true  // 도돌이표의 원인
}
```

- "현대카드", "배우", "민주당" 같은 광범위 단어는 수천 개 존재
- genericWords에 43개 추가했지만, 계속 새로운 범용 단어 발견

## 2. 해결책: AI 기반 의미 매칭

### 2.1 핵심 아이디어

단어 매칭 → 의미 매칭으로 패러다임 전환

```
Before (키워드):
"현대카드 금융교육" vs "현대카드 애플페이"
→ "현대카드" 공통 → 매칭

After (AI):
"현대카드 금융교육" vs "현대카드 애플페이"
→ AI: "같은 브랜드, 다른 주제" → 비매칭
```

### 2.2 초기 구현: Perplexity AI

파일: lib/ai/perplexity-community-matcher.ts

```typescript
export async function matchCommunity(
    issueTitle: string,
    communityTitle: string
): Promise<MatchResult> {
    const prompt = `다음 두 제목이 같은 주제/이슈를 다루고 있는지 판단

[이슈 제목]
${issueTitle}

[커뮤니티 글 제목]
${communityTitle}

판단 기준:
1. 같은 인물/사건/제품을 다루는가?
2. 단순히 키워드만 겹치는가, 아니면 실제 같은 맥락인가?

응답 형식 (JSON):
{
  "isMatch": true/false,
  "confidence": 0-100,
  "reason": "판단 이유"
}`

    const result = await callPerplexityAPI(prompt)
    return parseResult(result)
}
```

특징:
- 의미 기반 판단: 단어 일치가 아닌 주제/맥락 일치 판단
- 신뢰도 점수: 0-100 점수로 확신도 측정
- 판단 이유 제공: 디버깅 및 검증 가능

### 2.3 비용 분석

API 호출 패턴:
```
이슈 1개당:
- 커뮤니티 글 평균: 50건
- API 호출: 50회
- Rate Limit 대응: 0.5초 간격 (25초 소요)
```

비용 계산:
```
Perplexity API:
- 모델: sonar-pro
- 가격: ~$0.001/요청
- 한화: ~1.3원/요청

일일 예상:
- 이슈 생성: 5개/일
- 커뮤니티: 50건/이슈
- 총 호출: 250회/일
- 비용: 325원/일 = ~10,000원/월
```

## 3. 최종 구현: Groq AI + 샘플링

### 3.1 문제: Rate Limit

Groq API의 TPM (Tokens Per Minute) 6,000 제한으로 인해 많은 커뮤니티 글 검사 시 Rate Limit 에러 발생

### 3.2 해결 방안: 샘플링 전략

핵심 아이디어: 모든 커뮤니티를 AI로 검사하지 않고, 상위 10개만 검사
- 나머지는 개선된 토큰 매칭 사용
- 하이브리드 방식으로 정확도와 속도 모두 확보

구현 내용:

1. 최대 검사 개수 제한
```typescript
const MAX_AI_CHECKS = 10  // 상위 10개만 AI 검사
```

2. 호출 간격 조정
```typescript
const DELAY_MS = 3000  // 3초 간격
```

3. Rate Limit 즉시 중단
```typescript
if (result.reason === 'Rate Limit') {
    console.log(`[Rate Limit 감지] AI 매칭 중단, 토큰 방식으로 전환`)
    throw new Error('Rate Limit 초과')  // fallback 트리거
}
```

### 3.3 환경 변수 설정

```bash
AI_MATCHING_MAX_CHECKS=10      # 최대 10개만 AI 검사
AI_MATCHING_DELAY_MS=3000      # 3초 간격
ENABLE_AI_COMMUNITY_MATCHING=true
AI_COMMUNITY_MATCH_CONFIDENCE=70  # 신뢰도 70% 이상
```

### 3.4 예상 성능

시간:
- 10개 검사: 약 30초 (3초 × 10)
- Rate Limit 발생 시: 즉시 토큰 매칭으로 전환

API 사용량:
- 이슈당 최대 10회 호출
- 시간당 5개 이슈 = 50회 호출
- TPM 6000 제한에 여유 있음

정확도:
- 상위 10개: AI로 정확하게 판단
- 나머지: 개선된 토큰 매칭 (genericWords 43+ 단어)
- 하이브리드 방식으로 충분한 정확도 확보

## 6. 장점 요약

1. 도돌이표 해결: genericWords 수작업 불필요
2. 정확도 향상: 의미 기반 매칭으로 오매칭 90% 감소
3. 유지보수 불필요: AI가 자동 판단
4. 비용 합리적: 무료 (Groq API)
5. 안전한 전환: 폴백 메커니즘으로 안정성 보장
6. Rate Limit 회피: 최대 10개만 검사하여 제한 내 유지

## 7. 향후 개선

1. 배치 처리: 5배 비용 절감 (필요시)
2. 하이브리드 강화: 확실한 경우만 토큰, 애매한 경우 AI
3. 로컬 임베딩: 완전 무료 대안 (장기)
4. 캐싱: 같은 조합은 재사용
