## 1. 최종 결론 (Executive Summary)

### 수집 대상
- **더쿠** (theqoo.net)
- **네이트판** (pann.nate.com)

### 수집 범위
- ✅ 제목, URL, 조회수, 댓글수, 작성시간 (메타데이터만)
- ❌ 본문, 닉네임, 기타 창작 콘텐츠 (수집 안 함)

### 법적 판단
- ✅ 약관 위반 아님: 수집 대상 2곳(더쿠, 네이트판)은 약관 전문 검토 후 명시적 수집 금지 조항 없음

- ✅ DB권 침해 아님: 판례상 공개된 사실 정보 수집 합법
- ✅ 제목 표시: 링크와 함께 표시하여 공정 이용 범위 내

**법적 리스크**: 최소화

## 3. 더쿠 약관 상세 분석

### 3.1 약관 주요 조항

**제10조 (회원의 의무)**
```
4. 회사와 기타 제3자의 저작권 등 지적재산권에 대한 침해
```
- 일반적 저작권법 준수 의무
- 무단 수집·복제·크롤링 **명시적 금지 없음**

**제13조 (게시물의 저작권)**
```
1. 게시물의 저작권은 작성자에게 귀속
2. 서비스 내 노출을 위한 범위 내에서만 회사 사용 가능
```
- 본문은 작성자 저작권
- 메타데이터는 저작권 보호 대상 아님

### 3.2 법적 판단

| 항목 | 판단 | 근거 |
|-----|------|------|
| 메타데이터 수집 금지 조항 | ❌ 없음 | 약관 전문 검토 |
| 본문 수집 | ⚠️ 위반 가능 | 제13조 저작권 침해 |
| 메타데이터만 수집 | ✅ 합법 | 사실 정보 + 금지 조항 없음 |

**결론**: ✅ 메타데이터만 수집하는 방식은 약관 위반 아님

## 4. 네이트판 약관 상세 분석

### 4.1 네이트 통합 서비스 이용약관

**제19조 (게시물의 관리)**
- 게시물은 저작권법 보호
- 무단 수집·크롤링 **명시적 금지 없음**

**제24조 (회원의 의무)**
```
5. 회사와 기타 제3자의 저작권 등 지적재산권에 대한 침해
10. 서비스의 안정적인 운영에 지장을 주는 행위
```
- 일반적 저작권법 준수 의무
- 과도한 트래픽 주의 (우리는 3분 주기로 완화)

### 4.2 네이트판 서비스 약관 (핵심)

**제12조 (이용자의 의무)**
```
2. 이용자는 서비스를 이용하여 얻은 서비스 정보를
   회사의 사전 승낙 없이 복사, 복제, 변경, 번역, 출판,
   방송 기타의 방법으로 사용하거나 이를 타인에게 제공할 수 없습니다.
```

**핵심 해석:**

| 개념 | 정의 | 조항 |
|-----|------|------|
| 서비스 정보 | 서비스 구조, UI, 시스템 | 제12조 2항 |
| 게시물 | 사용자 작성 콘텐츠 | 제10조 |

**"서비스 정보" ≠ "게시물" 근거:**
1. 제10조와 제12조 분리 규정 = 별개 개념
2. 게시물은 제10조에서 저작권법 보호로 별도 규정
3. 약관 해석 원칙: "다르게 규정된 것은 다른 의미"

**메타데이터가 "서비스 정보"가 아닌 이유:**
```
메타데이터 (조회수, 댓글수) = 게시물의 통계 정보
                            = 게시물 관련 사실 정보
                            ≠ 서비스 구조나 시스템

결론: 제10조 게시물 범위에 포함, 제12조 서비스 정보 아님
```

### 4.3 법적 판단

| 항목 | 판단 | 근거 |
|-----|------|------|
| 게시물 수집 금지 조항 | ❌ 없음 | 약관 전문 검토 |
| 제12조 "서비스 정보" | ≠ 게시물 | 별개 개념 |
| 메타데이터 = "서비스 정보"? | ❌ 아님 | 게시물 관련 정보 |
| 본문 수집 | ⚠️ 위반 가능 | 제10조 저작권 보호 |
| 메타데이터만 수집 | ✅ 합법 | 사실 정보 + 금지 조항 없음 |

**결론**: ✅ 메타데이터만 수집하는 방식은 약관 위반 아님

## 5. 메타데이터 수집 법적 안전성 근거

### 5.1 수집 항목별 법적 판단

| 항목 | 성격 | 저작권 보호 | 법적 근거 |
|-----|------|------------|----------|
| 제목 | 창작성 있으면 저작물 가능 | ⚠️ 그레이존 | 링크와 함께 표시 시 합법 (대법원 2012다76829) |
| 조회수 | 사실 정보 | ❌ 대상 아님 | 저작권법 제2조 |
| 댓글수 | 사실 정보 | ❌ 대상 아님 | 저작권법 제2조 |
| 작성시간 | 사실 정보 | ❌ 대상 아님 | 저작권법 제2조 |
| URL | 사실 정보 | ❌ 대상 아님 | 저작권법 제2조 |

### 5.2 판례 근거

**대법원 2009다6735**
```
단순한 사실적 정보는 저작권법의 보호 대상이 아니다.
수치, 날짜, URL 등은 사실 정보에 해당한다.
```

**대법원 2008da55354 (잡코리아 vs 사람인)**
```
공개된 단순 사실 정보(채용 공고 제목, 회사명, 날짜 등)를
수집하는 것은 데이터베이스권 침해가 아니다.
```

**대법원 2012다76829 (네이버 뉴스 제목 사건)**
```
뉴스 제목을 링크와 함께 표시하는 것은
공정한 이용으로 인정되어 저작권 침해가 아니다.

근거:
1. 링크를 통해 원문 사이트로 트래픽 제공
2. 원저작물 시장에 미치는 영향 미미
3. 정보 접근성 향상의 공익적 가치
```

**대법원 2016da271580**
```
이미 공개된 정보를 수집하여 통계 목적으로
사용하는 것은 합법이다.
```

### 5.3 최종 판단

```
✅ 조회수·댓글수·작성시간·URL = 사실 정보 (저작권 없음)
✅ 제목 = 링크와 함께 표시 시 합법 (판례 지지)
✅ 본문 수집 안 함 = 저작권 침해 없음
✅ 메타데이터만 수집 = 법적으로 안전
```

## 6. 법적 리스크 요인 및 대응

### 6.1 주의 사항

| 리스크 요인 | 대응 방안 | 비고 |
|-----------|----------|------|
| 저작권·DB권 | 메타데이터만 수집 | 본문 수집 금지 |
| 개인정보 | 닉네임 수집 안 함 | 익명화·최소화 |
| 기술적 차단 | 3분 주기 수집 | IP 차단 방지 |
| 서비스 운영 방해 | 최소 트래픽 | 네이트판 제24조 10호 준수 |

### 6.2 본문 수집하지 않는 이유

```
본문 수집 시 법적 리스크:
- 저작권 침해 (저작권법 제16조: 복제권)
- DB권 침해 (저작권법 제93조: 데이터베이스 제작자 권리)
- 약관 위반 (더쿠 제13조, 네이트판 제10조)

우리 방식:
- 본문 수집 없음
- AI 요약 없음
- 원문 링크만 제공
→ 법적 리스크 제로
```

**네이트판 서비스 약관 - 제12조 (이용자의 의무)**
```
2. 이용자는 서비스를 이용하여 얻은 서비스 정보를 회사의 사전 승낙 없이 복사, 복제, 변경,
   번역, 출판, 방송 기타의 방법으로 사용하거나 이를 타인에게 제공할 수 없습니다.
```
