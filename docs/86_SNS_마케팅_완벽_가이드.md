## 3. 카드뉴스 완전 자동화

Canva 같은 수동 도구 없이, 데이터 조회 → AI 텍스트 생성 → 이미지 렌더링 → SNS 업로드까지 전부 코드로 처리하는 파이프라인.

---

### 3.1 전체 흐름

```
GitHub Actions Cron (매주 수/토 오전 9시)
         ↓
① Supabase → 주간 TOP 3 이슈 조회
         ↓
② Groq AI → 슬라이드별 텍스트 자동 생성
         ↓
③ HTML/CSS 템플릿에 텍스트 주입
         ↓
④ Playwright → 브라우저로 렌더링 → PNG 이미지 8장 생성
         ↓
⑤ Instagram Graph API → 캐러셀(여러 장) 게시물 업로드
         ↓
⑥ Threads API → 캐러셀 게시물 업로드 (30초 대기 후 발행)
         ↓
⑦ Twitter API → 연계 트윗 자동 발행
```

비용: **전부 무료** (Groq 무료 티어, GitHub Actions 무료, Instagram/Threads/Twitter API 기본 무료)

---

### 3.2 각 단계 상세 설명

#### ① 데이터 조회 (Supabase)

이미 있는 코드 패턴 그대로 사용. 댓글 수, 반응 수 기준으로 이번 주 인기 이슈 TOP 5 선별.

```typescript
const { data: issues } = await supabase
  .from('issues')
  .select('id, title, category, reaction_count, comment_count')
  .order('reaction_count', { ascending: false })
  .limit(5)
```

---

#### ② AI 텍스트 생성 (Groq)

각 슬라이드에 들어갈 텍스트를 Groq에게 맡김.

```typescript
import Groq from 'groq-sdk'

const groq = new Groq({ apiKey: process.env.GROQ_API_KEY })

async function generateSlideTexts(issue: Issue) {
  const res = await groq.chat.completions.create({
    model: 'llama3-8b-8192',
    messages: [{
      role: 'user',
      content: `
        이슈: "${issue.title}"
        다음 슬라이드 텍스트를 JSON으로 생성해줘:
        - cover: 커버 문구 (15자 이내)
        - summary: 한 줄 요약 (30자 이내)
        - point1: 핵심 포인트 1 (20자 이내)
        - point2: 핵심 포인트 2 (20자 이내)
        - cta: 행동 유도 문구 (예: "전체 타임라인 보기")
      `
    }]
  })
  return JSON.parse(res.choices[0].message.content!)
}
```

#### ④ Playwright로 PNG 생성

Node.js에서 headless Chrome(브라우저)을 띄워 HTML을 1080×1350 이미지로 찍는 방식.

```typescript
import { chromium } from 'playwright'

async function renderSlide(html: string, outputPath: string) {
  const browser = await chromium.launch({ args: ['--no-sandbox'] })
  const page = await browser.newPage()
  await page.setViewportSize({ width: 1080, height: 1350 })
  await page.setContent(html, { waitUntil: 'networkidle' })
  await page.screenshot({ path: outputPath, type: 'png' })
  await browser.close()
}

// 슬라이드 8장 생성
for (let i = 0; i < slides.length; i++) {
  const html = template.replace('{{cover}}', slides[i].text)
  await renderSlide(html, `output/slide_${i}.png`)
}
```

**결과물:** `slide_0.png` ~ `slide_7.png` (1080×1350px 완성 이미지)

#### ⑤ Instagram Graph API로 업로드

인스타그램 캐러셀(여러 장 게시물) 업로드는 3단계.

**전제 조건:**
- Instagram Business/Creator 계정
- Facebook Page와 연결
- Meta for Developers에서 앱 생성
- 필요 권한: `instagram_content_publish`, `pages_read_engagement`

```typescript
const IG_USER_ID = process.env.IG_USER_ID!
const ACCESS_TOKEN = process.env.IG_ACCESS_TOKEN!

async function uploadCarousel(imagePaths: string[], caption: string) {
  // 1단계: 이미지별 미디어 컨테이너 생성
  const mediaIds: string[] = []
  for (const imagePath of imagePaths) {
    const imageUrl = await uploadToStorage(imagePath) // Supabase Storage 활용
    const res = await fetch(
      `https://graph.facebook.com/v18.0/${IG_USER_ID}/media`,
      {
        method: 'POST',
        body: new URLSearchParams({
          image_url: imageUrl,
          is_carousel_item: 'true',
          access_token: ACCESS_TOKEN,
        }),
      }
    )
    const { id } = await res.json()
    mediaIds.push(id)
  }

  // 2단계: 캐러셀 컨테이너 생성
  const carouselRes = await fetch(
    `https://graph.facebook.com/v18.0/${IG_USER_ID}/media`,
    {
      method: 'POST',
      body: new URLSearchParams({
        media_type: 'CAROUSEL',
        children: mediaIds.join(','),
        caption,
        access_token: ACCESS_TOKEN,
      }),
    }
  )
  const { id: carouselId } = await carouselRes.json()

  // 3단계: 게시물 발행
  await fetch(
    `https://graph.facebook.com/v18.0/${IG_USER_ID}/media_publish`,
    {
      method: 'POST',
      body: new URLSearchParams({
        creation_id: carouselId,
        access_token: ACCESS_TOKEN,
      }),
    }
  )
}
```

**이미지 URL 문제:** Instagram API는 외부 URL을 받음 → 생성된 PNG를 Supabase Storage에 업로드 후 public URL 전달.

#### ⑥ Threads API로 업로드

Threads는 Instagram과 거의 동일하지만 Base URL과 대기 시간이 다름.

**전제 조건:**
- Meta for Developers 앱 (Instagram과 동일 앱 사용 가능)
- Tech Provider Verification (약 1주일)
- 필요 권한: `threads_basic`, `threads_content_publish`
- 60일 유효 액세스 토큰 (Instagram과 동일)

```typescript
const THREADS_USER_ID = process.env.THREADS_USER_ID!
const THREADS_ACCESS_TOKEN = process.env.THREADS_ACCESS_TOKEN!

async function uploadToThreads(imagePaths: string[], caption: string) {
  // 1단계: 이미지별 미디어 컨테이너 생성
  const itemIds: string[] = []
  for (const imagePath of imagePaths) {
    const imageUrl = await uploadToStorage(imagePath)
    const res = await fetch(
      `https://graph.threads.net/v1.0/${THREADS_USER_ID}/threads`,
      {
        method: 'POST',
        body: new URLSearchParams({
          media_type: 'IMAGE',
          image_url: imageUrl,
          is_carousel_item: 'true',
          access_token: THREADS_ACCESS_TOKEN,
        }),
      }
    )
    const { id } = await res.json()
    itemIds.push(id)
  }

  // 2단계: 캐러셀 컨테이너 생성
  const carouselRes = await fetch(
    `https://graph.threads.net/v1.0/${THREADS_USER_ID}/threads`,
    {
      method: 'POST',
      body: new URLSearchParams({
        media_type: 'CAROUSEL',
        children: itemIds.join(','),
        text: caption,
        access_token: THREADS_ACCESS_TOKEN,
      }),
    }
  )
  const { id: carouselId } = await carouselRes.json()

  // 3단계: 최소 30초 대기 후 게시물 발행
  await new Promise(resolve => setTimeout(resolve, 30000))
  await fetch(
    `https://graph.threads.net/v1.0/${THREADS_USER_ID}/threads_publish`,
    {
      method: 'POST',
      body: new URLSearchParams({
        creation_id: carouselId,
        access_token: THREADS_ACCESS_TOKEN,
      }),
    }
  )
}
```

**Instagram과의 차이:**
- Base URL: `graph.threads.net` (Instagram은 `graph.facebook.com`)
- 캐러셀 최대: 20개 (Instagram은 10개)
- 발행 전 대기: 최소 30초 필수
- Rate Limit: 250개/일 명확히 명시

### 4.3 인플루언서 노출 전략

```
"○○○ 사건" 시간순 정리

#○○○
```

→ 해당 이슈에 관심 있는 인플루언서가 타임라인에서 자연 발견 → 리트윗

## 부록 A: 트위터 자동 트윗 스크립트

```typescript
// scripts/auto-tweet-hot-issues.ts
import { TwitterApi } from 'twitter-api-v2'
import { createAdminClient } from '@/lib/supabase/server'

const twitter = new TwitterApi({
    appKey: process.env.TWITTER_API_KEY!,
    appSecret: process.env.TWITTER_API_SECRET!,
    accessToken: process.env.TWITTER_ACCESS_TOKEN!,
    accessSecret: process.env.TWITTER_ACCESS_SECRET!,
})

export async function tweetHotIssues() {
    const supabase = createAdminClient()

    // 최근 1시간 내 등록된 이슈 중 아직 트윗 안 된 것
    const { data: issues } = await supabase
        .from('issues')
        .select('*')
        .gte('created_at', new Date(Date.now() - 60 * 60 * 1000).toISOString())
        .eq('tweeted', false)
        .order('reaction_count', { ascending: false })

    for (const issue of issues || []) {
        const tweet = `
🚨 지금 커뮤니티 난리남!

"${issue.title}"

반응 ${issue.reaction_count}개 | 댓글 ${issue.comment_count}개
타임라인 보기 👉 https://whynali.com/issue/${issue.id}

#왜난리 #이슈 #${issue.category}
        `.trim()

        await twitter.v2.tweet(tweet)

        await supabase
            .from('issues')
            .update({ tweeted: true })
            .eq('id', issue.id)

        await new Promise(r => setTimeout(r, 60000))
    }
}
```

## 핵심 원칙

1. **자동화 > 수동** — 반복 작업은 코드로
2. **일관성 > 빈도** — 매일 3회보다 주 3회 꾸준히
3. **품질 > 양** — 평범한 트윗 10개보다 강력한 트윗 1개
4. **참여 > 노출** — 팔로워 1만명보다 참여율 5%
