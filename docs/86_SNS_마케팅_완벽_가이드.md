### 1.4 트윗 작성 가이드

#### 트윗 유형별 공식

**1. 클릭 유도 트윗**

**나쁜 예시:**
```
❌ 새로운 이슈가 등록되었습니다.
   "○○○ 논란"
   확인하세요.
   https://whynali.com/issue/123
```

**문제점:**
- 지루함
- 클릭 동기 없음
- 로봇 같음

---

**좋은 예시 - 긴급성:**
```
✅ 🚨 방금 화력 95 돌파!

   "○○○ 발언 논란"

   10분 전 새 전개 추가
   지금 커뮤니티 난리남

   실시간 타임라인 👉 [링크]

   #왜난리 #○○○
```

**성공 요소:**
- 🚨 긴급 신호
- "방금", "10분 전" = 실시간성
- "난리남" = 호기심
- 명확한 행동 유도

---

**좋은 예시 - 논쟁 유발:**
```
✅ "○○○ 사건, 누구 잘못?"

   A측 주장: "△△△가 먼저..."
   B측 주장: "아니, ○○○가..."

   전체 타임라인 보고 판단하세요
   👉 [링크]

   #○○○논란
```

**성공 요소:**
- 질문으로 시작
- 양측 의견 제시 → 공정함
- "판단하세요" = 참여 유도

---

**좋은 예시 - 숫자 활용:**
```
✅ 3시간 만에 댓글 500개 돌파

   "○○○ 논란"
   지금 가장 뜨거운 이슈

   여론 67% vs 33%로 팽팽
   당신은 어느 쪽?

   투표 참여 👉 [링크]
```

**성공 요소:**
- 숫자 = 객관성
- "67% vs 33%" = 구체적
- 경쟁 구도 = 흥미

---

**2. 참여 유도 트윗**

**나쁜 예시:**
```
❌ 이번주 가장 뜨거운 이슈는 무엇인가요?
   댓글로 알려주세요.
```

**문제점:**
- 너무 평범
- 답변 동기 없음

---

**좋은 예시 - 이모지 투표:**
```
✅ 🔥 논란의 중심

   "○○○ 발언"

   이게 문제일까요, 아닐까요?

   문제다 → ❤️
   아니다 → 🔁

   #○○○논란
```

**성공 요소:**
- 명확한 선택지
- 클릭 한 번으로 참여
- 결과 보고 싶어짐

---

**좋은 예시 - 예측 게임:**
```
✅ 🎯 예측해봅시다

   "○○○ 논란" 어떻게 끝날까요?

   A. 사과문 발표
   B. 법적 대응
   C. 침묵으로 종결
   D. 반전 폭로

   댓글로 투표!
   가장 많은 선택은?
```

**성공 요소:**
- 재미있음
- 여러 선택지
- 다른 사람 의견 궁금

---

**좋은 예시 - 리트윗 유도:**
```
✅ 이 이슈 알아야 합니다

   "○○○ 사건" 타임라인 정리
   1. 발단 (3일 전): ...
   2. 전개 (2일 전): ...
   3. 파생 (어제): ...
   4. 현재: ...

   친구들에게도 공유 부탁드립니다 🙏
   RT = 정보 확산
```

**성공 요소:**
- 유용한 정보 = 공유 동기
- "친구들에게도" = 명확한 요청
- RT 이유 설명

### 1.5 해시태그 전략

**1. 브랜드 태그 (항상)**


### 1.8 자동화 vs 수동 운영

#### 자동화 추천 (70%)

**자동화할 것:**
1. 화력 80+ 이슈 자동 트윗
2. 일일 TOP 3 자동 스레드
3. 주간 요약 자동 생성

**장점:**
- 시간 절약
- 일관성 유지
- 놓치지 않음

**도구:**
- GitHub Actions Cron
- 자체 스크립트
- 비용: 무료

---

#### 수동 운영 (30%)

**수동으로 할 것:**

3. 바이럴 노리는 특별 트윗
4. 커뮤니티 인터랙션

**장점:**
- 인간미
- 유연한 대응
- 관계 구축

---

#### 실무 운영 조합

- 자동 트윗 3회

### 4.2 논쟁 유도 방법

**템플릿:**

**왜 효과적:**

```
- (근거 1)

당신 생각은?
```

**왜 효과적:**

### 4.3 인플루언서 멘션 전략

```
   @△△△ 측 반박

   👉 [링크]
```

**부적절한 멘션:**
```
❌ @유명인A @유명인B @유명인C
   왜난리 봐주세요!
```

#### 인플루언서 리트윗 받기

**전략:**
1. 인플루언서가 관심 가질 이슈 선택
2. 유용한 타임라인 정리
3. 멘션 없이 트윗
4. 자연스럽게 발견되길 기대

```
#○○○
```

→ ○○○에 관심 있는 인플루언서가 리트윗

### 4.4 리트윗 유도 기법

#### 직접 요청

**템플릿:**
```
이 정보 많은 분들이 알아야 합니다

"○○○ 사건" 전말 정리

(유용한 정보)

친구들에게 공유 부탁드립니다 🙏
RT = 정보 확산
```

**성공 요소:**
- "알아야 합니다" = 사명감
- 유용한 정보
- RT 이유 명확

---

#### 인센티브 제공

**템플릿:**
```
🎁 이벤트

RT + 팔로우 하신 분 중
추첨 5명 스타벅스 기프티콘

"○○○ 이슈" 가장 재밌는
댓글도 선정! 1만원 상품권

~5/12까지
```

**주의:**
- 초기엔 비추천 (효과 대비 비용)
- 팔로워 1,000명 이후 고려

---

#### 시리즈 스레드

**템플릿:**
```
🧵 "○○○ 사건" 완벽 정리

1/10 발단
   (내용)

2/10 첫 반응
   (내용)

...

10/10 현재 상황
    전체 보기 👉 [링크]

도움 되셨다면 RT 부탁드립니다 🙏
```

**성공 요소:**
- 스레드 = 정보 밀도 높음
- 북마크 + RT 유도
- 권위 구축

## 부록 A: 자동화 스크립트

### 화력 80+ 이슈 자동 트윗

```typescript
// scripts/auto-tweet-hot-issues.ts
import { TwitterApi } from 'twitter-api-v2'
import { createAdminClient } from '@/lib/supabase/server'

const twitter = new TwitterApi({
    appKey: process.env.TWITTER_API_KEY!,
    appSecret: process.env.TWITTER_API_SECRET!,
    accessToken: process.env.TWITTER_ACCESS_TOKEN!,
    accessSecret: process.env.TWITTER_ACCESS_SECRET!,
})

export async function tweetHotIssues() {
    const supabase = createAdminClient()

    const { data: issues } = await supabase
        .from('issues')
        .select('*')
        .gte('heat', 80)
        .gte('created_at', new Date(Date.now() - 60 * 60 * 1000).toISOString())
        .eq('tweeted', false)

    for (const issue of issues || []) {
        const tweet = `
🔥 화력 ${issue.heat} 급상승!

"${issue.title}"

반응 ${issue.reaction_count}개 | 댓글 ${issue.comment_count}개
자세히 보기 👉 https://whynali.com/issue/${issue.id}

#왜난리 #이슈 #${issue.category}
        `.trim()

        await twitter.v2.tweet(tweet)

        await supabase
            .from('issues')
            .update({ tweeted: true })
            .eq('id', issue.id)

        await new Promise(r => setTimeout(r, 60000))
    }
}
```

## 마치며

**핵심 원칙:**

1. **일관성 > 빈도**
   - 매일 3회보다 주 3회 꾸준히

2. **품질 > 양**
   - 10개 평범한 트윗보다 1개 강력한 트윗

3. **참여 > 노출**
   - 팔로워 1만명보다 참여율 5%

4. **실험 > 완벽**
   - 완벽한 계획보다 빠른 실행
