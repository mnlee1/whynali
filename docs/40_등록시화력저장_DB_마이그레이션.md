# 등록 시 화력 저장 DB 마이그레이션

**날짜**: 2026-03-06

## 목적

이슈 등록 당시의 화력 점수를 저장하여 관리자가 시간 경과 후에도 등록 기준을 명확히 이해할 수 있도록 개선

## 문제점

- 이슈 등록 시: 화력 18점 (등록 조건 충족)
- 2시간 후: 화력 12점 표시
- 관리자 오해: "15점 미만인데 왜 등록되어 있지?"

## 해결 방안

등록 시점 화력을 별도 저장하고 "18점 → 12점" 형태로 표시

---

## SQL 마이그레이션

```sql
-- issues 테이블에 등록 시점 화력 컬럼 추가
ALTER TABLE issues 
ADD COLUMN IF NOT EXISTS created_heat_index INTEGER;

-- 컬럼 설명
COMMENT ON COLUMN issues.created_heat_index IS '이슈 등록 당시의 화력 점수 (등록 기준 확인용)';

-- 기존 데이터 마이그레이션 (현재 화력을 등록 시 화력으로 복사)
UPDATE issues 
SET created_heat_index = heat_index 
WHERE created_heat_index IS NULL AND heat_index IS NOT NULL;
```

---

## 실행 방법

### 1. Supabase Dashboard에서 실행

1. Supabase Dashboard → SQL Editor
2. 위 SQL 복사 붙여넣기
3. Run 실행

### 2. 로컬 확인

```bash
# 컬럼 추가 확인
psql $DATABASE_URL -c "\d issues"

# 데이터 확인
psql $DATABASE_URL -c "SELECT id, title, created_heat_index, heat_index FROM issues LIMIT 5;"
```
