# 숏폼 자동 생성·배포 설계 가이드

MVP 이후 확장 단계에서 "방법을 모르는" 상태로 숏폼 자동 생성·배포를 진행할 때, 무엇을 정하고 어떤 순서로 하면 되는지 정리한 문서다. 실제 구현 코드는 다루지 않고, 설계·준비·선택지만 다룬다.

---

## 1. 목적과 범위

**목적**: 인스타 릴스, 유튜브 쇼츠, 틱톡에 짧은 영상을 올려서 **본 서비스(whynali)로 유입**을 늘린다.

**범위**
- 플랫폼: 인스타 릴스, 유튜브 쇼츠, 틱톡 (최소 3곳).
- 콘텐츠: 이슈 한 건당 또는 전환 시점당 숏폼 1개. "이 이슈가 뭔지 + 본 서비스 링크"가 들어가면 됨.
- 트리거(기획): 이슈 생성 시, 이슈 상태 전환 시(예: 점화→논란 중) 등. 구체 트리거는 설계 단계에서 확정.

**사용하지 않는 것**: 본문 수집·본문 기반 AI 요약은 법적 검토상 사용하지 않는다. 숏폼에도 본문 요약을 넣지 않고, **이슈명·상태·출처 개수·화력·서비스 링크** 수준만 사용한다.

---

## 2. 전체 흐름 (입력 → 영상 → 배포)

```
[우리 서비스 DB]
  이슈 제목, 상태, 화력, 타임라인 출처 개수, 이슈 상세 URL
        ↓
  텍스트/이미지 초안 생성 (메타데이터만 사용, 본문 미사용)
        ↓
  숏폼 영상 1개 생성 (템플릿 + 데이터 주입, 또는 AI 영상 툴)
        ↓
  플랫폼별 업로드 (유튜브/인스타/틱톡 API 또는 수동)
        ↓
  영상 설명·댓글에 "자세히 보기: https://whynali.com/issue/xxx" 링크
```

설계 시 정할 것:
- 입력으로 DB에서 **어떤 필드**를 가져올지 (이슈 제목, 상태, 화력 등).
- 영상을 **어떤 방식**으로 만드는지 (템플릿 기반 vs AI 영상 툴).
- 업로드를 **전부 자동**으로 할지, **초안만 자동 생성하고 업로드는 수동**으로 할지.

---

## 3. 입력 데이터 (우리 쪽에서 쓸 수 있는 것)

법적 검토·기획에 맞게 **본문은 쓰지 않는다**. 아래만 사용한다.

| 항목 | 설명 | 숏폼에서 쓰는 예 |
|------|------|------------------|
| 이슈 제목 | 우리가 붙인 이슈명 | "OO 논란", "OO 사건" |
| 이슈 상태 | 점화/논란 중/종결 | "지금 논란 중", "종결" |
| 화력 지수·등급 | 숫자·높음·보통·낮음 | "화력 높음" |
| 관련 출처 수 | 뉴스 N건, 커뮤니티 M건 | "뉴스 5건, 게시 12건" |
| 이슈 상세 URL | whynali.com/issue/xxx | "자세히 보기" 링크 |

**쓰지 않는 것**: 기사·게시글 본문, 본문 요약문, 타임라인 문장(본문 기반).

---

## 4. 숏폼이 무엇인지 (포맷)

- **세로 영상**. 가로:세로 = 9:16 비율.
- **길이**: 플랫폼마다 다름. 유튜브 쇼츠 60초 이하, 인스타 릴스·틱톡도 60초 이하~90초 수준. **15~60초**로 통일해 두면 3곳 모두 사용 가능.
- **내용**: 텍스트 위주(자막) + 배경 음악 또는 무음. 실제 인물 출연 없이도 가능. "이슈 제목 + 한두 줄 설명 + 자세히 보기 링크"만 있어도 됨.

---

## 5. 영상 만드는 방법 (두 가지 큰 갈래)

### 5.1 템플릿 + 데이터 주입 (추천: 먼저 시도)

- **방식**: 미리 "자막 위치, 배경, 길이"가 정해진 **영상 템플릿**을 하나 만든다. 이슈마다 "이슈 제목", "상태", "화력" 등만 바꿔 넣어서 영상 파일를 만든다.
- **장점**: 비용 적고, 출력이 예측 가능하다. 법적·정책 리스크도 작다.
- **필요한 것**:
  - 영상 편집 툴로 템플릿 1개 제작 (예: DaVinci Resolve, Canva 비디오, CapCut 무료판).
  - 스크립트(코드) 또는 no-code 툴로 "데이터(이슈명 등) → 텍스트/이미지 → 영상 프레임 또는 최종 영상" 생성. (예: FFmpeg로 이미지+자막 합성, 또는 Canva API 등.)
- **순서**: 먼저 **수동으로** 이슈 1개 골라서 숏폼 1개 만들고, 그 구성을 "템플릿"으로 고정한 뒤, 그다음 자동화(데이터 주입)를 붙인다.

### 5.2 AI 영상 툴 사용

- **방식**: 텍스트(이슈 제목 + 한두 줄 설명)를 입력하면 AI가 영상을 만들어 주는 서비스 사용. (Runway, Pictory, Lumen5, Canva AI 등.)
- **장점**: 사람이 영상을 짜지 않아도 됨.
- **단점**: 비용 발생, 플랫폼별 한도·정책, 출력 품질·안정성 차이. **입력 텍스트는 반드시 메타데이터 기반 사실 서술만** (본문 요약 금지).
- **순서**: 툴 하나 정한 뒤, "이슈 제목 + 상태 + 자세히 보기 링크"만 넣어서 테스트 영상 1개 만들어 보고, 품질·비용 괜찮으면 자동화(API 연동) 검토.

---

## 6. 플랫폼별로 필요한 것

| 플랫폼 | 영상 포맷 | 업로드 방식 | 사전 준비 |
|--------|-----------|-------------|-----------|
| 유튜브 쇼츠 | 세로 9:16, 60초 이하 | YouTube Data API v3 (업로드) | Google 계정, YouTube 채널, API 키·OAuth |
| 인스타 릴스 | 세로 9:16 | Meta Graph API (비즈니스/크리에이터) | 인스타 비즈니스/크리에이터 계정, 앱 등록 |
| 틱톡 | 세로 9:16 | TikTok Content Posting API | 틱톡 개발자 계정, 앱 등록 |

**처음에는 한 플랫폼만** 정해서, "영상 1개 자동 생성 → 그 플랫폼에만 업로드"까지 만든 뒤, 나머지 두 곳은 같은 영상 파일을 수동 업로드해도 된다. 전부 API로 자동화하는 건 그다음 단계로 두면 된다.

---

## 7. 설계·진행 순서 (방법 모를 때 따라갈 단계)

| 단계 | 할 일 | 산출물 |
|------|--------|--------|
| 1 | **채널/계정 만들기** | 유튜브 채널 1개, 인스타 계정 1개, 틱톡 계정 1개. 모두 "왜난리" 전용. |
| 2 | **숏폼 1개 수동 제작** | 이슈 하나 골라서, "제목 + 상태 + 화력 + 자세히 보기 링크"만 넣은 15~60초 세로 영상 1개를 수동으로 만든다. 편집 툴(Canva, CapCut 등) 사용. |
| 3 | **포맷 확정** | 위 영상의 "구성(몇 초에 뭘 보여 줄지, 자막 위치, 링크 노출 방법)"을 문서로 정리. 이게 나중의 "템플릿 설계"가 됨. |
| 4 | **입력 데이터 정의** | DB에서 가져올 항목 목록 확정. 예: 이슈 ID, 제목, 상태, 화력 등급, 이슈 URL. (3절 표 참고.) |
| 5 | **템플릿 기반 1개 자동 생성 시도** | 스크립트 또는 no-code로 "입력 데이터 → 텍스트/이미지 → 영상 파일 1개"가 나오게 한다. FFmpeg, Canva API, 또는 팀이 쓰기 쉬운 툴 선택. |
| 6 | **한 플랫폼만 업로드 연동** | 만든 영상 파일을 한 곳(예: 유튜브)에 API로 업로드하는 흐름을 만든다. 설명란에 "자세히 보기: [이슈 URL]" 반드시 포함. |
| 7 | **트리거 연결** | "이슈 생성 시" 또는 "이슈 상태 전환 시"에 5~6번이 돌도록 스케줄/이벤트 연결. 관리자 승인 단계를 넣을지 여부는 팀 정책에 따름. |
| 8 | **나머지 플랫폼** | 같은 영상으로 인스타·틱톡은 먼저 수동 업로드. 괜찮으면 각 플랫폼 API 연동을 추가. |

---

## 8. 트리거 구체화 (언제 숏폼을 만들지)

기획: "이슈 생성 시, 이슈의 특정 전환 시점 발생 시."

**설계 시 정할 것**:
- **이슈 생성 시**: 새 이슈가 승인·공개될 때마다 1개 생성할지, "화력이 N 이상"일 때만 할지.
- **전환 시점**: 상태가 "점화→논란 중", "논란 중→종결" 바뀔 때만 만들지, 전 상태 변경마다 만들지.
- **빈도**: "하루 수십 개"가 부담이면 "하루 1~5개"처럼 상한을 두고, 우선순위(화력 순 등) 규칙을 정한다.

---

## 9. 비용·도구 (참고)

- **템플릿 + FFmpeg/스크립트**: 서버만 쓰면 비용은 거의 없음. Canva API 등은 유료 플랜 시 월 비용 발생.
- **AI 영상 툴**: Runway, Pictory 등은 사용량당 과금. "하루 수십 개"면 비용이 커질 수 있으니, 먼저 "하루 1~3개"로 테스트하는 편이 안전함.
- **플랫폼 API**: 유튜브/메타/틱톡 업로드 API는 보통 무료 할당량 안에서 사용 가능. 각 개발자 문서에서 할당량 확인.

---

## 10. 주의사항

- **저작권**: 기사·게시글 본문, 본문 요약문은 숏폼에도 넣지 않는다. 이슈명·상태·화력·출처 개수·우리 서비스 링크만 사용.
- **플랫폼 정책**: 인스타·유튜브·틱톡 모두 자동화·스팸 정책이 있다. 반복 업로드·동일 콘텐츠 대량 업로드 시 제재될 수 있으니, 빈도·중복을 낮추는 게 좋다.
- **관리자 승인**: 기획에 "AI 가공 콘텐츠는 승인 후 릴리스"가 있으므로, 숏폼도 "생성 후 목록 보여 주고, 승인한 것만 업로드"하는 흐름을 두면 정책과 맞다.
