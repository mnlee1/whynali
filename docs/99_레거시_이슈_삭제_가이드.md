# 레거시 이슈 삭제 가이드

## 개요
트랙 A 기본화에 따라 기존 프로세스로 생성된 이슈들을 정리합니다.

## 삭제 대상
- `source_track`이 NULL이거나 'track_a'가 아닌 모든 이슈
- 해당 이슈와 연결된 타임라인, 토론, 투표 데이터
- 연결된 뉴스/커뮤니티 데이터의 `issue_id` 해제

## 삭제 방법

### Option 1: Supabase SQL Editor (추천)

1. Supabase 대시보드 접속
2. SQL Editor 메뉴 선택
3. `supabase/migrations/009_cleanup_legacy_issues.sql` 파일 내용을 복사하여 붙여넣기
4. 실행 버튼 클릭

### Option 2: 로컬에서 실행

```bash
# Supabase CLI 사용 (설치되어 있다면)
supabase migration up --file supabase/migrations/009_cleanup_legacy_issues.sql
```

## 실행 결과 예시

```
=== 이슈 정리 시작 ===
트랙 A 이슈: 15 건 (유지)
레거시 이슈: 89 건 (삭제 예정)

UPDATE news_data: 245 rows affected
UPDATE community_data: 178 rows affected
DELETE FROM timeline_points: 267 rows affected
DELETE FROM discussions: 12 rows affected
DELETE FROM votes: 8 rows affected
DELETE FROM issues: 89 rows affected

=== 정리 완료 ===
남은 이슈: 15 건 (모두 트랙 A)
```

## 주의사항

⚠️ **이 작업은 되돌릴 수 없습니다!**

안전을 위해:
1. 프로덕션 환경에서는 먼저 백업 생성
2. 개발/스테이징 환경에서 먼저 테스트
3. 삭제 대상 확인 후 실행

백업이 있다면:
```sql
-- 백업에서 복원 (백업 방법에 따라 다름)
```

백업이 없다면 되돌릴 수 없으므로 신중하게 실행하세요.
