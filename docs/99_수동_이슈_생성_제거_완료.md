# 수동 이슈 생성 기능 제거 완료

날짜: 2026-03-16

## 제거 내용

### 1. API 엔드포인트 제거

**파일:** `app/api/issues/route.ts`

**변경:**
```typescript
// 제거됨
export async function POST(request: NextRequest) { ... }

// 주석으로 대체
// POST 메서드는 제거됨 - 이슈는 트랙 A 프로세스를 통해서만 생성됨
// 수동 생성 기능은 실제 사용 사례가 없어 2026-03-16에 제거됨
```

**영향:**
- POST /api/issues 엔드포인트 사용 불가
- GET /api/issues는 정상 작동 (이슈 목록 조회)

### 2. 검증 로직 간소화

**파일:** `lib/validation/issue-creation.ts`

**변경:**
```typescript
// Before
export type SourceTrack = 'track_a' | 'manual'

// After
export type SourceTrack = 'track_a'
```

**검증 로직:**
- source_track은 'track_a'만 허용
- 'manual' 제거
- 명확한 에러 메시지

### 3. 타입 정의 업데이트

**파일:** `types/issue.ts`

**변경:**
```typescript
// Before
source_track?: 'news_collection' | 'community_burst'

// After
source_track?: 'track_a'
```

**이유:**
- 레거시 타입 정리
- 현재 시스템에 맞는 타입 정의

## 제거 근거

### 1. 실제 사용 없음
```
DB 현황 (2026-03-16):
- 총 이슈: 0개
- track_a: 0개
- manual: 0개
```

### 2. UI 없음
- 관리자 페이지에 수동 생성 버튼 없음
- page.tsx.legacy에만 흔적 존재
- 프론트엔드에서 POST 호출 없음

### 3. 과거 사용 사례 분석
```
manual 이슈 5건 (모두 삭제됨):
1. 블랙핑크 지수 (0점) - 버그 테스트
2. 이탈리아 WBC (0점) - 버그 테스트
3. 피는 못 속여 (0점) - 버그 테스트
4. WBC 한국 8강 (1점) - 버그 테스트
5. 올림픽 메달 (1점) - 버그 테스트

→ 모두 화력 계산 버그 관련
→ 실제 프로덕션 이슈 없음
```

### 4. 트랙 A로 충분
```
트랙 A 프로세스 (30분 주기):
1. 커뮤니티 급증 감지
2. AI 검증 (신뢰도 70%)
3. 네이버 뉴스 팩트 체크
4. 중복 체크 (4단계)
5. AI 필터링 + 제목 생성
6. 타임라인 자동 생성
7. 화력 계산 → 자동 승인/대기
```

## DB 정리 (선택적)

### 현재 상태
```sql
-- source_track 컬럼은 유지됨
-- 값: 'track_a' 또는 null
-- 기존 데이터 호환성 유지
```

### 향후 정리 (선택)
```sql
-- CHECK 제약 추가 (필요시)
ALTER TABLE issues 
ADD CONSTRAINT issues_source_track_check 
CHECK (source_track = 'track_a');
```

**권장:** 현재는 유지, 필요시 나중에 정리
