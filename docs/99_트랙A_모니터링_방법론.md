### 2. 실시간 모니터링 (선택)

GitHub Actions 워크플로우에 헬스 체크 추가:

```yaml
# .github/workflows/health-check.yml
name: Track-A Health Check

on:
  schedule:
    - cron: '0 9 * * *'  # 매일 오전 9시 (KST 18시)
  workflow_dispatch:

jobs:
  health-check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
      - run: npm install
      - run: npx tsx scripts/monitor_track_a_health.ts
        env:
          NEXT_PUBLIC_SUPABASE_URL: ${{ secrets.NEXT_PUBLIC_SUPABASE_URL }}
          SUPABASE_SERVICE_ROLE_KEY: ${{ secrets.SUPABASE_SERVICE_ROLE_KEY }}
```

## 생성률 저하 원인 및 해결 방법

### 원인 1: 커뮤니티 급증 감지 임계값이 너무 높음

**현재 설정** (.env.local):
```
COMMUNITY_BURST_THRESHOLD=10      # 최소 10건
COMMUNITY_BURST_WINDOW_MINUTES=10  # 10분 내
```

**문제**: 10분 내 같은 키워드로 10건 이상은 너무 엄격한 조건

**해결책**: 임계값 조정
```
COMMUNITY_BURST_THRESHOLD=5       # 5건으로 완화
COMMUNITY_BURST_WINDOW_MINUTES=30  # 30분으로 확대
```

### 원인 2: AI 검증이 너무 엄격함

트랙A는 AI가 "진짜 이슈인지" 판단하는데, AI가 보수적으로 판단할 수 있습니다.

**확인 방법**:
```bash
# AI 검증 통과율 확인
npx tsx scripts/analyze_ai_rejection_rate.ts
```

**해결책**:
- AI 프롬프트 조정 (신뢰도 임계값 낮추기)
- AI 판단을 우회하고 뉴스 검색 결과로만 판단하는 옵션 추가

### 원인 3: 뉴스 검색이 실패함

AI가 이슈라고 판단해도 네이버 뉴스에서 관련 뉴스를 찾지 못하면 이슈가 생성되지 않습니다.

**확인 방법**:
```bash
# 뉴스 검색 실패율 확인
npx tsx scripts/analyze_news_search_failure.ts
```

**해결책**:
- 네이버 뉴스 API 키워드 확장 (동의어, 유사어 포함)
- 다른 뉴스 소스 추가 (다음 뉴스, 구글 뉴스 등)

### 원인 4: 중복 이슈 체크에서 걸림

새로운 키워드가 감지되어도 기존 이슈와 중복으로 판단되면 생성되지 않습니다.

**확인 방법**:
```bash
# 중복 체크 통계
npx tsx scripts/analyze_duplicate_check_rate.ts
```

**해결책**:
- 중복 체크 시간 범위 축소 (현재 24시간 → 12시간)
- 중복 판단 신뢰도 임계값 상향 (80% → 90%)

## 권장 개선 조치 (우선순위순)

### 1단계: 즉시 적용 가능 (설정 변경)

**.env.local 수정**:
```bash
# 커뮤니티 급증 감지 완화
COMMUNITY_BURST_THRESHOLD=5           # 10 → 5
COMMUNITY_BURST_WINDOW_MINUTES=30     # 10 → 30

# 중복 체크 범위 축소
DUPLICATE_CHECK_WINDOW_HOURS=12       # 기본 24 → 12

# 중복 판단 신뢰도 상향
DUPLICATE_CHECK_AI_CONFIDENCE=90      # 80 → 90
```

**예상 효과**: 생성률 2~3배 증가 (하루 0.3개 → 1개)

### 2단계: 코드 수정 (중기)

1. **AI 검증 로직 개선**
   - 신뢰도 낮은 경우에도 뉴스가 많으면 이슈 생성
   - "보류" 상태 추가 (관리자 검토 후 승인)

2. **뉴스 검색 강화**
   - 키워드 변형 자동 생성 (띄어쓰기, 순서 변경)
   - 다음/구글 뉴스 API 통합

3. **급증 감지 알고리즘 개선**
   - 시간대별 기준 다르게 적용 (심야 vs 낮)
   - 사이트별 가중치 (더쿠 > 네이트판)
