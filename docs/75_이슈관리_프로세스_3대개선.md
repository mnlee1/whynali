# 이슈 등록/관리 프로세스 3대 개선

**작업일**: 2025-03-11
**목적**: 중복 이슈 방지 및 긴급 이슈 자동 승인 체계 확립

## 개선 사항 요약

### 1. AI 중복 체크 활성화

**배경**
- `lib/candidate/issue-candidate.ts` 약 321번째 줄 AI 중복 체크가 TODO 주석으로 비활성화됨
- Exact title match만 동작하여 표현이 다른 같은 사건이 별개 이슈 2개로 등록됨

**구현**
- `checkForDuplicateIssue()` 함수에서 `ENABLE_AI_DUPLICATE_CHECK` 환경변수 기반 AI 중복 체크 활성화
- `lib/candidate/duplicate-checker.ts`의 4단계 검증 연결
    1. Exact match (정확한 제목 일치)
    2. 키워드 필터링 (공통 키워드 2개 이상)
    3. 안전 필터 (반대어, 숫자 차이)
    4. AI 정밀 비교 (Groq, 신뢰도 80% 이상)

**에러 처리**
- AI 호출 실패, 타임아웃 시 보수적으로 중복 아님으로 판정
- 로그 기록 후 이슈 등록 프로세스 계속 진행
- 중복 판정 시 기존 이슈에 수집 건 연결

**활성화 방법**

```bash
# .env.local
ENABLE_AI_DUPLICATE_CHECK=true
DUPLICATE_CHECK_AI_CONFIDENCE=80
DUPLICATE_CHECK_WINDOW_HOURS=1
```

### 2. 긴급 이슈 타임아웃 자동 승인

**배경**
- 연예, 정치 카테고리는 화력이 높아도 무조건 수동 승인 대기
- 관리자 부재 시 핫한 이슈가 수 시간씩 미노출됨

**구현**
- `app/api/cron/auto-create-issue/route.ts`에 `autoApproveUrgentIssues()` 추가
- `evaluateCandidates()` 호출 전 실행

**조건**
- `approval_status = '대기'`
- `is_urgent = true`
- `created_at < N시간 전` (환경변수 `URGENT_AUTO_APPROVE_HOURS`, 기본 2시간)

**처리**
- `approval_status = '승인'`
- `approval_type = 'auto'`
- `approved_at = now()`

**로그 예시**

```
[긴급 자동승인] 윤석열 대통령 긴급 기자회견 
(카테고리: 정치, 화력: 45점, 대기시간: 2시간+)
```

**활성화 방법**

```bash
# .env.local
URGENT_AUTO_APPROVE_HOURS=2
```

## 성능 영향

### AI 중복 체크
- 호출 빈도: 이슈 후보 생성 시 1회
- 비용: Groq 무료 플랜 (월 500,000 토큰)
- 토큰 사용: 후보당 약 200토큰
- 예상: 하루 4개 후보 × 200토큰 = 800토큰 (무료 플랜의 0.16%)

- 호출 빈도: 30분마다 (크론 실행 시)
- DB 쿼리: SELECT + UPDATE (대상 건수만큼)
- 평균 대상: 1~2건/일
- 성능 영향: 무시 가능


2. 수동 확인
    - 두 이슈가 실제로 중복인지 확인
    - 잘못 판정된 경우 AI 신뢰도 임계값 조정

3. 설정 조정
    ```bash
    # 중복 판정 더 엄격하게
    DUPLICATE_CHECK_AI_CONFIDENCE=90
    
    # 비교 대상 시간 확대
    DUPLICATE_CHECK_WINDOW_HOURS=2
    ```
