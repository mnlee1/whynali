# 서비스 노출 법적 검토

본 문서는 왜난리 서비스에서 수집한 데이터를 관리자 화면 및 최종 사용자 화면에 노출할 때의 법적 이슈를 검토한다.

---

## 1. 핵심 결론

### 1.1 기본 원칙

- **수집 범위**: 제목, 링크, 조회수, 댓글수, 작성시간만 수집 (본문 수집 안 함)
- **관리자 화면**: 합법적으로 수집한 데이터를 내부에서만 확인하는 것은 법적 문제 없음
- **서비스 노출**: 승인 후 사용자에게 노출되는 내용이 핵심 판단 기준

### 1.2 법적으로 안전한 노출 방식

**허용되는 것:**
- 이슈 주제명
- 출처 링크 (원문 URL)
- 제목 (링크와 함께 표시)
- 메타데이터 기반 분석 (화력 지수, 급상승 등)
- 사용자 생성 콘텐츠 (댓글, 투표)

**금지되는 것:**
- 게시글/기사 본문
- 본문 기반 AI 요약문
- 본문 재가공 텍스트

---

## 2. 관리자 화면 노출

### 2.1 법적 판단

**관리자 화면에서 수집 데이터를 확인하는 것:**
- 합법적으로 수집한 데이터를 내부 업무용으로 열람하는 것은 **공중송신권·배포권 침해가 아님**
- 수집 자체가 합법이면 관리자 화면 표시는 추가 법적 이슈 없음

### 2.2 주의사항

| 구분 | 문서 설계 | 법적 판단 |
|------|-----------|----------|
| 더쿠·네이트판 메타데이터 | ✅ 수집 | 문제 없음 |
| 클리앙 데이터 | ❌ 수집 제외 | 약관 위반 (제13조) |
| 본문 텍스트 | ❌ 수집 안 함 | 수집 시 저작권 침해 |

**관리자 화면이 스크린샷과 다른 점:**
- 스크린샷에 클리앙 데이터가 보이는 경우 → 문서와 불일치, 약관 위반 가능
- 본문이 표시되는 경우 → 문서와 불일치, 저작권 침해 가능

**결론**: 관리자 화면은 문서 설계대로 **메타데이터만** 표시해야 법적 이슈 없음

---

## 3. 서비스(사용자) 화면 노출

### 3.1 승인 후 노출 흐름

```
관리자 화면에서 확인
    ↓
관리자 승인
    ↓
서비스(일반 사용자)에 노출
```

**핵심**: 무엇이 노출되는가에 따라 법적 판단이 달라짐

### 3.2 법적으로 안전한 노출 (문서 설계 기준)

**노출 가능한 것:**
- 이슈 주제/타이틀
- 출처 링크 (원문 URL)
- 제목 (링크와 함께)
- 화력 지수/급상승 등 메타데이터 기반 분석
- 투표·댓글 등 사용자 참여 콘텐츠

**법적 근거:**
- 대법원 2012다76829: 제목+링크 표시는 공정 이용
- 대법원 2008da55354: 사실 정보 수집·분석은 합법
- 화력 분석은 메타데이터 기반 우리 쪽 저작물

### 3.3 법적 리스크가 있는 노출

**금지되는 것:**
- 게시글/기사 본문
- 본문 기반 AI 요약문
- 본문 내용 재가공 텍스트

**법적 근거:**
- 저작권법 제16조: 복제권 침해
- 저작권법 제21조: 2차적 저작물 작성권 침해
- 저작권법 제18조·제19조: 배포권·공중송신권 침해
- 네이버 뉴스스탠드 판례(2019가합573789): "요약문도 저작물, 출처 링크만으로 면책 안 됨"

## 4. AI 요약문 법적 검토

### 4.1 본문 기반 AI 요약

**상황**: 커뮤니티 게시글/뉴스 기사 본문을 AI에 넣어 3줄 요약 생성

**법적 판단**: ❌ **위법 가능성 큼**

**침해되는 권리:**
- 복제권 (본문 수집·저장)
- 2차적 저작물 작성권 (AI 요약 생성)
- 배포권·공중송신권 (요약문 서비스 노출)

**판례**:
```
네이버 뉴스스탠드 사건 (2019가합573789)
"뉴스 요약문도 저작물, 무단 사용 시 침해"
"출처 링크 제공해도 소용없음"
```

### 4.2 메타데이터 기반 AI 설명

**상황**: 제목, 링크, 조회수, 댓글수만 AI에 넣어 이슈 설명 생성

**예시**:
```
"이슈: OOO 논란. 관련 뉴스 5건, 커뮤니티 게시 12건.
조회 1.2만, 댓글 340건. 반응 급상승 중."
```

**법적 판단**: 상대적으로 안전

**근거**:
- 조회수·댓글수·링크는 사실 정보 (저작권 보호 대상 아님)
- AI 출력이 사실 나열이면 우리 쪽 창작물
- 본문 이용 없으므로 복제권·2차적 저작물 침해 없음

**주의사항**:
- 제목을 AI 입력에 사용하는 것은 **이론적 그레이존**
- 제목에 창작성이 있으면 저작물로 볼 가능성
- 하지만 AI 출력이 사실 서술형이면 실무적 리스크는 낮음

### 4.3 네이버 API 요약문 재가공

**상황**: 네이버 API로 받은 요약문을 AI로 다시 3줄 요약

**법적 판단**: ❌ **리스크 있음**

**근거**:
- 네이버 API 이용약관 제5조 3호: "검색 결과를 조작하지 않을 것"
- API 요약문 자체가 저작물이면 2차적 저작물 작성 문제

**안전한 방법**:
- 네이버 API 데이터는 "그대로" 표시 (추가 가공 없음)
- 우리 쪽 AI 요약은 메타데이터만 입력으로 사용

### 4.3.1 요약문(description)을 수집·저장하지 않는 경우

**방법**: API 응답에서 **요약문(description) 필드는 저장·노출하지 않고**, 제목(title), 링크(originallink), 출처, 날짜 등만 수집·저장.

**법적·저작권 판단**: 이슈 없음. 요약문 텍스트를 보관·표시·AI 입력에 쓰지 않으므로 해당 텍스트에 대한 복제권·2차적 저작물·네이버 약관("검색 결과 조작") 논의가 **원천적으로 발생하지 않음**. 수집 범위를 "제목, 링크, 출처, 날짜"로 한정하면 저작권·약관 리스크는 최소화됨.

**트레이드오프**: 타임라인·상세에서 "네이버가 제공한 한 줄 요약문"은 노출할 수 없음. 제목+링크+출처+날짜만 표시. 상세 페이지 AI 요약은 메타데이터(건수, 반응 등)만으로 생성(4.4)하면 됨.

**결론**: 요약문을 빼고 수집(저장)하면 요약문에 관한 법적·저작권 문제는 없음. 구현 시 API 응답은 받되 description 필드는 DB에 넣지 않고, 화면에도 사용하지 않으면 됨.

### 4.4 상세 페이지 AI 요약 (수집 데이터 활용)

**상황**: 이슈 상세 페이지에 "이 이슈를 한 줄로 요약한 문장"을 노출. 입력으로 쓰는 데이터는 (1) 외부 커뮤니티(더쿠·네이트판) 수집 데이터, (2) 네이버 뉴스 API로 수집한 데이터.

**수집 범위 (01·03 기준)**:
- **외부 커뮤니티**: 제목, URL, 조회수, 댓글수, 작성시간만(메타데이터). 본문 수집 안 함.
- **네이버 뉴스 API**: API는 제목·링크·요약문·출처·날짜를 제공하나, 01·4.3.1에 따라 **요약문은 수집·저장·표시하지 않고** 제목·링크·출처·날짜만 수집·사용함.

**법적·저작권 판단**:

| AI 입력으로 쓰는 것 | 법적 판단 | 비고 |
|--------------------|----------|------|
| **메타데이터만** (뉴스 건수, 커뮤니티 건수, 조회수 합·댓글수, 반응 속도, 이슈명 등) | 이슈 없음 | 4.2와 동일. 사실 정보만 사용, 본문 미이용. 출력 예: "OO 논란. 관련 뉴스 5건, 커뮤니티 12건, 반응 급상승." |
| **네이버 API 요약문**을 AI에 넣어 재요약·재구성 | 이슈 있음 | 4.3과 동일. 약관 "검색 결과 조작 금지", 2차적 저작물 리스크. |
| **커뮤니티/뉴스 본문** | 이슈 있음 | 4.1과 동일. 현재 기획에서도 본문 수집 안 함. |
| **제목**을 AI 입력에 포함 | 그레이존 | 7.1과 동일. 출력이 사실 나열형이면 실무적 리스크 낮음. |

**결론**: 상세 페이지 AI 요약은 **메타데이터만**(건수, 조회수, 댓글수, 반응 정도, 이슈명 등)을 입력으로 사용해 **사실 서술형 한 줄**로 만들면 법적·저작권 이슈 없음. 네이버 API **요약문**을 AI에 넣어 다시 요약하거나, 커뮤니티/뉴스 **본문**을 사용하면 저작권·약관 리스크 있음.

## 5. 실제 서비스 화면 검토

### 5.1 법적으로 안전한 구성 요소

| 항목 | 내용 | 법적 근거 |
|------|------|----------|
| 관련 출처 | 제목+링크 나열 | 대법원 2012다76829 |
| 투표·댓글 | 사용자 생성 콘텐츠 | 플랫폼 자체 콘텐츠 |
| 분류·태그 | 카테고리 라벨 | 메타데이터 |
| 출처 수 | "뉴스 5건, 게시 12건" | 사실 정보 |

### 5.2 리스크가 있는 구성 요소

| 항목 | 리스크 조건 | 법적 문제 |
|------|------------|----------|
| AI요약 | 본문 기반 생성 시 | 2차적 저작물 작성권 침해 |
| 핵심 정보 | 본문에서 추출한 요약 | 복제권·2차적 저작물 침해 |
| 주장·쟁점 | 본문 분석 결과 | 복제권·2차적 저작물 침해 |

**안전하게 만들려면**:
- AI요약: "이슈명 + 출처 수 + 반응 정도" 같은 **메타데이터 기반 사실 서술**
- 핵심 정보: 본문 아닌 **제목·통계만** 참조한 **사실 정리**
- 주장·쟁점: **출처 링크만** 제공, 본문 내용 요약·추출 금지

## 6. 수집 범위 확인

### 6.1 문서상 수집 범위

**수집하는 것:**
- 제목
- URL (링크)
- 조회수
- 댓글수
- 좋아요 수
- 작성시간

**수집하지 않는 것:**
- 본문
- 닉네임
- 기타 창작 콘텐츠

### 6.2 법적 판단

**메타데이터만 수집:**
- 저작권: 본문 미수집으로 복제권 침해 없음
- DB권: 사실 정보 수집, 판례상 합법
- 약관: 더쿠·네이트판 명시적 금지 조항 없음

**결론**: 수집 단계는 법적 문제 없음

## 7. 제목 사용 그레이존

### 7.1 제목 표시 vs 제목 이용

**제목을 링크와 함께 표시:**
- 대법원 2012da76829: 공정 이용, 저작권 침해 아님
- 우리 서비스: 출처 목록에 제목+링크 → **법적 문제 없음**

**제목을 AI 입력으로 사용:**
- 제목에 창작성 있으면 저작물 가능
- AI가 제목을 원료로 새 문장 생성 → **이론적으로 2차적 저작물 논의 가능**
- 하지만 실무적으로:
    - 제목이 짧고 사실적이면 창작성 인정 어려움
    - AI 출력이 사실 나열형이면 제목 표현을 복제한 것 아님
    - **완전히 위법이라 단정할 근거 없음**, 그레이존

### 7.2 권장 사항

**가장 안전한 방법:**
- 제목은 "표시용"으로만 사용 (링크와 함께 나열)
- AI 입력에는 **수치 데이터만** 사용 (조회수, 댓글수, 출처 개수 등)
- AI 출력: "이슈명 + N건 수집 + 반응 급증" 같은 사실 서술

**현실적 타협안:**
- 제목을 AI 입력에 넣되, 출력이 **제목 내용을 요약·재현하지 않고** 단순 **사실 나열**이면 실무적 리스크 낮음
- 본문만 절대 넣지 않으면 큰 문제 발생 가능성 낮음

## 8. 커뮤니티 기능(서비스 내 UGC) 법적 검토 — 왜난리 기획·정책 기준

**UGC**: User Generated Content(사용자 생성 콘텐츠). 서비스 이용자가 직접 작성·등록한 콘텐츠.

**왜난리 서비스에서 UGC 범위 (02 가정 6.4·6.6 기준):**
- **이슈 상세 댓글**: 이슈별 사용자 댓글. 베스트(score 상위)·리스트(최신순/좋아요순/싫어요순). 저장 시 세이프티봇 적용.
- **커뮤니티(토론) 댓글**: 토론 주제 상세 화면에서만 작성 가능. 토론 주제는 **관리자 승인된 이슈**에 한해 AI가 **메타데이터만**(이슈 제목·상태·카테고리·화력 등)으로 생성(본문·본문 요약 미사용) → 관리자 승인 후 등록(02 6.4). 토론 댓글도 저장 시 세이프티봇 적용.
- **수집 데이터**: 외부 커뮤니티(더쿠·네이트판)는 03 문서대로 **메타데이터만** 수집. 본문 수집·AI 요약 없음. 본 절은 **우리 플랫폼에 사용자가 올리는 댓글·토론 댓글**에 대한 법적 검토.

### 8.1 정보통신망법상 의무 — 왜난리 정책과의 연결

- **세이프티봇(02 6.6)**: 댓글·토론 댓글 저장 시 금칙어 목록 매칭(단어 경계, 목록 최소화). 매칭 시 자동 비공개·검토 대기. 관리자 검토는 일 1회 또는 검토 대기 10건 이상 시 배치로 공개/삭제(02 6.5). 이는 **불법정보가 그대로 노출되는 것**을 막는 1차선이며, 정보통신망법 제44조의2·제44조의7 방향과 부합함.
- **신고 절차는 별도 필수**: 세이프티봇은 **저장 시점**에 금칙어에 걸린 글만 걸러냄. **권리자·이용자 신고**(저작권 침해, 명예훼손 등)는 이미 노출된 글에 대한 사후 대응이므로, 동법 제44조의5 등에 따른 **신고 접수 방법(이메일/전용 URL)·처리 기한·삭제 기준**을 오픈 전 수립해야 함. 01_AI기획 170행 "통신비밀보호법·정보통신망법상 불법정보·권리침해 대응(신고 절차, 삭제 기준 등)은 서비스 오픈 전 운영 단계에서 수립"과 동일.
- **삭제 기준**: 02 6.5 관리자 운영 원칙(배치 검토, 공개/삭제 결정)과 맞추어, "욕설·비방·혐오·저작권 침해·불법정보·권리자 신고에 의한 삭제" 등을 문서화하고, 신고 접수 시 해당 글 비공개·삭제 후 검토하도록 연동.

### 8.2 저작권 — 이슈·토론 댓글 / AI 토론 주제 구분

- **이슈 댓글·토론 댓글**: 이용자 글이 타인 저작물을 무단 복제한 경우 작성자 1차 책임. 플랫폼은 **저작권법 제102조**(온라인서비스제공자 책임 제한)에 따라, **권리자 신고 접수 후 해당 글 삭제·접근차단** 절차를 두면 손해배상 책임 감면에 유리. 신고 절차(8.1)와 동일 창구로 처리 가능.
- **토론 주제(AI 생성)**: 02 가정 6.4에 따라 **이슈 제목·상태·카테고리·화력 등 메타데이터만** AI 입력, 본문·본문 요약 미사용. 따라서 04 섹션 4.2(메타데이터 기반 AI 설명)와 동일하게 **2차적 저작물·복제권 침해 리스크 없음**. 토론 주제 문장은 우리가 생성한 문장으로만 노출하므로, 외부 수집 본문과 무관. **기획 이탈 시에만 리스크**: 토론 주제 생성에 본문 또는 제목을 요약·재현하는 형태로 쓰면 04 섹션 4.1·4.3과 동일한 위법 논의 가능.

### 8.3 명예훼손·모욕 — 서비스 성격 반영

- 왜난리는 **논쟁 이슈**(연예·정치·사회 등)와 **철학적 관점 토론**(02 6.4)을 다루므로, 특정인 비방·모욕 댓글이 올 가능성이 있음. 민·형사상 책임은 작성자에게 있으나, **피해자 신고 시 해당 글 삭제·비공개** 절차를 두지 않으면 플랫폼도 정보통신망법상 의무 미이행·책임 논의 가능.
- **이용약관**: 게시 시 **금지 행위**(욕설·비방·명예훼손·혐오·저작권 침해·불법정보 유포·개인정보 무단 게시 등)를 명시하고, 위반 시 사전 통보 없이 삭제·비공개·이용 제한할 수 있음을 적어 두는 것이 필요. 02 6.6과 맞춰 "검토 후 공개·삭제" 가능하다는 사전 안내와 일치시키면 됨.

### 8.4 약관·운영정책 — 02·01과 맞출 사항

| 항목 | 왜난리 기획·정책과의 연결 |
|------|---------------------------|
| 게시물 저작권·사용 허락 | 작성자 귀속 + 서비스 운영·노출을 위한 비독점적 사용 허락. 이슈 댓글·토론 댓글 동일 적용. |
| 금지 행위 | 욕설·비방·명예훼손·저작권 침해·음란·혐오·스팸·개인정보 무단 게시. 02 6.6 금칙어 목록·오탐 방지 정책과 방향 일치. |
| 삭제·제재 | 위반 시 사전 통보 없이 삭제·비공개·이용 제한. 02 6.5 관리자 공개/삭제 결정과 동일. |
| 권리자 신고 | 저작권·명예훼손 등 신고 방법(이메일/URL)·처리 절차 안내. 8.1 신고 절차와 동일. |
| 사전 안내 | 02 6.6 결정 문구와 일치: "등록되었습니다. 내용 검토 후 공개되거나 삭제될 수 있습니다." |

### 8.5 세이프티봇·신고 절차 역할 구분

- **세이프티봇(02 6.6)**: 저장 시 금칙어 매칭 → 비공개/검토 대기 → 관리자 배치 검토(일 1회 또는 10건 이상) 후 공개/삭제. **선제적** 불법정보 노출 차단.
- **신고 절차(8.1)**: 권리자·이용자 신고 접수 → 해당 글 비공개·삭제 후 검토. **사후** 대응. 세이프티봇에 안 걸린 글(예: 금칙어 목록에 없는 비방·저작권 침해)에 필수.
- 오픈 전: **신고 접수 창구**, **삭제 기준** 문서화, **이용약관**에 게시물·금지 행위·삭제 권한·권리자 신고 안내 반영. 01 170·02 6.5와 연동.

### 8.6 요약: 왜난리 설계 기준으로 문제 될 부분·필수 조치

| 구분 | 왜난리 설계 | 법적 이슈 | 필수 조치 |
|------|-------------|----------|----------|
| UGC 범위 | 이슈 댓글, 토론 댓글만. 토론 주제는 AI 생성·관리자 승인(02 6.4) | 정보통신망법상 신고·삭제 의무 | 신고 접수·처리 절차·삭제 기준 오픈 전 수립(01 170). 세이프티봇(02 6.6)은 유지 |
| 저작권 | 댓글만 UGC. 토론 주제는 메타데이터만 AI 입력(02 6.4) | 권리자 신고 시 미삭제 시 플랫폼 책임 가능 | 신고 접수 후 해당 글 삭제·접근차단. 토론 주제는 본문/제목 요약 사용 금지 유지 |
| 명예훼손 | 논쟁 이슈·철학적 토론 특성상 비방 댓글 가능 | 피해자 신고 시 미대응 시 책임 논의 | 이용약관 금지 조항 + 신고 시 삭제·비공개 절차 |
| AI 토론 주제 | 메타데이터만 입력, 본문 미사용(02 6.4) | 본문 사용 시 2차적 저작물 리스크 | 현재 기획 유지. AI 입력에 본문·제목 요약 추가 시 04 4.1·4.3 검토 필요 |
| 약관 | 02 6.6 검토 대기 문구 확정됨 | 금지 행위·삭제 권한 불명확 시 분쟁 시 불리 | 오픈 전 이용약관에 게시물·신고 조항 반영, 02 6.6 문구와 일치 |

**결론**: 왜난리 기획·02·01 정책을 지키는 전제에서 (1) **세이프티봇**은 선제 차단, (2) **권리자·이용자 신고 절차**는 사후 대응으로 반드시 별도 마련, (3) **토론 주제**는 메타데이터만 AI 입력 유지, (4) **이용약관**에 게시물·금지 행위·삭제 권한·권리자 신고 안내를 02 6.6 문구와 맞춰 반영하면, 커뮤니티 기능에서 법적으로 문제 될 부분을 기획과 정책에 맞게 최소화할 수 있음.

## 9. 체크리스트

### 9.1 수집 단계

- [ ] 더쿠·네이트판만 수집 (클리앙 등 금지 사이트 제외)

- [ ] 닉네임 수집 안 함

### 9.2 관리자 화면

- [ ] 수집된 메타데이터만 표시
- [ ] 본문 표시 안 함
- [ ] 금지 사이트 데이터 표시 안 함

### 9.3 서비스(사용자) 화면

- [ ] 핵심 정보: 본문 요약·추출 안 함
- [ ] 본문 텍스트 노출 안 함
- [ ] 네이버 API 데이터: 그대로 표시 (재가공 안 함)

### 9.4 커뮤니티(UGC) 운영 (왜난리 기획·02 기준)

- [ ] 토론 주제 AI 생성 시 이슈 메타데이터만 입력, 본문·본문 요약 미사용(02 6.4)

## 10. 최종 권고사항

### 10.1 법적으로 완전히 안전한 설계

```
수집: 제목, URL, 조회수, 댓글수, 작성시간만
    ↓
관리자 화면: 메타데이터 확인
    ↓
AI 처리: 수치 데이터만 입력 → 사실 서술 생성
    ↓
서비스 노출:
- 이슈명 + "관련 뉴스 N건, 커뮤니티 M건"
- 출처 링크 (제목+URL)
- 화력 지수 (메타데이터 기반)
- 투표·댓글 (사용자 생성)
```

### 10.2 피해야 할 것

- 본문 수집·저장·표시
- 본문 기반 AI 요약·재가공
- 네이버 API 요약문 재가공
- 금지 사이트(클리앙 등) 데이터 수집
- 출처 표시 없이 콘텐츠 노출

### 10.3 진실만 말하기

**질문: "제목·링크·댓글수·좋아요·조회수만 수집하면 AI 3줄 요약 괜찮은가?"**

**답변**:
- **수집 단계**: 메타데이터만 수집하면 법적 문제 없음 ✅
- **AI 요약**: 
    - 수치만 입력 → 문제 없음 ✅
    - 제목도 입력 → 이론적 그레이존 ⚠️ (하지만 실무적 리스크는 낮음)
    - 본문 입력 → 위법 가능성 큼 ❌
- **서비스 노출**: 본문 기반이 아니면 문서 설계와 일치, 실무적으로 안전

**결론**: 본문만 절대 쓰지 않으면 현재 설계는 법적으로 방어 가능한 수준
