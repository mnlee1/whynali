# 늑구 이슈 오분류 분석 및 해결방안 검토

## 원인 요약

'늑구 탈출 후 생포' 이슈가 "연예" 카테고리로 오분류된 이유:

1. AI가 "늑구" 키워드 분석 시 "국민 늑대", "스타" 같은 표현을 연예인 수식어로 오인
2. searchNaverNewsByKeyword()가 AI 판단 카테고리를 모든 뉴스에 일괄 적용
3. 12개 뉴스 중 6개가 "연예"로 저장되어 이슈도 "연예"로 생성됨

## 제안한 해결방안별 문제점

### 1. AI 프롬프트에 명시적 규칙 추가

**제안:**
```
- 동물원 탈출, 야생동물 관련 → 무조건 "사회"
- "국민 OO", "스타 OO"는 비유적 표현
- 동물 이름 → "연예" 절대 금지
```

**문제점:**

1. **과도한 규칙화로 유연성 상실**
   - "호랑이 선수" (야구 별명) → 스포츠인데 사회로 오분류 가능
   - "탈출" 키워드: "탈출 성공" (예능 프로그램명)도 존재
   - 규칙이 많아질수록 다른 케이스 오분류 위험 증가

2. **프롬프트 길이 증가**
   - 이미 797-805라인에 우선순위 규칙 존재
   - 추가 규칙으로 프롬프트가 너무 길어져 AI 성능 저하 가능

3. **근본 문제 미해결**
   - AI가 맥락을 제대로 이해하지 못하는 게 원인
   - 규칙 추가는 임시방편일 뿐

### 2. 사후 검증 로직 추가

**제안:**
```typescript
const animalKeywords = ['동물원', '탈출', '포획', '생포', '늑대', ...]
if (animalKeywords.some(k => keyword.includes(k)) && category === '연예') {
    category = '사회'
}
```

**심각한 문제점:**

1. **정상 케이스 오분류 위험**
   - "동물농장" (SBS 예능) 관련 이슈 → 연예가 맞는데 사회로 강제 변경
   - 실제로는 드물지만, 시스템 신뢰도 하락

2. **키워드 단순 매칭의 한계**
   - "탈출": 예능 프로그램 "탈출 성공"과 실제 동물 탈출 구분 불가
   - "포획": "관심 포획" 같은 비유적 표현도 있음

3. **유지보수 부담**
   - 예외 케이스 발견할 때마다 키워드 리스트 수정 필요
   - 하드코딩된 규칙은 장기적으로 버그의 원인

### 3. 카테고리 키워드에서 "스타", "셀럽" 제거

**제안:**
```typescript
// lib/config/categories.ts
keywords: [
    '배우', '가수', '아이돌', ... 
    // '스타', '셀럽' 제거
]
```

**심각한 문제점:**

1. **정상 연예 이슈 놓침**
   - "아이돌 스타 열애설" → 연예가 맞는데 카테고리 매칭 실패
   - "셀럽 결혼 소식" → 연예 키워드 부족으로 오분류

2. **근본 원인과 무관**
   - 이 키워드들은 현재 시스템에서 영향이 미미
   - AI 판단 오류가 주원인이므로 키워드 수정은 효과 없음

3. **부작용**
   - 키워드 기반 분류 정확도 하락
   - 다른 이슈들까지 영향받을 가능성

## 더 근본적인 문제점

### 현재 시스템의 구조적 결함

1. **AI 판단을 모든 뉴스에 일괄 적용**
   ```typescript
   // lib/collectors/naver-news.ts:160
   category: category,  // AI가 판단한 카테고리를 모든 뉴스에 강제 적용
   ```
   - 같은 사건도 언론사마다 다른 섹션에 배치함
   - 네이버 뉴스 원본 카테고리 무시하고 덮어씀

2. **카테고리 결정 시점 문제**
   - 커뮤니티 키워드만 보고 판단 (뉴스 내용 확인 전)
   - 실제 뉴스 수집 후에도 재검증 없음

3. **다수결 로직 부재**
   - 뉴스 12개 중 6개 사회, 6개 연예인데 이슈는 연예로 결정
   - AI 초기 판단만 신뢰, 실제 뉴스 분포 무시

## 올바른 해결 방안

### 방안 A: 네이버 뉴스 원본 카테고리 활용 (권장)

```typescript
// 네이버 뉴스 API 응답에 원본 카테고리 포함 여부 확인 필요
// 만약 있다면:
const newsCategory = extractNaverCategory(item) || category
```

**장점:**
- 네이버가 이미 분류한 정확한 카테고리 활용
- AI 오판 영향 최소화

**단점:**
- 네이버 API가 카테고리 제공하지 않을 수 있음

### 방안 B: 뉴스 수집 후 다수결 재분류

```typescript
// 뉴스 수집 완료 후
const categoryCounts = newsItems.reduce((acc, news) => {
    acc[news.category] = (acc[news.category] || 0) + 1
    return acc
}, {})

// 가장 많은 카테고리 선택
const finalCategory = Object.entries(categoryCounts)
    .sort((a, b) => b[1] - a[1])[0][0]

// 이슈 생성 시 finalCategory 사용
```

**장점:**
- 실제 뉴스 분포 반영
- AI 초기 오판을 뉴스 데이터로 보정

**단점:**
- 로직 복잡도 증가
- 여전히 뉴스 카테고리가 잘못되어 있으면 소용없음

### 방안 C: 각 뉴스를 개별적으로 AI 재분류

```typescript
// 뉴스 수집 후 각 뉴스 제목/내용으로 개별 분류
for (const news of newsItems) {
    const newsCategory = await classifyNewsByAI(news.title)
    news.category = newsCategory
}

// 다수결로 이슈 카테고리 결정
```

**장점:**
- 가장 정확한 분류 가능
- 뉴스별 맥락 정확히 파악

**단점:**
- API 비용 폭증 (뉴스마다 AI 호출)
- 처리 시간 증가

### 방안 D: 최소 개입 - AI 프롬프트 최적화만 (추천)

현재 프롬프트 797-805 라인 수정:

```
⚠️ 카테고리 판단 우선순위:
1. 동물 관련 사건사고 (동물원, 야생동물, 포획, 탈출) → "사회"
2. 스포츠 관련 (경기, 선수) → "스포츠"
3. 기업 비즈니스 관련 → "경제"
4. 실제 연예인/드라마/영화 → "연예"
   (주의: "국민 OO", "스타급" 등은 비유적 표현일 수 있음)
5. 사건사고/법원/교육/생활/문화 → "사회"
...
```

**장점:**
- 최소한의 수정으로 문제 해결
- 기존 시스템 구조 변경 없음
- 부작용 최소화

**단점:**
- 완벽한 해결은 아님
- 향후 유사 케이스 재발 가능
