# 이슈 후보 알림 기준 미달 분석

작성일: 2026-03-13
상태: 분석 완료, UI 개선 적용

## 문제 상황

**캡처 이미지:**
- 제목: "고려대학, '인터배터리 2026'서 이차전지 소재 전방물질 기술력 선보인다"
- 표시: "최근 3시간 뉴스 3건 (4건 → 1건 + 커뮤니티 0)"

**문제점:**
- `ALERT_THRESHOLD = 5` (최소 5건 필요)
- 실제 건수: 3-4건
- **기준 미달인데 알림창에 노출됨**

---

## 초기 분석

### 원인 시나리오

#### 시나리오 A: 중복 그루핑 후 일부 제거

**가능성:**
1. 초기 그루핑: Group A (3건) + Group B (2건) = 5건
2. 739번 라인 체크: `recentCount = 5` → 통과
3. AI 중복 체크: Group B의 2건이 제거됨
4. 최종: `newsIds.length = 3`이지만 `recentCount = 5`로 유지

**결론:** 병합이지 제거가 아니므로 건수가 줄어들지 않음

#### 시나리오 B: 뉴스 건수와 커뮤니티 건수의 혼동

```typescript
// 737번 라인
const recentCount = group.items.length

// group.items에는 뉴스와 커뮤니티가 모두 포함될 수 있음
```

**검증 결과:** `group.items`에는 뉴스만 포함되므로 문제 없음

#### 시나리오 C: 이미 등록된 이슈에 추가 수집 (정상)

기존 이슈에 새로운 수집 건 연결하는 경우, `recentCount`와 `newsIds.length`가 일치하므로 문제 없음.

#### 시나리오 D: UI 표시 로직 문제 (가능성 높음)

알림창에 표시되는 "뉴스 3건 (4건 → 1건 + 커뮤니티 0)"의 의미가 불명확.

## 코드 분석 결과

### 이슈 후보 알림 생성 흐름

파일: `lib/candidate/issue-candidate.ts`

```typescript
// 1단계: 뉴스만 그루핑
const newsRawItems = (newsItems ?? []).map(n => ({ ...n, type: 'news' }))
groups = groupItems(newsRawItems)

// 2단계: 그룹별 처리
for (const [groupIndex, group] of groups.entries()) {
    const recentCount = group.items.length  // 737번 라인
    
    // 기준 체크 (739번 라인)
    if (recentCount < ALERT_THRESHOLD) continue  // 5건 미만 필터링
    
    const newsIds = group.items.map(i => i.id)  // 803번 라인
    
    // 알림 추가
    result.alerts.push({
        title: representativeTitle,
        count: recentCount,              // group.items.length
        newsCount: newsIds.length,       // group.items.length와 동일
        communityCount: communityIds.length,
    })
}
```

### 검증 결과

**`recentCount`와 `newsIds.length`는 항상 동일:**
- `recentCount = group.items.length`
- `newsIds = group.items.map(i => i.id)`
- `newsIds.length === group.items.length === recentCount`

**739번 라인에서 5건 미만은 모두 필터링:**
```typescript
if (recentCount < ALERT_THRESHOLD) continue
```

**결론:** 현재 코드로는 3-4건의 이슈가 알림에 표시될 수 없음

## 가능한 원인

### 원인 A: 다른 버전의 코드

캡처 이미지의 UI 텍스트 형식이 현재 코드와 다름:

**현재 코드:**
```typescript
최근 3시간 {alert.count}건 (뉴스 {alert.newsCount} + 커뮤니티 {alert.communityCount})
```

**캡처 이미지:**
```
뉴스 3건 (4건 → 1건 + 커뮤니티 0)
```

→ **다른 버전의 UI일 가능성**

### 원인 B: 데이터 캐싱 문제

파일: `app/api/admin/candidates/route.ts`

```typescript
const cached = getCachedCandidates()
if (cached) {
    return NextResponse.json({
        alerts: cached.alerts,
        evaluated: cached.evaluated,
        cached: true,
    })
}
```

→ 이전 버전의 데이터가 캐시되어 표시될 가능성

### 원인 C: Race Condition

코드 배포 중 또는 캐시 갱신 중에 발생한 일시적 오류

### 원인 D: 수동 테스트 데이터

개발자가 테스트용으로 수동으로 `alerts` 배열에 추가한 데이터
