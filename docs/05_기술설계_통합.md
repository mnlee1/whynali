## 2. 숏폼 설계 가이드

### 2.1 개요
왜난리 서비스의 확장 단계로 숏폼 콘텐츠 생성 기능 추가

### 2.2 핵심 방향
- MVP 완성 후 진행 (확장 단계)
- 이슈 → 숏폼 자동 생성 파이프라인
- 법적 리스크 최소화 (메타데이터 기반)

### 2.3 제작 파이프라인

```
1. 이슈 선별
   - 화력 50점 이상
   - 상태: 논란중
   - 출처 10건 이상
   ↓
2. 스크립트 생성 (AI)
   - 입력: 메타데이터만 (제목, 통계)
   - 출력: 30초 분량 대본
   ↓
3. 음성 합성 (TTS)
   - VITS, Google TTS
   ↓
4. 영상 편집
   - 템플릿 기반 자동 편집
   ↓
5. 자막 삽입
   ↓
6. 배포
   - 유튜브 Shorts
   - 인스타그램 Reels
```

### 2.4 법적 주의사항
- 본문 사용 금지
- 메타데이터만 활용
- 출처 명시 필수
- 링크 제공 (트래픽 환원)

## 3. 이슈 등록 화력 정렬 규격

### 3.1 이슈 등록 기준

**환경변수:**
```bash
CANDIDATE_ALERT_THRESHOLD=5           # 5건 이상 → 대기 등록
CANDIDATE_AUTO_APPROVE_THRESHOLD=10   # 10건 이상 → 자동 승인
CANDIDATE_WINDOW_HOURS=24             # 24시간 집계 창
CANDIDATE_MIN_HEAT_TO_REGISTER=30     # 화력 30점 이상만 등록
CANDIDATE_MIN_UNIQUE_SOURCES=2        # 출처 2곳 이상
CANDIDATE_COMMUNITY_MATCH_THRESHOLD=2 # 커뮤니티 키워드 2개 이상
```

### 3.2 화력 지수 계산

**공식:**
```
화력 = newsCredibility × (0.3 + 0.7 × communityAmp)

newsCredibility = Σ(뉴스 건수 × 출처 가중치)
communityAmp = 1 + (커뮤니티 반응 점수 / 100)
```

**뉴스 출처 가중치:**
- 주요 언론: 1.5 (연합뉴스, KBS, MBC, SBS, 조선일보, 중앙일보, 한겨레 등)
- 일반 언론: 1.0
- 기타: 0.5

**커뮤니티 반응 점수:**
```
반응 점수 = Σ(조회수 × 0.001 + 댓글수 × 0.1)
```

**화력 등급:**
- 🔥🔥🔥 강함 (70점 이상): 전국구 이슈
- 🔥🔥 보통 (40-69점): 지역/분야별 이슈
- 🔥 약함 (15-39점): 소규모 이슈
- 등록 안 됨 (15점 미만)

### 3.3 목록 정렬

**정렬 옵션:**
- `latest`: 등록 최신순 (created_at DESC)
- `heat`: 화력 높은 순 (heat_index DESC)

**기본 정렬:** `heat`

## 4. 이슈 상태 전환 규격

### 4.1 상태 정의

**점화 (ignite)**
- 초기 등록 상태
- 화력 40점 미만 또는 등록 6시간 미만

**논란중 (debate)**
- 활발한 논의 진행 중
- 화력 40점 이상 + 등록 6시간 이상

**종결 (closed)**
- 논의 종료
- 48시간 신규 데이터 없음 + 화력 10점 이하

### 4.2 자동 전환 조건

**점화 → 논란중:**
```
(등록 후 6시간 경과) AND (화력 40점 이상)
```

**논란중 → 종결:**
```
(48시간 신규 데이터 없음) AND (화력 10점 이하)
```

**종결 → 논란중 (재점화):**
```
급증 감지:
(10분간 신규 데이터 50건 이상)

OR

점진적 재점화:
(48시간 이내 신규 수집) AND (화력 40점 이상)
```

### 4.3 환경변수

```bash
STATUS_IGNITE_TO_DEBATE_HOURS=6       # 점화→논란중 시간
STATUS_IGNITE_MIN_HEAT=40             # 점화→논란중 화력
STATUS_CLOSED_IDLE_HOURS=48           # 종결 판단 시간
STATUS_CLOSED_MAX_HEAT=10             # 종결 판단 화력
STATUS_REIGNITE_RATE_PER_MINUTE=5     # 재점화 분당 건수
STATUS_REIGNITE_DURATION_MINUTES=10   # 재점화 감지 시간
```

## 5. 조건부 자동 승인

### 5.1 카테고리별 정책

| 카테고리 | 자동 승인 | 이유 |
|---------|---------|------|
| 사회 | 허용 | 객관적 사건/사고, 리스크 낮음 |
| 기술 | 허용 | 제품 출시, 업계 동향 등 사실 중심 |
| 스포츠 | 허용 | 경기 결과, 이적 등 명확한 사실 |
| 연예 | 금지 | 루머, 프라이버시 민감 |
| 정치 | 금지 | 정치적 편향 리스크 |

### 5.2 환경변수

```bash
AUTO_APPROVE_CATEGORIES=사회,기술,스포츠
```

## 6. 그루핑 및 매칭 개선

### 6.1 핵심 인물 리스트

**정치:** 윤석열, 이재명, 한동훈, 이준석, 추경호, 박찬대, 권영세
**연예:** 민희진, 뉴진스, 하이브, 어도어, 방시혁, 옥택연, 김연경
**스포츠:** 손흥민, 이강인, 황희찬, 김하성, 류현진, 오타니
**기업:** 삼성전자, LG전자, SK하이닉스, 네이버, 카카오

### 6.2 불용어 확장

**정당/정파:**
민주당, 공화당, 여당, 야당, 보수, 진보, 좌파, 우파, 당, 의원, 지지율, 여론조사

**국가/지역:**
한국, 미국, 중국, 일본, 북한, 러시아, 유럽, 아시아

**범용 동사/명사:**
발언, 언급, 주장, 입장, 반박, 비판, 반발, 있다, 없다, 한다, 된다, 이다, 같다

### 6.3 키워드 매칭 임계값 개선

**뉴스/커뮤니티 링커 임계값:**
- 기존: 키워드 60% 일치 + 최소 2개
- 개선: 키워드 70% 일치 + 최소 3개
- 인물명 키워드 포함 시: 무조건 최소 3개 이상 일치 필수

**인물명 키워드 목록:**
김연경, 손흥민, 이강인, 옥택연, 민희진, 뉴진스, 윤석열, 이재명, 한동훈, 황희찬, 김하성, 류현진

**개선 효과:**
- "김연경" 키워드 하나만으로 무관한 뉴스가 연결되던 문제 해결
- IOC GEDI 챔피언 선정 이슈에 배구 경기 관련 뉴스가 섞이는 현상 방지
- 인물명 키워드는 최소 3개 키워드 중 하나로만 기능하도록 제한

### 6.4 그루핑 로직

```
1단계: groupItems - 키워드 기반 초기 그루핑
조건: (키워드 2개 이상 && 같은 카테고리) OR (키워드 1개 이상 && 핵심 인물 공통)

2단계: mergeRelatedGroups - Jaccard 유사도 기반 병합
Jaccard 유사도 >= 0.4 OR 핵심 인물 공통 → 병합
```
