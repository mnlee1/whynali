# 왜난리 이슈 이미지 시스템 가이드

## 개요

Unsplash API를 활용하여 이슈마다 대표 이미지를 자동으로 생성하고 관리하는 시스템입니다.
관리자는 3개의 이미지 중 대표 이미지를 선택하거나 재검색할 수 있습니다.

## 시스템 아키텍처

### 데이터 구조

**issues 테이블**:
- `thumbnail_urls`: 이미지 URL 배열 (최대 3개)
- `primary_thumbnail_index`: 대표 이미지 인덱스 (0-2, 기본값 0)

**마이그레이션**: `supabase/migrations/add_primary_thumbnail_index.sql`

```sql
ALTER TABLE issues 
ADD COLUMN IF NOT EXISTS primary_thumbnail_index SMALLINT DEFAULT 0;
```

### 타입 정의

**파일**: `types/issue.ts`

```typescript
export interface Issue {
    thumbnail_urls?: string[] | null
    primary_thumbnail_index?: number | null
}
```

## 이미지 생성 프로세스

### 효율성 분석 (2026-04-13 기준)

**프로덕션 데이터**:
- 전체 이슈: 83개
- 승인: 70개 (84.3%)
- 반려: 0개 (0%)
- 승인율: 100%

**결론**: 승인율이 100%이므로 이슈 생성 시 자동으로 이미지 생성하는 것이 효율적입니다.

| 항목 | Before (승인 시 생성) | After (생성 시 자동) |
|------|---------------------|---------------------|
| 승인 속도 | 2-3초 대기 | 즉시 완료 (0.5초) |
| 미리보기 | 이미지 없음 | 이미지 확인 가능 |
| API 낭비 | 0회 | 0회 (승인율 100%) |
| 관리자 UX | 느림 | 빠름 |

### 1. 신규 이슈 생성 시 (자동)

**프로세스**:

```
트랙 A 크론 실행 (매 정각)
  ↓
/api/cron/track-a
  ↓
1. 커뮤니티 급증 감지
2. AI로 진짜 이슈인지 검증
3. 네이버 뉴스 검색 및 연결
4. 이슈 후보 등록 (approval_status='대기')
5. 이미지 자동 생성
   ├─ Groq API로 영문 키워드 추출
   ├─ Unsplash API로 이미지 검색
   ├─ 사람 얼굴 제외
   ├─ 중복 이미지 제외
   └─ 랜덤 3개 선택
6. thumbnail_urls 업데이트
7. primary_thumbnail_index = 0 설정
  ↓
관리자 미리보기 (이미지 미리 확인 가능)
  ↓
승인 버튼 클릭 → 즉시 완료
```

**코드 위치**: `app/api/cron/track-a/route.ts` (1262번째 줄 이후)

**장점**:
- 승인이 즉시 완료 (2-3초 → 0.5초)
- 미리보기에서 이미지 미리 확인 가능
- 승인율 100%이므로 API 낭비 없음

### 2. 관리자 승인 시 (백업)

이미지가 없는 경우에만 생성합니다.

**프로세스**:

```
관리자가 승인 버튼 클릭
  ↓
/api/admin/issues/[id]/approve
  ↓
1. approval_status = '승인' 업데이트
2. thumbnail_urls 확인
3. 이미지 없으면 → fetchUnsplashImages 실행
4. 이미지 있으면 → 건너뛰기
  ↓
즉시 승인 완료
```

**코드 위치**: `app/api/admin/issues/[id]/approve/route.ts` (49-60번째 줄)

### 3. 기존 이슈 일괄 적용 (수동)

**URL**: `GET /api/test/unsplash-batch?limit=45`

```bash
# 로컬 테스트
curl http://localhost:3000/api/test/unsplash-batch?limit=10

# 실서버 (조심!)
curl https://whynali.com/api/test/unsplash-batch?limit=45
```

**동작**:
- thumbnail_urls가 없거나 빈 배열인 이슈를 찾음
- 최신 이슈부터 limit개만큼 처리
- 각 이슈마다 3초 대기 (Rate limit 방지)
- 결과 JSON 반환

**주의사항**:
- Unsplash Demo 플랜: 50회/시간
- 한 번에 45개까지만 처리 권장
- maxDuration: 300초 (5분)

## 이미지 필터링 시스템

### 1단계: 검색 쿼리 최적화

```
원본: "삼성 갤럭시"
  ↓
최적화: "삼성 갤럭시 architecture abstract object concept 
         -person -people -face -portrait -human -man -woman -selfie -crowd"
```

**제외 키워드**:
- `-person`, `-people`, `-face`, `-portrait`
- `-human`, `-man`, `-woman`, `-selfie`, `-crowd`

**포함 키워드**:
- `architecture`, `abstract`, `object`, `concept`

### 2단계: 결과 후처리

1. Unsplash에서 30개 결과 가져오기
2. 설명(description/alt_description)에 사람 키워드 포함 시 제외
3. 이미 사용 중인 이미지 URL 제외
4. 랜덤하게 3개 선택

**사람 키워드**:
```typescript
const PERSON_KEYWORDS = [
    'person', 'people', 'man', 'woman', 'face', 'portrait',
    'selfie', 'human', 'girl', 'boy', 'child', 'baby',
    'crowd', 'group', 'team', 'family', 'couple'
]
```

### 3단계: 중복 방지

**개발 환경**:
- 로컬 DB 이미지 조회
- 프로덕션 DB 이미지 조회 (`.env.local`에 설정된 경우)
- 모든 사용 중인 URL 제외

**프로덕션 환경**:
- 프로덕션 DB 이미지만 조회
- 제외 후 검색

**환경 변수** (`.env.local`):
```env
# 프로덕션 DB 접근 (로컬 개발 시 중복 체크용)

SUPABASE_PRODUCTION_SERVICE_ROLE_KEY=eyJxxx...
```

## 관리자 기능

### UI 구성

**파일**: `components/admin/IssuePreviewDrawer.tsx`

**기능**:
1. 대표 이미지 선택 (라디오 버튼)
2. 이미지 재검색 버튼
3. 이미지 제거 버튼

**추가된 상태**:
- `selectedThumbnailIndex`: 현재 선택된 대표 이미지 인덱스
- `isRefreshing`: 재검색 중인지 여부

**추가된 Props**:
- `onIssueUpdate?: () => void`: 이미지 변경 시 부모 컴포넌트 새로고침

### 대표 이미지 선택

**동작**:
1. 3개 이미지 중 라디오 버튼으로 선택
2. 선택된 이미지에 "대표" 뱃지 표시
3. `primary_thumbnail_index` 즉시 업데이트
4. 슬라이드에 반영

**API 호출**: `POST /api/admin/issues/[id]/primary-thumbnail`

```json
{
    "index": 0 | 1 | 2
}
```

### 이미지 재검색

**동작**:
1. "이미지 재검색" 버튼 클릭
2. Groq API로 영문 키워드 추출
3. Unsplash API로 새로운 이미지 3개 검색
4. thumbnail_urls 업데이트
5. primary_thumbnail_index는 0으로 초기화

**API 호출**: `POST /api/admin/issues/[id]/refresh-thumbnails`

### 이미지 제거

**동작**:
1. "이미지 제거" 버튼 클릭
2. thumbnail_urls를 빈 배열로 설정
3. primary_thumbnail_index를 0으로 설정
4. 슬라이드에서 그라디언트 배경 표시

**API 호출**: `POST /api/admin/issues/[id]/remove-thumbnails`

### 프론트엔드 표시

**파일**: `components/issues/HotIssueHighlight.tsx`

```typescript
// primary_thumbnail_index에 지정된 이미지를 표시
const heroImage = issue.thumbnail_urls?.[issue.primary_thumbnail_index ?? 0] ?? null
```

**파일**: `app/admin/(protected)/issues/page.tsx`

```tsx
<IssuePreviewDrawer
    issue={previewIssue}
    onClose={() => setPreviewIssue(null)}
    onApprove={handleApprove}
    onReject={handleReject}
    onIssueUpdate={fetchIssues}
/>
```

## 문제 해결

### 이미지에 사람이 포함됨

**프로덕션 DB 확인**:
```bash
npx tsx scripts/check-production-images.ts --limit=20
```

**로컬 DB 확인**:
```bash
npx tsx scripts/check-images-with-people.ts
```

**자동 수정** (로컬만):
```bash
npx tsx scripts/check-images-with-people.ts --fix
```

**스크립트 동작**:
1. Supabase에서 thumbnail_urls가 있는 이슈 조회
2. Unsplash API로 이미지 상세 정보 가져오기
3. description/alt_description에서 사람 키워드 검색
4. `--fix` 옵션 사용 시 자동으로 이미지 재검색

### 같은 이미지가 중복 사용됨

중복 방지 시스템이 자동으로 작동합니다.

**개발 환경**:
- 로컬 + 프로덕션 이미지 모두 제외

**프로덕션 환경**:
- 프로덕션 이미지만 제외

**확인 방법**:
```bash
# 중복 이미지 검사
npm run check:images
```

**Demo 플랜** (50회/시간):
- 1시간 대기 후 재시도
- 또는 Production 플랜으로 업그레이드 (무료)

**Production 플랜** (5,000회/시간):
- Unsplash 대시보드에서 업그레이드
