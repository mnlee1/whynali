# 화력 15점 미만 이슈 정리

날짜: 2026-03-11
상태: ✅ 완료

## 문제 상황

관리자 이슈 목록에 화력 15점 미만 이슈가 다수 존재하는 문제 발견.

### 발견된 이슈

```
화력 15점 이하 이슈: 20개 (샘플)
├─ 등록 당시부터 15점 미만: 16개 ← 문제!
└─ 등록 시 15점 이상 → 현재 15점 미만: 4개 ← 시간 가중치 영향 (정상)
```

### 설정 확인

```bash
# .env.local
CANDIDATE_MIN_HEAT_TO_REGISTER=15  # 최소 등록 화력
```

화력 15점 미만 이슈는 등록되지 않아야 하는데, `created_heat_index`가 `null`이거나 15점 미만인 이슈가 81개나 존재.

## 원인 분석

### 1. `created_heat_index` 미설정

#### lib/candidate/issue-candidate.ts (라인 959-967)

```typescript
// 신규 이슈 등록 시
const { error: updateError } = await supabaseAdmin
    .from('issues')
    .update({
        approval_status: approvalStatus,
        approval_type: isAutoApproved ? 'auto' : null,
        approval_heat_index: isAutoApproved ? actualHeat : null,
        approved_at: isAutoApproved ? now.toISOString() : null,
        // ❌ created_heat_index가 없음!
    })
    .eq('id', tempIssue.id)
```

**문제**:
- `created_heat_index`는 등록 시 화력을 기록하는 필드
- 설정되지 않아서 `null` 상태로 남음
- 화력 필터링 로직은 작동하지만, 히스토리 기록이 누락됨

### 2. 기존 이슈 업데이트 시에도 미설정

```typescript
// 기존 대기 이슈 자동승인 시
.update({ 
    approval_status: '승인', 
    approval_type: 'auto',
    approval_heat_index: actualHeat,
    approved_at: now.toISOString() 
    // ❌ created_heat_index가 없음!
})
```

## 해결 방안

### 1. 신규 이슈 등록 시 `created_heat_index` 설정

```typescript
const { error: updateError } = await supabaseAdmin
    .from('issues')
    .update({
        approval_status: approvalStatus,
        approval_type: isAutoApproved ? 'auto' : null,
        approval_heat_index: isAutoApproved ? actualHeat : null,
        approved_at: isAutoApproved ? now.toISOString() : null,
        created_heat_index: actualHeat,  // ✅ 등록 시 화력 기록
    })
    .eq('id', tempIssue.id)
```

### 2. 기존 이슈 업데이트 시 백필

```typescript
if (existingIssue.approval_status === '대기') {
    if (shouldAutoApprove(issueCategory, actualHeat)) {
        const updateData: any = { 
            approval_status: '승인', 
            approval_type: 'auto',
            approval_heat_index: actualHeat,
            approved_at: now.toISOString()
        }
        
        // ✅ created_heat_index가 null이면 현재 화력으로 설정
        if (!existingIssue.created_heat_index) {
            updateData.created_heat_index = actualHeat
        }
        
        await supabaseAdmin
            .from('issues')
            .update(updateData)
            .eq('id', existingIssue.id)
    } else {
        // 대기 유지 시에도 백필
        if (!existingIssue.created_heat_index) {
            await supabaseAdmin
                .from('issues')
                .update({ created_heat_index: actualHeat })
                .eq('id', existingIssue.id)
        }
    }
}
```

### 3. 기존 데이터 정리

```javascript
// created_heat_index가 null인 이슈 처리
const issues = await supabase
    .from('issues')
    .select('id, title, heat_index, created_heat_index')
    .is('created_heat_index', null)
    .in('approval_status', ['대기', '승인', '반려']);

for (const issue of issues) {
    if (issue.heat_index < 15) {
        // 현재 화력이 15점 미만이면 삭제
        await supabase
            .from('issues')
            .delete()
            .eq('id', issue.id);
    } else {
        // 15점 이상이면 현재 화력을 created_heat_index로 설정
        await supabase
            .from('issues')
            .update({ created_heat_index: issue.heat_index })
            .eq('id', issue.id);
    }
}
```

## 실행 결과

### 백필 및 삭제

```
created_heat_index가 null인 이슈: 81개

처리 결과:
├─ 백필 완료 (15점 이상): 15개
└─ 삭제 (15점 미만): 66개
```

### 삭제된 이슈 예시

```
- 김윤지, 동계 패럴림픽 한국 여자선수 사상 첫 금메달: 2점
- 이재명 "차이가 차별 되지 않는 사회": 3점
- 한동훈 "국힘 '절윤'?": 2점
- 무신사, 스포츠·IP 전문 큐레이션관: 0점
- 초고령사회 노인 돌봄 인력 부족: 0점
...
```

### 검증 결과

```
화력 14점 이하 이슈: 240개
  → 시간 가중치로 인해 현재 화력이 낮아진 경우 (정상)

created_heat_index가 null인 이슈: 0개 ✅
created_heat_index가 14점 이하인 이슈: 233개
  → 시간 가중치로 인해 현재 화력이 낮아진 경우 (정상)

총 이슈 수: 362개 → 269개 (93개 감소)
```

## 정리 효과

### 1. 데이터 품질 개선

- `created_heat_index` null 제거: 81개 → 0개
- 화력 15점 미만 이슈 삭제: 66개
- 데이터 무결성 확보

### 2. 이슈 목록 정확성 향상

**Before**:
```
관리자 목록에 화력 0점~14점 이슈 81개 노출
→ 사용자 혼란
→ 관리 부담 증가
```

**After**:
```
등록 시 화력 15점 이상만 목록에 노출
→ 명확한 기준
→ 관리 효율성 증가
```

### 3. 히스토리 추적 가능

```
등록 시 화력: 30점 (created_heat_index)
현재 화력: 10점 (heat_index, 시간 가중)

→ 관리자가 "등록 당시 → 현재" 추이 확인 가능
→ 화력 감소 추세 파악
```

## 향후 방지책

### 1. 이슈 등록 시 필수 필드 검증

```typescript
// lib/candidate/issue-candidate.ts
const updateData = {
    approval_status: approvalStatus,
    created_heat_index: actualHeat,  // 필수!
    // ...
};

// 검증
if (!updateData.created_heat_index) {
    console.error('[오류] created_heat_index 미설정');
    throw new Error('created_heat_index is required');
}
```

### 2. DB 제약 조건 추가 고려

```sql
-- created_heat_index NULL 방지 (옵션)
ALTER TABLE issues 
ADD CONSTRAINT chk_created_heat_index_not_null 
CHECK (
    approval_status IS NULL OR 
    created_heat_index IS NOT NULL
);
```

### 3. 정기 모니터링

```sql
-- created_heat_index가 null인 이슈 조회 (매주)
SELECT COUNT(*) 
FROM issues 
WHERE approval_status IN ('대기', '승인', '반려')
  AND created_heat_index IS NULL;
```
