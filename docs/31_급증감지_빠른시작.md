# 급증 감지 시스템 구현 완료

**날짜**: 2026-03-06  
**버전**: 1.0

2-Track 급증 감지 시스템 구현이 완료되었습니다.

---

## 핵심 개선

- **실시간성**: 30분 → 3분 (10배 향상)
- **등록 기준**: 15점 (일반/급증 동일)
- **자동 승인 기준**: 30점 (일관성 유지)
- **트랙**: 뉴스 only → 뉴스 + 커뮤니티

---

## Track 1: 뉴스 급증 감지

### 핵심 기능
- 5분간 5건 이상 급증 감지
- 가속도 패턴 감지 (15분 대비 5분 비율 50% 이상)
- 급증 레벨 산정 (0~3)
- 자동 승인 기준 동일 유지 (30점)

### 구현 파일
- `lib/candidate/burst-detector.ts` - 급증 감지 로직
- `lib/candidate/issue-candidate.ts` - 이슈 생성 연동

---

## Track 2: 커뮤니티 급증 감지

### 핵심 기능
- 10분간 키워드 10건 이상 급증 감지
- AI 이슈 검증 (Groq) - 밈/드립/스팸 필터링
- 네이버 뉴스 검색 연동 (최소 3건)
- 3분 주기 실행

### 구현 파일
- `lib/candidate/community-burst-detector.ts` - 커뮤니티 급증 로직
- `app/api/cron/detect-community-burst/route.ts` - API 엔드포인트
- `.github/workflows/prod-cron-detect-community-burst.yml` - GitHub Actions

## AI 기반 중복 체크

### 4단계 검증
1. 정확한 제목 일치 (빠른 체크)
2. 키워드 필터링 (공통 키워드 2개 이상)
3. 안전 장치 (반대어, 숫자 차이 감지)
4. Groq AI 정밀 비교 (신뢰도 80% 이상)

### 구현 파일
- `lib/candidate/duplicate-checker.ts` - 중복 체크 로직

### 통계 쿼리

```sql
-- 오늘 급증 이슈
SELECT 
    source_track,
    COUNT(*) as count,
    AVG(burst_level) as avg_level
FROM issues
WHERE is_urgent = true
  AND created_at >= CURRENT_DATE
GROUP BY source_track;

-- 시간대별 급증 이슈
SELECT 
    DATE_TRUNC('hour', created_at) as hour,
    source_track,
    COUNT(*) as count
FROM issues
WHERE is_urgent = true
  AND created_at >= CURRENT_DATE - INTERVAL '1 day'
GROUP BY hour, source_track
ORDER BY hour;
```
