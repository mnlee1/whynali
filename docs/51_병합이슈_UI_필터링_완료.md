# 병합 이슈 UI 필터링 구현 완료

## 문제 인식

병합된 이슈가 목록에 계속 표시되는 문제 발견:
- 병합 스크립트로 이슈 9개를 병합했지만, 목록에는 여전히 14개가 표시됨
- 사용자는 병합된 이슈(secondary)는 보이지 않고 원본 이슈(primary)만 보기를 원함

## 구현 내용

### 1. 사용자용 API 필터링

#### `/api/issues` (이슈 목록 API)
```typescript
// app/api/issues/route.ts
let query = supabaseAdmin
    .from('issues')
    .select('*', { count: 'exact' })
    .eq('approval_status', '승인')
    .eq('visibility_status', 'visible')
    .is('merged_into_id', null) // 병합된 이슈 제외
    .gte('heat_index', MIN_HEAT_TO_REGISTER)
```

#### `/api/search` (검색 API)
```typescript
// app/api/search/route.ts
let issueQuery = admin
    .from('issues')
    .select('id, title, status, category, created_at')
    .eq('approval_status', '승인')
    .eq('visibility_status', 'visible')
    .is('merged_into_id', null) // 병합된 이슈 제외
    .gte('heat_index', MIN_HEAT_TO_REGISTER)
```

### 2. 이슈 상세 페이지 리다이렉트

병합된 이슈로 직접 접근 시 원본 이슈로 자동 리다이렉트:

```typescript
// app/issue/[id]/page.tsx
const { data: issue } = await adminClient
    .from('issues')
    .select('*')
    .eq('id', id)
    .single()

// 병합된 이슈인 경우 원본 이슈로 리다이렉트
if (issue.merged_into_id) {
    redirect(`/issue/${issue.merged_into_id}`)
}

// 승인되지 않았거나 숨김 처리된 이슈는 표시하지 않음
if (issue.approval_status !== '승인' || issue.visibility_status !== 'visible') {
    return <ErrorMessage />
}
```

### 3. 관리자 페이지 개선

#### 필터 추가
관리자는 병합된 이슈도 확인할 수 있도록 "병합됨" 필터 추가:

```typescript
const filters = [
    { value: '', label: '전체' },
    { value: '대기', label: '대기' },
    { value: '승인', label: '승인 전체' },
    { value: '승인:auto', label: '자동 승인' },
    { value: '승인:manual', label: '관리자 승인' },
    { value: '반려', label: '반려 전체' },
    { value: '반려:auto', label: '자동 반려' },
    { value: '반려:manual', label: '관리자 반려' },
    { value: '병합됨', label: '병합됨' }, // 새로 추가
]
```

#### 병합 상태 표시
```typescript
const getApprovalDisplay = (issue: Issue) => {
    if (issue.approval_status === '병합됨') {
        return {
            label: '병합됨',
            className: 'bg-purple-100 text-purple-700 border-purple-200'
        }
    }
    // ...
}
```

#### 병합된 이슈 제목 표시
병합된 이슈는 취소선 + 원본 이슈 링크로 표시:

```typescript
{issue.merged_into_id ? (
    <div className="flex flex-col gap-1">
        <span className="text-gray-500 line-through">
            {decodeHtml(issue.title)}
        </span>
        <a
            href={`/issue/${issue.merged_into_id}`}
            target="_blank"
            className="text-xs text-purple-600 hover:underline"
        >
            → 병합된 이슈로 이동
        </a>
    </div>
) : (
    <a href={`/issue/${issue.id}`} target="_blank">
        {decodeHtml(issue.title)}
    </a>
)}
```

## 동작 방식

### 일반 사용자 경험
1. 홈/카테고리 페이지: 병합된 이슈는 목록에서 제외됨
2. 검색: 병합된 이슈는 검색 결과에서 제외됨
3. 직접 URL 접근: 병합된 이슈 URL로 접근 시 원본 이슈로 자동 리다이렉트

### 관리자 경험
1. 전체 이슈 확인 가능: "병합됨" 필터로 병합된 이슈만 볼 수 있음
2. 병합 상태 구분: 제목 취소선 + 원본 이슈 링크로 명확히 구분
3. 원본 이슈 추적: 클릭 한 번으로 원본 이슈 확인 가능

## 수정된 파일

1. `app/api/issues/route.ts` - 사용자용 이슈 목록 API 필터링
2. `app/api/search/route.ts` - 검색 API 필터링
3. `app/issue/[id]/page.tsx` - 병합된 이슈 리다이렉트 처리
4. `app/admin/issues/page.tsx` - 관리자 UI 개선 (필터 + 표시)
5. `types/issue.ts` - 타입 정의 업데이트

## 테스트 방법

1. 홈페이지에서 이슈 목록 확인 → 병합된 이슈가 보이지 않는지 확인
2. 병합된 이슈 URL 직접 접근 → 원본 이슈로 리다이렉트되는지 확인
3. 관리자 페이지 → "병합됨" 필터 → 병합된 이슈만 표시되는지 확인
4. 관리자 페이지에서 병합된 이슈의 "병합된 이슈로 이동" 링크 클릭 → 원본 이슈로 이동하는지 확인

## 결과

병합된 이슈는 사용자에게 노출되지 않고, 관리자만 필터를 통해 확인할 수 있음.
병합 기능이 완전히 UI에 통합되어 자연스러운 사용자 경험 제공.
