# 이슈 로직 개선 통합

본 문서는 이슈 등록, 그루핑, 커뮤니티 매칭, AI 시도 관련 개선 사항을 통합한 문서입니다.

**최근 업데이트**: 2026-03-11 - 무신사 이슈 오분류 해결 (상세: `60_무신사_이슈_오분류_해결.md`)

## 1. 이슈 등록 로직 개선 전체

### 1.1 기존 문제점

1. 제목 기반 매칭의 한계: 키워드 토큰화만으로는 같은 이슈를 정확히 묶기 어려움
2. 무분별한 자동 승인: 모든 카테고리가 동일 기준으로 자동 승인되어 연예/정치 같은 민감 분야에서 오등록 위험
3. 재점화 미감지: 종결된 이슈에 대한 신규 데이터 급증 시 자동 상태 전환 없음

### 1.2 개선 방안

**조건부 자동 승인:**
- 카테고리별 차별화된 승인 정책
- 환경변수: `AUTO_APPROVE_CATEGORIES=사회,기술,스포츠`

| 카테고리 | 자동 승인 | 이유 |
|---------|---------|------|
| 사회 | 허용 | 객관적 사건/사고, 리스크 낮음 |
| 기술 | 허용 | 제품 출시, 업계 동향 등 사실 중심 |
| 스포츠 | 허용 | 경기 결과, 이적 등 명확한 사실 |
| 연예 | 금지 | 루머, 프라이버시 민감 |
| 정치 | 금지 | 정치적 편향 리스크 |

**재점화 감지:**
```bash
STATUS_REIGNITE_RATE_PER_MINUTE=5
STATUS_REIGNITE_DURATION_MINUTES=10
```

동작:
1. 급증 감지 (최우선): 10분간 50건 이상 → 즉시 논란중
2. 점진적 재점화: 48시간 내 신규 수집 + 화력 40점 이상 → 논란중

### 1.3 테스트 결과

**자동 승인 검증:**
- 19개 이슈 자동 승인 (카테고리 기반)
- 4개 저화력 이슈 자동 반려
- 자동 승인률: 68% (19/28)

**저품질 이슈 정리:**
- 25개 저품질 이슈 반려 처리 (화력 7~19점)
- 최종 공개 이슈: 3개 (모두 화력 30점 이상)
    - 옥택연 결혼 (화력 58점, 연예)
    - 민희진 뉴진스 (화력 30점, 연예)
    - 갤럭시 S26 (화력 55점, 기술)

## 2. 그루핑 개선 및 AI 시도

### 2.1 최종 채택 방식: 키워드 개선

**핵심 인물 리스트 대폭 확장 (9명 → 30+명)**
```
정치: 윤석열, 이재명, 한동훈, 이준석, 추경호, 박찬대, 권영세
연예: 민희진, 뉴진스, 하이브, 어도어, 방시혁, 옥택연, 김연경
스포츠: 손흥민, 이강인, 황희찬, 김하성, 류현진, 오타니
기업: 삼성전자, LG전자, SK하이닉스, 네이버, 카카오
```

**최종 임계값 설정 (품질 중시)**
```bash
CANDIDATE_ALERT_THRESHOLD=5
CANDIDATE_AUTO_APPROVE_THRESHOLD=10
CANDIDATE_WINDOW_HOURS=24
CANDIDATE_MIN_HEAT_TO_REGISTER=30
CANDIDATE_MIN_UNIQUE_SOURCES=2
CANDIDATE_COMMUNITY_MATCH_THRESHOLD=2
```

**화력 30점 기준 근거:**
- 화력 공식: newsCredibility × (0.3 + 0.7 × communityAmp)
- 30점 = 약 뉴스 15건 + 커뮤니티 반응 적정
- 실제 이슈 사례:
    - 민희진: 30점 (연예, 이슈성 있음)
    - 옥택연: 69점 (결혼 발표, 핫이슈)
    - 대전충남 통합: 14점 (지역 이슈, 제외됨)
    - LG AI: 12점 (기업 뉴스, 제외됨)

### 2.2 Perplexity AI 그루핑 시도 (실패)

**목표:**
Perplexity AI를 사용해 뉴스를 같은 이슈끼리 자동으로 그룹화

**발견된 문제:**
1. 불완전한 응답 형식: 빈 배열(,,,)이 많음
2. 그루핑 품질 낮음: 100건 입력 → 거의 100개 그룹으로 반환
3. 성능: 1000건 처리에 약 12초 (키워드 방식: 약 2초)

**결론:**
Perplexity는 그루핑 작업에 부적합

이유:
- 채팅 모델은 구조화된 출력에 최적화되지 않음
- sonar-pro는 검색 증강 생성에 특화 (분류가 아님)
- 비용 대비 효과 낮음

최종 설정:
```bash
ENABLE_AI_GROUPING=false
ENABLE_AI_GROUP_VALIDATION=false
```

### 2.3 그루핑 오류 개선

**문제 사례:**
```
A: '민주당 갈라치기 안 먹혔다' 조국당 지지율 최저치 경신 (한국 정치)
B: 미 민주당 반박 연설 "트럼프, 전례 없는 부패 대통령" (미국 정치)
→ "민주당" 키워드만 겹쳐서 완전히 다른 주제가 같은 이슈로 묶임
```

**해결책: 불용어 확장**
```
정당/정파: 민주당, 공화당, 여당, 야당, 보수, 진보
국가/지역: 북한, 러시아, 영국, 프랑스, 유럽, 아시아
범용 정치 용어: 발언, 언급, 입장, 반박, 대응, 논평
```

**개선 효과:**
- 오묶임 비율: 15% → 3% (예상)
- 이슈 정확도: 70% → 90% (예상)

## 3. 커뮤니티 매칭 오류 개선

### 3.1 제품명 매칭 오류

**문제 사례:**
```
커뮤니티: "색 잘뽑은거 같은 갤럭시 버즈4 핑크 골드.JPG"
이슈: "멀티 AI 품은 갤럭시S26 공개 D-1"
→ "갤럭시" 브랜드명만 겹쳐서 완전히 다른 제품(버즈4 vs S26)이 매칭됨
```

**해결책:**
1. 커뮤니티 매칭 임계값 상향: 1 → 2
2. 제품명 특수 처리: 브랜드 + 모델명 조합은 완전 일치만 허용

### 3.2 범용 동사 오매칭

**문제 사례:**
```
이슈: "짐승남 옥택연, 품절남 된다 '4월의 신랑'"
뉴스: "옥택연, 10년 사귄 연인과 4월 24일 결혼"

잘못 매칭된 커뮤니티:
1. "한국에서 활동하는 외국인들에게도 적용된다는 아이돌 세대 법칙"
2. "한국영화계를 위해 왕사남이 흥행하면 안된다는 자칭 시네필들"
3. "멈추지않는지랄이계속된다"
4. "아이브 가을이가 명품엠버서더에 발탁된다면?"
```

**원인:**
"된다", "활동", "발탁" 같은 범용 동사가 불용어가 아님

**해결책:**
1. 범용 동사/조사 불용어 추가: 있다, 없다, 한다, 된다, 이다, 같다 등
2. 핵심 키워드 로직 개선: 3글자 이상 + 범용 단어 제외

**검증:**
```
옥택연 이슈:
Before: 4건 (전혀 관련 없는 커뮤니티)
After: 0건 (개선된 로직으로 모두 제거됨)
```

### 3.3 인물명 키워드 오연결 개선 (2026-03-11)

**문제 사례:**
```
이슈: "스포츠 통한 성평등·포용 가치 확산 공로' 김연경, IOC 'GEDI 챔피언"
→ IOC GEDI 챔피언 선정 관련 이슈

잘못 연결된 뉴스:
1. "김연경, 배구 국가대표 복귀전 3세트 MVP"
2. "흥국생명 김연경, V리그 올스타전 출전"
3. "김연경 30득점 폭발, 팀 승리 이끌어"
```

**원인:**
- 키워드 매칭 임계값이 60%로 너무 낮음
- "김연경" 키워드 하나만으로도 무관한 뉴스가 연결됨
- IOC 시상 관련 뉴스와 배구 경기 뉴스가 구분되지 않음

**해결책:**
1. **뉴스 링커 임계값 상향**: 60% → 70%, 최소 2개 → 3개
2. **커뮤니티 링커 임계값 상향**: 60% → 70%, 최소 2개 → 3개
3. **인물명 키워드 특별 처리**: 인물명이 포함된 경우 무조건 최소 3개 이상 일치 필수

**인물명 키워드 목록:**
```typescript
const personKeywords = [
    '김연경', '손흥민', '이강인', '옥택연', '민희진', '뉴진스',
    '윤석열', '이재명', '한동훈', '황희찬', '김하성', '류현진'
]
```

**개선 효과:**
```
김연경 IOC GEDI 챔피언 이슈:
Before: 12건 연결 (IOC 관련 5건 + 배구 경기 7건)
After: 5건 연결 (IOC 관련만 정확히 필터링)

오연결 비율: 58% → 0% (예상)
정확도: 42% → 100% (예상)
```

**적용 파일:**
- `lib/linker/issue-news-linker.ts`
- `lib/linker/issue-community-linker.ts`

**재검증 로직:**
- 기존 연결된 뉴스/커뮤니티도 매 cron 주기마다 새로운 기준으로 재검증
- 기준 미달 시 자동으로 issue_id 연결 해제
- unlinkInvalidNews, unlinkInvalidCommunity 함수에서 처리

### 3.4 이커머스/유통 키워드 오분류 방지 (2026-03-11)

**문제 사례:**
```
이슈: "무신사, 스포츠·IP 전문 큐레이션관 '팬 스토어' 론칭"
→ "스포츠·IP"는 스포츠 굿즈 상품 카테고리를 의미
→ 실제로는 패션 이커머스의 신규 서비스 론칭 → 기술 카테고리

잘못된 분류: 스포츠
→ 무관한 스포츠 뉴스 95건 잘못 연결 (어르신 스포츠 지원사업, 스포츠 중계권 등)

올바른 분류: 기술
→ 무신사 팬 스토어 관련 뉴스 10건만 연결
```

**해결책:**
1. **기술 카테고리 키워드 확장**: 무신사, 쿠팡, 이커머스, 큐레이션, 스토어 등 추가
2. **Context Rules 추가**: ['무신사', '큐레이션'], ['이커머스', '플랫폼'] 등
3. **스포츠 카테고리 키워드 정제**: 실제 경기/선수 관련만 포함 (MVP, K리그, KBO 등)
4. **AI 프롬프트 개선**: "스포츠·IP 큐레이션"은 유통 서비스로 명시

**개선 효과:**
```
Before: "스포츠" 키워드만으로 스포츠 카테고리 분류 → 오연결 95건
After: 맥락 규칙으로 기술 카테고리 분류 → 정확한 연결 10건

오연결 비율: 90% → 0%
정확도: 10% → 100%
```
