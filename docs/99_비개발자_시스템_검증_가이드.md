# 비개발자를 위한 시스템 검증 가이드

왜난리 프로젝트의 자동화 시스템이 제대로 작동하는지 확인하는 방법입니다.

## 가장 쉬운 방법: 관리자 대시보드 확인

### 1단계: 대시보드 접속

1. 브라우저에서 왜난리 관리자 페이지 접속
2. URL: `https://your-domain.com/admin` 또는 로컬 `http://localhost:3000/admin`
3. 로그인 (관리자 권한 필요)

### 2단계: 24시간 통계 확인

**"최근 24시간 요약" 섹션을 찾으세요.**

#### 정상적인 경우:

```
1단계: 데이터 수집
뉴스: 1,200건
커뮤니티: 500건

2단계: 이슈 생성
생성됨: 15건
승인: 3건 / 대기: 8건 / 반려: 4건

3단계: 이슈 연결
뉴스 연결률: 36%
커뮤니티 연결률: 32%
```

#### 문제가 있는 경우:

빨간색 또는 노란색 경고 배너가 표시됩니다:

**빨간색 (치명):**
- "뉴스 수집이 24시간 동안 10건 미만입니다"
- "수집 데이터는 많지만 이슈가 생성되지 않았습니다"

**노란색 (경고):**
- "커뮤니티 수집이 24시간 동안 50건 미만입니다"
- "승인 대기 이슈가 10건 이상 쌓여 있습니다"

### 3단계: 문제 해결

#### 경고: 수집이 너무 적음

**확인 사항:**
1. GitHub 저장소 접속
2. "Actions" 탭 클릭
3. 최근 워크플로우 실행 내역 확인
4. 빨간색 X 표시가 있으면 실패한 것

**해결 방법:**
- 실패한 워크플로우 클릭하여 에러 로그 확인
- 주로 API 키 문제 또는 외부 사이트 변경

#### 경고: 이슈가 생성되지 않음

**확인 사항:**
1. `/admin/collections` 페이지 접속
2. 수집 데이터가 실제로 있는지 확인
3. "최근 수집 데이터" 목록 확인

**해결 방법:**
- 수집 데이터는 있는데 이슈가 안 생성되면 → AI 자동 생성 로직 점검 필요
- 개발자에게 문의

#### 경고: 승인 대기가 많음

**해결 방법:**
1. `/admin/issues` 페이지 접속
2. "승인 대기" 탭 선택
3. 이슈 하나씩 검토하여 승인/반려 처리
4. 승인: 실제 서비스에 노출
5. 반려: 이슈로 적합하지 않음

## 보조 방법: 브라우저 개발자 도구

관리자 대시보드만으로 부족하면 개발자 도구를 사용하세요.

### 1. 개발자 도구 열기

- Windows/Linux: `F12` 또는 `Ctrl + Shift + I`
- Mac: `Cmd + Option + I`

### 2. Console 탭 확인

**정상:**
```
[Admin Dashboard] fetchAll 시작
[Admin Dashboard] 핵심 API 응답 받음
```

**에러:**
```
❌ Error: Failed to fetch
❌ 500 Internal Server Error
❌ Unauthorized
```

빨간색 에러가 보이면 스크린샷 찍어서 개발자에게 공유하세요.

### 3. Network 탭 확인

1. "Network" 탭 클릭
2. "새로고침" 버튼 클릭 (대시보드)
3. 목록에서 빨간색 항목 확인

**확인 항목:**
- `/api/admin/stats-24h` - 빨간색이면 통계 API 실패
- `/api/admin/collections` - 빨간색이면 수집 API 실패

## 고급: Supabase 직접 확인

관리자 대시보드 자체가 안 열리면 Supabase에서 직접 확인하세요.

### 1. Supabase 대시보드 접속

1. https://app.supabase.com/ 접속
2. 프로젝트 선택
3. 왼쪽 메뉴에서 "Table Editor" 클릭

### 2. 테이블 확인

#### news_data 테이블
- 최근 `created_at` 확인
- 24시간 이내 데이터가 있는지 확인

#### community_data 테이블
- 최근 `created_at` 확인
- 24시간 이내 데이터가 있는지 확인

#### issues 테이블
- `approval_status` 필터 적용
- "승인" 상태 데이터 확인
- "대기" 상태가 많으면 승인 작업 필요

### 3. SQL Editor로 통계 확인

왼쪽 메뉴 "SQL Editor" 클릭 후 아래 쿼리 실행:

```sql
-- 24시간 이내 수집 건수
SELECT 
    '뉴스' as 타입,
    COUNT(*) as 건수
FROM news_data
WHERE created_at > NOW() - INTERVAL '24 hours'

UNION ALL

SELECT 
    '커뮤니티' as 타입,
    COUNT(*) as 건수
FROM community_data
WHERE created_at > NOW() - INTERVAL '24 hours';
```

결과 예시:
```
타입       | 건수
---------|-----
뉴스      | 1234
커뮤니티   | 567
```

## 문제별 체크리스트

### 이슈가 화면에 안 보임

- [ ] Supabase `issues` 테이블에 데이터 있음
- [ ] `approval_status`가 "승인" 상태
- [ ] `merged_into_id`가 NULL (병합되지 않음)
- [ ] 홈페이지 새로고침 (Ctrl + F5)
- [ ] 브라우저 캐시 삭제

### 투표가 안 보임

- [ ] Supabase `votes` 테이블에 데이터 있음
- [ ] `phase`가 "진행중" 상태
- [ ] `approval_status`가 "승인" 상태
- [ ] 해당 이슈 ID와 연결되어 있음

### 댓글이 안 보임

- [ ] Supabase `comments` 테이블에 데이터 있음
- [ ] `visibility`가 "public" 상태
- [ ] 로그인 상태 확인

## 자주 묻는 질문

**Q: 경고가 떠도 서비스는 정상 작동하나요?**
A: 네. 경고는 "주의가 필요하다"는 신호일 뿐, 서비스가 멈춘 것은 아닙니다.

**Q: 24시간 통계가 0건이면 문제인가요?**
A: 새벽 시간대나 방금 배포한 경우 정상일 수 있습니다. 하루 이상 지속되면 문제입니다.

**Q: 연결률이 낮으면 문제인가요?**
A: 20~40% 정도는 정상 범위입니다. 모든 수집 데이터가 이슈와 연결되는 것은 아닙니다.

**Q: 승인 대기가 많으면 어떻게 하나요?**
A: `/admin/issues` 페이지에서 하나씩 검토하여 승인/반려 처리하세요.
