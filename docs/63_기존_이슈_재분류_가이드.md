# 기존 이슈 재분류 가이드

날짜: 2026-03-11
상태: ⏳ Rate Limit으로 내일 실행 예정

## 현재 상황

### Rate Limit 도달

```
Groq 사용량: 499,980 / 500,000 토큰 (99.99%)
→ 오늘 테스트/분석/수정으로 한도 거의 소진
→ 내일 (2026-03-12) 자동 리셋
```

### 재분류 대상

```
최근 7일 이슈: 31개
최근 1일 이슈: 17개

예상 소요:
- 17개 × 250토큰 = 4,250토큰
- 31개 × 250토큰 = 7,750토큰
- 전체 재분류: 충분히 가능
```

## 실행 방법

### 1. 내일 실행 (권장)

**Option 1: 최근 7일 전체 재분류**
```bash
npx tsx scripts/reclassify-all-recent-issues.ts --days 7 --apply
```

**Option 2: 최근 1일만**
```bash
npx tsx scripts/reclassify-all-recent-issues.ts --days 1 --apply
```

**Option 3: 특정 카테고리만**
```bash
# 스포츠 카테고리만
npx tsx scripts/reclassify-all-recent-issues.ts --category 스포츠 --apply

# 기술 카테고리만
npx tsx scripts/reclassify-all-recent-issues.ts --category 기술 --apply
```

### 2. Dry-Run으로 먼저 확인

```bash
# 실제 변경 없이 미리보기
npx tsx scripts/reclassify-all-recent-issues.ts --days 7

# 변경될 내용 확인 후
npx tsx scripts/reclassify-all-recent-issues.ts --days 7 --apply
```

### 3. 옵션

| 옵션 | 설명 | 예시 |
|-----|------|------|
| `--days N` | 최근 N일 이슈 | `--days 7` |
| `--category` | 특정 카테고리만 | `--category 스포츠` |
| `--apply` | 실제 적용 (없으면 dry-run) | `--apply` |
| `--delay` | API 대기 시간(ms) | `--delay 3000` |

## 자동화 옵션

### Cron으로 자동 재분류

**1. GitHub Actions 워크플로우 추가**

`.github/workflows/reclassify-old-issues.yml`:
```yaml
name: Reclassify Old Issues

on:
  schedule:
    # 매주 일요일 오전 3시 (KST 12시)
    - cron: '0 3 * * 0'
  workflow_dispatch:

jobs:
  reclassify:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
        with:
          node-version: '18'
      - run: npm ci
      - name: Reclassify Last 7 Days
        run: npx tsx scripts/reclassify-all-recent-issues.ts --days 7 --apply
        env:
          GROQ_API_KEY: ${{ secrets.GROQ_API_KEY }}
          NEXT_PUBLIC_SUPABASE_URL: ${{ secrets.NEXT_PUBLIC_SUPABASE_URL }}
          SUPABASE_SERVICE_ROLE_KEY: ${{ secrets.SUPABASE_SERVICE_ROLE_KEY }}
          ENABLE_AI_CATEGORY: true
          CATEGORY_STRATEGY: ai
```

**2. 수동 실행**
```bash
# GitHub Actions에서 workflow_dispatch로 수동 실행 가능
```


## 모니터링

### 변경 후 확인

```sql
-- 오늘 변경된 이슈 확인
SELECT id, title, category, updated_at
FROM issues
WHERE DATE(updated_at) = CURRENT_DATE
ORDER BY updated_at DESC;

-- 카테고리별 분포
SELECT category, COUNT(*) as count
FROM issues
WHERE approval_status = '승인'
GROUP BY category
ORDER BY count DESC;
```
