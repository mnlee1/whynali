# AI 대시보드 가이드

관리자 대시보드에서 AI 시스템 현황을 실시간으로 모니터링하는 방법.

## 표시되는 AI 시스템

### Groq AI (항상 표시)
- 모델: Llama 3.1 8B Instant
- 비용: 무료
- 활성 기능:
    - 카테고리 자동 분류
    - 중복 이슈 체크
    - 커뮤니티 급증 검증
    - 뉴스-이슈 연결 검증
    - 커뮤니티 글 매칭
    - 토론 주제 생성
    - 투표 생성

### Perplexity AI (사용 시에만 표시)
- 모델: Sonar
- 비용: 유료 (토큰 기반)
- 활성 기능:
    - 뉴스 그루핑 (선택적)
    - 이슈 후보 검증 (선택적)

## 조건부 렌더링

### 표시 조건
Perplexity 섹션은 **오늘 호출이 1회 이상**일 때만 자동으로 표시됩니다.

**백엔드 로직:**
```typescript
// lib/api-usage-tracker.ts
perplexity: perplexityCallsToday > 0 ? {
    today: perplexityCostToday,
    monthly: perplexityCostMonthly,
    calls: { ... },
    tokens: { ... }
} : null
```

**프론트엔드 렌더링:**
```tsx
// app/admin/page.tsx
{apiCosts.perplexity && (
    <div className="p-4 bg-purple-50">
        {/* Perplexity 상세 정보 */}
    </div>
)}
```

### 표시 정보

**Groq AI 카드:**
- 오늘: 호출 횟수 + 금액 ($0.00)
- 월 누적: 호출 횟수 + 금액 ($0.00)
- 활성 기능 목록 (녹색 배지)
- 성공률 (%)

**Perplexity AI 카드 (활성화 시):**
- 오늘: 호출 횟수 + 금액
- 월 누적: 호출 횟수 + 금액
- 토큰 사용량: 입력/출력/전체 (K 단위)
- 활성 기능 목록
- 성공률 (%)

## 활성화/비활성화

### Perplexity 활성화
1. 환경변수 설정
```bash
ENABLE_AI_GROUPING=true
ENABLE_AI_GROUP_VALIDATION=true
```

2. 서버 재시작
3. Perplexity API 호출 시작
4. **다음 날** 대시보드에 자동으로 표시

### Perplexity 비활성화
1. 환경변수 변경
```bash
ENABLE_AI_GROUPING=false
ENABLE_AI_GROUP_VALIDATION=false
```

2. 서버 재시작
3. API 호출 중단
4. **다음 날** 대시보드에서 자동으로 사라짐

완전히 즉시 숨기려면 DB에서 해당 날짜 데이터 삭제 필요.

## 장점

1. **자동 감지**: 환경변수만 변경하면 대시보드가 자동으로 업데이트
2. **깔끔한 화면**: 사용하지 않는 AI는 완전히 숨김
3. **실시간 모니터링**: 오늘/월별 사용량, 성공률 확인
4. **정확한 비용**: 실제 사용량 기반 표시

## 주의사항

- Perplexity 활성화 후 **다음 날부터** 대시보드에 표시
- 비활성화 후에도 **다음 날까지는** 표시될 수 있음
- 즉시 반영을 원하면 DB에서 해당 날짜 데이터 삭제 필요
- 이번 달 호출이 1회 이상이면 "사용 중"으로 표시
