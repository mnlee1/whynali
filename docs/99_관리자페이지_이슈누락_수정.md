# 관리자 페이지 이슈 누락 버그 수정

날짜: 2026-03-13
문제: 사이트에는 보이지만 관리자 페이지에는 안 보이는 승인 이슈 존재

## 문제 발견

### 현상
- 사이트에 승인된 이슈가 노출됨
- 관리자 페이지 목록에는 없음
- 관리 불가능 상태

### 확인된 누락 이슈

**이슈**: "대법, 윤석준 대구 동구청장 당선무효형 확정…정치자금법 위반"
- ID: `8b9bd371-5692-419a-ab55-16ea295e102e`
- approval_status: 승인
- visibility_status: visible
- **source_track: manual** ← 원인
- heat_index: 24점
- category: 정치

## 원인 분석

### 필터링 차이

**사이트 API** (`app/api/issues/route.ts`):
```typescript
.eq('approval_status', '승인')
.eq('visibility_status', 'visible')
.is('merged_into_id', null)
.gte('heat_index', MIN_HEAT_TO_REGISTER)
// source_track 필터 없음 ✅
```

**관리자 페이지** (`app/admin/issues/page.tsx`):
```typescript
params.append('source_track', 'track_a')  // ❌ 문제!
```

**관리자 API** (`app/api/admin/issues/route.ts`):
```typescript
if (sourceTrack) {
    query = query.or(`source_track.eq.${sourceTrack},source_track.is.null`)
}
```

### 문제 발생 시나리오

1. 관리자가 수동으로 이슈 생성
2. `source_track: 'manual'`로 저장됨
3. 관리자가 승인 처리
4. 사이트에 노출됨 ✅
5. **관리자 페이지는 `track_a` 필터만 적용** → 안 보임 ❌

## 해결 방법

### 코드 수정

**파일**: `app/admin/issues/page.tsx`

**변경 전**:
```typescript
const params = new URLSearchParams()
params.append('source_track', 'track_a')  // track_a만 표시
```

**변경 후**:
```typescript
const params = new URLSearchParams()
// source_track 필터 제거 - manual 이슈도 표시
// params.append('source_track', 'track_a')
```

### UI 수정

**제목 변경**:
- Before: "트랙 A 이슈 관리"
- After: "이슈 관리"

**설명 변경**:
- Before: "커뮤니티 급증 기반 AI 검증 이슈"
- After: "트랙 A (자동) + 수동 생성 이슈 전체"

## 결과

### Before
```
관리자 페이지: track_a 이슈만 표시
  - track_a: 5개 ✅
  - manual: 0개 ❌ (누락)
```

### After
```
관리자 페이지: 전체 이슈 표시
  - track_a: 5개 ✅
  - manual: 1개 ✅ (표시됨)
```

## 영향 범위

### 긍정적 효과
- 수동 생성 이슈도 관리자 페이지에서 관리 가능
- 승인 상태 불일치 해소
- 사이트 노출 이슈와 관리자 목록 일치

### 부작용 없음
- 기존 track_a 이슈는 그대로 표시
- 필터 기능(전체/대기/승인/반려) 정상 작동
- API 로직 변경 없음 (프론트엔드만 수정)

## 추가 고려사항

### source_track 값 정의

현재 시스템에서 사용되는 값:
- `track_a`: 트랙 A 프로세스 (자동 생성)
- `manual`: 관리자 수동 생성
- `null`: 레거시 데이터

### 향후 개선안

옵션 1: 탭으로 분리
```
[트랙 A] [수동 생성] [전체]
```

옵션 2: source_track 배지 표시
- 목록에서 각 이슈의 source_track을 배지로 표시
- 필터 옵션 추가

## 결론

관리자 페이지의 `source_track` 필터를 제거하여 수동 생성 이슈도 표시되도록 수정했습니다.

이제 사이트에 노출된 모든 이슈를 관리자 페이지에서 확인하고 관리할 수 있습니다.
