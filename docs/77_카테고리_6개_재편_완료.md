# 카테고리 재편: 8개 → 6개 완료

**작업일**: 2026-03-13
**목적**: 카테고리 단순화 및 명확화

## 변경 내역 요약

8개 카테고리 → 6개 카테고리로 단순화
- 유지: 연예, 스포츠, 정치, 사회, 경제, 세계
- 변경: IT과학 → 기술
- 제거: 생활문화 → 사회에 통합

## 주요 변경사항

### 1. IT과학 → 기술

**변경 이유:**
- "IT과학"은 범위가 애매하고 슬래시 표기 혼란
- "기술"이 더 직관적이고 명확함

**키워드:** IT, 과학, AI, 반도체, 스타트업, 연구 등 모두 유지

### 2. 생활문화 제거 → 사회 통합

**변경 이유:**
- 생활문화는 이슈성이 낮고 분류가 애매함
- 대부분의 생활문화 콘텐츠는 사회 카테고리로 충분히 커버 가능

**통합된 키워드:**
- 맛집, 여행, 뷰티, 육아, 교육, 건강, 의료
- 문화, 전시, 공연, 책, 독서
- 요리, 카페, 관광, 호텔, 화장품, 미용

## DB 마이그레이션 가이드

### 1. 백업 (필수)

```sql
-- 현재 데이터 백업
CREATE TABLE issues_backup AS SELECT * FROM issues;
CREATE TABLE news_data_backup AS SELECT * FROM news_data;
```

### 2. CHECK 제약 조건 삭제

```sql
ALTER TABLE issues DROP CONSTRAINT IF EXISTS issues_category_check;
ALTER TABLE news_data DROP CONSTRAINT IF EXISTS news_data_category_check;
```

### 3. 데이터 마이그레이션

```sql
-- IT과학 → 기술
UPDATE issues SET category = '기술' WHERE category = 'IT과학';
UPDATE news_data SET category = '기술' WHERE category = 'IT과학';

-- 생활문화 → 사회
UPDATE issues SET category = '사회' WHERE category = '생활문화';
UPDATE news_data SET category = '사회' WHERE category = '생활문화';

-- 변경 결과 확인
SELECT category, COUNT(*) FROM issues GROUP BY category;
SELECT category, COUNT(*) FROM news_data GROUP BY category;
```

### 4. CHECK 제약 조건 재설정

```sql
ALTER TABLE issues ADD CONSTRAINT issues_category_check 
  CHECK (category IN ('연예', '스포츠', '정치', '사회', '경제', '기술', '세계'));

ALTER TABLE news_data ADD CONSTRAINT news_data_category_check 
  CHECK (category IN ('연예', '스포츠', '정치', '사회', '경제', '기술', '세계'));
```

### 5. 검증

```sql
-- 잘못된 카테고리 확인 (결과가 0이어야 함)
SELECT * FROM issues 
WHERE category NOT IN ('연예', '스포츠', '정치', '사회', '경제', '기술', '세계');

SELECT * FROM news_data 
WHERE category NOT IN ('연예', '스포츠', '정치', '사회', '경제', '기술', '세계');
```
