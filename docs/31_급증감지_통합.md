## 1. 핵심 개요

### 시스템 목적
실시간 이슈 감지를 위한 통합 급증 감지 시스템 (트랙 A)

### 핵심 개선
- **실시간성**: 30분 주기로 커뮤니티 급증 감지 및 이슈 생성
- **등록 기준**: 15점 (화력 기준)
- **자동 승인 기준**: 카테고리별 차등 적용
- **통합 프로세스**: 커뮤니티 급증 → AI 판단 → 뉴스 검색 → 이슈 생성

## 2. Track A: 통합 급증 감지 프로세스

### 핵심 기능
- 10분간 키워드 10건 이상 급증 감지
- AI 이슈 검증 (Groq) - 밈/드립/스팸 필터링
- 네이버 뉴스 즉시 타겟 검색
- 뉴스 필터링 및 커뮤니티 필터링
- 최종 이슈 제목 AI 생성
- 긴급 이슈 타임아웃 자동 승인 (2시간)

### 동작 흐름
```
1. 긴급 이슈 자동 승인 (대기 2시간 이상)
   ↓
2. 최근 10분간 커뮤니티 글 수집
   ↓
3. 키워드 추출 및 빈도 계산
   ↓
4. 급증 키워드 필터링 (10건 이상)
   ↓
5. AI 이슈 검증 (Groq)
   - 신뢰도 70% 이상
   - 밈/드립/스팸 필터링
   - 카테고리 분류
   - 검색 키워드 생성
   ↓
6. 네이버 뉴스 타겟 검색
   - AI가 생성한 키워드로 검색
   - 최근 30일 이내 뉴스만
   ↓
7. AI 통합 필터링 (1회 호출로 통합)
   - 관련 뉴스 선별
   - 관련 커뮤니티 글 선별
   - 최종 이슈 제목 생성
   ↓
8. 중복 체크
   ↓
9. 이슈 후보 등록 (대기 상태)
   - 화력 15점 이상만
   - 타임라인 자동 생성
```

### 구현 파일
- `app/api/cron/track-a/route.ts` - 트랙 A 통합 프로세스
- `.github/workflows/cron-track-a.yml` - GitHub Actions (30분 주기)

## 5. AI 기반 중복 체크

### 4단계 검증
1. 정확한 제목 일치 (빠른 체크)
2. 키워드 필터링 (공통 키워드 2개 이상)
3. 안전 장치 (반대어, 숫자 차이 감지)
4. Groq AI 정밀 비교 (신뢰도 80% 이상)

### 안전 장치
**반대어 감지:**
- "복귀" vs "사퇴" → 별개
- "기소" vs "무혐의" → 별개

**숫자 차이 감지:**
- "1차 회견" vs "2차 회견" → 별개
- "재판 3일차" vs "4일차" → 별개

### 구현 파일
- `lib/candidate/duplicate-checker.ts` - 중복 체크 로직

## 6. 환경변수 설정

### 필수 환경변수
```bash
# Groq AI (필수 - 트랙 A 이슈 검증 및 필터링)
GROQ_API_KEY=your_groq_api_key_here

# 커뮤니티 급증 감지 (기본값: true)
ENABLE_COMMUNITY_BURST=true
```

### 선택 환경변수 (기본값 있음)
```bash
# 트랙 A 급증 임계값
COMMUNITY_BURST_THRESHOLD=10           # 키워드 급증 기준 (기본 10건)
COMMUNITY_BURST_WINDOW_MINUTES=10     # 관찰 시간 (기본 10분)

# 트랙 A Rate Limit 완화
TRACK_A_MAX_KEYWORDS=1                # 한 번에 처리할 키워드 수 (기본 1개)
TRACK_A_AI_DELAY_MS=10000            # AI 호출 간 대기 시간 (기본 10초)

# 이슈 등록 기준
CANDIDATE_MIN_HEAT_TO_REGISTER=15    # 최소 화력 (기본 15점)

# 긴급 이슈 자동 승인
URGENT_AUTO_APPROVE_HOURS=2          # 타임아웃 기준 (기본 2시간)

# AI 중복 체크
DUPLICATE_CHECK_AI_CONFIDENCE=80
DUPLICATE_CHECK_WINDOW_HOURS=1
```

**주의:** `ENABLE_TRACK_A` 환경변수는 제거되었습니다. 트랙 A는 이제 기본 프로세스입니다.

### 통계 쿼리

```sql
-- 오늘 트랙 A 이슈
SELECT 
    COUNT(*) as count,
    AVG(heat_index) as avg_heat,
    AVG(created_heat_index) as avg_initial_heat
FROM issues
WHERE source_track = 'track_a'
  AND created_at >= CURRENT_DATE;

-- 시간대별 트랙 A 이슈
SELECT 
    DATE_TRUNC('hour', created_at) as hour,
    COUNT(*) as count,
    AVG(heat_index) as avg_heat
FROM issues
WHERE source_track = 'track_a'
  AND created_at >= CURRENT_DATE - INTERVAL '1 day'
GROUP BY hour
ORDER BY hour;

-- 긴급 자동 승인 이슈
SELECT 
    title,
    category,
    heat_index,
    created_at,
    approved_at
FROM issues
WHERE approval_type = 'auto'
  AND is_urgent = true
  AND created_at >= CURRENT_DATE
ORDER BY created_at DESC;
```

## 12. 문제 해결

### 급증 감지 안 됨
```bash
# 환경변수 확인
echo $ENABLE_COMMUNITY_BURST

# 임계값 낮추기 (테스트)
COMMUNITY_BURST_THRESHOLD=3
```

### AI 검증 실패
```bash
# 로그 확인
vercel logs --follow | grep "AI 검증"

# 신뢰도 낮추기 (테스트)
# lib/candidate/community-burst-detector.ts
# confidence >= 60  // 70 → 60
```

### Rate Limit 초과
```typescript
// 처리 키워드 수 줄이기
burstKeywords.slice(0, 2)  // 3 → 2

// 대기 시간 늘리기
setTimeout(5000)  // 3초 → 5초
```
