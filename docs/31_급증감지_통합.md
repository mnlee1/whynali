# 급증 감지 시스템 통합 문서

**버전**: 1.0  
**최종 업데이트**: 2026-03-06

## 1. 핵심 개요

### 시스템 목적
실시간 이슈 감지를 위한 2-Track 급증 감지 시스템

### 핵심 개선
- **실시간성**: 30분 → 3분 (10배 향상)
- **등록 기준**: 15점 (일반/급증 동일)
- **자동 승인 기준**: 30점 (일관성 유지)
- **트랙**: 뉴스 only → 뉴스 + 커뮤니티

## 2. Track 1: 뉴스 급증 감지

### 핵심 기능
- 5분간 5건 이상 급증 감지
- 가속도 패턴 감지 (15분 대비 5분 비율 50% 이상)
- 급증 레벨 산정 (0~3)
- 자동 승인 기준 동일 유지 (30점)

### 급증 레벨
- 레벨 3 (강함): 5분간 10건 이상
- 레벨 2 (보통): 5분간 7건 이상  
- 레벨 1 (약함): 5분간 5건 이상 또는 가속도 50% 이상
- 레벨 0 (없음): 급증 없음

### 구현 파일
- `lib/candidate/burst-detector.ts` - 급증 감지 로직
- `lib/candidate/issue-candidate.ts` - 이슈 생성 연동

## 3. Track 2: 커뮤니티 급증 감지

### 핵심 기능
- 10분간 키워드 10건 이상 급증 감지
- AI 이슈 검증 (Groq) - 밈/드립/스팸 필터링
- 네이버 뉴스 검색 연동 (최소 3건)
- 3분 주기 실행

### 동작 흐름
```
1. 최근 10분간 커뮤니티 글 수집
   ↓
2. 키워드 추출 및 빈도 계산
   ↓
3. 급증 키워드 필터링 (10건 이상)
   ↓
4. AI 이슈 검증 (Groq)
   - 신뢰도 70% 이상
   - 밈/드립/스팸 필터링
   ↓
5. 네이버 뉴스 검색
   - 최소 3건 이상
   ↓
6. 중복 체크
   ↓
7. 긴급 이슈 생성
```

### AI 이슈 검증 기준
1. 사회적 논란/사건 → 이슈 O
2. 단순 유행어/밈 → 이슈 X
3. 스팸/홍보 → 이슈 X
4. 정보 가치 없는 반응 → 이슈 X

### 구현 파일
- `lib/candidate/community-burst-detector.ts` - 커뮤니티 급증 로직
- `app/api/cron/detect-community-burst/route.ts` - API 엔드포인트
- `.github/workflows/prod-cron-detect-community-burst.yml` - GitHub Actions

## 4. AI 기반 중복 체크

### 4단계 검증
1. 정확한 제목 일치 (빠른 체크)
2. 키워드 필터링 (공통 키워드 2개 이상)
3. 안전 장치 (반대어, 숫자 차이 감지)
4. Groq AI 정밀 비교 (신뢰도 80% 이상)

### 안전 장치
**반대어 감지:**
- "복귀" vs "사퇴" → 별개
- "기소" vs "무혐의" → 별개

**숫자 차이 감지:**
- "1차 회견" vs "2차 회견" → 별개
- "재판 3일차" vs "4일차" → 별개

### 구현 파일
- `lib/candidate/duplicate-checker.ts` - 중복 체크 로직

## 5. DB 마이그레이션

Supabase SQL Editor에서 실행:

```sql
ALTER TABLE issues 
ADD COLUMN IF NOT EXISTS is_urgent BOOLEAN DEFAULT FALSE,
ADD COLUMN IF NOT EXISTS burst_level INTEGER DEFAULT 0,
ADD COLUMN IF NOT EXISTS source_track TEXT;

CREATE INDEX IF NOT EXISTS idx_issues_is_urgent ON issues(is_urgent) WHERE is_urgent = true;
```

### 통계 쿼리

```sql
-- 오늘 급증 이슈
SELECT 
    source_track,
    COUNT(*) as count,
    AVG(burst_level) as avg_level
FROM issues
WHERE is_urgent = true
  AND created_at >= CURRENT_DATE
GROUP BY source_track;

-- 시간대별 급증 이슈
SELECT 
    DATE_TRUNC('hour', created_at) as hour,
    source_track,
    COUNT(*) as count
FROM issues
WHERE is_urgent = true
  AND created_at >= CURRENT_DATE - INTERVAL '1 day'
GROUP BY hour, source_track
ORDER BY hour;
```

## 12. 문제 해결

### 급증 감지 안 됨
```bash
# 환경변수 확인
echo $ENABLE_COMMUNITY_BURST

# 임계값 낮추기 (테스트)
COMMUNITY_BURST_THRESHOLD=3
```

### AI 검증 실패
```bash
# 로그 확인
vercel logs --follow | grep "AI 검증"

# 신뢰도 낮추기 (테스트)
# lib/candidate/community-burst-detector.ts
# confidence >= 60  // 70 → 60
```

### Rate Limit 초과
```typescript
// 처리 키워드 수 줄이기
burstKeywords.slice(0, 2)  // 3 → 2

// 대기 시간 늘리기
setTimeout(5000)  // 3초 → 5초
```
