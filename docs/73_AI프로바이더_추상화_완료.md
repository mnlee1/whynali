## 작업 완료 요약

### 1. Supabase 테이블 추가 ✅

**파일:** `supabase/schema.sql`

**추가된 테이블:**
```sql
CREATE TABLE ai_key_status (
    provider TEXT NOT NULL,
    key_hash TEXT NOT NULL,
    is_blocked BOOLEAN NOT NULL DEFAULT FALSE,
    blocked_until TIMESTAMPTZ,
    fail_count INT NOT NULL DEFAULT 0,
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    PRIMARY KEY (provider, key_hash)
);
```

**역할:**
- 서버리스 인스턴스 간 키 차단 상태 공유
- Rate Limit 발생 시 해당 키를 DB에 기록
- 다른 인스턴스는 차단된 키를 건너뛰고 사용 가능한 키 선택

**Supabase 대시보드에서 SQL 실행 필요**

## 모델 변경으로 인한 Rate Limit 개선

### 기존: llama-3.3-70b-versatile

```
TPM (Tokens Per Minute): 6,000
RPM (Requests Per Minute): 30
```

### 신규: llama-3.1-8b-instant

```
TPM (Tokens Per Minute): 30,000
RPM (Requests Per Minute): 30
```

**개선 효과:**
- TPM 5배 증가 (6,000 → 30,000)
- 더 빠른 응답 속도
- 카테고리 분류, 중복 체크 등 간단한 작업에 충분한 성능

## 기대 효과

### 1. Rate Limit 근본 해결

- 서버리스 인스턴스 간 키 상태 공유
- 차단된 키 자동 회피
- 사용 가능한 키로 자동 전환

### 2. 토큰 사용량 최적화

- 8b 모델로 전환: TPM 5배 증가
- 5개 키 사용: 일일 한도 2,500,000 토큰

### 3. 유지보수성 향상

- AI 프로바이더 추상화
- Perplexity 전환 시 환경변수만 변경
- 코드 수정 불필요

### 4. 하위 호환성 유지

- 기존 `callGroq()` 인터페이스 유지
- 점진적 마이그레이션 가능
