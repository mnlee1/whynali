# 병합 기능 제거

날짜: 2026-03-11
상태: ✅ 완료

## 배경

### 병합 기능이 필요했던 이유

초기에는 키워드 그루핑이 완벽하지 않아 같은 이슈가 여러 개로 쪼개져 등록되는 문제가 있었습니다.
이를 해결하기 위해 사후적으로 이슈를 병합하는 기능이 필요했습니다.

### 왜 이제 불필요한가?

#### 1. 키워드 그루핑 개선
- 공통 키워드 2개 이상 → 자동 그루핑
- GROUPING_STOPWORDS 대폭 강화 (범용어 30개+ 추가)
- 그루핑 조건 완화 (1개 + 같은 카테고리)

#### 2. AI 중복 체크 시스템
```
1단계: 정확한 제목 일치
2단계: 공통 키워드 2개 이상 필터링
3단계: 반대어 감지, 숫자 차이 감지
4단계: Groq AI 정밀 비교 (신뢰도 80% 이상)
```

#### 3. 급증 감지 시스템
- 뉴스 급증: 5분간 5건 이상 → AI 중복 체크 → 등록
- 커뮤니티 급증: 10분간 10건 이상 → AI 검증 → 등록
- 두 시스템 모두 최근 24시간 내 중복 이슈 자동 체크

### 결과

**Before**:
```
같은 이슈가 7-8개로 쪼개져 등록 → 관리자가 수동으로 병합
예: 이하늬 곰탕집 논란 8개
```

**After**:
```
처음부터 1개로 정확히 묶여서 등록 → 병합 불필요
```

## 제거 작업

### 1. 병합된 이슈 삭제

```sql
DELETE FROM issues WHERE merged_into_id IS NOT NULL;
```

**결과**: 9개 삭제
- 이하늬 곰탕집 관련 7개
- 민희진 관련 1개
- SK플라즈마 관련 1개

### 2. UI 코드 제거

#### app/admin/issues/page.tsx

**제거된 항목**:
1. STATUS_ORDER에서 '병합됨' 제거
   ```typescript
   // Before
   const STATUS_ORDER = { '대기': 0, '승인': 1, '병합됨': 2, '반려': 3 }
   
   // After
   const STATUS_ORDER = { '대기': 0, '승인': 1, '반려': 2 }
   ```

2. getApprovalDisplay에서 '병합됨' 케이스 제거
   ```typescript
   // 제거됨
   if (issue.approval_status === '병합됨') {
       return {
           label: '병합됨',
           className: 'bg-purple-100 text-purple-700 border-purple-200'
       }
   }
   ```

3. 필터에서 '병합됨' 버튼 제거
   ```typescript
   // Before: 9개 필터
   '전체', '대기', '승인 전체', '자동 승인', '관리자 승인', 
   '반려 전체', '자동 반려', '관리자 반려', '병합됨'
   
   // After: 8개 필터
   '전체', '대기', '승인 전체', '자동 승인', '관리자 승인', 
   '반려 전체', '자동 반려', '관리자 반려'
   ```

4. 테이블 행에서 병합 표시 제거
   ```typescript
   // 제거됨
   {issue.merged_into_id ? (
       <div className="flex flex-col gap-1">
           <span className="text-gray-500 line-through">{title}</span>
           <a href={`/issue/${issue.merged_into_id}`}>
               → 병합된 이슈로 이동
           </a>
       </div>
   ) : (
       <a href={`/issue/${issue.id}`}>{title}</a>
   )}
   ```

### 3. 남은 병합 관련 코드

#### DB 테이블
```sql
-- issues 테이블의 merged_into_id 컬럼은 유지
-- 기존 데이터와의 호환성을 위해 스키마는 그대로 두되
-- 새로운 이슈에는 항상 NULL로 유지
```

#### API 필터
- `app/api/issues/route.ts`: `.is('merged_into_id', null)` 조건 유지
- `app/api/search/route.ts`: `.is('merged_into_id', null)` 조건 유지
- 병합된 이슈가 노출되지 않도록 방어 로직 유지

## 효과

### 1. 관리 효율성 향상
- Before: 중복 이슈 발견 → 수동 병합 필요
- After: 처음부터 1개로 등록 → 병합 불필요

### 2. UI 단순화
- 필터 개수: 9개 → 8개
- 불필요한 상태 제거
- 코드 복잡도 감소

### 3. 데이터 일관성
- `merged_into_id` 컬럼은 항상 null
- 승인 상태는 대기/승인/반려 3가지만
- 상태 흐름 명확화

## 향후 방침

### 병합 기능 완전 제거는 보류

**이유**:
- DB 스키마 변경은 신중해야 함
- 기존 데이터와의 호환성
- 혹시 모를 예외 상황 대비

**유지**:
- `merged_into_id` 컬럼 유지 (NULL로)
- API 필터링 로직 유지 (병합 이슈 제외)

**제거**:
- UI에서 병합 상태 표시 제거
- 관리자 병합 기능 제거 (사용 안 함)

### 모니터링

```sql
-- 주간 점검: 중복 이슈 발생 여부 확인
SELECT title, COUNT(*) as cnt
FROM issues 
WHERE approval_status IN ('대기', '승인')
  AND merged_into_id IS NULL
GROUP BY title
HAVING COUNT(*) > 1;
```

만약 중복 이슈가 발생하면:
1. 키워드 그루핑 로직 개선
2. AI 중복 체크 임계값 조정
3. 필요시 병합 기능 재도입
