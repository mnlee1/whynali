# 팀 작업 분담 가이드

담당A(팀장)와 담당B(팀원)의 브랜치 작업 분담 및 작업 방법.

## 1단계: 최신 코드 받기

```bash
# 프로젝트 폴더로 이동
cd whynali

# 최신 코드 받기
git fetch origin

# develop 브랜치로 전환
git checkout develop

# 최신 상태로 업데이트
git pull origin develop
```

## 2단계: 담당 브랜치 확인

```bash
# 모든 브랜치 보기
git branch -a
```

## 4단계: 작업 시작

### 현재 브랜치 확인
```bash
# 현재 어느 브랜치에 있는지 확인
git branch

# * 표시가 있는 브랜치가 현재 브랜치
# 예: * feature/user-pages
```

### 파일 수정
해당 브랜치의 담당 파일만 수정합니다.

**예시: feature/user-pages 담당 파일**
- `app/page.tsx` (홈페이지)
- `app/issue/[id]/page.tsx` (이슈 상세)
- `app/search/page.tsx` (검색)
- `components/issues/*` (이슈 목록 컴포넌트)
- `components/issue/*` (이슈 상세 컴포넌트)

**주의:** 다른 브랜치의 담당 파일은 수정하지 않기!

## 6단계: 커밋하기

```bash
# 변경된 파일 스테이징
git add .

# 또는 특정 파일만 스테이징
git add app/page.tsx components/issues/IssueList.tsx

# 커밋 (의미있는 메시지 작성)
git commit -m "feat: 이슈 목록 페이지 UI 개선"
```

### 커밋 메시지 규칙
```
feat: 새로운 기능 추가
fix: 버그 수정
refactor: 코드 리팩토링
style: 코드 포맷팅, 세미콜론 등
docs: 문서 수정
test: 테스트 추가/수정
chore: 빌드, 패키지 매니저 수정
```

**예시:**
```bash
git commit -m "feat: 검색 페이지에 필터 기능 추가"
git commit -m "fix: 이슈 상세 페이지 로딩 버그 수정"
git commit -m "refactor: IssueCard 컴포넌트 성능 개선"
```

## 8단계: Pull Request (PR) 생성

### GitHub 웹사이트에서:
1. https://github.com/mnlee1/whynali 접속
2. "Pull requests" 탭 클릭
3. "New pull request" 버튼 클릭
4. **base:** `develop` ← **compare:** `feature/user-pages` 선택
5. PR 제목 작성 (예: "feat: 이슈 목록 페이지 UI 개선")
6. 변경사항 설명 작성
7. "Create pull request" 클릭

### PR 설명 예시:
```markdown
## 변경 내용
- 이슈 목록 페이지 레이아웃 개선
- 검색 필터 기능 추가
- 반응형 디자인 적용

## 테스트
- [x] 로컬에서 테스트 완료
- [x] 모바일 화면 확인

## 스크린샷
(스크린샷 첨부)
```

## 9단계: 코드 리뷰 및 머지

1. 팀장이 코드 리뷰
2. 수정 요청 있으면 → 로컬에서 수정 → 다시 푸시
3. 승인되면 → "Merge pull request" 클릭
4. develop 브랜치에 반영됨

## 10단계: 다음 작업 준비

```bash
# develop 브랜치로 전환
git checkout develop

# 최신 코드 받기 (방금 머지된 내용 포함)
git pull origin develop

# 작업 중이던 브랜치 업데이트
git checkout feature/user-pages
git merge develop

# 또는 새로운 브랜치로 작업 시작
git checkout feature/vote-system
```

## 자주 묻는 질문 (FAQ)

### Q1. 여러 브랜치에서 동시에 작업할 수 있나요?
**A:** 가능하지만 권장하지 않습니다. 한 번에 하나의 브랜치에 집중하는 것이 좋습니다.

### Q2. 실수로 다른 브랜치의 파일을 수정했어요!
**A:** 커밋하기 전이면 되돌릴 수 있습니다.
```bash
# 변경사항 취소
git checkout -- app/admin/page.tsx

# 모든 변경사항 취소
git reset --hard
```

### Q3. develop에 최신 코드가 추가되었어요. 제 브랜치에도 반영하려면?
**A:** 
```bash
# develop 최신 코드 받기
git checkout develop
git pull origin develop

# 내 브랜치로 전환
git checkout feature/user-pages

# develop 내용 가져오기
git merge develop
```

### Q4. 충돌(Conflict)이 발생했어요!
**A:** 같은 파일을 두 명이 수정했을 때 발생합니다.
```bash
# 충돌 파일 확인
git status

# 파일을 열어서 충돌 부분 수정
# <<<<<<< HEAD
# 내 코드
# =======
# 다른 사람 코드
# >>>>>>> develop

# 충돌 해결 후 커밋
git add .
git commit -m "merge: develop 브랜치 병합 및 충돌 해결"
```

### Q5. 푸시가 거부되었어요 (rejected)
**A:** 원격 브랜치에 새 커밋이 있을 때 발생합니다.
```bash
# 원격 브랜치 내용 먼저 받기
git pull origin feature/user-pages

# 충돌 있으면 해결 후
git push origin feature/user-pages
```
