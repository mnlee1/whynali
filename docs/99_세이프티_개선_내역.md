```typescript
function createWordBoundaryRegex(word: string): RegExp {
    const escaped = word.replace(/[.*+?^${}()|[\]\\]/g, '\\$&')
    return new RegExp(`(?:^|\\s|[^\\w가-힣])${escaped}(?:$|\\s|[^\\w가-힣])`, 'i')
}
```

### 4. Rate Limit 메모리 정리
**위치**: 
- `lib/safety.ts` - `cleanupRateLimit()` 함수 추가
- `app/api/cron/cleanup-rate-limit/route.ts` (신규)

**개선 전**:
- 만료된 Rate Limit 항목이 메모리에 계속 누적

**개선 후**:
- 만료된 항목을 주기적으로 정리하는 함수 추가
- Vercel Cron으로 10분마다 자동 정리 가능

```typescript
export function cleanupRateLimit(): void {
    const now = Date.now()
    for (const [userId, record] of rateLimitMap.entries()) {
        if (now > record.resetAt) {
            rateLimitMap.delete(userId)
        }
    }
}
```

**Vercel Cron 설정** (`vercel.json`):
```json
{
  "crons": [{
    "path": "/api/cron/cleanup-rate-limit",
    "schedule": "*/10 * * * *"
  }]
}
```

### 6. 에러 처리 강화
**위치**: `lib/safety.ts` - `loadBannedWords()` 함수

**개선 전**:
- DB 조회 실패 시 에러 전파

**개선 후**:
- try-catch로 에러 처리
- 실패 시 빈 배열 반환 (하드코딩 금칙어는 항상 적용)
- 콘솔 로그로 에러 추적

```typescript
export async function loadBannedWords(adminClient: SupabaseClient): Promise<string[]> {
    try {
        const { data, error } = await adminClient
            .from('safety_rules')
            .select('value')
            .eq('kind', 'banned_word')
        
        if (error) {
            console.error('금칙어 로드 실패:', error)
            return []
        }
        
        return (data ?? []).map((r: { value: string }) => r.value)
    } catch (e) {
        console.error('금칙어 로드 예외:', e)
        return []
    }
}
```
