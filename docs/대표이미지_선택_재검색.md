# 이슈 대표 이미지 선택 및 재검색 기능

## 개요

Unsplash에서 검색한 3개의 이미지 중에서 관리자가 슬라이드에 노출될 대표 이미지를 선택할 수 있고,
3개 모두 마음에 들지 않을 경우 새로운 이미지 세트를 재검색할 수 있는 기능입니다.

## 구현 내용

### 1. DB 마이그레이션

**파일**: `supabase/migrations/add_primary_thumbnail_index.sql`

```sql
ALTER TABLE issues 
ADD COLUMN IF NOT EXISTS primary_thumbnail_index SMALLINT DEFAULT 0;
```

issues 테이블에 `primary_thumbnail_index` 컬럼 추가
- 기본값: 0 (첫 번째 이미지)
- thumbnail_urls 배열 내에서 대표 이미지 위치를 지정

### 2. 타입 정의 수정

**파일**: `types/issue.ts`

```typescript
export interface Issue {
    // ... 기존 필드
    thumbnail_urls?: string[] | null
    primary_thumbnail_index?: number | null
}
```

### 3. API 엔드포인트 추가

#### 3-1. 대표 이미지 선택 API

**파일**: `app/api/admin/issues/[id]/primary-thumbnail/route.ts`

```http
POST /api/admin/issues/[id]/primary-thumbnail
Content-Type: application/json

{
    "index": 0 | 1 | 2
}
```

thumbnail_urls 배열 중 어떤 이미지를 대표로 사용할지 설정

#### 3-2. 이미지 재검색 API

**파일**: `app/api/admin/issues/[id]/refresh-thumbnails/route.ts`

```http
POST /api/admin/issues/[id]/refresh-thumbnails
```

Unsplash에서 새로운 이미지 3개를 다시 검색
- 성공 시 thumbnail_urls 업데이트
- primary_thumbnail_index는 0으로 초기화

### 4. 관리자 UI 수정

**파일**: `components/admin/IssuePreviewDrawer.tsx`

#### 변경사항:

**추가된 상태**:
- `selectedThumbnailIndex`: 현재 선택된 대표 이미지 인덱스
- `isRefreshing`: 재검색 중인지 여부

**추가된 함수**:
- `handlePrimaryThumbnailChange(index)`: 대표 이미지 선택
- `handleRefreshThumbnails()`: 이미지 재검색

**UI 개선**:
- 3개 이미지를 라디오 버튼으로 선택 가능
- 선택된 이미지는 "대표" 뱃지 표시
- "이미지 재검색" 버튼 추가

**Props 추가**:
- `onIssueUpdate?: () => void`: 이미지 변경 시 부모 컴포넌트 새로고침

### 5. 프론트엔드 수정

**파일**: `components/issues/HotIssueHighlight.tsx`

```typescript
// 변경 전
const heroImage = issue.thumbnail_urls?.[0] ?? null

// 변경 후
const heroImage = issue.thumbnail_urls?.[issue.primary_thumbnail_index ?? 0] ?? null
```

슬라이드에서 primary_thumbnail_index에 지정된 이미지를 표시

**파일**: `app/admin/(protected)/issues/page.tsx`

```tsx
<IssuePreviewDrawer
    issue={previewIssue}
    onClose={() => setPreviewIssue(null)}
    onApprove={handleApprove}
    onReject={handleReject}
    onIssueUpdate={fetchIssues}
/>
```

onIssueUpdate prop 추가로 이미지 변경 시 목록 새로고침

### 6. 이슈 승인 시 초기화

**파일**: `app/api/admin/issues/[id]/approve/route.ts`

이슈 승인 시 Unsplash 이미지 생성할 때 primary_thumbnail_index를 0으로 초기화

```typescript
if (thumbnailUrls.length > 0) {
    await supabaseAdmin.from('issues').update({ 
        thumbnail_urls: thumbnailUrls,
        primary_thumbnail_index: 0,
    }).eq('id', id)
}
```

## 사용 방법

### 관리자 페이지에서

1. 이슈 관리 페이지 접속
2. 이슈 "미리보기" 클릭
3. 대표 이미지 섹션에서 3개 이미지 중 하나 선택
4. 마음에 들지 않으면 "이미지 재검색" 버튼 클릭

### 동작 방식

**대표 이미지 선택**:
- 라디오 버튼 클릭 시 즉시 저장
- 선택된 이미지에 "대표" 뱃지 표시
- primary_thumbnail_index 업데이트

**이미지 재검색**:
- "이미지 재검색" 버튼 클릭
- Groq API로 영문 키워드 추출 (무료)
- Unsplash API로 새로운 이미지 3개 검색
- thumbnail_urls 업데이트
- primary_thumbnail_index는 0으로 초기화

## 비용 영향

### 현재 API 한도

**Groq API**:
- 비용: 무료
- 한도: 일일 14,400건
- 현재 사용량: 하루 수백 건

**Unsplash API**:
- Demo 플랜: 50회/시간 (3,600회/일)
- Production 플랜: 5,000회/시간 (무료)

### 예상 비용

**대표 이미지 선택**:
- 비용: $0 (DB 업데이트만)

**이미지 재검색**:
- Groq API: 무료 (키워드 추출 1회)
- Unsplash API: 무료 (Demo 플랜 한도 내)
- 예상 재검색 빈도: 하루 10~20건
- **총 비용: $0**

### 결론

관리자만 사용하므로 남용 우려 없음
무료 한도 내에서 충분히 운영 가능
비용 걱정 없이 사용 가능
