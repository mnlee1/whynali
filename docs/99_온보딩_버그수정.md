# 온보딩 연계 버그 수정 완료

## 수정 개요

신규 유저 display_name 실명 노출 방지 및 신고 배치 날짜 필터 추가

## 수정 내용

### 1. 신규 유저 display_name 실명 노출 방지

#### 문제
- 신규 유저가 소셜 로그인하면 OAuth 실명(name/full_name)이 display_name에 즉시 저장됨
- 온보딩 완료 전까지 실명이 노출될 수 있음
- 온보딩이 display_name을 랜덤 닉네임으로 덮어쓰기 전 상태에서 노출 발생

#### 수정 사항

**A. Auth 트리거 수정**

파일: `supabase/migrations/fix_users_trigger_no_display_name.sql`

- `handle_new_user()` 함수에서 display_name을 NULL로 설정
- 온보딩에서 랜덤 닉네임으로 설정하기 전까지 NULL 유지

```sql
INSERT INTO public.users (id, provider, provider_id, display_name)
VALUES (
    NEW.id,
    provider_name,
    provider_user_id,
    NULL  -- 온보딩에서 랜덤 닉네임으로 설정
)
ON CONFLICT (id) DO NOTHING;
```

**B. 네이버 콜백 수정**

파일: `app/auth/callback/naver/route.ts` (157번째 줄)

```typescript
// 변경 전
await admin.from('users').insert({
    id: userId,
    provider: '네이버',
    provider_id: naverId,
    display_name: name,
})

// 변경 후
await admin.from('users').insert({
    id: userId,
    provider: '네이버',
    provider_id: naverId,
    display_name: null,
})
```

### 2. 신고 일일 배치 날짜 필터 추가

#### 문제
- `sendDailyReportSummary()`에서 comment_reports 조회 시 날짜 필터 없음
- 매일 누적된 전체 신고가 반복 알림됨
- 처리된 신고도 계속 포함될 수 있음

#### 수정 사항

파일: `app/api/cron/daily-generate-content/route.ts` (46-52번째 줄)

```typescript
// 변경 전
const { data: reports } = await supabaseAdmin
    .from('comment_reports')
    .select('comment_id, reason, comments!inner(id, body, visibility, issue_id, discussion_topic_id)')
    .neq('reason', '욕설/혐오')
    .eq('comments.visibility', 'public')

// 변경 후
const yesterday = new Date(Date.now() - 24 * 60 * 60 * 1000).toISOString()

const { data: reports } = await supabaseAdmin
    .from('comment_reports')
    .select('comment_id, reason, comments!inner(id, body, visibility, issue_id, discussion_topic_id)')
    .neq('reason', '욕설/혐오')
    .eq('comments.visibility', 'public')
    .gte('created_at', yesterday)
```

**효과:**
- 최근 24시간 내 신고만 조회
- 중복 알림 방지
- 매일 새로운 신고만 Dooray 전송

## 테스트 방법

### 1. display_name NULL 테스트

1. DB 마이그레이션 실행: `fix_users_trigger_no_display_name.sql`
2. 신규 이메일로 소셜 로그인
3. DB에서 users 테이블 확인: `display_name IS NULL`
4. 온보딩 페이지로 리다이렉트 확인
5. 온보딩 완료 후 display_name에 랜덤 닉네임 설정 확인

### 2. 신고 배치 날짜 필터 테스트

1. comment_reports 테이블에 테스트 데이터 추가
   - 24시간 이내 신고
   - 24시간 이전 신고
2. `/api/cron/daily-generate-content` 엔드포인트 호출
3. Dooray 알림에 24시간 이내 신고만 포함 확인
