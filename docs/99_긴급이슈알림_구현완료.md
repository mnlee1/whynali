### 7. 이슈 등록 시점 즉시 알림

- 뉴스 수집 루트와 커뮤니티 급증 루트 모두에 즉시 알림 로직 추가
- 조건: 화력 30점 이상 + 연예/정치 카테고리 + 대기 상태
- 비동기 처리로 알림 실패 시에도 이슈 등록은 정상 진행

## 알림 조건

**즉시 처리 필요 = 화력 30점 이상 + 카테고리 IN ('연예', '정치')**

### 배경
- 화력 15-29점: 자동 반려 대상 아니지만 즉시 알림 불필요
- 화력 30점 이상:
  - 사회/기술/스포츠: 자동 승인 (알림 불필요)
  - 연예/정치: 관리자 승인 필수 (긴급 알림 필요)

## 알림 채널

### 1. UI 배너 (실시간)
- 관리자 페이지 접속 시 즉시 표시
- 1시간 동안 해제 가능
- 대기 목록으로 바로 이동 기능

### 2. Dooray 메신저 (배치)
- 1시간마다 자동 전송 (GitHub Actions Cron)
- 긴급 이슈 목록과 화력 정보 포함
- 화력에 따른 색상 구분

### 3. Dooray 즉시 알림 (이슈 등록 시)
- 이슈 등록 직후 즉시 Dooray 전송
- 뉴스 수집 + 커뮤니티 급증 모두 지원

## 테스트 방법

### 로컬 테스트

#### 1. Dooray 알림 유틸리티 테스트
```bash
# Node.js 환경에서 직접 테스트
node -e "
const { sendDoorayUrgentAlert } = require('./lib/dooray-notification.ts')
sendDoorayUrgentAlert([{
  id: 'test-id',
  title: '테스트 이슈',
  category: '연예',
  heat_index: 35,
  created_at: new Date().toISOString()
}])
"
```

#### 2. Cron API 직접 호출
```bash
curl -X GET http://localhost:3000/api/cron/send-urgent-alerts \
  -H "Authorization: Bearer YOUR_CRON_SECRET"
```

#### 3. UI 배너 테스트
1. 관리자 페이지 접속: http://localhost:3000/admin/issues
2. 화력 30점 이상 + 연예/정치 이슈가 대기 중이면 배너 표시 확인
3. 닫기 버튼 클릭 후 localStorage 저장 확인
4. 1시간 후 재접속 시 배너 재표시 확인

### 프로덕션 배포 후

#### 1. GitHub Actions Cron 확인
- https://github.com/[your-repo]/actions
- `긴급 이슈 Dooray 알림 (Production)` 워크플로우 확인
- 매시 정각 실행 여부 확인

#### 2. 실제 이슈 발생 시나리오
1. 연예 또는 정치 카테고리 뉴스가 5건 이상 수집됨
2. 화력이 30점 이상으로 계산됨
3. 이슈가 '대기' 상태로 등록됨
4. 즉시 Dooray 알림 전송 (콘솔 로그 확인)
5. 관리자 페이지에 배너 표시
6. 1시간 후 배치 알림 재전송 (중복 가능)

## 주의사항

1. **DOORAY_WEBHOOK_URL 보안**
   - `.env.local.example`에는 포함하지 않음
   - Vercel 환경변수에 직접 설정 필요

2. **알림 중복**
   - 즉시 알림 + 배치 알림으로 같은 이슈가 2번 알림될 수 있음
   - 의도된 동작 (긴급성 강조)

3. **CRON_SECRET 설정**
   - Vercel 환경변수에 설정 필요
   - 로컬 테스트 시에는 주석 처리 가능

4. **화력 재계산**
   - 이슈 등록 후 화력이 변동될 수 있음
   - 배치 알림은 현재 시점의 화력 기준으로 전송
