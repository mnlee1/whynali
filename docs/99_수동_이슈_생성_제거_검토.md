# 수동 이슈 생성 기능 제거 검토

날짜: 2026-03-16

## 현재 상태 분석

### 1. DB 상태
```
총 이슈: 0개
- track_a: 0개
- manual: 0개
- null: 0개
```

**결론:** 현재 DB에 manual 이슈가 전혀 없음

### 2. 코드 현황

**구현되어 있는 부분:**
- `app/api/issues/route.ts` - POST 메서드 (수동 생성 API)
- `lib/validation/issue-creation.ts` - manual 검증 로직
- source_track 타입 정의에 'manual' 포함

**사용되지 않는 부분:**
- 프론트엔드 UI 없음 (제거됨, page.tsx.legacy만 존재)
- 어떤 페이지에서도 POST /api/issues를 호출하지 않음
- 관리자 페이지에 수동 생성 버튼/폼 없음

- 2026-03-16 전체 정리 완료

## 제거 검토

### 제거해도 되는 이유

**1. 트랙 A가 모든 케이스 커버**
```
트랙 A 프로세스 (30분 주기):
1. 커뮤니티 급증 감지 (10분간 10건 이상)
2. AI 검증 (신뢰도 70%)
3. 네이버 뉴스 확인 (팩트 체크)
4. 중복 체크 (4단계)
5. AI 필터링 + 제목 생성
6. 타임라인 자동 생성
7. 화력 계산 → 자동 승인/대기 판단
```

**2. 수동 생성이 필요 없는 이유**
- 커뮤니티 반응이 있는 이슈 = 트랙 A가 자동 감지
- 커뮤니티 반응이 없는 이슈 = 왜난리 서비스 목적에 맞지 않음
  - "대중이 논란하는 이슈"가 핵심 가치
  - 커뮤니티 반응 없으면 논란이 아님

**3. 실제 사용 사례 없음**
- 프론트엔드 UI 없음 (의도적 제거)
- DB에 manual 이슈 0건
- 과거 manual 이슈는 모두 테스트/버그 케이스
- 2개월간 프로덕션 운영에서 필요성 없었음

**4. 유지보수 부담**
- 사용하지 않는 코드 유지
- 검증 로직 복잡도 증가
- 문서화/테스트 대상 증가

### 제거하면 안 되는 케이스 (가상)

**1. 긴급 속보 이슈**
- 커뮤니티 반응이 아직 없는 중요 뉴스
- 예: 대통령 긴급 기자회견 발표 (커뮤니티 반응 전)

**반박:** 
- 트랙 A가 30분마다 실행
- 속보 발표 후 30분 이내에 커뮤니티 반응 발생
- 실제로 필요한 케이스 없었음

**2. 외부 제보 이슈**
- 관리자가 직접 입력하는 이슈

**반박:**
- 왜난리는 "커뮤니티 반응 기반" 서비스
- 커뮤니티 반응 없는 이슈 = 서비스 취지에 맞지 않음
- 외부 제보는 트랙 A가 감지할 때까지 대기

**3. 특정 이벤트/공지**
- 서비스 공지, 특별 이벤트

**반박:**
- 이런 용도라면 별도 기능으로 구현해야 함
- 일반 이슈와 다른 성격

## 제거 계획

### Phase 1: 코드 제거

**1. API 제거**
```bash
# POST 메서드만 제거, GET은 유지
app/api/issues/route.ts
- export async function POST() 전체 제거
```

**2. 검증 로직 간소화**
```bash
lib/validation/issue-creation.ts
- SourceTrack 타입: 'track_a'만 허용
- validateIssueCreation: manual 관련 코드 제거
```

**3. 타입 정의 업데이트**
```bash
모든 파일에서:
- source_track: 'track_a' | 'manual' → 'track_a'
```

### Phase 2: DB 정리

**1. 기존 테이블 확인**
```sql
-- manual 이슈가 없는지 최종 확인
SELECT COUNT(*) FROM issues WHERE source_track = 'manual';
-- 결과: 0건 확인됨
```

**2. CHECK 제약 업데이트 (선택)**
```sql
-- source_track 컬럼의 CHECK 제약 변경
ALTER TABLE issues DROP CONSTRAINT IF EXISTS issues_source_track_check;
ALTER TABLE issues ADD CONSTRAINT issues_source_track_check 
  CHECK (source_track = 'track_a');
```

### Phase 3: 문서 업데이트

**수정 필요 문서:**
- `README.md` - 이슈 생성 방식 설명
- `docs/01_AI기획.md` - 아키텍처 설명
- `docs/05_기술설계_통합.md` - API 명세
- 모든 `source_track` 관련 문서
