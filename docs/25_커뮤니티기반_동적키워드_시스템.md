# 커뮤니티 기반 동적 키워드 추출 시스템

## 변경 일자
2026-02-27

## 구현 내용

### 핵심 아이디어
사용자 요구사항: **실시간으로 키워드를 추출해서 광범위하게 뉴스를 수집**

### 해결 방법
**커뮤니티 인기글 = 실시간 트렌드 감지기**

## 시스템 구조

### 1단계: 커뮤니티에서 키워드 추출
```
더쿠/네이트판 인기글 (최근 1시간)
↓
제목 분석 → 키워드 추출
↓
빈도 분석 + 가중치 계산
↓
상위 15개 키워드 반환
```

**예시:**
```
인기글 제목:
- "원덬이 보고 놀란 오늘 새벽 방탄 정국 라방"
- "정국이랑 윈터"
- "전정국 왤케변함"

추출 키워드:
→ "정국" (점수: 15.2)
→ "방탄" (점수: 8.5)
→ "윈터" (점수: 6.3)
```

### 2단계: 네이버 뉴스 수집
```
기본 키워드 + 커뮤니티 키워드
↓
네이버 API 호출
↓
카테고리별 뉴스 수집
```

**호출 흐름:**
```typescript
// 연예 카테고리
queries = [
    '연예',           // 기본
    '논란', '사과',   // 일반 패턴
    '정국', '윈터',   // 커뮤니티 트렌드!
    '블랙핑크'        // 커뮤니티 트렌드!
]

// 각 키워드당 100건씩 수집
```

## 핵심 기능

### 1. 키워드 추출기 (`dynamic-keyword-extractor.ts`)

#### 기능
- 최근 1시간 인기글 100개 분석
- 제목에서 키워드 추출
- 불용어 제거 (대박, 진짜, ㅋㅋㅋ 등)
- 가중치 계산 (조회수/댓글 수 반영)

#### 점수 계산
```typescript
점수 = 1 + (조회수 / 10,000) + (댓글수 / 100)

예:
- 조회수 50,000 + 댓글 200 = 8점
- 조회수 10,000 + 댓글 50 = 2.5점
```

#### 필터링
```typescript
최소 점수: 2 (2개 이상 글에서 등장)
상위 개수: 15개
카테고리 매칭: 연예/스포츠/정치 자동 분류
```

### 2. 뉴스 수집기 개선 (`naver-news.ts`)

#### 변경 사항
```typescript
// 기존: 고정 키워드
queries = ['연예', '논란', '사과', '민희진', '뉴진스']

// 개선: 동적 키워드
queries = [
    '연예', '논란', '사과',  // 기본
    ...trendingKeywords       // 실시간 추가!
]
```

#### 에러 처리
```typescript
try {
    const keywords = await extractTrendingKeywords()
    queries.push(...keywords)
} catch (error) {
    // 실패 시 기본 키워드만 사용
    console.error('키워드 추출 실패, 기본 키워드 사용')
}
```

## 작동 시나리오

### Case 1: 정국 라방 이슈

```
13:00 - 커뮤니티 반응 시작
    더쿠: "원덬이 보고 놀란 오늘 새벽 방탄 정국 라방" (조회 5만)
    네이트판: "정국이랑 윈터" (조회 3만)

13:05 - 뉴스 수집 Cron 실행
    1) 커뮤니티 분석 → "정국" 키워드 추출 (점수 15.2)
    2) 네이버 API 호출: "정국"
    3) 뉴스 10건 수집 (BTS 정국 음주 라이브 등)

13:10 - 이슈 생성
    뉴스 10건 그루핑 → 이슈 후보 생성
    커뮤니티 자동 연결
    ✅ 정국 라방 이슈 등록 완료
```

### Case 2: 신인 이슈

```
커뮤니티: "신인 배우 A씨 대박" (조회 4만)
↓
키워드: "신인", "배우", "A씨"
↓
네이버 API: "A씨" 검색
↓
뉴스 수집 성공
↓
이슈 생성
```

## 장점

### 1. 실시간 트렌드 포착
- ✅ 커뮤니티가 빠름 (뉴스보다 1~3시간 빠름)
- ✅ "정국", "윈터" 같은 신규 이슈 자동 감지
- ✅ 키워드 관리 불필요

### 2. 법적 안전
- ✅ 네이버 API만 사용
- ✅ 커뮤니티는 이미 수집 중 (기존 기능)
- ✅ HTML 크롤링 없음

### 3. 노이즈 제거
- ✅ 불용어 필터 ("대박", "ㅋㅋㅋ" 제거)
- ✅ 최소 점수 2 (2개 이상 글에서 등장)
- ✅ 기존 화력 필터 유지

### 4. 성능 효율
- ✅ API 호출 증가: +10~15회/5분 (허용 범위)
- ✅ 처리 시간: +2~3초 (키워드 추출)
- ✅ Rate Limit 안전

## 잠재적 이슈

### 1. 커뮤니티 노이즈
**상황:** "대박ㅋㅋ" 같은 무의미 키워드
**대응:** 불용어 필터 (50개 이상)

### 2. 카테고리 오분류
**상황:** 연예 키워드가 스포츠로
**대응:** 카테고리 매칭 로직 (`getCategoryForKeyword`)

### 3. API 호출 증가
**상황:** 키워드 15개 추가 → 호출 증가
**대응:** 
- 카테고리별 필터링 (최대 10개)
- 네이버 제한: 25,000회/일 (여유 있음)

## 롤백 계획

문제 발생 시:
1. `extractTrendingKeywords` 호출 제거
2. 기본 키워드만 사용
3. 또는 `minScore` 높이기 (2 → 5)
