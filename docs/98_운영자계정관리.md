# 운영자 계정 관리

## 운영자 권한 구조

운영자 여부는 Supabase `auth.users` 테이블의 `raw_app_meta_data.is_admin` 플래그로 관리합니다.
이메일 도메인에 의존하지 않으므로, 어떤 방식으로 가입한 계정이든 수동으로 권한을 부여해야 합니다.

## 운영자 추가

Supabase 대시보드 → **SQL Editor** → 아래 쿼리 실행 (이메일과 닉네임만 변경)

```sql
-- 1) admin 플래그 설정
UPDATE auth.users
SET raw_app_meta_data = raw_app_meta_data || '{"is_admin": true}'::jsonb
WHERE email = '추가할_이메일@nhnad.com';

-- 2) 닉네임 등록 (public.users)
INSERT INTO public.users (id, provider, provider_id, display_name)
SELECT id, NULL, id::TEXT, '운영자F'
FROM auth.users
WHERE email = '추가할_이메일@nhnad.com'
ON CONFLICT (id) DO UPDATE SET display_name = '운영자F';
```

> 변경 사항은 다음 로그인(토큰 갱신) 시 적용됩니다.
> 즉시 적용이 필요하면 해당 계정으로 로그아웃 후 재로그인.

## 전체 운영자 일괄 등록 (초기화 후 재등록 시)

```sql
-- 1) admin 플래그 일괄 설정
UPDATE auth.users
SET raw_app_meta_data = raw_app_meta_data || '{"is_admin": true}'::jsonb
WHERE email IN (
    'mnlee@nhnad.com',
    'jeongyun.seo@nhnad.com',
    'ks.kim@nhnad.com'
);

-- 2) 닉네임 일괄 등록
INSERT INTO public.users (id, provider, provider_id, display_name)
SELECT id, NULL, id::TEXT, display_name
FROM auth.users
JOIN (VALUES
    ('mnlee@nhnad.com',          '운영자A'),
    ('jeongyun.seo@nhnad.com',   '운영자B'),
    ('ks.kim@nhnad.com',         '운영자C')
) AS mapping(email, display_name) USING (email)
ON CONFLICT (id) DO UPDATE SET display_name = EXCLUDED.display_name;
```

## 운영자 제거

```sql
-- 운영자 권한 제거 (이메일 기준)
UPDATE auth.users
SET raw_app_meta_data = raw_app_meta_data - 'is_admin'
WHERE email = '제거할_이메일@nhnad.com';
```

## 현재 운영자 목록 확인 (SQL)

```sql
SELECT a.id, a.email, u.display_name, u.provider, u.provider_email, a.created_at, a.last_sign_in_at
FROM auth.users a
LEFT JOIN public.users u ON a.id = u.id
WHERE a.raw_app_meta_data->>'is_admin' = 'true'
ORDER BY a.created_at;
```
