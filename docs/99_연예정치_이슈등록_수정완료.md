# 연예/정치 이슈 등록 문제 수정 완료

> 수정일: 2026-03-11
> 관련 문서: `docs/99_연예정치_이슈등록_검증보고서.md`

## 수정 내역

### 1. approval_status null 이슈 긴급 수정 ✅

**문제:**
- 화력 42점, 65점의 고화력 정치 이슈가 approval_status = null
- null 상태 이슈는 사용자에게 노출되지 않음

**수정 작업:**

```bash
npx ts-node --esm scripts/fix-null-approval-status.ts
```

**결과:**
- null 상태 이슈 12건 발견
- 화력 15점 이상 3건 → 대기 상태로 변경
  - "국힘 尹 정치적 복귀 반대" (65.0점)
  - "국민의힘, '절윤' 공식화" (42.0점)
  - "아이티센클로잇 스포츠 SaaS" (18.0점)
- 화력 15점 미만 9건 → 반려 상태로 변경
- **남은 null 이슈: 0건**

### 2. DB 스키마 기본값 추가 ✅

**문제:**
- approval_status 컬럼에 기본값이 없어 null 발생 가능

**수정:**

```sql
ALTER TABLE issues 
ALTER COLUMN approval_status SET DEFAULT '대기';
```

**파일:** `supabase/migrations/add-approval-status-default.sql`

**실행 방법:**
1. Supabase 대시보드 접속
2. SQL Editor 열기
3. 마이그레이션 파일 내용 복사하여 실행

### 3. 연예 카테고리 자동 승인 허용 ✅

**문제:**
- 연예 카테고리가 자동 승인 대상에서 제외
- 화력 30점 이상이어도 관리자 승인 대기
- 논란 이슈의 실시간성 약화

**수정:**

```bash
# 변경 전
AUTO_APPROVE_CATEGORIES=사회,경제,IT과학,생활문화,세계,스포츠

# 변경 후
AUTO_APPROVE_CATEGORIES=사회,경제,IT과학,생활문화,세계,스포츠,연예
```

**파일:** `.env.local`

**정책 변경 사유:**
- 연예 이슈는 정치에 비해 상대적으로 민감도가 낮음
- 화력 30점 이상이면 충분히 논란 가치가 있음
- Dooray 알림은 유지하여 관리자가 사후 확인 가능
- 실시간 논란 확인이라는 서비스 가치 강화

**정치 카테고리:**
- 정치는 여전히 관리자 승인 필수 유지
- 높은 민감도로 인한 리스크 관리

## 효과 검증

### 변경 전

```
연예: 6건 (평균 화력 13.0점)
- 자동 승인: 0건
- 대기: 다수 (화력 30점 미만)

정치: 10건 (평균 화력 20.7점)
- null 상태: 2건 (화력 42점, 65점)
- 대기: 2건 (화력 23-24점)
```

### 변경 후 예상

```
연예: 
- 화력 30점 이상 → 자동 승인 (실시간 노출)
- 화력 15-29점 → 대기 (관리자 승인)
- Dooray 알림 유지 (화력 30점 이상)

정치:
- null 상태 해소 (대기 상태로 변경)
- 화력 30점 이상 → Dooray 알림 + 관리자 승인
- 타임아웃 자동 승인 검토 필요
```
