# 스포츠/기술 이슈 등록 문제 분석 보고서

작성일: 2026-04-10
분석자: Claude AI Assistant

## 요약

**스포츠와 기술 이슈가 잘 등록되지 않는 원인: 커뮤니티 데이터 수집 크론이 정상 작동하지 않음**

로직 자체는 정상이나, 데이터 수집 파이프라인이 간헐적으로만 실행되어 Track A가 처리할 데이터가 없는 상태입니다.

모든 이슈가 [테스트] 태그가 붙어있어 수동으로 생성된 것으로 추정됩니다.

### 2. Track A 실행 현황

**최근 7일간 Track A 로그: 0건**

- Track A 크론이 실행되지 않았거나, 처리할 데이터가 없어서 로그가 생성되지 않음

### 3. 커뮤니티 데이터 수집 현황

#### 수집량 통계 (최근 7일)

```
총 수집: 1000건
  더쿠: 882건
  클리앙: 98건
  네이트판: 20건

일별 수집:
  2026-04-08: 1000건
```

모든 글이 2026-04-08에 한꺼번에 수집되었습니다.

#### 수집 시간 분석

```
최근 10개 게시글의 작성 시간 vs 수집 시간:

[네이트판] 보고싶다
  작성: 2026-04-08 22:21
  수집: 2026-04-09 05:18
  시차: 6시간

[네이트판] 요즘은 열심히 살아도 불안한 이유가...
  작성: 2026-04-09 04:52
  수집: 2026-04-09 05:18
  시차: 0시간

[네이트판] 라이즈 성찬의 전생 시절...
  작성: 2026-04-08 18:59
  수집: 2026-04-09 05:18
  시차: 10시간
```

**핵심 발견:**
- 모든 글이 정확히 같은 시간(05:18:10)에 수집됨
- 작성 시간은 0~10시간 전으로 다양함
- 수집 크론이 오랫동안 실행되지 않다가 한 번에 실행된 것으로 보임

**최근 1시간 내 수집: 0건**
- 마지막 수집(05:18) 이후 새로운 수집이 없음

### 4. Track A 로직 분석

Track A는 다음과 같이 작동합니다:

```typescript
// 최근 WINDOW_MINUTES(10분) 내 커뮤니티 글 조회
const { data: recentPosts } = await supabaseAdmin
    .from('community_data')
    .select('id, title, created_at, source_site')
    .gte('updated_at', cutoffTime)  // updated_at 기준!
    .is('issue_id', null)
```

**문제점:**
- Track A는 `updated_at`(수집 시간) 기준으로 최근 10분 내 글만 조회
- 커뮤니티 수집이 간헐적으로만 실행되면, 대부분의 시간대에 처리할 데이터가 없음
- 따라서 Track A가 실행되어도 급증 키워드를 감지할 수 없음

## 결론

### 문제의 근본 원인

**커뮤니티 수집 크론(/api/cron/collect-community)이 정상 작동하지 않습니다.**

1. Vercel cron 설정: 5분마다 실행 ("*/5 * * * *")
2. 실제 작동: 간헐적으로만 실행됨 (최근 1시간 동안 0회)
3. 결과: Track A가 처리할 신규 데이터가 없어 이슈 생성 불가

### 로직 검증 결과

**스포츠와 기술 이슈 등록 로직은 정상입니다:**

- ✅ 자동 승인 카테고리에 포함됨
- ✅ AI 카테고리 분류 프롬프트 명확함
- ✅ 카테고리 키워드 충분함
- ✅ Track A 로직 정상 작동
- ✅ 네이버 뉴스 검색 로직 정상

### 실제 원인

**데이터 수집 파이프라인 장애**

커뮤니티 수집이 정상 작동하지 않아서 어떤 카테고리든 이슈가 생성될 수 없는 상태입니다. 
스포츠/기술만의 문제가 아니라, 전체 Track A 시스템이 멈춰있습니다.

## 해결 방안

### 1. 즉시 조치 (긴급)

Vercel 대시보드에서 크론 실행 로그 확인:
- Vercel Dashboard → Project → Cron Jobs
- `/api/cron/collect-community` 실행 이력 확인
- 에러 로그 확인

### 2. 임시 조치

수동으로 커뮤니티 수집 API 실행:
```bash
curl -X GET "https://whynali.com/api/cron/collect-community" \
  -H "Authorization: Bearer YOUR_CRON_SECRET"
```

### 3. 근본 해결

#### 원인 1: Vercel Cron 설정 문제
- Vercel 환경변수 `CRON_SECRET` 확인
- Vercel Cron Jobs 활성화 여부 확인
- 플랜 제한 확인 (Hobby 플랜은 제한 있을 수 있음)

#### 원인 2: 크론 핸들러 에러
```typescript
// app/api/cron/collect-community/route.ts
export async function GET(request: NextRequest) {
    const authError = verifyCronRequest(request)
    if (authError) return authError  // 인증 실패 시 403 반환
    
    try {
        const result = await collectAllCommunity()
        // ...
    } catch (error) {
        // 에러 처리
    }
}
```

- `verifyCronRequest` 인증 로직 확인
- `collectAllCommunity` 함수 에러 확인

#### 원인 3: 외부 API 장애
- 더쿠/네이트판/클리앙 API 상태 확인
- Rate Limiting 여부 확인
- IP 차단 여부 확인
