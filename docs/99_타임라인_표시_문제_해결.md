# 타임라인 표시 문제 해결 완료

## 발견된 문제

관리자 이슈 목록에서 `source_track` 필터가 제대로 적용되지 않아, **레거시 이슈**(트랙 A 이전 방식으로 생성된 이슈)들이 함께 표시되고 있었습니다.

### 레거시 이슈의 특징
- `source_track` 컬럼이 `NULL` 또는 `track_a`가 아닌 값
- 타임라인이 없거나 불완전할 수 있음
- 구 프로세스(auto-create-issue, detect-community-burst)로 생성됨

## 적용된 수정

### 1. 관리자 API 필터 추가
**파일**: `app/api/admin/issues/route.ts`

```typescript
// source_track 필터 적용 (Track A 이슈만 표시)
if (sourceTrack) {
    query = query.eq('source_track', sourceTrack)
}
```

프론트엔드에서 보내는 `source_track=track_a` 파라미터를 이제 정상적으로 처리합니다.

### 2. TimelineSection 디버깅 로그 추가
**파일**: `components/issue/TimelineSection.tsx`

브라우저 콘솔에서 타임라인 로딩 상태를 확인할 수 있도록 로그를 추가했습니다.

```typescript
console.log('[TimelineSection] Fetching timeline for issue:', issueId)
console.log('[TimelineSection] Timeline response:', response)
console.log('[TimelineSection] Timeline data count:', response.data?.length ?? 0)
```

## 확인 방법

### 1. 관리자 페이지에서 확인
1. `/admin/issues` 접속
2. 이제 **Track A 이슈만** 표시됩니다
3. 각 이슈는 타임라인이 있어야 합니다 (최소 1개 이상)

### 2. 이슈 상세 페이지에서 확인
현재 DB에 있는 Track A 이슈:
- **ID**: `c16a001d-3ad0-4d64-8b23-e5426d9507c1`
- **제목**: "WBC 한국 8강 진출"
- **타임라인**: 5개 포인트

확인 URL:
```
/issue/c16a001d-3ad0-4d64-8b23-e5426d9507c1
```

F12 개발자 도구 → Console 탭에서 다음 로그를 확인:
```
[TimelineSection] Fetching timeline for issue: c16a001d-3ad0-4d64-8b23-e5426d9507c1
[TimelineSection] Timeline response: {data: Array(5)}
[TimelineSection] Timeline data count: 5
```

### 3. DB에서 직접 확인
스크립트 실행:
```bash
npx tsx scripts/check-timeline-status.ts
```

## 레거시 이슈 정리

레거시 이슈들이 여전히 DB에 남아있다면 다음 마이그레이션을 실행하세요:

```bash
supabase db push --file supabase/migrations/009_cleanup_legacy_issues.sql
```

이 마이그레이션은:
- `source_track != 'track_a'` 인 이슈들 삭제
- 관련 타임라인, 뉴스, 커뮤니티 연결 해제
- Track A 이슈만 남김

## 예상 결과

이제 모든 이슈 목록 페이지와 상세 페이지에서:
- ✅ Track A 이슈만 표시됨
- ✅ 모든 이슈에 타임라인이 정상적으로 표시됨
- ✅ 타임라인은 발단 → 전개 → 파생 → 진정 순서로 표시됨
- ✅ 각 타임라인 포인트에 제목, 시간, 출처 링크가 있음
