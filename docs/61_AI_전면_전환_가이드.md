# AI 전면 전환 가이드

날짜: 2026-03-11
상태: ✅ 구현 완료

## 문제 인식

키워드 기반 카테고리 분류 방식의 한계:

```
문제점:
├─ 새로운 케이스마다 키워드 추가 필요 ❌
├─ 맥락 이해 부족으로 오분류 발생 ❌
└─ 지속적인 유지보수 부담 ❌

예시:
- "무신사, 스포츠·IP 큐레이션" → 스포츠 오분류
- "스포츠마케팅" → 스포츠 오분류
- 앞으로도 계속 발생 가능...
```

## AI 전면 전환 가능성 분석

### 1. 현재 이슈 생성량

```
최근 7일간: 28개 이슈
하루 평균: 4개 이슈

카테고리별 분포:
- 기술: 12개 (42.9%)
- 정치: 7개 (25.0%)
- 스포츠: 4개 (14.3%)
- 연예: 3개 (10.7%)
- 사회: 2개 (7.1%)
```

### 2. Groq API 사용량 예측

```
토큰 소비:
- 이슈당: ~250 토큰
- 하루: ~1,000 토큰
- 한달: ~30,000 토큰

Groq 무료 플랜 대비:
- 하루 사용률: 0.2% (50만 토큰 중 1,000토큰)
- 한달 사용률: 0.2% (1,500만 토큰 중 30,000토큰)
```

### 3. 결론

**✅ Groq 무료 플랜으로 충분히 감당 가능!**

- 현재 이슈 생성량이 적음
- API 비용 부담 없음
- 유지보수 간편화 효과가 큼

## 구현 방안

### Option 1: AI 전용 모드 (권장) ⭐

**장점:**
- ✅ 키워드 유지보수 완전 불필요
- ✅ 맥락 이해로 정확도 향상
- ✅ 새로운 케이스 자동 대응
- ✅ 하루 1,000토큰 (무료 플랜의 0.2%)

**단점:**
- API 호출 증가 (하루 ~4회)
- 응답 속도 약간 느림 (1-2초)

**설정:**
```bash
# .env.local
ENABLE_AI_CATEGORY=true
CATEGORY_STRATEGY=ai
```

### Option 2: 하이브리드 모드 (기본값)

**장점:**
- API 호출 최소화 (신뢰도 낮을 때만)
- 빠른 응답 속도

**단점:**
- ❌ 키워드 유지보수 필요
- ❌ 새로운 케이스마다 키워드 추가
- ❌ 맥락 이해 부족으로 오분류 발생

**설정:**
```bash
# .env.local
ENABLE_AI_CATEGORY=true
CATEGORY_STRATEGY=hybrid  # 또는 설정 안 함 (기본값)
```

### Option 3: 키워드 전용 모드

**장점:**
- AI API 호출 없음
- 매우 빠른 응답

**단점:**
- ❌ 키워드 유지보수 필수
- ❌ 오분류 발생 가능성 높음

**설정:**
```bash
# .env.local
ENABLE_AI_CATEGORY=false  # 또는 설정 안 함
# CATEGORY_STRATEGY는 무시됨
```

## FAQ

### Q1. Rate Limit에 걸리면?

**A:** 자동으로 '사회' 카테고리로 폴백됩니다. 수동으로 재분류하거나 하이브리드 모드로 변경하세요.

### Q2. 비용이 걱정됩니다.

**A:** 하루 1,000토큰 (무료 플랜의 0.2%)으로 매우 낮습니다. Groq는 무료이며, 이슈 생성량이 100배 증가해도 20% 사용률입니다.

### Q3. 키워드 방식도 유지해야 하나요?

**A:** AI 전용 모드에서는 키워드가 완전히 무시됩니다. 하지만 롤백 대비용으로 키워드는 그대로 유지하는 것을 권장합니다.

### Q4. 하이브리드 vs AI 전용?

**A:** 현재 이슈 생성량(하루 4개)에서는 AI 전용을 권장합니다. 이슈가 하루 50개 이상 생성되면 하이브리드를 고려하세요.

### Q5. 정확도는?

**A:** AI 전용 모드가 키워드 방식보다 높습니다. 맥락을 이해하기 때문입니다.
