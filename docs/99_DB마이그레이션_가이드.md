# DB 마이그레이션 실행 가이드

왜난리 프로젝트의 Supabase DB 마이그레이션 실행 방법.

## 방법 1: Supabase 대시보드 (권장)

가장 간단하고 안전한 방법. 운영 DB에 직접 적용.

### 실행 단계

1. **Supabase 대시보드 접속**
   ```
   https://supabase.com/dashboard
   ```

2. **프로젝트 선택**
   - 왜난리 프로젝트 클릭

3. **SQL Editor 열기**
   - 왼쪽 메뉴에서 "SQL Editor" 클릭
   - 또는 "Database" → "SQL Editor"

4. **마이그레이션 SQL 복사**
   ```bash
   # 터미널에서 실행
   cat supabase/migrations/add_vote_status_and_timestamps.sql
   ```

5. **SQL Editor에 붙여넣기**
   - "New query" 버튼 클릭
   - 복사한 SQL 붙여넣기
   - 쿼리 이름 입력 (예: "투표 상태 및 타임스탬프 추가")

6. **실행**
   - "Run" 버튼 (Ctrl/Cmd + Enter) 클릭
   - 성공 메시지 확인

7. **검증**
   ```sql
   -- votes 테이블 구조 확인
   SELECT column_name, data_type, is_nullable
   FROM information_schema.columns
   WHERE table_name = 'votes'
   ORDER BY ordinal_position;
   ```

### 주의사항

- 운영 DB에 직접 적용되므로 신중히 실행
- 백업 권장 (Supabase는 자동 백업 제공)
- 실행 전 SQL 내용 검토 필수

## 방법 2: Supabase CLI (고급)

로컬 개발 환경과 연동하여 마이그레이션 관리.

### 사전 준비

1. **Supabase CLI 설치 확인**
   ```bash
   supabase --version
   ```

2. **프로젝트 초기화 (최초 1회)**
   ```bash
   cd /Users/nhn/Documents/pub/@react/whynali
   supabase init
   ```

3. **Supabase 프로젝트 연결**
   ```bash
   # 프로젝트 ID 확인: Supabase 대시보드 → Settings → General
   supabase link --project-ref [프로젝트-ID]
   
   # 또는 대화형으로 연결
   supabase login
   supabase link
   ```

### 마이그레이션 실행

```bash
# 1. 로컬에서 테스트 (선택)
supabase start  # 로컬 Supabase 시작
supabase db reset  # 마이그레이션 적용

# 2. 원격 DB에 적용
supabase db push

# 3. 특정 마이그레이션만 실행 (수동)
supabase db push --dry-run  # 미리보기
```

### 새 마이그레이션 생성

```bash
# 새 마이그레이션 파일 생성
supabase migration new [마이그레이션-이름]

# 예시
supabase migration new add_vote_feature
```

## 방법 3: SQL 파일 직접 실행 (간편)

psql 또는 다른 PostgreSQL 클라이언트 사용.

### 1. 연결 정보 확인

Supabase 대시보드 → Settings → Database → Connection string

```
postgresql://[USER]:[PASSWORD]@[HOST]:[PORT]/[DATABASE]
```

### 2. psql로 실행

```bash
# 연결 후 실행
psql "postgresql://[연결문자열]" -f supabase/migrations/add_vote_status_and_timestamps.sql
```

## 현재 프로젝트 마이그레이션 목록

```bash
# 마이그레이션 파일 확인
ls -l supabase/migrations/

# 최신 마이그레이션
add_vote_status_and_timestamps.sql  # 투표 상태 및 타임스탬프 추가
```

## 마이그레이션 검증

실행 후 반드시 검증:

```sql
-- 1. votes 테이블 구조 확인
\d votes

-- 2. 새 컬럼 확인
SELECT 
    column_name, 
    data_type, 
    is_nullable,
    column_default
FROM information_schema.columns
WHERE table_name = 'votes'
    AND column_name IN (
        'issue_status_snapshot',
        'started_at',
        'ended_at'
    );

-- 3. 기존 데이터 마이그레이션 확인
SELECT 
    id,
    phase,
    started_at,
    created_at
FROM votes
LIMIT 5;

-- 4. 인덱스 확인
SELECT 
    indexname,
    indexdef
FROM pg_indexes
WHERE tablename = 'votes';
```

## 롤백 (필요 시)

문제 발생 시 롤백 SQL:

```sql
-- 컬럼 삭제
ALTER TABLE votes DROP COLUMN IF EXISTS issue_status_snapshot;
ALTER TABLE votes DROP COLUMN IF EXISTS started_at;
ALTER TABLE votes DROP COLUMN IF EXISTS ended_at;

-- 인덱스 삭제
DROP INDEX IF EXISTS idx_votes_phase;
DROP INDEX IF EXISTS idx_votes_issue_status_snapshot;
```

## 트러블슈팅

### 연결 오류
```
Error: Failed to connect to database
```
→ 연결 정보 확인, 방화벽 설정 확인

### 권한 오류
```
Error: permission denied
```
→ Supabase 대시보드에서 실행 권장

### 컬럼 중복 오류
```
Error: column already exists
```
→ 이미 실행됨. 검증 쿼리로 확인

## 권장 방법 요약

**운영 환경**: 방법 1 (Supabase 대시보드)
- 가장 안전하고 간단
- 웹 인터페이스로 즉시 실행
- 오류 시 즉시 확인 가능

**개발 환경**: 방법 2 (Supabase CLI)
- 버전 관리 용이
- 로컬 테스트 가능
- 팀 협업에 적합
