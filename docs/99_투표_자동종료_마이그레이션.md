# 투표 자동 종료 기능 마이그레이션 가이드

## 문제 상황

관리자 투표 관리 페이지에서 "투표 조회 실패" 오류가 발생했습니다.

**원인**: 
- 코드에서 `auto_end_date`, `auto_end_participants` 컬럼을 조회하려 했으나
- DB에 해당 컬럼이 아직 생성되지 않음

## 해결 방법

### 1단계: 현재 상태 (임시 수정 완료)

자동 종료 기능 관련 코드를 모두 주석 처리하여 투표 관리 페이지가 정상 동작하도록 수정했습니다.

**수정된 파일:**
- `app/api/admin/votes/route.ts` - auto_end 컬럼 조회 제거
- `components/issue/VoteSection.tsx` - 자동 종료 UI 주석 처리
- `app/admin/votes/page.tsx` - 자동 종료 정보 표시 주석 처리
- `types/index.ts` - Vote 타입에서 auto_end 필드 주석 처리

### 2단계: DB 마이그레이션 실행

Supabase 대시보드에서 아래 SQL을 실행하세요:

```sql
-- 투표 자동 종료 옵션 필드 추가
ALTER TABLE votes ADD COLUMN IF NOT EXISTS auto_end_date TIMESTAMP WITH TIME ZONE;
ALTER TABLE votes ADD COLUMN IF NOT EXISTS auto_end_participants INTEGER;

-- 인덱스 추가 (크론잡 조회 성능 향상)
CREATE INDEX IF NOT EXISTS idx_votes_auto_end_date ON votes(auto_end_date) WHERE phase = '진행중';

-- 코멘트
COMMENT ON COLUMN votes.auto_end_date IS '자동 종료 날짜. 설정된 경우 해당 시각에 투표가 자동으로 마감됨.';
COMMENT ON COLUMN votes.auto_end_participants IS '목표 참여자 수. 설정된 경우 해당 인원 도달 시 투표가 자동으로 마감됨.';
```

또는 Supabase CLI 사용:

```bash
cd /Users/nhn/Documents/pub/@react/whynali

```
