# 트랙A 자동 모니터링 시스템 구축 가이드

**작성일**: 2026-03-13  
**소요 시간**: 10분

## 자동 모니터링 개요

더 이상 수동으로 모니터링 스크립트를 실행할 필요가 없습니다!

### 자동화된 작업

| 작업 | 주기 | 알림 방법 |
|-----|------|---------|
| 일일 헬스 체크 | 매일 오전 9시 | Dooray 메신저 |
| 주간 리포트 | 매주 월요일 10시 | Dooray 메신저 |
| 긴급 알림 | 문제 발생 시 즉시 | Dooray 메신저 |

## 설정 방법

### 1단계: GitHub Secrets 설정 (5분)

GitHub 저장소 → Settings → Secrets and variables → Actions

다음 3개의 Secret을 추가:

```
NEXT_PUBLIC_SUPABASE_URL
SUPABASE_SERVICE_ROLE_KEY  
DOORAY_WEBHOOK_URL
```

**DOORAY_WEBHOOK_URL은 `.env.local`에 있는 값을 복사**하면 됩니다.

### 2단계: GitHub Actions 활성화 확인

저장소에 다음 파일들이 추가되었습니다:

```
.github/workflows/
  ├── track-a-health-check.yml     # 일일 헬스 체크
  └── track-a-weekly-report.yml    # 주간 리포트
```

커밋 후 푸시하면 자동으로 활성화됩니다:

```bash
git add .github/workflows/
git add scripts/send_dooray_health_report.ts
git commit -m "feat: 트랙A 자동 모니터링 시스템 구축"
git push origin develop
```

### 3단계: 수동 테스트 (선택)

GitHub 저장소 → Actions 탭 → "Track-A Health Check" 선택 → "Run workflow"

Dooray로 알림이 오는지 확인하세요.

## 자동 모니터링 내용

### 일일 헬스 체크 (매일 오전 9시)

**체크 항목**:
- ✅ 최근 24시간 트랙A 이슈 생성 개수
- ✅ 커뮤니티 데이터 수집 상태
- ✅ AI API 키 사용 가능 여부
- ✅ 시스템 전반 상태

**Dooray 알림 예시**:

정상 시:
```
✅ 트랙A 일일 헬스 체크

📊 통계 (최근 24시간)
- 생성된 이슈: 2개
- 수집 커뮤니티: 548건
- AI API 키: 5/5개 사용 가능

최근 생성 이슈
- WBC 점수 조작 죄송…대만에서 혐한 마케팅...
- 블랙핑크 지수, 심각한 연기력 오히려 화제...
```

문제 발생 시:
```
⚠️ 트랙A 일일 헬스 체크

📊 통계 (최근 24시간)
- 생성된 이슈: 0개
- 수집 커뮤니티: 0건
- AI API 키: 3/5개 사용 가능

⚠️ 발견된 문제
- 트랙A 이슈 생성 없음 (24시간)
- 커뮤니티 데이터 수집 중단

[GitHub Actions 로그 보기]
```

### 주간 리포트 (매주 월요일 10시)

**포함 내용**:
1. 7일간 트랙A 이슈 생성 트렌드
2. 퍼널 분석 (전환율)
3. 이슈 품질 검증
4. 전체 시스템 무결성

**Dooray 알림**:
- 요약만 메시지로 전송
- 전체 리포트는 GitHub Actions 아티팩트로 저장 (30일간 보관)

## 문제 해결

### Q1: Dooray 알림이 안 와요

**확인 사항**:
1. GitHub Secrets에 `DOORAY_WEBHOOK_URL`이 제대로 등록되었는지
2. GitHub Actions 탭에서 워크플로우가 실행되었는지
3. 워크플로우 로그에서 에러 메시지 확인

**테스트**:
```bash
# 로컬에서 직접 테스트
npx tsx scripts/send_dooray_health_report.ts
```

### Q2: 워크플로우가 실행 안 돼요

**확인 사항**:
1. `.github/workflows/` 폴더가 `main` 또는 `develop` 브랜치에 있는지
2. GitHub Actions가 활성화되어 있는지 (Settings → Actions)
3. YAML 파일 문법 오류 확인

### Q3: 매일 오전 9시가 아니라 다른 시간에 받고 싶어요

`.github/workflows/track-a-health-check.yml` 수정:

```yaml
on:
  schedule:
    - cron: '0 1 * * *'  # 오전 10시 (UTC 1시)
    - cron: '0 13 * * *' # 오후 10시 (UTC 13시)
```
