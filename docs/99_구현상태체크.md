#### 이슈 생성 기준 (02_AI기획_판단포인트.md §4 기준)

**자동 등록 조건:**
- 최근 1시간 동안 동일 이슈 후보로 묶인 수집 건수(뉴스+커뮤니티)가 **5건 이상**
- 자동으로 `issues` 테이블에 `approval_status='대기'`로 등록
- 수집 건에 `issue_id` 자동 연결
- 관리자 대시보드에 해제형 배너 알람 표시

**중복 방지:**
- 최근 24시간 내 같은 제목 이슈가 이미 존재하면 중복 등록 방지
- 승인/반려된 이슈는 재처리하지 않음

**관리자 처리:**
- 이슈 관리 대기 목록에서 승인/반려 처리
- B 방식 적용: 8건 이상이고 6시간 이상 무대응 시 자동 승인
    - 기존 대기 이슈가 있으면 `'승인'`으로 업데이트
    - 없으면 바로 `'승인'`으로 신규 생성
- 5~7건 구간: 자동 승인 대상 아님, 관리자 배치 처리(일 1~2회)에서만 승인/반려

**화력 계산:**
- 이슈 등록 직후 연결된 수집 데이터 기준으로 화력(heat_index) 즉시 계산
- 최소 등록 화력: 15점 미만 자동 반려
- 자동 승인 조건: 화력 30점 이상 + 허용 카테고리(사회/기술/스포츠)

#### ⚠️ 개선 필요

**배너 알람 미구현**
- 기획: 이슈 대기 등록 시 관리자 대시보드 상단에 해제형 배너 표시
- 현재: 배너 없음, 대기 목록만 조회 가능
- 수정 필요: 관리자 레이아웃에 배너 컴포넌트 추가

## 2. 이슈 등록·화력 분석 (07 규격 준수 여부)

### 2.1 이슈 등록 기준

| 항목 | 규격 | 구현 상태 |
|------|------|----------|
| 집계 시간 창 | 최근 3시간 (환경변수 조정 가능) | ✅ CANDIDATE_WINDOW_HOURS (현재 24시간) |
| 알람 조건 | 5건 이상 + 고유 출처 2곳 이상 | ✅ 구현됨 |
| 커뮤니티 반응 필수 | 원래 필수였으나 → 선택으로 완화 | ✅ 매칭 0건도 등록 허용, 화력으로 필터링 |
| 화력 최소 기준 | 15점 미만 자동 반려 | ✅ CANDIDATE_MIN_HEAT_TO_REGISTER=15 |
| 자동 승인 조건 | 화력 30점 이상 + 허용 카테고리 | ✅ 사회/기술/스포츠만 자동 승인 |
| 중복 방지 | 최근 24시간 내 같은 제목 | ✅ AI 기반 중복 체크 (선택) |

### 2.2 화력 지수 계산

| 항목 | 규격 | 구현 상태 |
|------|------|----------|
| 입력 | 메타데이터만 (본문·요약문 미사용) | ✅ 준수 |
| 출력 범위 | 0–100 (정수) | ✅ 준수 |
| 등급 구간 | 높음(70+), 보통(30-69), 낮음(30-) | ✅ 준수 |
| 계산 방식 | 커뮤니티 점화 구조 | ✅ `heat = newsCredibility × (0.3 + 0.7 × community_amp)` |
| 관리자 승인 | 자동 반영 (승인 불필요) | ✅ 준수 |

**화력 계산식 (구현 확인됨)**
```typescript
// lib/analysis/heat.ts 68-74번 줄
const communityAmp = communityHeat <= 3
    ? 0
    : Math.min(1, Math.sqrt(Math.max(0, communityHeat - 3) / 70))

const heatIndex = Math.round(
    Math.min(100, Math.max(0, newsCredibility * (0.3 + 0.7 * communityAmp)))
)
```

### 2.3 이슈 목록 정렬

| 항목 | 규격 | 구현 상태 |
|------|------|----------|
| 기본 정렬 | 최신순 (updated_at DESC) | ✅ `sort=latest` 기본값 |
| 정렬 옵션 | 최신순·화력순 전환 가능 | ⚠️ UI에 화력순 옵션 없음 (API는 지원) |
| 상태 우선순위 | 점화·논란중 > 종결 | ⚠️ 복합 정렬 미구현 (현재 단순 최신순) |

**수정 필요**: 이슈 목록에 화력순 정렬 옵션 추가, 복합 정렬(상태+최신+화력) 구현 검토
