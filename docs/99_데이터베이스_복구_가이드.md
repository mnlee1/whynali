# 데이터베이스 복구 가이드

Supabase 프로젝트가 삭제되거나 데이터 손실이 발생한 경우 복구 방법을 설명합니다.

## 복구 방법 선택

삭제 후 경과 시간과 플랜에 따라 적절한 방법을 선택하세요.

| 상황 | 복구 방법 | 성공 가능성 |
|------|----------|------------|
| 7일 이내 삭제 | 대시보드 복구 | 높음 |
| 7일 경과 (Free) | Support 문의 | 낮음 |
| 7일 경과 (Pro+) | Support 문의 | 중간 |
| 완전 삭제 | 새 프로젝트 생성 | 해당없음 |

## 방법 1: 대시보드에서 복구 (7일 이내)

### 1단계: 삭제된 프로젝트 찾기

1. Supabase 대시보드 접속
   https://supabase.com/dashboard

2. 프로젝트 목록 확인
   - "삭제된 프로젝트" 탭 또는 "Paused" 탭
   - Settings > General에서 "Restore" 옵션

3. 복구 실행
   - "Restore this project" 버튼 클릭
   - 복구 완료까지 몇 분 소요
   - 프로젝트 상태가 "Active"로 변경 확인

### 2단계: 환경변수 원복

.env.local 파일에서 기존 프로젝트 정보 활성화:

```
# 기존 프로젝트 (주석 제거)
NEXT_PUBLIC_SUPABASE_URL=https://your-project-ref.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key-here
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key-here

# 새 프로젝트 (주석 처리)
# NEXT_PUBLIC_SUPABASE_URL=https://new-project-ref.supabase.co
# NEXT_PUBLIC_SUPABASE_ANON_KEY=new-anon-key-here
# SUPABASE_SERVICE_ROLE_KEY=new-service-role-key-here
```

### 3단계: 연결 테스트

```bash
npm run dev
```

확인 사항:
- http://localhost:3000 접속
- 로그인 테스트
- 이슈 목록 정상 표시
- 댓글, 투표 기능 동작

## 방법 2: Support 문의 (7일 경과)

### 문의 방법

1. Support 페이지 접속
   https://supabase.com/dashboard/support

2. 또는 이메일 발송
   help@supabase.io

### 문의 템플릿

```
Subject: Data Recovery Request - Deleted Project

Project Reference: [your-project-ref]
Project URL: https://[your-project-ref].supabase.co
Deletion Date: [삭제한 날짜]
Organization: [조직명]
Current Plan: [Free/Pro/Team/Enterprise]

I accidentally deleted my Supabase project and need to recover the data.
Is there any way to restore the database backup?

Key tables I need:
- issues
- users
- votes
- comments
- news_data
- collectors
```

### 플랜별 복구 가능성

**Free 플랜**
- 7일 보관
- 7일 이후 완전 삭제
- Support 문의 시 복구 어려움

**Pro/Team/Enterprise 플랜**
- Point-in-Time Recovery (PITR) 지원
- 백업 보관 기간 더 김
- Support 팀 복구 가능성 높음

### 삭제 일시 확인

1. 이메일에서 "Project deleted" 검색
2. 정확한 삭제 시간 확인
3. Support 문의 시 정확한 시간 전달

## 방법 3: 새 프로젝트 생성 및 복원

데이터 복구가 불가능한 경우 스키마만 재구축합니다.

### 1단계: 새 프로젝트 생성

1. Supabase 대시보드 접속
   https://supabase.com/dashboard

2. 프로젝트 생성
   - "New Project" 클릭
   - Project name: whynali
   - Database Password: 안전한 비밀번호 설정 (저장 필수)
   - Region: Northeast Asia (Seoul) 권장

3. 프로젝트 생성 완료 대기 (약 2분)

### 2단계: API 정보 복사

Settings > API에서 다음 정보 복사:

- Project URL
- anon/public key
- service_role key

### 3단계: 환경변수 업데이트

.env.local 파일 수정:

```
NEXT_PUBLIC_SUPABASE_URL=<새 Project URL>
NEXT_PUBLIC_SUPABASE_ANON_KEY=<새 anon key>
SUPABASE_SERVICE_ROLE_KEY=<새 service_role key>
```

### 4단계: 스키마 복원

1. Supabase 대시보드에서 SQL Editor 열기

2. supabase/dev_setup.sql 파일 내용 복사

3. SQL Editor에 붙여넣기 후 RUN 실행

4. 실행 완료 확인
   - 테이블 생성 확인
   - RLS 정책 적용 확인
   - 함수 생성 확인

### 5단계: 인증 설정

Authentication > Providers에서 소셜 로그인 설정:

**Google**
- Client ID 입력
- Client Secret 입력
- Redirect URL: <새 Project URL>/auth/v1/callback

**Kakao**
- Client ID 입력
- Redirect URL: <새 Project URL>/auth/v1/callback

**Naver**
- Client ID 입력
- Client Secret 입력
- Redirect URL: <새 Project URL>/auth/v1/callback

### 6단계: Storage 버킷 생성

Storage > Create bucket:

- Bucket name: shortform
- Public bucket: 체크
- 생성 완료

### 7단계: RLS 정책 적용

1. SQL Editor에서 supabase/rls_setup.sql 실행

2. 정책 적용 확인
   - Table Editor에서 각 테이블 선택
   - Policies 탭 확인

### 8단계: 초기 데이터 입력

개발 환경에서 테스트 데이터 필요 시:

1. SQL Editor에서 supabase/dummy_data.sql 실행

2. 금지어 데이터 입력
   - supabase/seeds/initial_banned_words.sql 실행

### 9단계: 연결 테스트

```bash
npm run dev
```

확인 사항:
- 로그인 테스트 (Google/Kakao/Naver)
- 이슈 등록 테스트
- 댓글 작성 테스트
- 투표 기능 테스트
- 프로필 수정 테스트

## 대안: 부분 데이터 복구

데이터 전체 복구가 불가능한 경우 일부 데이터를 복구할 수 있습니다.

### 브라우저 캐시 확인

1. 개발자 도구 열기 (F12)

2. Application 탭
   - Local Storage 확인
   - Session Storage 확인
   - IndexedDB 확인

3. 데이터 추출
   - JSON 형태로 저장된 데이터 복사
   - SQL INSERT문으로 변환

### 로컬 개발 데이터

1. .next/cache 폴더 확인
   - 페이지 캐시 데이터

2. node_modules/.cache 확인
   - 빌드 캐시 데이터

### 팀원 데이터

다른 팀원이 로컬에서 개발 중이었다면:
- 로컬 Supabase 인스턴스 데이터
- 개발 중 생성한 테스트 데이터

### 백업 파일

프로젝트 폴더에서 백업 파일 확인:
- *.dump 파일
- *.sql 파일
- backups/ 폴더

## 복구 후 필수 작업

### 백업 설정

자동 백업 설정으로 재발 방지:

1. 프로덕션 데이터 정기 백업
   - 매일 자동 백업 스크립트 실행
   - pg_dump 사용

2. 중요 데이터 Export
   - CSV 형태로 정기 다운로드
   - 별도 저장소에 보관

### 모니터링 설정

1. Supabase 대시보드에서 알림 설정
   - Database > Settings > Alerts
   - 프로젝트 상태 이메일 알림

2. 개발 환경과 프로덕션 분리
   - 개발: 로컬 Supabase
   - 프로덕션: 클라우드 Supabase

### 삭제 보호 설정

1. Organization Settings > Projects
   - 프로젝트 삭제 보호 활성화 (Pro 플랜 이상)

2. 팀원 권한 관리
   - Owner 권한 최소화
   - Developer 권한으로 제한

## 주의사항

1. 삭제 후 빠르게 복구 시도
   - 시간이 지날수록 복구 가능성 낮아짐

2. 프로덕션 환경 보호
   - 절대 프로덕션 프로젝트 삭제 금지
   - 삭제 전 반드시 백업 확인

3. 환경변수 관리
   - .env.local은 절대 git에 커밋 금지
   - API 키 노출 주의

4. 정기 백업
   - 중요 데이터는 별도 백업
   - 자동화 스크립트 사용 권장
