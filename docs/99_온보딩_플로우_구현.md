# 소셜 로그인 온보딩 플로우 구현 완료

## 구현 개요

신규 유저가 소셜 로그인 후 약관 동의와 닉네임 설정을 완료하는 온보딩 플로우를 구현했습니다.

## 구현 파일 목록

### 1. DB 마이그레이션

**supabase/migrations/add_terms_columns_to_users.sql**
- users 테이블에 terms_agreed_at, marketing_agreed 컬럼 추가
- terms_agreed_at IS NULL = 신규 유저 (온보딩 필요)

**supabase/migrations/create_users_on_auth_signup.sql**
- Auth 유저 가입 시 users 테이블 자동 생성 트리거
- Google, Kakao, Naver 모든 소셜 로그인 지원

### 2. 랜덤 닉네임 생성 유틸

**lib/random-nickname.ts**
- 형식: 형용사 + 동물 + 4자리숫자
- generateRandomNickname(): 랜덤 닉네임 1개 생성
- generateUniqueNickname(supabase): DB 중복 확인 후 유니크 닉네임 반환 (최대 5회 시도)
- 형용사 25개, 동물 25개 풀 제공

### 3. 온보딩 페이지

**app/onboarding/page.tsx**
- 서버 컴포넌트: 로그인 확인, terms_agreed_at 체크
- 미로그인 시 /login 리다이렉트
- 이미 온보딩 완료 시 / 리다이렉트
- 초기 닉네임 생성

**app/onboarding/OnboardingClient.tsx**
- 클라이언트 컴포넌트: UI 인터랙션
- 닉네임 재생성 버튼 (최대 3회)
- 약관 동의 섹션 (전체 동의, 필수 2개, 선택 1개)
- 약관 내용 토글 (서비스 이용약관, 개인정보 처리방침)
- 시작하기 버튼 (필수 약관 미동의 시 비활성)

### 4. 온보딩 API

**app/api/onboarding/route.ts**
- POST: 온보딩 완료 처리
- 닉네임 중복 확인 (409 + 새 닉네임 제안)
- users 테이블 업데이트: display_name, terms_agreed_at, marketing_agreed

**app/api/onboarding/nickname/route.ts**
- GET: 온보딩 중 닉네임 재생성

### 5. 닉네임 변경 API

**app/api/users/nickname/route.ts**
- PATCH: 마이페이지용 닉네임 변경
- 2~10자 유효성 검증
- 특수문자 제한 (한글, 영문, 숫자, 언더스코어만)
- 중복 확인

### 6. OAuth 콜백 수정

**app/auth/callback/route.ts**
- Google, Kakao OAuth 콜백
- 신규 유저 (terms_agreed_at IS NULL) 감지 후 /onboarding 리다이렉트
- 기존 유저는 next 파라미터로 리다이렉트

**app/auth/callback/naver/route.ts**
- Naver OAuth 콜백
- 신규 유저 생성 시 users 테이블에 레코드 추가

**app/auth/verify/page.tsx**
- Naver OAuth 매직 링크 verify 페이지
- 세션 설정 후 terms_agreed_at 체크
- 신규 유저는 /onboarding 리다이렉트

## 사용 흐름

1. 사용자가 소셜 로그인 (Google, Kakao, Naver)
2. OAuth 콜백 처리
3. **신규 유저**: users 테이블 생성 (트리거) + /onboarding 리다이렉트
4. **기존 유저**: / 또는 next 파라미터로 리다이렉트
5. 온보딩 페이지:
   - 랜덤 닉네임 표시
   - 다른 닉네임 추천 버튼 (최대 3회)
   - 약관 동의 체크박스
   - 시작하기 버튼 클릭
6. 온보딩 완료 후 메인 페이지로 이동

## 테스트 방법

1. DB 마이그레이션 실행
2. 로컬 서버 실행: npm run dev
3. 신규 이메일로 소셜 로그인
4. /onboarding 페이지로 자동 리다이렉트 확인
5. 닉네임 재생성 테스트 (최대 3회)
6. 약관 동의 후 시작하기 클릭
7. 메인 페이지로 이동 확인
8. 동일 계정으로 재로그인 시 온보딩 스킵 확인

## 주의 사항

1. DB 마이그레이션을 먼저 실행해야 합니다.
2. 기존 users 테이블에 terms_agreed_at 컬럼이 없으면 모든 유저가 온보딩으로 리다이렉트됩니다.
3. 기존 유저는 terms_agreed_at = NOW()로 업데이트하여 온보딩 스킵 처리해야 합니다.

## 기존 유저 마이그레이션 쿼리

기존 유저가 온보딩을 스킵하도록 하려면:

UPDATE users SET terms_agreed_at = created_at WHERE terms_agreed_at IS NULL;
