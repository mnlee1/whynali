# 오픈ID (소셜 로그인) 가이드

구글, 네이버, 카카오 소셜 로그인 설정 및 운영 가이드.

## 키 발급 순서 (카카오 → 네이버 → 구글)

### 0. Supabase 콜백 URL 확인 (공통)

1. [Supabase 대시보드](https://supabase.com/dashboard) → 프로젝트 선택
2. **Authentication** → **URL Configuration**
3. **Redirect URLs**에 전부 등록:
   - `http://localhost:3000/auth/callback`
   - `http://localhost:3000/auth/verify`
   - `https://dev-whynali.vercel.app/auth/callback`
   - `https://dev-whynali.vercel.app/auth/verify`
   - `https://whynali.vercel.app/auth/callback`
   - `https://whynali.vercel.app/auth/verify`
4. **Site URL**: `http://localhost:3000` (로컬) 또는 `https://whynali.vercel.app` (운영)

**Supabase 콜백 URL 복사:**
- **Authentication** → **Providers** → **Kakao** 또는 **Google**
- 상단 안내된 **Callback URL** (예: `https://xxxx.supabase.co/auth/v1/callback`) 복사

## 1. 카카오

### 앱 생성
1. [카카오 개발자 콘솔](https://developers.kakao.com) 로그인
2. **내 애플리케이션** → **애플리케이션 추가하기**
3. 앱 이름·사업자명 입력 후 저장

### 카카오 로그인 활성화
1. 왼쪽 **제품** → **카카오 로그인**
2. **활성화 설정** ON

### 동의항목 설정
왼쪽 **제품** → **카카오 로그인** → **동의항목**:

| 항목 | 설정 | 비고 |
|------|------|------|
| 닉네임 | 필수 동의 | 댓글·화면에 표시 |
| 프로필 사진 | 필수 또는 선택 | 선택으로 변경 가능 |
| 카카오계정(이메일) | 권한 없음 | 비활성화 상태 (설정 불가) |
| 친구목록 | 사용 안 함 | 반드시 사용 안 함 |
| 나머지 항목 | 권한 없음 | 건드리지 않음 |

### 리다이렉트 URI 등록
**왼쪽 [앱] → [플랫폼 키]:**
1. **REST API 키** 복사 (Supabase Client ID에 사용)
2. **로그인 리다이렉트 URI** 클릭
3. Supabase 콜백 URL 입력 (`https://...supabase.co/auth/v1/callback`)
4. **저장**

### 사이트 도메인 등록
**플랫폼 키** → **web** 섹션:
- `http://localhost:3000`
- `https://dev-whynali.vercel.app`
- `https://whynali.vercel.app`

### 클라이언트 시크릿 생성
왼쪽 **제품** → **카카오 로그인** → **보안**:
1. **클라이언트 시크릿** 영역에서 **코드 생성**
2. 상태 **사용함**
3. 생성된 값 복사 (한 번만 표시)

### Supabase 입력
1. Supabase **Authentication** → **Providers** → **Kakao**
2. **Enable Sign in with Kakao** ON
3. **Client ID**: REST API 키
4. **Client Secret**: 위에서 생성한 값
5. **Save**

### KOE205 오류 해결
증상: "설정하지 않은 동의 항목: account_email"

해결: 로그인 요청 시 `scopes`를 `profile_nickname`, `profile_image`만 지정
(이미 `app/login/page.tsx`에 적용됨)

## 2. 네이버

### 애플리케이션 등록
1. [네이버 개발자센터](https://developers.naver.com/apps) 로그인
2. 기존 앱에 **네이버 로그인** 추가 또는 새로 등록
3. **Callback URL** 등록:
   - `http://localhost:3000/auth/callback/naver`
   - `https://dev-whynali.vercel.app/auth/callback/naver`
   - `https://whynali.vercel.app/auth/callback/naver`

### 제공 정보 선택
| 항목 | 설정 | 이유 |
|------|------|------|
| 이메일 | 필수 | Supabase 계정 생성·식별 |
| 이름 | 필수 | 사용자 표시 |
| 닉네임 | 선택 | 이름 없을 때 대신 표시 |
| 프로필 사진 | 선택 | 헤더 프로필 이미지 |
| 성별, 연령대 등 | 요청 안 함 | 미사용 |

### Client ID·Secret 확인
애플리케이션 정보에서 **Client ID**, **Client Secret** 복사

### .env.local 설정
```
NAVER_CLIENT_ID=여기에_Client_ID
NAVER_CLIENT_SECRET=여기에_Client_Secret
```

서버 재시작 후 테스트 (Supabase 설정 불필요)

## 3. 구글

### OAuth 동의 화면
1. [Google Cloud Console](https://console.cloud.google.com) 로그인
2. 프로젝트 생성 또는 선택
3. **API 및 서비스** → **OAuth 동의 화면**
4. **외부** 선택 → **만들기**
5. 앱 정보 입력
6. **범위 추가 또는 삭제**에서 체크:
   - `.../auth/userinfo.email`
   - `.../auth/userinfo.profile`
   - `openid`
7. **저장 후 계속**

### OAuth 클라이언트 ID
1. **API 및 서비스** → **사용자 인증 정보**
2. **+ 사용자 인증 정보 만들기** → **OAuth 클라이언트 ID**
3. **애플리케이션 유형**: 웹 애플리케이션
4. **승인된 리디렉션 URI**: Supabase 콜백 URL 입력
   (예: `https://프로젝트참조ID.supabase.co/auth/v1/callback`)
5. **만들기** → **클라이언트 ID**, **클라이언트 보안 비밀** 복사

### Supabase 입력
1. Supabase **Authentication** → **Providers** → **Google**
2. **Enable Sign in with Google** ON
3. **Client ID**: 구글 클라이언트 ID
4. **Client Secret**: 구글 클라이언트 보안 비밀
5. **Save**

## 요약표

| 순서 | 플랫폼 | 키 발급처 | Redirect URI | 키 넣는 곳 |
|------|--------|-----------|--------------|------------|
| 1 | 카카오 | developers.kakao.com | Supabase 콜백 URL | Supabase Providers → Kakao |
| 2 | 네이버 | developers.naver.com | `.../auth/callback/naver` | .env.local |
| 3 | 구글 | console.cloud.google.com | Supabase 콜백 URL | Supabase Providers → Google |

## 공통 유의사항

### 리다이렉트 URI
- 정확히 등록 필요 (오타·만료 URL 불가)
- 네이버는 완전 일치 검증 (앞부분만 맞으면 안 됨)

### Client Secret 보안
- 서버에서만 사용
- `NEXT_PUBLIC_` 환경 변수에 넣지 말 것
- 클라이언트 노출 시 앱 도용 위험

### CSRF 방지
- OAuth 요청 시 `state` 파라미터 사용 권장

### 토큰·세션
- Supabase Auth가 세션을 쿠키로 관리
- 액세스 토큰은 서버 또는 Supabase가 관리

## 배포 전 체크리스트

- [ ] 네이버: Callback URL 정확 등록, 동의 항목 최소화
- [ ] 카카오: Redirect URI(Supabase 콜백) 등록, 동의 항목 설정
- [ ] 구글: 리디렉션 URI(Supabase 콜백) 등록, 동의 화면 구성
- [ ] Supabase: Google·Kakao 활성화 및 키 입력
- [ ] Supabase: Redirect URLs에 `/auth/callback`, `/auth/verify` 포함
- [ ] 환경 변수: Client Secret은 서버 전용, 저장소 노출 방지
