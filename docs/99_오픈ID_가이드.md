# 오픈ID (소셜 로그인) 가이드

구글, 네이버, 카카오 소셜 로그인 설정 및 운영 가이드.

## 개인정보 수집 및 사용 목적

### 수집하는 정보 요약

| 플랫폼 | 수집 정보 | 저장 정보 | 사용 목적 |
|--------|-----------|-----------|-----------|
| 구글 | 이메일, 프로필 이름, Google User ID | 이메일, 프로필 이름, provider_id | 로그인 인증, 닉네임 표시, 관리자 권한 확인 |
| 카카오 | 이메일(선택), 닉네임, 프로필 사진(선택), Kakao User ID | 이메일, 닉네임, provider_id | 로그인 인증, 닉네임 표시, 관리자 권한 확인 |
| 네이버 | 이메일, 이름, 닉네임(선택), 프로필 사진(선택), Naver User ID | 이메일, 이름 또는 닉네임, provider_id | 로그인 인증, 닉네임 표시, 관리자 권한 확인 |

### 구글 (Google OAuth 2.0)

**제공받는 정보 (OAuth 범위)**:
- 이메일 주소 (`userinfo.email`)
- 기본 프로필 정보 (`userinfo.profile`): 이름, 프로필 사진 URL, Google User ID
- OpenID 식별자 (`openid`)

**실제 저장 정보**:
- 이메일 주소
- 프로필 이름 (full_name 또는 name)
- Google User ID (provider_id)
- 로그인 제공자: "구글"

**사용 목적**:
- 로그인 인증 및 계정 식별
- 댓글/투표 작성자 닉네임 표시
- 관리자 권한 확인 (ADMIN_EMAILS 비교)

**수집하지 않는 정보**:
- 전화번호, 주소, 생년월일, 성별
- 구글 서비스 데이터 (연락처, 캘린더, 드라이브 등)

### 카카오 (Kakao OAuth 2.0)

**제공받는 정보 (동의 항목)**:
- 닉네임 (필수 동의)
- 프로필 사진 (선택 동의)
- 카카오계정 이메일 (권한 없음, 현재 수집 불가)
- Kakao User ID

**실제 저장 정보**:
- 닉네임
- Kakao User ID (provider_id)
- 이메일 (수집 가능하면 저장, 현재는 권한 없음)
- 로그인 제공자: "카카오"

**사용 목적**:
- 로그인 인증 및 계정 식별
- 댓글/투표 작성자 닉네임 표시
- 관리자 권한 확인 (이메일 있는 경우)

**수집하지 않는 정보**:
- 친구 목록, 카카오톡 메시지
- 성별, 연령대, 생일
- 전화번호, 주소

**참고**:
- 카카오는 이메일 수집 권한이 없어 닉네임으로만 식별
- 앱 검수 진행 중으로 일시적으로 로그인 제한됨

### 네이버 (Naver OAuth 2.0)

**제공받는 정보 (제공 항목)**:
- 이메일 (필수)
- 이름 (필수)
- 닉네임 (선택)
- 프로필 사진 (선택)
- Naver User ID

**실제 저장 정보**:
- 이메일 주소
- 이름 또는 닉네임 (우선순위: nickname → name)
- Naver User ID (provider_id)
- 로그인 제공자: "네이버"

**사용 목적**:
- 로그인 인증 및 계정 식별
- 댓글/투표 작성자 닉네임 표시
- 관리자 권한 확인 (ADMIN_EMAILS 비교)

**수집하지 않는 정보**:
- 성별, 연령대, 생일
- 전화번호, 주소
- 네이버 서비스 데이터

**참고**:
- 테스트 단계로 테스터 ID 등록 필요

### 공통 데이터 보관 및 보안

**저장 위치**:
- Supabase Auth 테이블: 이메일, provider 정보 (암호화)
- public.users 테이블: 닉네임, 약관 동의 시각

**보안**:
- OAuth 2.0 표준 프로토콜 사용
- 비밀번호 저장하지 않음
- 세션 암호화된 쿠키로 관리
- RLS (Row Level Security) 정책으로 데이터 접근 제어

**개인정보 제3자 제공**:
- 제3자에게 제공하지 않음
- 법원 명령, 수사기관 요청 등 법적 의무가 있는 경우에만 제공

**사용자 권리**:
- 정보 확인: 로그인 후 프로필에서 본인 정보 확인 가능
- 정보 수정: 닉네임 변경 가능 (설정 페이지)
- 계정 삭제: 탈퇴 시 개인정보 삭제, 작성 콘텐츠는 "탈퇴한 사용자"로 표시

## 키 발급 순서 (카카오 → 네이버 → 구글)

### 0. Supabase 콜백 URL 확인 (공통)

1. [Supabase 대시보드](https://supabase.com/dashboard) → 프로젝트 선택
2. **Authentication** → **URL Configuration**
3. **Redirect URLs**에 전부 등록:
   - `http://localhost:3000/auth/callback`
   - `http://localhost:3000/auth/verify`
   - `https://dev-whynali.vercel.app/auth/callback`
   - `https://dev-whynali.vercel.app/auth/verify`
   - `https://whynali.vercel.app/auth/callback`
   - `https://whynali.vercel.app/auth/verify`
4. **Site URL**: `http://localhost:3000` (로컬) 또는 `https://whynali.vercel.app` (운영)

**Supabase 콜백 URL 복사:**
- **Authentication** → **Providers** → **Kakao** 또는 **Google**
- 상단 안내된 **Callback URL** (예: `https://xxxx.supabase.co/auth/v1/callback`) 복사

## 1. 카카오

### 앱 생성
1. [카카오 개발자 콘솔](https://developers.kakao.com) 로그인
2. **내 애플리케이션** → **애플리케이션 추가하기**
3. 앱 이름·사업자명 입력 후 저장

### 카카오 로그인 활성화
1. 왼쪽 **제품** → **카카오 로그인**
2. **활성화 설정** ON

### 동의항목 설정
왼쪽 **제품** → **카카오 로그인** → **동의항목**:

| 항목 | 설정 | 비고 |
|------|------|------|
| 닉네임 | 필수 동의 | 댓글·화면에 표시 |
| 프로필 사진 | 필수 또는 선택 | 선택으로 변경 가능 |
| 카카오계정(이메일) | 권한 없음 | 비활성화 상태 (설정 불가) |
| 친구목록 | 사용 안 함 | 반드시 사용 안 함 |
| 나머지 항목 | 권한 없음 | 건드리지 않음 |

### 리다이렉트 URI 등록
**왼쪽 [앱] → [플랫폼 키]:**
1. **REST API 키** 복사 (Supabase Client ID에 사용)
2. **로그인 리다이렉트 URI** 클릭
3. Supabase 콜백 URL 입력 (`https://...supabase.co/auth/v1/callback`)
4. **저장**

### 사이트 도메인 등록
**플랫폼 키** → **web** 섹션:
- `http://localhost:3000`
- `https://dev-whynali.vercel.app`
- `https://whynali.vercel.app`

### 클라이언트 시크릿 생성
왼쪽 **제품** → **카카오 로그인** → **보안**:
1. **클라이언트 시크릿** 영역에서 **코드 생성**
2. 상태 **사용함**
3. 생성된 값 복사 (한 번만 표시)

### Supabase 입력
1. Supabase **Authentication** → **Providers** → **Kakao**
2. **Enable Sign in with Kakao** ON
3. **Client ID**: REST API 키
4. **Client Secret**: 위에서 생성한 값
5. **Save**

### KOE205 오류 해결
증상: "설정하지 않은 동의 항목: account_email"

해결: 로그인 요청 시 `scopes`를 `profile_nickname`, `profile_image`만 지정
(이미 `app/login/page.tsx`에 적용됨)

## 2. 네이버

### 애플리케이션 등록
1. [네이버 개발자센터](https://developers.naver.com/apps) 로그인
2. 기존 앱에 **네이버 로그인** 추가 또는 새로 등록
3. **Callback URL** 등록:
   - `http://localhost:3000/auth/callback/naver`
   - `https://dev-whynali.vercel.app/auth/callback/naver`
   - `https://whynali.vercel.app/auth/callback/naver`

### 제공 정보 선택
| 항목 | 설정 | 이유 |
|------|------|------|
| 이메일 | 필수 | Supabase 계정 생성·식별 |
| 이름 | 필수 | 사용자 표시 |
| 닉네임 | 선택 | 이름 없을 때 대신 표시 |
| 프로필 사진 | 선택 | 헤더 프로필 이미지 |
| 성별, 연령대 등 | 요청 안 함 | 미사용 |

### Client ID·Secret 확인
애플리케이션 정보에서 **Client ID**, **Client Secret** 복사

### .env.local 설정
```
NAVER_CLIENT_ID=여기에_Client_ID
NAVER_CLIENT_SECRET=여기에_Client_Secret
```

서버 재시작 후 테스트 (Supabase 설정 불필요)

## 3. 구글

### OAuth 동의 화면
1. [Google Cloud Console](https://console.cloud.google.com) 로그인
2. 프로젝트 생성 또는 선택
3. **API 및 서비스** → **OAuth 동의 화면**
4. **외부** 선택 → **만들기**
5. 앱 정보 입력
6. **범위 추가 또는 삭제**에서 체크:
   - `.../auth/userinfo.email`
   - `.../auth/userinfo.profile`
   - `openid`
7. **저장 후 계속**

### OAuth 클라이언트 ID
1. **API 및 서비스** → **사용자 인증 정보**
2. **+ 사용자 인증 정보 만들기** → **OAuth 클라이언트 ID**
3. **애플리케이션 유형**: 웹 애플리케이션
4. **승인된 리디렉션 URI**: Supabase 콜백 URL 입력
   (예: `https://프로젝트참조ID.supabase.co/auth/v1/callback`)
5. **만들기** → **클라이언트 ID**, **클라이언트 보안 비밀** 복사

### Supabase 입력
1. Supabase **Authentication** → **Providers** → **Google**
2. **Enable Sign in with Google** ON
3. **Client ID**: 구글 클라이언트 ID
4. **Client Secret**: 구글 클라이언트 보안 비밀
5. **Save**

## 공통 유의사항

### 리다이렉트 URI
- 정확히 등록 필요 (오타·만료 URL 불가)
- 네이버는 완전 일치 검증 (앞부분만 맞으면 안 됨)

### Client Secret 보안
- 서버에서만 사용
- `NEXT_PUBLIC_` 환경 변수에 넣지 말 것
- 클라이언트 노출 시 앱 도용 위험

### CSRF 방지
- OAuth 요청 시 `state` 파라미터 사용 권장

### 토큰·세션
- Supabase Auth가 세션을 쿠키로 관리
- 액세스 토큰은 서버 또는 Supabase가 관리

## 배포 전 체크리스트

- [ ] 네이버: Callback URL 정확 등록, 동의 항목 최소화
- [ ] 카카오: Redirect URI(Supabase 콜백) 등록, 동의 항목 설정
- [ ] 구글: 리디렉션 URI(Supabase 콜백) 등록, 동의 화면 구성
- [ ] Supabase: Google·Kakao 활성화 및 키 입력
- [ ] Supabase: Redirect URLs에 `/auth/callback`, `/auth/verify` 포함
- [ ] 환경 변수: Client Secret은 서버 전용, 저장소 노출 방지
- [ ] 개인정보처리방침: 각 플랫폼별 수집 정보 명시
- [ ] 로그인 페이지: 수집 정보 안내문 표시

## 개인정보 처리 관련 참고사항

### 닉네임 표시 우선순위

코드 내부에서 닉네임을 추출하는 우선순위 (`lib/ensure-user.ts`):

1. `nickname` (카카오, 네이버가 제공하는 닉네임)
2. `user_name` (일부 플랫폼)
3. `full_name` (구글이 제공하는 전체 이름)
4. `name` (네이버가 제공하는 이름)

**실명 부담 완화**: 이메일은 작성자명으로 사용하지 않음. 닉네임이 없으면 "사용자 ab****" 형태로 마스킹 표시.

### 로그인 페이지 개선 권장사항

현재 로그인 페이지에는 간단한 안내문만 있습니다:
```
로그인 시 서비스 이용약관에 동의하는 것으로 간주됩니다.
```

더 명확한 정보 제공을 위해 아래와 같이 개선 권장:
```
로그인 시 수집되는 정보:
- 구글: 이메일 주소, 프로필 이름
- 카카오: 닉네임, 프로필 사진(선택)
- 네이버: 이메일 주소, 이름, 닉네임(선택)

로그인 시 [서비스 이용약관] 및 [개인정보처리방침]에 동의하는 것으로 간주됩니다.
```

### 법적 요구사항

1. **개인정보처리방침 작성 필수**
   - 수집 항목, 수집 목적, 보유 기간 명시
   - 각 소셜 로그인별 수집 정보 구분 기재

2. **로그인 전 고지**
   - 로그인 버튼 클릭 전 수집 정보 안내
   - 약관 및 개인정보처리방침 링크 제공

3. **최소 수집 원칙**
   - 서비스 운영에 필요한 최소한의 정보만 수집
   - 선택 항목은 거부해도 서비스 이용 가능하도록 설계

4. **데이터 보존 기간**
   - 회원 탈퇴 시 즉시 삭제 (법적 보존 의무 제외)
   - 부정 이용 방지를 위한 최소 정보만 일정 기간 보관

### 관련 문서

- `lib/ensure-user.ts`: 사용자 정보 추출 및 저장 로직
- `app/auth/callback/route.ts`: OAuth 콜백 처리 및 온보딩 분기
- `app/auth/callback/naver/route.ts`: 네이버 OAuth 수동 처리
- `app/login/page.tsx`: 로그인 페이지 UI
