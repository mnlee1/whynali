# 커뮤니티 매칭 오류 분석 및 해결 전체 히스토리

날짜: 2026-02-26

## 1. 갤럭시 버즈4 vs 갤럭시 S26 오류

### 1.1 문제 사례

```
커뮤니티: "색 잘뽑은거 같은 갤럭시 버즈4 핑크 골드.JPG"
이슈: "멀티 AI 품은 갤럭시S26 공개 D-1"
→ "갤럭시" 브랜드명만 겹쳐서 완전히 다른 제품(버즈4 vs S26)이 매칭됨
```

### 1.2 원인 분석

토큰화 결과:
```
커뮤니티: ["잘뽑은거", "같은", "갤럭시", "버즈4", "핑크", "골드", "JPG"]
이슈: ["멀티", "AI", "품은", "갤럭시S26", "공개"]
```

매칭 과정:
```typescript
// commonKeywordCount의 부분 문자열 매칭
"갤럭시s26".includes("갤럭시") → true
→ 공통 키워드 1개로 카운트
→ COMMUNITY_MATCH_THRESHOLD=1이므로 매칭됨
```

### 1.3 해결책

해결책 1: 커뮤니티 매칭 임계값 상향
```bash
CANDIDATE_COMMUNITY_MATCH_THRESHOLD=1 → 2
```
- 브랜드명 1개만으로는 매칭 안 됨
- 최소 2개 이상의 공통 키워드 필요

해결책 2: 제품명 특수 처리
```typescript
// 제품명 패턴 감지 (브랜드 + 모델명)
const productNamePattern = /^(갤럭시|아이폰|갤럭시버즈|에어팟)([a-z0-9]+)$/i

// 제품명끼리는 부분 매칭 금지, 완전 일치만 허용
if (aIsProduct && bIsProduct) {
    // "갤럭시S26" vs "갤럭시버즈4" → 매칭 안 됨
    continue
}
```

## 2. 옥택연 커뮤니티 오매칭

### 2.1 문제 사례

```
이슈: "짐승남 옥택연, 품절남 된다 '4월의 신랑'"
뉴스: "옥택연, 10년 사귄 연인과 4월 24일 결혼"

잘못 매칭된 커뮤니티:
1. "한국에서 활동하는 외국인들에게도 적용된다는 아이돌 세대 법칙"
2. "한국영화계를 위해 왕사남이 흥행하면 안된다는 자칭 시네필들"
3. "멈추지않는지랄이계속된다"
4. "아이브 가을이가 명품엠버서더에 발탁된다면?"
```

### 2.2 원인 분석

핵심 키워드:
```
["옥택연", "10년", "사귄", "연인과", "결혼"]
```

문제:
1. "된다", "활동", "발탁" 같은 범용 동사가 불용어가 아님
2. 부분 문자열 매칭으로 "발탁된다" ⊃ "된다" 매칭
3. 핵심 키워드 로직이 3글자 이상 모든 단어를 포함 → "사귄", "연인과", "결혼" 같은 범용 단어도 핵심 키워드로 인식

### 2.3 해결책

해결책 1: 범용 동사/조사 불용어 추가
```typescript
// 추가된 불용어
'있다', '없다', '한다', '된다', '된다는', '된다면', '한다는', '한다면',
'이다', '아니다', '같다', '다르다', '많다', '적다', '크다', '작다',
'좋다', '나쁘다', '높다', '낮다', '빠르다', '느리다',
```

해결책 2: 핵심 키워드 로직 개선
```typescript
// Before: 3글자 이상 모두 핵심 키워드
const coreKeywords = representativeTokens.filter(t => t.length >= 3)

// After: 3글자 이상 + 범용 단어 제외
const genericWords = new Set([
    '있다', '없다', '한다', '된다', '이다', '같다',
    '사귄', '연인과', '결혼', '활동', '발탁', '적용', '흥행',
])
const coreKeywords = representativeTokens.filter(t => 
    t.length >= 3 && !genericWords.has(t)
)
```

검증:
```
옥택연 이슈:
Before: 4건 (전혀 관련 없는 커뮤니티)
After: 0건 (개선된 로직으로 모두 제거됨)
```

## 3. 강민아 이슈 오매칭

### 3.1 문제 사례

```
이슈: "강민아, 공감세포서 국민 아이돌 출신 배우로 변신"
화력: 40점 (뉴스 0건, 커뮤니티 2건)

잘못 매칭된 커뮤니티:
1. "영화 첫 인터뷰라고 무신사에서 할인쿠폰 먹여서 산 가죽자켓입고 온 배우.jpg"
   (조회수 4281, 댓글 26)
2. "친한 배우 형 결혼식에서 와이프 X로 의심 받은 남배우"
   (조회수 3761, 댓글 22)
```

### 3.2 원인 분석

토큰화 결과:
```
이슈: ["강민아", "공감세포서", "국민", "아이돌", "출신", "배우로", "변신"]
핵심 키워드: ["강민아", "공감세포서", "국민", "아이돌", "출신", "배우로", "변신"]
```

매칭 과정:
```
커뮤니티1: "...배우.jpg"
→ "배우로" vs "배우" → 부분 매칭!

커뮤니티2: "...남배우"
→ "배우" 포함!
```

핵심 문제: "배우"라는 초광범위 단어가 핵심 키워드로 인식되어 모든 배우 관련 글이 매칭됨

### 3.3 해결책

범용 명사를 genericWords에 추가:
```typescript
const genericWords = new Set([
    // 기존...
    '있다', '없다', '한다', '된다', '이다', '같다',
    
    // 추가: 범용 명사
    '배우', '배우로', '남배우', '여배우', '아이돌', '가수', '연예인', '탤런트',
    '국민', '출신', '변신', '등장', '공개', '출연', '캐스팅', '데뷔',
    '인터뷰', '영화', '드라마', '작품', '이번', '올해', '내년', '최근',
    '형', '누나', '동생', '선배', '후배', '친구', '동료',
])
```

적용 결과:
```
Before: 핵심 키워드 ["강민아", "공감세포서", "국민", "아이돌", "출신", "배우로", "변신"]
After:  핵심 키워드 ["강민아", "공감세포서"] (고유명사만)
```

재매칭 실행:
```bash
curl -X POST "http://localhost:3000/api/admin/rematch-community"

{
  "issue_title": "강민아, 공감세포서 국민 아이돌 출신 배우로 변신",
  "matched_count": 0  # 2건 → 0건
}
```

최종 상태:
```
- 뉴스: 0건
- 커뮤니티: 0건 (오매칭 2건 모두 제거)
- 화력: 20점
- 상태: 반려 (자동 반려)
```

## 4. 현대카드 이슈 오류

### 4.1 문제 사례

```
이슈: "현대카드, '1사1교 금융교육 시상식' 금융감독원장 수상"
- 뉴스: 0건
- 커뮤니티: 1건 ("현대카드 없이 애플페이 사용하는 꿀팁")
- 화력: null
```

문제:
1. 뉴스가 없는데 이슈가 생성됨
2. 전혀 관련 없는 커뮤니티 글이 매칭됨
3. 화력이 계산되지 않음

### 4.2 원인 분석

공통 키워드:
```
["현대카드"]
```

문제:
- "현대카드"가 핵심 키워드로 인식됨
- 커뮤니티 제목에 "현대카드" 포함
- 핵심 키워드 1개 매칭으로 연결됨

### 4.3 해결책

브랜드명을 genericWords에 추가:
```typescript
'현대카드', '삼성카드', '신한카드', '국민카드', '롯데카드',
'현대', '삼성', 'LG', 'SK', '네이버', '카카오',
```

적용 결과:
```
Before:
- 커뮤니티: 1건 ('현대카드 없이 애플페이 사용하는 꿀팁')
- 화력: null
- 상태: 대기

After:
- 커뮤니티: 0건 (오매칭 제거)
- 화력: 15점 (뉴스 0건 기준)
- 상태: 반려 (자동)
```

## 5. 커뮤니티 매칭 최종 해결

### 5.1 개선 효과

1. 오매칭 제거: 다수 건 → 0건
2. 화력 정상화: 부풀려진 화력 제거
3. 자동 반려: 화력 부족으로 자동 반려 처리
4. 범용 단어 필터링: '배우', '아이돌', '현대카드' 등 43개+ 범용 명사 추가

### 5.2 적용 파일

- lib/candidate/issue-candidate.ts: genericWords 확장 (10개 → 43개)
- app/api/admin/rematch-community/route.ts: 대기 이슈 포함 재매칭
- app/api/admin/force-recalc-heat/[id]/route.ts: 화력 강제 재계산 API 신규 생성

### 5.3 교훈

1. 범용 단어의 위험성: '배우', '아이돌', '국민', '현대카드' 같은 단어는 너무 광범위함
2. 고유명사 우선: 핵심 키워드는 인명, 고유 작품명만 인정해야 함
3. 화력 재계산 필요성: 매칭 변경 후 반드시 화력 재계산 실행
4. 지속적 모니터링: 새로운 범용 단어 발견 시 즉시 추가 필요
