# 급증 감지 시스템 DB 마이그레이션

급증 감지 기능을 위한 issues 테이블 컬럼 추가

날짜: 2026-03-06

## 추가할 컬럼

```sql
-- issues 테이블에 급증 관련 컬럼 추가
ALTER TABLE issues 
ADD COLUMN IF NOT EXISTS is_urgent BOOLEAN DEFAULT FALSE,
ADD COLUMN IF NOT EXISTS burst_level INTEGER DEFAULT 0 CHECK (burst_level BETWEEN 0 AND 3),
ADD COLUMN IF NOT EXISTS source_track TEXT CHECK (source_track IN ('news_collection', 'community_burst'));

-- 컬럼 설명
COMMENT ON COLUMN issues.is_urgent IS '급증 이슈 여부 (5분간 급증 감지됨)';
COMMENT ON COLUMN issues.burst_level IS '급증 강도 레벨 (0: 없음, 1: 약함, 2: 보통, 3: 강함)';
COMMENT ON COLUMN issues.source_track IS '이슈 생성 출처 트랙 (news_collection: 뉴스 수집, community_burst: 커뮤니티 급증)';

-- 인덱스 추가 (급증 이슈 빠른 조회)
CREATE INDEX IF NOT EXISTS idx_issues_is_urgent ON issues(is_urgent) WHERE is_urgent = true;
CREATE INDEX IF NOT EXISTS idx_issues_source_track ON issues(source_track);
```

## Supabase Dashboard에서 실행

1. Supabase Dashboard 접속
2. SQL Editor 이동
3. 위 SQL 실행
4. 완료 후 테이블 구조 확인
