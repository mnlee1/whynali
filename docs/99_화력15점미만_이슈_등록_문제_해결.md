# 화력 15점 미만 이슈 등록 문제 해결 완료

날짜: 2026-03-16

## 문제 상황

관리자 이슈 목록에 **화력 15점 미만 이슈**가 자꾸 등록되는 문제 발생

## 문제 분석

### 1. 발견된 비정상 이슈: 6개

```
모두 다음 특징을 가짐:
- source_track: NULL
- created_heat_index: 0점
- 생성일: 2026-03-16 11:34-11:35 (거의 동시)
- 뉴스/커뮤니티 연결: 대부분 0건
```

**구체적 이슈:**
1. 이제훈 연애 (0→15점, 대기)
2. 김윤지 패럴림픽 (0→7점, 반려)
3. 사랑의열매 유출 (0→1점, 반려)
4. 연예인 세금 (0→3점, 반려)
5. 하버마스 별세 (0→7점, 반려)
6. 기아 e스포츠 (0→23점, 대기)

### 2. 원인 분석

**트랙 A 코드는 정상:**
```typescript
// app/api/cron/track-a/route.ts (858-875번 라인)
const issueValidation = validateIssueCreation({
    title: finalIssueTitle,
    category,
    source_track: 'track_a',  // ✅ 명시적으로 설정
    approval_status: '대기',
    status: '점화',
})

// 검증 후 삽입
.insert(issueValidation.validated!)
```

**문제점:**
- source_track이 NULL인 이슈가 생성됨
- created_heat_index가 0점으로 생성됨
- 트랙 A가 아닌 **다른 경로**에서 생성됨

**가능한 원인:**
1. 테스트 스크립트 실행
2. 직접 DB 삽입
3. 알 수 없는 코드 경로
4. API 직접 호출 (하지만 POST /api/issues는 제거됨)

### 3. 왜 트랙 A가 아닌가?

트랙 A로 생성되었다면:
- ✅ source_track = 'track_a'
- ✅ created_heat_index ≥ 15점
- ✅ 뉴스 1건 이상
- ✅ 커뮤니티 1건 이상
- ✅ 타임라인 1개 이상

실제 이슈:
- ❌ source_track = NULL
- ❌ created_heat_index = 0점
- ❌ 뉴스/커뮤니티 대부분 0건

→ **트랙 A 프로세스를 거치지 않음**

## 해결 방법

### 1단계: 즉시 조치 - 비정상 이슈 삭제 ✅

**스크립트:** `scripts/delete_null_source_track_issues.ts`

**결과:**
```
삭제 성공: 6개
삭제 실패: 0개
```

### 2단계: 근본 방지 - DB 트리거 추가 ✅

**파일:** `supabase/migrations/002_prevent_low_heat_and_null_source_track.sql`

**방어 메커니즘:**

**1) source_track NULL 방지**
```sql
CREATE TRIGGER trigger_set_default_source_track
    BEFORE INSERT ON issues
    FOR EACH ROW
    EXECUTE FUNCTION set_default_source_track();
```

동작:
- source_track이 NULL이면 자동으로 'track_a'로 설정
- WARNING 로그 출력
- 이슈 생성은 허용 (레거시 호환)

**2) 화력 15점 미만 방지**
```sql
CREATE TRIGGER trigger_prevent_low_heat_creation
    BEFORE INSERT ON issues
    FOR EACH ROW
    EXECUTE FUNCTION prevent_low_heat_creation();
```

동작:
- created_heat_index < 15점이면 이슈 생성 거부
- EXCEPTION 발생
- 이슈 생성 차단

### 3단계: 모니터링 스크립트 ✅

**파일:** `scripts/check_low_heat_issues.ts`

**기능:**
- 화력 15점 미만 이슈 자동 감지
- 등록 시점(created_heat_index) 확인
- source_track NULL 체크
- 상세 진단 정보 제공
