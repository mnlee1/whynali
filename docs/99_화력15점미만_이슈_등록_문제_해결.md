# 화력 15점 미만 이슈 등록 문제 해결 완료

날짜: 2026-03-16

## 문제 상황

관리자 이슈 목록에 **화력 15점 미만 이슈**가 자꾸 등록되는 문제 발생

## 문제 분석

### 1. 발견된 비정상 이슈: 6개

```
모두 다음 특징을 가짐:
- source_track: NULL
- created_heat_index: 0점
- 생성일: 2026-03-16 11:34-11:35 (거의 동시)
- 뉴스/커뮤니티 연결: 대부분 0건
```

**구체적 이슈:**
1. 이제훈 연애 (0→15점, 대기)
2. 김윤지 패럴림픽 (0→7점, 반려)
3. 사랑의열매 유출 (0→1점, 반려)
4. 연예인 세금 (0→3점, 반려)
5. 하버마스 별세 (0→7점, 반려)
6. 기아 e스포츠 (0→23점, 대기)

### 2. 원인 분석

**트랙 A 코드는 정상:**
```typescript
// app/api/cron/track-a/route.ts (858-875번 라인)
const issueValidation = validateIssueCreation({
    title: finalIssueTitle,
    category,
    source_track: 'track_a',  // ✅ 명시적으로 설정
    approval_status: '대기',
    status: '점화',
})

// 검증 후 삽입
.insert(issueValidation.validated!)
```

**문제점:**
- source_track이 NULL인 이슈가 생성됨
- created_heat_index가 0점으로 생성됨
- 트랙 A가 아닌 **다른 경로**에서 생성됨

**가능한 원인:**
1. 테스트 스크립트 실행
2. 직접 DB 삽입
3. 알 수 없는 코드 경로
4. API 직접 호출 (하지만 POST /api/issues는 제거됨)

### 3. 왜 트랙 A가 아닌가?

트랙 A로 생성되었다면:
- ✅ source_track = 'track_a'
- ✅ created_heat_index ≥ 15점
- ✅ 뉴스 1건 이상
- ✅ 커뮤니티 1건 이상
- ✅ 타임라인 1개 이상

실제 이슈:
- ❌ source_track = NULL
- ❌ created_heat_index = 0점
- ❌ 뉴스/커뮤니티 대부분 0건

→ **트랙 A 프로세스를 거치지 않음**

## 해결 방법

### 1단계: 즉시 조치 - 비정상 이슈 삭제 ✅

**스크립트:** `scripts/delete_null_source_track_issues.ts`

**결과:**
```
삭제 성공: 6개
삭제 실패: 0개
```

### 2단계: 근본 방지 - DB 트리거 추가 ✅

**파일:** `supabase/migrations/002_prevent_low_heat_and_null_source_track.sql`

**방어 메커니즘:**

**1) source_track NULL 방지**
```sql
CREATE TRIGGER trigger_set_default_source_track
    BEFORE INSERT ON issues
    FOR EACH ROW
    EXECUTE FUNCTION set_default_source_track();
```

동작:
- source_track이 NULL이면 자동으로 'track_a'로 설정
- WARNING 로그 출력
- 이슈 생성은 허용 (레거시 호환)

**2) 화력 15점 미만 방지**
```sql
CREATE TRIGGER trigger_prevent_low_heat_creation
    BEFORE INSERT ON issues
    FOR EACH ROW
    EXECUTE FUNCTION prevent_low_heat_creation();
```

동작:
- created_heat_index < 15점이면 이슈 생성 거부
- EXCEPTION 발생
- 이슈 생성 차단

### 3단계: 모니터링 스크립트 ✅

**파일:** `scripts/check_low_heat_issues.ts`

**기능:**
- 화력 15점 미만 이슈 자동 감지
- 등록 시점(created_heat_index) 확인
- source_track NULL 체크
- 상세 진단 정보 제공

## 추가 정리 (2026-03-17)

### 등록 시 화력점수가 변동되듯 보이던 현상

**원인:** 관리자 이슈 페이지에서 `created_heat_index`가 null일 때 **현재 화력(heat_index)**으로 대체해 "등록시 N점"을 표시하고 있었음. 화력 재계산 Cron이 10분마다 `heat_index`만 갱신하므로, "등록시"로 보이던 값이 같이 바뀌는 것처럼 보임.

**수정:** `app/admin/issues/page.tsx`에서 등록 시점 화력은 `created_heat_index`만 사용. null이면 "—" 표시(현재 화력으로 대체하지 않음). 백엔드는 생성 시 한 번만 `created_heat_index`를 설정하며, recalculate-heat 등 어떤 Cron도 `created_heat_index`를 수정하지 않음.

### 15점 미만(2점, 7점, 5점 등) 이슈가 보이는 이유

1. **레거시 데이터:** 트랙 A/커뮤니티 급증에 "화력 15점 미만 시 삭제" 로직을 넣기 전, 또는 마이그레이션 002 적용 전에 생성된 이슈. DB에는 그대로 남아 있을 수 있음.
2. **UI 표시 오류(수정 완료):** `created_heat_index`가 null인 이슈는 이전에 "등록시"를 현재 화력으로 표시했기 때문에, 현재 2/7/5점인 이슈가 "등록시 2점"처럼 보였을 수 있음. 수정 후에는 null이면 "—"로 표시됨.
3. **DB 트리거 한계:** `prevent_low_heat_creation`은 **INSERT 시** `created_heat_index`가 **설정된 경우**에만 15점 미만을 차단함. 트랙 A/급증은 INSERT 시점에는 `created_heat_index`를 넣지 않고, 화력 계산 후 UPDATE로 설정하며, 그 전에 15점 미만이면 삭제하므로 정상 경로에서는 15점 미만 이슈가 남지 않음.

현재 코드: 트랙 A(`app/api/cron/track-a/route.ts`), 커뮤니티 급증(`lib/candidate/community-burst-detector.ts`) 모두 화력 15점 미만이면 이슈 삭제 후 continue. 이미 등록된 15점 미만 이슈가 있다면 레거시이거나 비정상 경로로 생성된 것이므로, `scripts/check_low_heat_issues.ts`로 확인 후 필요 시 수동 정리.
