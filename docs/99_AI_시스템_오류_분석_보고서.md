# AI 시스템 오류 분석 및 해결 보고서

작성일: 2026-04-09

## 원인 파악

### 현재 상황
- Claude AI: 67회 호출 중 8회 실패 (성공률 88.06%)
- Groq AI: 13회 호출 중 1회 실패 (성공률 92.31%)

### 오류 판단 로직
대시보드는 다음 기준으로 AI 상태를 표시합니다:

```typescript
- 정상: 실패 건수 0건
- 일부 오류: 성공률 >= 90% (예: 92% → "일부 오류 1건")
- 오류: 성공률 < 90% (예: 88% → "오류 8건")
```

### 실패 건수 집계 방식
- 집계 기간: **당월 누적** (4월 1일~현재)
- 데이터 출처: `api_usage` 테이블의 `fail_count` 컬럼
- 매월 1일 자정에 자동 초기화되지 않음

## 해결 내역

### 1. Groq AI 실패 추적 추가
**문제**: GroqProvider가 실패 건수를 기록하지 않음
**해결**: `lib/ai/groq-provider.ts`에 실패 추적 로직 추가

```typescript
// 성공 시
incrementApiUsage('groq', {
    calls: 1,
    successes: 1,
}).catch(() => {})

// 실패 시
incrementApiUsage('groq', { calls: 1, failures: 1 }).catch(() => {})
```

### 2. Claude AI 에러 로깅 개선
**문제**: 실패 원인을 파악하기 어려움
**해결**: `lib/ai/claude-provider.ts`에 상세 에러 로깅 추가

```typescript
console.error(`[ClaudeProvider] API 호출 실패:`, {
    status: error.status,
    type: error.type,
    message: error.message,
    model,
})
```

### 3. 대시보드 UI 개선
**문제**: 당월 누적 실패 건수라는 것이 명확하지 않음
**해결**: `app/admin/(protected)/page.tsx` 개선

변경 사항:
- "AI 상태" → "AI 상태 (4월 누적)"으로 표시
- 성공률 표시 추가 (예: "88.1% (59/67)")
- 성공률 90% 미만 시 "모니터링 필요" 경고 표시

## 현재 상태 평가

### Claude AI (성공률 88.06%)
- 상태: 주의 필요
- 평가: 일시적인 네트워크 오류나 API 이슈로 추정
- 권장: 향후 24시간 모니터링

### Groq AI (성공률 92.31%)
- 상태: 정상 범위
- 평가: 정상 운영 중

## 향후 권장사항

### 1. 실패 원인 모니터링
실서버 API 사용 현황 조회 스크립트로 현재 상태를 확인할 수 있습니다.

```bash
# 실서버 API 사용 현황 조회
npm run check:api

# 또는 직접 실행
node scripts/check-api-failures.mjs
```

서버 로그에서 `[ClaudeProvider] API 호출 실패` 메시지를 확인하여 실패 원인 파악

```bash
# Vercel 로그 확인
vercel logs production --follow
```

### 2. 월간 데이터 관리 (선택 사항)
매월 1일에 api_usage 데이터를 초기화하거나 아카이빙

옵션 1: 수동 초기화
```bash
# reset-api-usage-monthly.sql 실행
```

옵션 2: 자동 초기화 (Cron Job)
- 매월 1일 00:00에 실행되는 API 엔드포인트 추가
- `/api/cron/reset-api-usage` 생성

### 3. 알림 시스템 추가 (선택 사항)
성공률이 85% 미만으로 떨어지면 알림 발송
- 두레이 웹훅 연동
- 이메일 알림
