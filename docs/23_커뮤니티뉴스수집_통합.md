# 커뮤니티 뉴스 수집 통합

본 문서는 커뮤니티 수집, 뉴스 수집, 동적 키워드 시스템을 통합한 문서입니다.

## 1. 커뮤니티 수집 업데이트 및 수정

### 1.1 수집 대상
- 더쿠 (theqoo.net)
- 네이트판 (pann.nate.com)

### 1.2 수집 범위
- 제목
- URL
- 조회수
- 댓글수
- 작성시간

**수집하지 않는 것:**
- 본문
- 닉네임
- 기타 창작 콘텐츠

### 1.3 수집 주기
- 3분마다 수집 (급증 감지를 위해)
- 과도한 트래픽 방지 (법적 준수)

### 1.4 기술적 구현
- Puppeteer 또는 Cheerio 사용
- Rate Limiting 적용
- User-Agent 설정
- 에러 핸들링

## 2. 뉴스 수집 광범위 전략 개선

### 2.1 기존 문제
- 키워드 기반 수집의 한계
- 새로운 이슈 발견 어려움
- 특정 분야 편향

### 2.2 개선 방안

**광범위 수집 전략:**
1. 키워드 없이 최신 뉴스 수집
2. 카테고리별 분산 수집
3. 주요 언론사 우선 수집
4. 시간대별 균등 수집

**네이버 뉴스 API 활용:**
```typescript
// 카테고리별 최신 뉴스
const categories = ['정치', '경제', '사회', '생활/문화', 'IT/과학', '세계']
for (const category of categories) {
    await fetchNaverNews({
        query: category,
        sort: 'date',
        display: 100
    })
}
```

### 2.3 수집 빈도
- 3분마다 실행
- 카테고리당 100건
- 총 600건/3분 = 200건/분

## 3. 커뮤니티 기반 동적 키워드 시스템

### 3.1 목적
커뮤니티에서 급증하는 키워드를 자동으로 감지하여 뉴스 검색 키워드로 활용

### 3.2 동작 흐름

```
1. 커뮤니티 글 수집 (3분마다)
   ↓
2. 키워드 추출 및 빈도 계산
   ↓
3. 급증 키워드 감지 (10건 이상)
   ↓
4. 네이버 뉴스 검색
   ↓
5. 이슈 후보 등록
```

### 3.3 키워드 추출 로직

**불용어 필터링:**
```typescript
const STOP_WORDS = [
    '있다', '없다', '한다', '된다', '이다', '같다',
    '하는', '되는', '있는', '없는',
    '민주당', '공화당', '여당', '야당',
    '발언', '언급', '주장', '입장'
]
```

**키워드 추출:**
```typescript
// 1. 3글자 이상
// 2. 불용어 제외
// 3. 빈도 10건 이상
// 4. 급증 패턴 (이전 시간대 대비 50% 이상 증가)
```

### 3.4 테스트 결과

**급증 키워드 예시:**
- "민희진" (15건) → 뉴스 8건 발견 → 이슈 등록
- "옥택연" (20건) → 뉴스 12건 발견 → 이슈 등록
- "갤럭시S26" (10건) → 뉴스 5건 발견 → 이슈 등록

**필터링된 키워드:**
- "ㅋㅋㅋ" (30건) → 의미 없는 반응
- "레전드" (25건) → 범용 표현
- "민주당" (18건) → 불용어

## 4. 이슈 자동 연결 (Auto-Link)

### 4.1 개요

수집된 뉴스·커뮤니티 데이터를 이슈와 자동으로 연결합니다.
매 cron 주기마다 실행되어 새로운 데이터를 연결하고, 기존 연결을 재검증합니다.

### 4.2 뉴스 링커 (issue-news-linker)

**매칭 기준:**
- 이슈 카테고리와 동일한 뉴스만 대상 (카테고리 필터)
- 키워드 70% 이상 일치 + 최소 3개
- 인물명 키워드 포함 시: 무조건 최소 3개 이상 일치 필수
- 이슈 생성일 기준 전 1일/후 2일 이내 뉴스만 대상

**인물명 키워드 목록:**
```typescript
const personKeywords = [
    '김연경', '손흥민', '이강인', '옥택연', '민희진', '뉴진스',
    '윤석열', '이재명', '한동훈', '황희찬', '김하성', '류현진'
]
```

**환경변수:**
```bash
LINKER_NEWS_BEFORE_DAYS=1  # 이슈 생성일 이전 N일
LINKER_NEWS_AFTER_DAYS=2   # 이슈 생성일 이후 N일
```

### 4.3 커뮤니티 링커 (issue-community-linker)

**매칭 기준:**
- 키워드 70% 이상 일치 + 최소 3개
- 인물명 키워드 포함 시: 무조건 최소 3개 이상 일치 필수
- 이슈 생성일 기준 전 1일/후 7일 이내 글만 대상

**환경변수:**
```bash
LINKER_COMMUNITY_BEFORE_DAYS=1  # 이슈 생성일 이전 N일
LINKER_COMMUNITY_AFTER_DAYS=7   # 이슈 생성일 이후 N일
```

### 4.4 재검증 로직

**unlinkInvalidNews / unlinkInvalidCommunity:**
- 이미 연결된 데이터도 매 cron 주기마다 재검증
- 기준 미달(키워드 불일치, 날짜 범위 초과, 카테고리 불일치) 시 자동 해제
- 대량 해제 방지 안전 장치 (50% 이상 해제 시 경고)

**적용 시기:**
- auto-link cron 실행 시마다 자동 실행
- 잘못 연결된 데이터는 다음 실행 시 자동으로 정리됨

### 4.5 개선 히스토리 (2026-03-11)

**문제:**
- "김연경" 키워드 하나만으로 무관한 뉴스가 연결됨
- IOC GEDI 챔피언 이슈에 배구 경기 뉴스가 섞임

**해결:**
- 키워드 매칭 임계값: 60% → 70%, 최소 2개 → 3개
- 인물명 키워드 특별 처리 추가
- 재검증 로직에도 동일 기준 적용

**효과:**
```
김연경 IOC GEDI 챔피언 이슈:
Before: 12건 연결 (IOC 5건 + 배구 경기 7건)
After: 5건 연결 (IOC 관련만)
오연결 비율: 58% → 0%
```

## 8. 모니터링

### 8.1 수집 통계 쿼리

```sql
-- 오늘 수집 통계
SELECT 
    source,
    COUNT(*) as count,
    MIN(created_at) as first,
    MAX(created_at) as last
FROM collected_data
WHERE created_at >= CURRENT_DATE
GROUP BY source;

-- 시간대별 수집량
SELECT 
    DATE_TRUNC('hour', created_at) as hour,
    source,
    COUNT(*) as count
FROM collected_data
WHERE created_at >= CURRENT_DATE - INTERVAL '1 day'
GROUP BY hour, source
ORDER BY hour;
```

### 8.2 동적 키워드 추적

```sql
-- 급증 키워드 기록
CREATE TABLE dynamic_keywords (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    keyword TEXT NOT NULL,
    frequency INTEGER NOT NULL,
    news_count INTEGER,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 오늘의 급증 키워드
SELECT 
    keyword,
    frequency,
    news_count
FROM dynamic_keywords
WHERE created_at >= CURRENT_DATE
ORDER BY frequency DESC
LIMIT 20;
```

### 8.3 링커 통계 쿼리

```sql
-- 이슈별 연결된 뉴스/커뮤니티 건수
SELECT 
    i.id,
    i.title,
    i.category,
    COUNT(DISTINCT n.id) as news_count,
    COUNT(DISTINCT c.id) as community_count
FROM issues i
LEFT JOIN news_data n ON n.issue_id = i.id
LEFT JOIN community_data c ON c.issue_id = i.id
WHERE i.approval_status = '승인'
GROUP BY i.id, i.title, i.category
ORDER BY news_count + community_count DESC
LIMIT 20;

-- 연결되지 않은 수집 데이터 (최근 24시간)
SELECT 
    'news' as type,
    COUNT(*) as unlinked_count
FROM news_data
WHERE issue_id IS NULL
  AND created_at >= NOW() - INTERVAL '24 hours'
UNION ALL
SELECT 
    'community' as type,
    COUNT(*) as unlinked_count
FROM community_data
WHERE issue_id IS NULL
  AND written_at >= NOW() - INTERVAL '24 hours';
```
