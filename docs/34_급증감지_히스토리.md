# 급증 감지 시스템 개발 히스토리

**날짜**: 2026-03-06

이 문서는 급증 감지 시스템 개발 과정과 의사결정 내용을 기록합니다.

---

## 문제 인식

### 기존 방식의 한계

```
뉴스 수집: 30분 주기
이슈 등록 조건: 5건 이상 + 화력 30점 이상
→ 최대 30분 지연 + 높은 등록 기준
```

**실제 문제**:
- "지금 막 터진" 이슈를 30분 후에야 감지
- 화력 30점 = 이미 어느 정도 쌓인 후
- 커뮤니티 "반응 속도"를 제대로 활용 안 함

---

## 해결 방안: 2-Track 시스템

### Track 1: 뉴스 급증 감지

**핵심 아이디어**: "속도"를 측정하자

```typescript
// 기존: 건수만 봄
if (newsCount >= 5) → 이슈 등록

// 개선: 속도를 봄
if (5분간 5건 급증) → 🔥 긴급 이슈!
if (30분간 5건 누적) → 일반 이슈
```

### Track 2: 커뮤니티 급증 감지

**핵심 아이디어**: 커뮤니티가 더 빠르다

```typescript
// 커뮤니티 우선 감지
10분간 키워드 10건 급증
  ↓
AI 검증 (Groq): 진짜 이슈인가?
  ↓
네이버 뉴스 검색 (3건 이상)
  ↓
긴급 이슈 생성
```

## 구현 단계

### Week 1: 뉴스 급증 감지

**구현**:
1. `burst-detector.ts` - 급증 감지 로직
2. `duplicate-checker.ts` - AI 중복 체크
3. `issue-candidate.ts` 수정 - 연동

**결과**:
- 5분 급증 감지 성공
- 급증 레벨 0-3 산정
- 자동 승인 기준 30점 유지 (일관성)

### Week 2: 커뮤니티 급증 감지

**구현**:
1. `community-burst-detector.ts` - 커뮤니티 로직
2. `detect-community-burst/route.ts` - API
3. GitHub Actions Workflow - Cron

**결과**:
- 3분 주기 감지 성공
- AI 검증으로 밈/드립 필터링
- 네이버 뉴스 확인으로 신뢰도 향상

## 기술적 의사결정

### 1. Perplexity AI 포기

**시도**: Perplexity AI로 뉴스 그루핑
**결과**: 실패

**이유**:
- 품질 문제: 관련 없는 뉴스를 같은 그룹으로 묶음
- 성능 문제: 응답 시간 느림
- 비용 문제: 호출당 비용

**대안**: 키워드 기반 그루핑 + Groq AI 중복 체크

### 2. Groq AI 선택

**이유**:
- **완전 무료**: 일일 14,400회
- **빠른 응답**: Llama 3.3 70B
- **높은 정확도**: 중복 체크, 이슈 검증

**사용처**:
- 커뮤니티 매칭
- 중복 이슈 체크
- 커뮤니티 급증 검증

### 3. GitHub Actions 선택

**고려 사항**:
- Vercel Cron: 무료 플랜 제한
- cron-job.org: 외부 서비스 의존
- GitHub Actions: 무료 + 안정적

**결정**: GitHub Actions 사용
- Public 저장소: 무제한
- Private: 월 2,000분 (충분)

## 실패와 학습

### 실패 1: AI 그루핑

**시도**: Perplexity AI로 뉴스 자동 그루핑
**실패**: 품질 문제로 롤백

**학습**: 
- AI는 만능이 아님
- 키워드 방식도 충분히 효과적
- AI는 검증/보조 용도로 활용

### 실패 2: 과도한 AI 사용

**시도**: 모든 단계에 AI 적용
**문제**: Rate Limit, 비용, 속도

**학습**:
- AI는 꼭 필요한 곳에만
- 키워드 필터링으로 AI 호출 줄이기
- 안전 장치로 불필요한 AI 호출 방지

### 실패 3: 너무 낮은 임계값

**시도**: 3분간 3건으로 테스트
**문제**: 거짓 양성(False Positive) 과다

**학습**:
- 임계값은 실제 데이터로 튜닝
- 초기에는 보수적으로 설정
- 점진적으로 완화

## 성과

### 정량적 성과

- 실시간성: 30분 → 3분 (10배 향상)
- 등록 기준: 15점 (일반/급증 동일)
- 자동 승인 기준: 30점 (일관성 유지)
- 커버리지: 뉴스 only → 뉴스 + 커뮤니티
- AI 비용: 일일 1,800회 (무료 한도 12.5%)

### 정성적 성과

- "지금 막 터지는" 이슈 빠르게 캐치
- 커뮤니티 반응 속도 활용
- 중복 이슈 자동 방지
- 밈/드립 자동 필터링
