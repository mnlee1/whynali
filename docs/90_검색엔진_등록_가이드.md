# 검색엔진 등록 가이드

## 구글 서치 콘솔 등록

### 1단계: 속성 추가
```
1. search.google.com/search-console 접속
2. "URL 접두어" 선택
3. URL 입력: https://whynali.com
4. "계속" 클릭
```

### 2단계: 소유권 확인
```
"HTML 태그" 방식 선택

인증 코드 (이미 설정됨):
J9cnf6UOrn5T_W38YOde3BnpgoLRpxbzMPjuM23QazE

또는

oLeSnP_W1iS3crjqf9RtO1koomeIm860DTAP-WSclWg

→ app/layout.tsx에 이미 설정되어 있음
→ "확인" 클릭하면 즉시 완료
```

### 3단계: 사이트맵 제출
```
1. 왼쪽 메뉴 "Sitemaps" 클릭
2. 새 사이트맵 추가: sitemap.xml
3. "제출" 클릭

완료!
```

### 4단계: 확인
```
2-3일 후 확인:
- 왼쪽 메뉴 "실적" 클릭
- 노출, 클릭 데이터 확인
- "색인 생성" → "페이지" 에서 색인된 페이지 수 확인
```

## 네이버 서치어드바이저 등록

### 1단계: 사이트 추가
```
1. searchadvisor.naver.com 접속
2. "사이트 추가" 클릭
3. URL 입력: https://whynali.com
```

### 2단계: 소유권 확인
```
"HTML 태그" 선택

인증 코드 (이미 설정됨):
7308faf20b068eccbe33fbe3794875d9db9f676d

→ app/layout.tsx에 이미 설정되어 있음
→ "확인" 클릭
```

### 3단계: 사이트맵 제출
```
1. "요청" 탭 클릭
2. "사이트맵 제출" 클릭
3. URL 입력: https://whynali.com/sitemap.xml
4. "확인" 클릭

완료!
```

### 4단계: 수집 요청
```
1. "요청" → "URL 수집 요청"
2. 메인 페이지 입력: https://whynali.com
3. "확인" 클릭

→ 빠른 색인을 위해 추가 요청
```

## 트러블슈팅

### Q1: "소유권을 확인할 수 없습니다"
```
원인: 메타태그가 배포되지 않음
해결: 
1. 코드 수정 후 git push
2. Vercel 배포 완료 대기 (1-2분)
3. 브라우저에서 소스 보기로 태그 확인
4. 다시 "확인" 클릭
```

### Q2: "사이트맵을 읽을 수 없습니다"
```
원인: URL 오타 또는 형식 오류
해결:
1. https://whynali.com/sitemap.xml 직접 접속
2. XML이 정상 표시되는지 확인
3. 404 오류 시 Next.js 빌드 확인
```

### Q3: "색인이 생성되지 않습니다"
```
정상: 신규 사이트는 2-4주 소요
가속:
1. URL 검사 도구로 개별 페이지 요청
2. 소셜 공유로 외부 링크 생성
3. 콘텐츠 지속 업데이트
```
