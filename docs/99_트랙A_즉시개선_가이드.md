## 핵심 문제

### 전환율이 2.2%로 극히 낮음

```
커뮤니티 글 548건 수집
    ↓
급증 키워드 89개 감지 (임계값 2건 이상)
    ↓
⚠️  전환율 2.2%
    ↓
이슈 2개만 생성
```

**정상 전환율**: 20~30%  
**현재 전환율**: 2.2%  
**병목 구간**: AI 검증 또는 뉴스 검색 단계

## 즉시 적용 가능한 해결책

### 1단계: 환경 변수 조정 (5분)

`.env.local` 파일을 열고 다음 값을 수정:

```bash
# === 커뮤니티 급증 감지 완화 ===
# 기존: 10분 내 10건 → 30분 내 5건으로 완화
COMMUNITY_BURST_THRESHOLD=5           # 기존 10 → 5
COMMUNITY_BURST_WINDOW_MINUTES=30     # 기존 10 → 30

# === 중복 체크 완화 ===
# 24시간 → 12시간으로 축소 (같은 사건의 후속 이슈 허용)
DUPLICATE_CHECK_WINDOW_HOURS=12       # 새로 추가

# 중복 판단을 더 엄격하게 (신뢰도 80% → 90%)
DUPLICATE_CHECK_AI_CONFIDENCE=90      # 기존 80 → 90

# === 트랙A 처리량 증가 ===
# 한 번에 1개 → 3개 키워드 동시 처리
TRACK_A_MAX_KEYWORDS=3                # 기존 1 → 3

# AI 호출 간격 단축 (10초 → 5초)
TRACK_A_AI_DELAY_MS=5000              # 기존 10000 → 5000
```

## 추가 개선 (선택)

### A. AI 검증 완화 (코드 수정 필요)

현재 AI가 "진짜 이슈인지" 판단하는데, 너무 보수적일 수 있습니다.

**수정 파일**: `app/api/cron/track-a/route.ts`

```typescript
// 현재: AI가 "NO"라고 하면 무조건 스킵
if (!verificationResult.isIssue) {
    continue
}

// 개선: 신뢰도가 낮아도 뉴스가 많으면 이슈 생성
if (!verificationResult.isIssue && verificationResult.confidence < 0.4) {
    // 뉴스 검색해서 3건 이상이면 이슈로 인정
    const newsCheck = await searchNaverNewsByKeyword(keyword)
    if (newsCheck.length < 3) {
        continue
    }
}
```

### B. 뉴스 검색 키워드 확장

**수정 파일**: `app/api/cron/track-a/route.ts`

```typescript
// AI가 뽑은 키워드 외에 변형 키워드도 검색
const keywords = [
    searchKeyword,
    searchKeyword.replace(/\s+/g, ''),  // 띄어쓰기 제거
    burst.keyword,  // 원본 키워드
]

// 모든 키워드로 뉴스 검색
const allNews = []
for (const kw of keywords) {
    const news = await searchNaverNewsByKeyword(kw)
    allNews.push(...news)
}
```

## FAQ

### Q1: 설정을 너무 완화하면 쓰레기 이슈가 많아지지 않나요?

**A**: 아닙니다. 여전히 다음 필터들이 작동합니다:
- AI 검증 (진짜 이슈인지 판단)
- 뉴스 검색 (뉴스 1건 이상 필수)
- 중복 체크 (기존 이슈와 중복 방지)
- 관리자 승인 (대기 → 승인 단계)

오히려 **더 많은 후보를 수집하고 관리자가 선별**하는 것이 안전합니다.

### Q2: 트랙A 크론이 너무 느려지지 않나요?

**A**: `TRACK_A_MAX_KEYWORDS=3`으로 늘려도 괜찮습니다:
- 현재: 1개 키워드 처리 → 약 30초
- 변경: 3개 키워드 동시 처리 → 약 60초
- 크론 실행: 30분마다 1회

충분히 여유가 있습니다.

### Q3: 언제 다시 확인하면 되나요?

**A**: 24시간 후 (2026-03-14 오후)에 다음 명령 실행:

