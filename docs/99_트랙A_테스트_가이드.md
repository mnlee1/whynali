# 트랙 A 신규 아키텍처 테스트 가이드

## 개요

기존 이슈를 모두 삭제하고 새로운 트랙 A 구조가 제대로 작동하는지 테스트하는 방법을 안내합니다.

## 사전 준비

### 1. 환경변수 설정 (.env.local)

```bash
# 트랙 A 활성화
ENABLE_TRACK_A=true

# 급증 감지 설정
COMMUNITY_BURST_THRESHOLD=10          # 급증 임계값 (최소 건수)
COMMUNITY_BURST_WINDOW_MINUTES=10     # 감지 시간 창 (분)

# 이슈 등록 설정
CANDIDATE_MIN_HEAT_TO_REGISTER=15     # 최소 등록 화력

# AI 설정 (이미 있음)
GROQ_API_KEY=gsk_...
ENABLE_AI_CATEGORY=true

# 네이버 뉴스 API (이미 있음)
NAVER_CLIENT_ID=...
NAVER_CLIENT_SECRET=...

# 크론 인증
CRON_SECRET=your-secret-here
```

### 2. 로컬 서버 실행

```bash
npm run dev
```

## 테스트 시나리오

### 시나리오 1: 전체 리셋 후 테스트 (권장)

깨끗한 상태에서 트랙 A가 제대로 작동하는지 확인합니다.

#### Step 1: 데이터 상태 확인

```bash

```

- 급증 키워드가 있는가?
- 현재 이슈 개수

**결과 예시:**
```

  • "손흥민": 28건
  ...
```

#### Step 4: Supabase에서 결과 확인

Supabase 대시보드 > Table Editor > issues

**확인 사항:**
- `approval_status = '대기'` 이슈가 생성되었는가?
- `source_track = 'track_a'` 인가?
- `category`가 올바르게 분류되었는가?
- `heat_index >= 15` 인가?

**SQL 쿼리:**
```sql
SELECT 
    id,
    title,
    category,
    approval_status,
    source_track,
    heat_index,
    created_at
FROM issues
WHERE source_track = 'track_a'
ORDER BY created_at DESC
LIMIT 10;
```

#### Step 5: 관련 데이터 확인

**뉴스 연결 확인:**
```sql
SELECT 
    i.title as issue_title,
    COUNT(n.id) as news_count
FROM issues i
LEFT JOIN news_data n ON n.issue_id = i.id
WHERE i.source_track = 'track_a'
GROUP BY i.id, i.title;
```

**커뮤니티 연결 확인:**
```sql
SELECT 
    i.title as issue_title,
    COUNT(c.id) as community_count
FROM issues i
LEFT JOIN community_data c ON c.issue_id = i.id
WHERE i.source_track = 'track_a'
GROUP BY i.id, i.title;
```

#### Step 6: 관리자 페이지에서 확인

2. "대기" 탭에서 트랙 A 이슈 확인
3. 이슈 상세보기:
   - 연결된 뉴스 개수
   - 연결된 커뮤니티 글 개수
   - 화력 지수
4. 승인/반려 테스트

## 트랙 A 처리 흐름 상세

### 1단계: 커뮤니티 급증 감지
- 최근 N분간 수집된 community_data 조회 (`issue_id IS NULL`)
- 제목에서 키워드 추출 (토큰화)
- 키워드별 등장 빈도 계산
- 임계값 이상인 키워드 필터링

### 2단계: AI 이슈 검증
- Groq AI (llama-3.1-8b-instant) 호출
- 진짜 이슈인지 판단 (신뢰도 70% 이상)
- 네이버 뉴스 검색용 키워드 추출

### 3단계: 네이버 뉴스 타겟 검색
- AI가 추출한 키워드로 즉시 검색
- `sort=sim`, `display=20`
- **뉴스 0건 = 등록 보류** (루머 가능성)
- 뉴스 1건 이상 = 다음 단계 진행

### 4단계: 이슈 생성
- 제목: `"{키워드} 관련 논란"`
- 중복 체크 (제목 유사도)
- 카테고리 분류 (키워드 기반)
- `approval_status = '대기'`
- `source_track = 'track_a'`

### 5단계: 화력 계산
- 뉴스 + 커뮤니티 건수로 화력 계산
- **화력 < 15점 = 이슈 삭제**
- 화력 >= 15점 = 이슈 유지

## 문제 해결

### "최근 커뮤니티 데이터가 없습니다"

**원인:** 커뮤니티 수집이 안 되었거나 시간 창이 너무 짧음

**해결:**
1. 커뮤니티 수집 크론 실행
2. `COMMUNITY_BURST_WINDOW_MINUTES=60`으로 늘리기

### "급증 키워드가 없습니다"

**원인:** 임계값이 너무 높거나 데이터가 분산됨

**해결:**
1. `COMMUNITY_BURST_THRESHOLD=5`로 낮추기
2. 시간 창 늘리기

### "인증 실패 (401)"

**원인:** CRON_SECRET이 올바르지 않음

**해결:**
```bash
# .env.local 확인
CRON_SECRET=your-secret-here
```

### "로컬 서버 연결 실패"

**원인:** Next.js 개발 서버가 실행 중이지 않음

**해결:**
```bash
npm run dev
```
