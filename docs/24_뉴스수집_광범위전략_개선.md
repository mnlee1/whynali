# 뉴스 수집 광범위 전략 개선

## 변경 일자
2026-02-27

## 문제 상황

### 발견된 이슈
"정국 라방" 커뮤니티 글이 이슈에 연결되지 않음

### 원인 분석
1. **뉴스 수집 단계에서 누락**
   - 네이버 뉴스 수집 키워드: '민희진', '뉴진스', '하이브' 등 특정 인물만
   - 'BTS', '정국', '지민' 등이 키워드에 없어서 관련 뉴스 1건만 수집
   - 이슈 후보 생성 기준(5건 이상) 미달로 이슈 자동 생성 실패

2. **근본적 구조 문제**
   ```
   뉴스 수집 누락 → 이슈 생성 실패 → 커뮤니티 연결 불가
   ```

3. **AI 도입의 한계**
   - AI 중요도 판별 비용: 월 $1,188 (₩1,584,000)
   - Groq Rate Limit: 분당 85건 (5분 500건 처리 불가)
   - 결국 샘플링(10건)만 AI 사용 → 대부분 키워드 방식

## 해결 방안

### 1. 뉴스 수집 전략 변경

#### 변경 전 (특정 키워드)
```typescript
if (category === '연예') {
    queries.push(
        '민희진', '뉴진스', '하이브',  // 특정 인물/그룹만
        '논란', '사과', '기자회견'
    )
}
```

**문제:**
- 새로운 이슈 인물 등장 시 키워드 수동 추가 필요
- 정국, 윈터 등 놓침

#### 변경 후 (광범위 수집)
```typescript
if (category === '연예') {
    queries.push(
        '논란', '사과', '컴백', '열애'  // 일반 이슈 키워드만
    )
}
// 특정 인물명 제거 → 모든 연예 이슈 자동 포착
```

**장점:**
- 모든 연예인 자동 커버 (정국, 윈터, 신인 등)
- 키워드 관리 불필요
- 타이밍 손실 없음

### 2. 노이즈 제거 메커니즘 (이미 구현됨)

#### 1단계: 키워드 그루핑
- 공통 키워드 2개 이상 → 같은 그룹
- 5건 미만 그룹 → 이슈 생성 안 됨

#### 2단계: 출처 다양성 필터
- 최소 2개 언론사 이상 보도 필수
- 단독 오보 방지

#### 3단계: 화력 필터
- 화력 30점 미만 → 자동 삭제
- 뉴스 신뢰도 + 커뮤니티 반응 종합 평가

#### 4단계: 데이터 Cleanup
- 7일 경과 + 이슈 미연결 → 자동 삭제

### 3. 카테고리별 수집 전략

#### 연예
```typescript
'연예', '논란', '사과', '컴백', '열애'
```

- 이슈 패턴 키워드만

#### 스포츠

## 예상 효과

### 긍정적 효과
1. **이슈 누락 방지**
   - 정국, 윈터 등 모든 연예 이슈 자동 포착
   - 신인 이슈도 자동 감지

2. **관리 부담 감소**
   - 키워드 수동 추가 작업 불필요
   - 타이밍 손실 없음

3. **비용 효율**
   - AI 비용 $0 (키워드 그루핑만 사용)
   - 네이버 API 호출 수 동일 (쿼리만 변경)

### 예상 단점 및 대응

#### 1. DB 증가
**예상:**
- 기존: 카테고리당 ~200건/일
- 변경 후: 카테고리당 ~400건/일 (2배)
- 전체: 하루 1,000건 → 2,000건

**대응:**
- 화력 필터(30점)로 저품질 자동 삭제
- 7일 cleanup으로 용량 관리
- PostgreSQL 여유 충분 (현재 사용량 < 10%)

#### 2. 노이즈 증가
**예상:**
- 홍보성 기사, 단신 뉴스 유입

**대응:**
- 5건 미만 그룹 → 이슈 생성 안 됨
- 최소 2개 언론사 필수
- 화력 30점 미만 삭제
- 3단계 필터링으로 노이즈 차단

#### 3. 처리 속도
**예상:**
- 그루핑 시간 증가 (2배 데이터)

**대응:**
- 현재 그루핑 시간: ~2초 (1000건)
- 예상 시간: ~4초 (2000건)
- 5분 Cron 여유 충분 (60초 제한)
