# 기존 이슈 재분류 가이드

**날짜**: 2026-03-06

기존에 잘못 분류된 이슈들을 AI로 일괄 재분류합니다.

---

## 사용 방법

### 1. Dry Run (테스트 모드)

실제 업데이트 없이 어떻게 변경될지 미리 확인:

```bash
npx tsx scripts/reclassify-issues.ts --dry-run
```

### 2. 스포츠 카테고리만 재분류

```bash
# Dry Run
npx tsx scripts/reclassify-issues.ts --category 스포츠 --dry-run

# 실제 적용
npx tsx scripts/reclassify-issues.ts --category 스포츠
```

### 3. 최근 100개 이슈 재분류

```bash
npx tsx scripts/reclassify-issues.ts
```

### 4. 전체 이슈 재분류 (주의!)

```bash
# Dry Run 먼저 확인
npx tsx scripts/reclassify-issues.ts --all --dry-run

# 실제 적용
npx tsx scripts/reclassify-issues.ts --all
```

### 5. 개수 제한

```bash
npx tsx scripts/reclassify-issues.ts --limit 50
```

---

## 옵션

| 옵션 | 설명 | 예시 |
|------|------|------|
| `--dry-run` | 테스트 모드 (실제 업데이트 안 함) | `--dry-run` |
| `--category` | 특정 카테고리만 | `--category 스포츠` |
| `--all` | 전체 이슈 | `--all` |
| `--limit` | 개수 제한 (기본 100) | `--limit 50` |

## 작동 방식

### 1. 이슈 조회
```
- 최근 100개 (기본)
- 특정 카테고리만 (옵션)
- 전체 (--all)
```

### 2. AI 재분류
```
각 이슈 제목 → AI 분석 → 카테고리 + 신뢰도
```

### 3. 업데이트 조건
```
- 신뢰도 80% 이상
- 현재 카테고리와 다름
- Dry Run이 아님
```

### 4. Rate Limit
```
각 이슈 간 2초 대기 (Groq API 보호)
```

---

## 주의사항

### 1. Dry Run 먼저 실행

실제 적용 전에 항상 Dry Run으로 결과 확인:

```bash
npx tsx scripts/reclassify-issues.ts --category 스포츠 --dry-run
```

### 2. Rate Limit

- 100개 이슈 = 약 3-4분 소요
- Groq API 무료 한도: 14,400회/일
- 안전 여유: 1000개까지 OK

### 3. 백업 권장

대량 재분류 전 DB 백업:

```sql
-- Supabase Dashboard → Database → Backups
```

### 4. 신뢰도 임계값

- 80% 이상: 자동 변경
- 80% 미만: 변경 안 함 (수동 확인 필요)

## 문제 해결

### Q1: "Module not found" 에러

```bash
# 의존성 설치 확인
npm install
```

### Q2: "GROQ_API_KEY" 에러

```bash
# .env.local 확인
cat .env.local | grep GROQ_API_KEY

# 없으면 추가
echo "GROQ_API_KEY=your-key" >> .env.local
```

### Q3: 너무 느림

```bash
# 개수 제한
npx tsx scripts/reclassify-issues.ts --limit 10
```

### Q4: 특정 이슈만 재분류

```sql
-- Supabase Dashboard → SQL Editor
-- 해당 이슈의 ID 확인 후 수동 업데이트
UPDATE issues 
SET category = '연예' 
WHERE id = 'issue-id-here';
```
