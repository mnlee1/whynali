# 세이프티 가이드

왜난리 프로젝트의 세이프티봇 시스템 운영 가이드.

## 개선된 기능

### 1. 단어 경계 매칭 (오탐 방지)
**위치**: `lib/safety.ts`

정규식 기반 단어 경계 매칭으로 오탐 최소화:
- 공백, 구두점, 단어 경계에서만 매칭
- 한글과 영문 모두 지원
- 다른 단어의 일부와 겹치지 않음

```typescript
function createWordBoundaryRegex(word: string): RegExp {
    const escaped = word.replace(/[.*+?^${}()|[\]\\]/g, '\\$&')
    return new RegExp(`(?:^|\\s|[^\\w가-힣])${escaped}(?:$|\\s|[^\\w가-힣])`, 'i')
}
```

**테스트 예시:**
```
금칙어: "나쁨"
✅ "이건 나쁨" → 차단 (공백 경계)
✅ "이건나쁨" → 차단 (문장 경계)
✅ "남바다" → 통과 (다른 단어 일부)
```

### 2. DB 금칙어 검증 완전 통합
모든 쓰기 API에서 동일한 금칙어 정책 적용:
- 댓글 작성/수정
- 토론 주제 작성
- 토론 댓글 작성/수정

### 3. 관리자 알림 자동 발송
**위치**: `lib/safety-notification.ts`

검토 대기 댓글이 10건 이상일 때 자동으로 관리자 이메일 발송:
- 10분 쿨다운으로 중복 알림 방지
- 비동기 처리로 응답 속도에 영향 없음

### 4. Rate Limit 메모리 정리
**위치**: `lib/safety.ts`, `app/api/cron/cleanup-rate-limit/route.ts`

만료된 Rate Limit 항목을 주기적으로 정리:
- Vercel Cron으로 10분마다 자동 정리
- 메모리 누적 방지

### 5. 관리자 UI 개선
**위치**: `app/admin/safety/page.tsx`

- 검토 대기 10건 이상 시 빨간색 경고 표시
- 금칙어 정책 상세 안내
- 매칭 방식, 검토 배치 정책 명시

### 6. 에러 처리 강화
DB 조회 실패 시:
- try-catch로 에러 처리
- 빈 배열 반환 (하드코딩 금칙어는 항상 적용)
- 콘솔 로그로 에러 추적

## 금칙어 관리

### 금칙어 추가
Supabase Table Editor → `safety_rules` 테이블:
```sql
INSERT INTO safety_rules (kind, value)
VALUES ('banned_word', '금칙어');
```

### 금칙어 목록
하드코딩 금칙어 + DB 금칙어 결합:
- 하드코딩: `lib/safety.ts` 내부 배열
- DB: `safety_rules` 테이블 `kind='banned_word'`

## 검토 대기 처리

### 자동 알림 조건
- 검토 대기 댓글 10건 이상
- 10분 쿨다운 (중복 알림 방지)

### 관리자 검토
1. `/admin/safety` 페이지 접속
2. 검토 대기 목록 확인
3. 각 댓글 공개/삭제 처리

### 검토 배치 정책
- 일 1회 또는 10건 이상 시 배치 검토
- 10건 미만은 일 1회 처리로 충분

## 비용

모든 개선 사항은 **무료 범위 내에서 구현**:
- 단어 경계 매칭: 정규식 처리 (비용 없음)
- DB 금칙어 통합: 기존 DB 쿼리 활용 (비용 없음)
- 관리자 알림: 기존 이메일 기능 + 10분 쿨다운 (무료)
- Rate Limit 정리: Vercel Cron 무료 한도 충분 (월 100회 = 4,320회)
