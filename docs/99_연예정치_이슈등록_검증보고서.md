# 연예/정치 카테고리 이슈 등록 검증 보고서

> 작성일: 2026-03-11
> 검증 대상: 왜난리의 핵심 가치인 "논란 빠른 확인" 기능 검증

## 검증 배경

왜난리는 논란 이슈를 빠르게 확인할 수 있는 서비스이며, 논란은 주로 **연예/정치 카테고리**에서 발생합니다.
실제로 이런 논란 이슈들이 제대로 등록되고 있는지 검증이 필요했습니다.

## 실제 데이터 분석 결과

### 1. 카테고리별 이슈 현황

```
카테고리 | 전체 | 대기 | 승인 | 반려 | 평균화력
연예       |    6 |    0 |    4 |    1 |   13.0점
정치       |   10 |    2 |    2 |    1 |   20.7점
사회       |   18 |    4 |    2 |    6 |   15.3점
IT과학     |   14 |    6 |    3 |    2 |   12.4점
스포츠      |    5 |    0 |    2 |    0 |   13.6점
```

### 2. 뉴스 수집 현황 (최근 24시간)

```
연예: 65건
정치: 112건
사회: 172건
IT과학: 123건
스포츠: 110건
```

### 3. 연예+정치 비율

```
16/53 이슈 = 30.2%
```

## 핵심 발견 사항

### ✅ 긍정적 요소

1. **뉴스 수집은 충분함**
   - 연예 65건, 정치 112건으로 24시간 기준 충분한 양 수집 중
   
2. **논란성 카테고리 비율 적절**
   - 연예+정치가 전체의 30.2%로 적절한 비중 유지
   
3. **이슈 등록 자체는 작동함**
   - 연예 6건, 정치 10건으로 최소한의 등록은 이루어지고 있음

### ⚠️ 문제점

#### 문제 1: 정치 이슈 중 approval_status가 null인 경우 다수 발견

```
5. [null] 국민의힘, '절윤' 공식화…"계엄 사과·尹정치적 복귀 반대"
   화력: 42.0점
   
6. [null] 국힘 "尹 정치적 복귀 요구에 명확히 반대"…의총서 결의문 발표
   화력: 65.0점
```

**심각도: 높음**
- 화력 42점, 65점으로 매우 높은 이슈가 approval_status가 null
- null 상태인 이슈는 사용자에게 노출되지 않음
- 왜난리의 핵심 가치인 "논란 빠른 확인" 기능이 작동하지 않음

**원인 추정:**
- DB 스키마에 approval_status 기본값이 없을 가능성
- 이슈 생성 시 approval_status를 명시하지 않은 코드 존재
- 마이그레이션 또는 수동 생성 시 누락

#### 문제 2: 연예 카테고리 평균 화력이 낮음 (13.0점)

```
연예 평균 화력: 13.0점 (최소 등록 기준 15점 미만)
```

**심각도: 중간**
- 연예 이슈가 화력 15점을 넘기기 어려움
- 실제 논란이 많은 카테고리인데 등록이 부족할 수 있음

**원인 추정:**
- 연예 뉴스는 수집되지만(65건) 특정 이슈로 그루핑되지 않음
- 분산된 연예 뉴스가 하나의 이슈로 묶이지 못함
- 화력 계산 시 연예 카테고리 가중치 부족

#### 문제 3: 정치 이슈 대기 상태 방치

```
2. [대기] 한동훈 "尹 복귀 반대? 절연 본질 가려…"
   화력: 23.0점 | 생성: 2026. 3. 11. AM 11:49:38
   
3. [대기] 한동훈 "국힘, 尹어게인 절연?…"
   화력: 24.0점 | 생성: 2026. 3. 11. AM 10:50:54
```

**심각도: 중간**
- 화력 23-24점으로 논란성이 있는 이슈가 대기 상태
- 자동 승인 기준(30점)에 미달하여 관리자 승인 대기 중
- 2시간 타임아웃 자동 승인 기능도 작동하지 않음

## 자동 승인 정책 검증

### 현재 정책

```typescript
AUTO_APPROVE_CATEGORIES=사회,경제,IT과학,생활문화,세계,스포츠
AUTO_APPROVE_THRESHOLD=30점
```

**연예/정치는 자동 승인 제외 → 관리자 승인 필수**

### 정책의 문제점

1. **화력 30점 기준이 너무 높음**
   - 연예 평균 13점, 정치 평균 20.7점
   - 대부분의 이슈가 30점을 넘기지 못함
   
2. **긴급 이슈 타임아웃 자동 승인 미작동**
   - 2시간 타임아웃 기능이 있으나 실제로 작동하지 않음
   - 환경변수: URGENT_AUTO_APPROVE_HOURS=2
   
3. **관리자 승인 병목**
   - 연예/정치 이슈가 대기 상태로 쌓임
   - 실시간 논란 확인이라는 서비스 가치 훼손

## 권장 개선 방안

### 1. approval_status null 이슈 긴급 조치

**즉시 실행:**

```sql
-- null 상태 이슈 조회
SELECT id, title, category, heat_index, created_at
FROM issues
WHERE approval_status IS NULL
ORDER BY heat_index DESC;

-- 화력 15점 이상 이슈를 대기 상태로 변경
UPDATE issues
SET approval_status = '대기'
WHERE approval_status IS NULL
AND heat_index >= 15;

-- 화력 15점 미만은 반려
UPDATE issues
SET approval_status = '반려'
WHERE approval_status IS NULL
AND heat_index < 15;
```

**근본 해결:**

```sql
-- approval_status 기본값 설정
ALTER TABLE issues
ALTER COLUMN approval_status SET DEFAULT '대기';
```
