# 투표 기능 구현 가이드

왜난리 프로젝트의 투표 기능 구현 세부사항 문서.
토론 주제와 동일한 패턴(직접 생성 + AI 생성)을 적용.

## 1. 개요

기획 문서(02_AI기획_판단포인트.md §3, §6.4)에 따라 투표 기능은 필수 항목으로:
- 이슈당 복수 투표 가능 (시점별 여론 변화 추적)
- 2-4개 선택지
- 실시간 득표 비율 그래프
- 이슈 상태(점화/논란중/종결)별 투표 구분

## 2. 구현 항목

### 2.1 AI 투표 생성기

**파일**: `lib/ai/vote-generator.ts`

이슈 메타데이터(제목, 카테고리, 상태, 화력)만 사용하여 투표 후보를 생성.
토론 주제 생성기와 동일한 패턴 사용.

법적 안전 원칙:
- 본문·요약문 절대 미사용
- 특정인 실명·특정 집단 직접 지목 금지
- 생성된 투표는 phase='대기'로 저장, 관리자 승인 후 활성화

### 2.2 API 엔드포인트

#### 관리자 - 투표 목록 조회
```
GET /api/admin/votes?phase={대기|진행중|마감}&limit=50
```

#### 관리자 - 투표 생성 (직접)
```
POST /api/admin/votes
Body: { 
    issue_id, 
    title, 
    choices: string[],
    auto_end_date?: string,
    auto_end_participants?: number
}
```
- phase='대기'로 생성
- issue_status_snapshot 자동 저장
- 자동 종료 옵션 설정 가능
  - auto_end_date: ISO 날짜 문자열 (특정 날짜에 자동 종료)
  - auto_end_participants: 숫자 (목표 참여자 수 도달 시 자동 종료)

#### 관리자 - AI 투표 생성
```
POST /api/admin/votes/generate
Body: { issue_id, count?: number }
```
- count: 1-3개 (기본 2)
- 승인된 이슈만 대상
- 생성 후 phase='대기' 상태로 저장

#### 관리자 - 투표 승인
```
POST /api/admin/votes/{id}/approve
```
- phase='대기' → '진행중'
- started_at 현재 시각으로 설정

#### 관리자 - 투표 반려
```
POST /api/admin/votes/{id}/reject
```
- approval_status를 '반려'로 변경
- 투표는 삭제되지 않고 유지됨
- '대기' 상태만 반려 가능

#### 관리자 - 투표 삭제
```
DELETE /api/admin/votes/{id}
```
- 투표 및 선택지 완전 삭제
- '대기' 또는 '반려' 상태만 삭제 가능
- '진행중' 또는 '마감' 상태는 삭제 불가

#### 공개 - 투표 목록
```
GET /api/votes?issue_id={id}
```
- phase='진행중' 또는 '마감'만 반환 (대기 제외)

#### 크론 - 자동 종료
```
GET /api/cron/auto-end-votes
```
- 매 시간마다 실행 (GitHub Actions)
- 날짜 기준 자동 종료: auto_end_date가 현재 시각 이전인 투표를 마감
- 참여자 수 기준 자동 종료: auto_end_participants 목표 달성한 투표를 마감
- 워크플로우: `.github/workflows/cron-auto-end-votes.yml` (개발)
- 워크플로우: `.github/workflows/prod-cron-auto-end-votes.yml` (운영)

#### 관리자 - 수동 종료
```
POST /api/admin/votes/{id}/close
```
- 진행 중인 투표를 즉시 마감 처리
- phase를 '마감'으로 변경, ended_at 설정

#### 이슈 연동 자동 종료

**이슈 상태 전환**:
- `app/api/cron/recalculate-heat/route.ts`: 이슈가 '종결' 상태로 전환될 때 관련 투표 자동 마감
- `lib/analysis/status-transition.ts`: 상태 전환 로직에 투표 마감 연동

**이슈 삭제/숨김**:
- `app/api/admin/issues/[id]/hide/route.ts`: 이슈 숨김 시 관련 투표 자동 마감

**유틸리티**:
- `lib/vote-auto-closer.ts`: 투표 자동 마감 공통 로직
  - `closeVotesOnIssueClosed()`: 이슈 종결 시 호출
  - `closeVotesOnIssueDeleted()`: 이슈 삭제 시 호출

### 2.3 DB 스키마 확장

**마이그레이션**: 
- `supabase/migrations/add_vote_status_and_timestamps.sql`
- `supabase/migrations/add_vote_auto_end_options.sql`
- `supabase/migrations/add_approval_status_to_votes.sql`

추가 필드:
- `issue_status_snapshot`: 투표 생성 당시 이슈 상태 저장
- `started_at`: 투표 시작(승인) 시각
- `ended_at`: 투표 종료(마감) 시각
- `auto_end_date`: 자동 종료 날짜
- `auto_end_participants`: 목표 참여자 수
- `approval_status`: 관리자 검토 상태 (대기/승인/반려)

phase 값:
- '대기': 승인 대기 중
- '진행중': 사용자 투표 가능
- '마감': 투표 종료

approval_status 값:
- '대기': 검토 전
- '승인': 승인됨 (사용자 노출)
- '반려': 거부됨 (삭제 전 상태)

### 2.4 관리자 페이지

**파일**: `app/admin/votes/page.tsx`

토론 주제 관리 페이지와 동일한 구조:

직접 생성 폼:
1. 이슈 검색·선택 (승인된 이슈만)
2. 투표 제목 입력 (40자 이내)
3. 선택지 입력 (2-6개, 각 20자 이내)
4. 자동 종료 옵션 설정
   - 날짜 기준: 특정 날짜/시각 지정
   - 참여자 수 기준: 목표 참여자 수 지정

AI 생성 폼:
1. 이슈 검색·선택 (승인된 이슈만)
2. 생성 개수 선택 (1-3개)
3. 자동 종료 옵션 설정 (선택)
4. AI 생성 버튼 → 대기 목록에 추가

필터 탭:
- 전체 / 대기 / 진행중 / 마감 / 반려

대기 투표:
- 승인 버튼 → approval_status='승인', phase='진행중', started_at 설정
- 반려 버튼 → approval_status='반려' (삭제하지 않음)
- 삭제 버튼 → 투표·선택지 완전 삭제

반려 투표:
- 삭제 버튼만 표시
- 재검토 후 수동으로 대기 상태로 변경 가능 (DB에서)

### 2.5 사용자 화면 개선

**파일**: `components/issue/VoteSection.tsx`

참여 유도:
- 비로그인 사용자: 강화된 CTA 카드 표시
  - 전체 참여자 수 강조
  - 눈에 띄는 "로그인하고 투표하기" 버튼
- 로그인 사용자: 간단한 안내 문구

진행중 투표:
- 상단 강조 표시
- 자동 종료 정보 표시
  - 24시간 이내 종료: 깜박이는 배지 + 남은 시간
  - 참여자 목표: 진행률 바 + 퍼센트
- 로그인 시 참여/취소 가능

이전 투표 (접이식):
- 이슈 상태별 블록으로 구분
  - 점화 시기 투표
  - 논란중 시기 투표
  - 종결 시기 투표
  - 기타
- 각 블록별로 투표 목록 표시
- 득표 결과 표시 (비율 그래프)

### 2.6 타입 정의

**파일**: `types/index.ts`

```typescript
export interface Vote {
    id: string
    issue_id: string
    title?: string
    phase?: '대기' | '진행중' | '마감'
    issue_status_snapshot?: '점화' | '논란중' | '종결' | null
    started_at?: string | null
    ended_at?: string | null
    created_at: string
}
```

## 3. 사용 흐름

### 관리자 흐름

1. 투표 생성
   - 방법 A: 직접 생성 (제목·선택지 입력)
   - 방법 B: AI 생성 (이슈 선택 → AI가 후보 생성)

2. 대기 목록 확인
   - AI가 생성한 투표 후보 검토
   - 직접 생성한 투표도 대기 상태로 저장됨

3. 승인/반려/삭제
   - 승인: approval_status='승인', phase='진행중', 사용자에게 노출
   - 반려: approval_status='반려', 투표는 유지됨
   - 삭제: 투표·선택지 완전 삭제 (대기/반려 상태만 가능)

4. 진행중 투표 관리
   - 득표 현황 확인
   - 수동 종료: 관리자가 즉시 투표 마감 처리
   - 자동 종료: 조건 충족 시 자동 마감 (날짜/참여자 수)

5. 투표 자동 종료 조건
   - 설정된 날짜/시각 도달
   - 목표 참여자 수 달성
   - 이슈가 '종결' 상태로 전환
   - 이슈가 삭제/숨김 처리

### 사용자 흐름

1. 이슈 상세 페이지 진입

2. 진행중 투표 참여
   - 로그인 필수
   - 선택지 클릭 → 즉시 반영
   - 재클릭 시 취소
   - 다른 선택지로 변경 불가 (취소 후 재선택)

3. 이전 투표 보기
   - 접이식 영역 확장
   - 이슈 상태별로 구분된 과거 투표 확인
   - 당시 여론 분포 확인

## 4. 법적 안전성

토론 주제 생성과 동일한 원칙 적용:
- 메타데이터만 사용 (본문·요약문 미사용)
- 특정인 실명·특정 집단 직접 지목 금지
- 관리자 승인 단계 필수 (phase='대기' → '진행중')
- 중립적이고 명확한 투표 문항 생성

## 5. 운영 가이드

### AI 생성 활용

적합한 경우:
- 이슈가 명확하고 여론 조사가 필요한 경우
- 시점별 여론 변화 추적이 필요한 경우
- 빠른 투표 생성이 필요한 경우

직접 생성 권장:
- 특수한 문항이 필요한 경우
- 세밀한 선택지 구성이 필요한 경우
- AI 생성 결과가 부적합한 경우

### 승인 기준

승인 조건:
- 중립적이고 명확한 질문
- 상호 배타적인 선택지
- 실명·특정 집단 직접 지목 없음
- 이슈와 연관성이 있음

반려 조건:
- 편향된 질문
- 애매모호한 선택지
- 실명·특정 집단 직접 지목
- 이슈와 무관한 내용

### 자동 종료 옵션 활용

날짜 기준 종료 권장:
- 특정 시점까지만 의견을 받고 싶은 경우
- 이벤트/행사와 연동된 투표
- 주간/월간 정기 투표

참여자 수 기준 종료 권장:
- 빠른 결과가 필요한 경우
- 일정 샘플 크기만 필요한 경우
- 참여 유도를 위한 목표 설정

둘 다 사용:
- 날짜 또는 참여자 수 중 먼저 도달하는 조건으로 종료
- 최대 기한 + 최소 참여자 수 보장
