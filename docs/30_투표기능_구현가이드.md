# 투표 기능 구현 가이드

왜난리 프로젝트의 투표 기능 구현 세부사항 문서.
토론 주제와 동일한 패턴(직접 생성 + AI 생성)을 적용.

## 1. 개요

기획 문서(02_AI기획_판단포인트.md §3, §6.4)에 따라 투표 기능은 필수 항목으로:
- 이슈당 복수 투표 가능 (시점별 여론 변화 추적)
- 2-4개 선택지
- 실시간 득표 비율 그래프
- 이슈 상태(점화/논란중/종결)별 투표 구분

## 2. 구현 항목

### 2.1 AI 투표 생성기

**파일**: `lib/ai/vote-generator.ts`

이슈 메타데이터(제목, 카테고리, 상태, 화력)만 사용하여 투표 후보를 생성.
토론 주제 생성기와 동일한 패턴 사용.

법적 안전 원칙:
- 본문·요약문 절대 미사용
- 특정인 실명·특정 집단 직접 지목 금지
- 생성된 투표는 phase='대기'로 저장, 관리자 승인 후 활성화

### 2.2 API 엔드포인트

#### 관리자 - 투표 목록 조회
```
GET /api/admin/votes?phase={대기|진행중|마감}&limit=50
```

#### 관리자 - 투표 생성 (직접)
```
POST /api/admin/votes
Body: { issue_id, title, choices: string[] }
```
- 기존 로직 유지, phase='대기'로 생성하도록 수정
- issue_status_snapshot 자동 저장

#### 관리자 - AI 투표 생성
```
POST /api/admin/votes/generate
Body: { issue_id, count?: number }
```
- count: 1-3개 (기본 2)
- 승인된 이슈만 대상
- 생성 후 phase='대기' 상태로 저장

#### 관리자 - 투표 승인
```
POST /api/admin/votes/{id}/approve
```
- phase='대기' → '진행중'
- started_at 현재 시각으로 설정

#### 관리자 - 투표 반려
```
POST /api/admin/votes/{id}/reject
```
- 투표 및 선택지 완전 삭제
- phase='대기'인 것만 반려 가능

#### 공개 - 투표 목록
```
GET /api/votes?issue_id={id}
```
- phase='진행중' 또는 '마감'만 반환 (대기 제외)

### 2.3 DB 스키마 확장

**마이그레이션**: `supabase/migrations/add_vote_status_and_timestamps.sql`

추가 필드:
- `issue_status_snapshot`: 투표 생성 당시 이슈 상태 저장
- `started_at`: 투표 시작(승인) 시각
- `ended_at`: 투표 종료(마감) 시각

phase 값 확장:
- 기존: '진행중' | '마감'
- 개선: '대기' | '진행중' | '마감'

### 2.4 관리자 페이지

**파일**: `app/admin/votes/page.tsx`

토론 주제 관리 페이지와 동일한 구조:

직접 생성 폼:
1. 이슈 검색·선택 (승인된 이슈만)
2. 투표 제목 입력 (40자 이내)
3. 선택지 입력 (2-6개, 각 20자 이내)

AI 생성 폼:
1. 이슈 검색·선택 (승인된 이슈만)
2. 생성 개수 선택 (1-3개)
3. AI 생성 버튼 → 대기 목록에 추가

필터 탭:
- 전체 / 대기 / 진행중 / 마감

대기 투표:
- 승인 버튼 → phase='진행중', started_at 설정
- 반려 버튼 → 투표·선택지 삭제

### 2.5 사용자 화면 개선

**파일**: `components/issue/VoteSection.tsx`

진행중 투표:
- 상단 강조 표시
- 로그인 시 참여/취소 가능

이전 투표 (접이식):
- 이슈 상태별 블록으로 구분
  - 점화 시기 투표
  - 논란중 시기 투표
  - 종결 시기 투표
  - 기타
- 각 블록별로 투표 목록 표시
- 득표 결과 표시 (비율 그래프)

### 2.6 타입 정의

**파일**: `types/index.ts`

```typescript
export interface Vote {
    id: string
    issue_id: string
    title?: string
    phase?: '대기' | '진행중' | '마감'
    issue_status_snapshot?: '점화' | '논란중' | '종결' | null
    started_at?: string | null
    ended_at?: string | null
    created_at: string
}
```

## 3. 사용 흐름

### 관리자 흐름

1. 투표 생성
   - 방법 A: 직접 생성 (제목·선택지 입력)
   - 방법 B: AI 생성 (이슈 선택 → AI가 후보 생성)

2. 대기 목록 확인
   - AI가 생성한 투표 후보 검토
   - 직접 생성한 투표도 대기 상태로 저장됨

3. 승인/반려
   - 승인: phase='진행중', 사용자에게 노출
   - 반려: 투표·선택지 삭제

4. 진행중 투표 모니터링
   - 득표 현황 확인
   - 필요 시 수동 종료 (향후 기능)

### 사용자 흐름

1. 이슈 상세 페이지 진입

2. 진행중 투표 참여
   - 로그인 필수
   - 선택지 클릭 → 즉시 반영
   - 재클릭 시 취소
   - 다른 선택지로 변경 불가 (취소 후 재선택)

3. 이전 투표 보기
   - 접이식 영역 확장
   - 이슈 상태별로 구분된 과거 투표 확인
   - 당시 여론 분포 확인

## 4. 법적 안전성

토론 주제 생성과 동일한 원칙 적용:
- 메타데이터만 사용 (본문·요약문 미사용)
- 특정인 실명·특정 집단 직접 지목 금지
- 관리자 승인 단계 필수 (phase='대기' → '진행중')
- 중립적이고 명확한 투표 문항 생성

## 5. 운영 가이드

### AI 생성 활용

적합한 경우:
- 이슈가 명확하고 여론 조사가 필요한 경우
- 시점별 여론 변화 추적이 필요한 경우
- 빠른 투표 생성이 필요한 경우

직접 생성 권장:
- 특수한 문항이 필요한 경우
- 세밀한 선택지 구성이 필요한 경우
- AI 생성 결과가 부적합한 경우

### 승인 기준

승인 조건:
- 중립적이고 명확한 질문
- 상호 배타적인 선택지
- 실명·특정 집단 직접 지목 없음
- 이슈와 연관성이 있음

반려 조건:
- 편향된 질문
- 애매모호한 선택지
- 실명·특정 집단 직접 지목
- 이슈와 무관한 내용
