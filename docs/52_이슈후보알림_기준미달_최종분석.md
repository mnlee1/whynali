# 이슈 후보 알림 기준 미달 버그 최종 분석

## 문제 재정의

**캡처 이미지 내용:**
```
수집 데이터 기반 이슈 후보 1건 — 즉시 처리 필요
고려대학, '인터배터리 2026'서 이차전지 소재 전방물질 기술력 선보인다
뉴스 3건 (4건 → 1건 + 커뮤니티 0)
```

**문제:** `ALERT_THRESHOLD = 5`이지만 3-4건만 있는 이슈가 알림에 표시됨

## 코드 분석 결과

### 1. 이슈 후보 알림 생성 흐름

```typescript
// lib/candidate/issue-candidate.ts

// 1단계: 뉴스만 그루핑 (568-595번 라인)
const newsRawItems = (newsItems ?? []).map(n => ({ ...n, type: 'news' }))
groups = groupItems(newsRawItems)

// 2단계: 그룹별 처리 (735-1014번 라인)
for (const [groupIndex, group] of groups.entries()) {
    const recentCount = group.items.length  // 737번 라인
    
    // 기준 체크 (739번 라인)
    if (recentCount < ALERT_THRESHOLD) continue  // 5건 미만 필터링
    
    // ... (필터링: 뉴스 존재, 출처 다양성 등)
    
    const newsIds = group.items.map(i => i.id)  // 803번 라인
    
    // 알림 추가 (846, 984번 라인)
    result.alerts.push({
        title: representativeTitle,
        count: recentCount,              // group.items.length
        newsCount: newsIds.length,       // group.items.length와 동일
        communityCount: communityIds.length,
    })
}
```

### 2. 검증 결과

**`recentCount`와 `newsIds.length`는 항상 동일합니다:**
- `recentCount = group.items.length`
- `newsIds = group.items.map(i => i.id)`
- `newsIds.length === group.items.length === recentCount`

**739번 라인에서 5건 미만은 모두 필터링됩니다:**
```typescript
if (recentCount < ALERT_THRESHOLD) continue
```

### 3. 불가능한 이유

현재 코드로는 **3-4건의 이슈가 알림에 표시될 수 없습니다.**

## 가능한 원인

### 원인 A: 다른 버전의 코드
캡처 이미지의 UI 텍스트 형식이 현재 코드와 다릅니다:

**현재 코드:**
```typescript
최근 3시간 {alert.count}건 (뉴스 {alert.newsCount} + 커뮤니티 {alert.communityCount})
```

**캡처 이미지:**
```
뉴스 3건 (4건 → 1건 + 커뮤니티 0)
```

→ **다른 버전의 UI일 가능성**

### 원인 B: 데이터 캐싱 문제
```typescript
// app/api/admin/candidates/route.ts (25-37번 라인)
const cached = getCachedCandidates()
if (cached) {
    return NextResponse.json({
        alerts: cached.alerts,
        evaluated: cached.evaluated,
        cached: true,
    })
}
```

→ 이전 버전의 데이터가 캐시되어 표시될 가능성

### 원인 C: Race Condition
코드 배포 중 또는 캐시 갱신 중에 발생한 일시적 오류

### 원인 D: 수동 테스트 데이터
개발자가 테스트용으로 수동으로 `alerts` 배열에 추가한 데이터

## 해결 방법

### 1. 즉시 적용 (완료)
관리자 UI에 기준 미달 경고 추가:

```typescript
{!isValid && (
    <span className="ml-2 text-red-600 font-semibold">
        ⚠️ 기준 미달 (최소 {threshold}건 필요)
    </span>
)}
```

### 2. 디버깅 스크립트 생성 (완료)
`scripts/check-candidate-alerts.ts` 생성하여 실제 데이터 확인 가능

### 3. 추가 안전 장치 (권장)
API 응답 시점에 최종 검증 추가:

```typescript
// app/api/admin/candidates/route.ts
const result = await evaluateCandidates()

// 최종 필터링: 기준 미달 알림 제거
const validAlerts = result.alerts.filter(alert => 
    alert.count >= parseInt(process.env.CANDIDATE_ALERT_THRESHOLD ?? '5')
)

return NextResponse.json({
    alerts: validAlerts,
    evaluated: result.evaluated,
    cached: false,
})
```

### 4. 로깅 강화 (권장)
기준 미달 데이터 발견 시 즉시 로그 출력
