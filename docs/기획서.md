## 1. 프로젝트 개요

**서비스명**: 왜난리 (WhyNali)

**서비스 유형**: 한국 이슈 내비게이션 웹 서비스

**배포 환경**: Vercel (https://whynali.vercel.app)

**개발 기간**: 2주 MVP 완료 후 지속 운영 중 (2026-03 기준 95% 구현 완료)

### 핵심 콘셉트

뉴스·커뮤니티에 분산된 한국 이슈 정보를 한 곳에 모아 시계열(발단 → 전개 → 파생 → 진정)로 제공하는 이슈 내비게이션 서비스.

---

## 2. 배경 및 문제 정의

### 문제

- 연예·스포츠·정치·사회·기술 등 이슈 정보가 뉴스/커뮤니티/SNS에 분산
- "지금 무슨 일이 벌어졌고 왜 커졌는지"를 한번에 파악하기 어려움
- 이슈의 진행 과정(발단 → 전개 → 파생 → 진정)을 시계열로 정리해주는 서비스 부재
- 이슈 탐색 비용이 높아 재방문 동기가 약함

### 해결 방향

- 이슈를 이벤트 단위로 묶어 타임라인 + 이슈 포인트 제공
- "현재 상황"과 "왜 이렇게 되었는지"를 동시 제공
- 감정 표현·댓글·투표로 여론 변화 확인 가능하게 설계

---

## 3. 서비스 방향

| 방향 | 내용 |
|------|------|
| 이슈 내비게이션형 | 단순 기사 모음이 아닌, 이벤트 단위 타임라인 제공 |
| 모바일 퍼스트 | 반응형 웹 설계 |
| 선택적 로그인 | 이슈·타임라인 조회는 비로그인 가능, 감정·댓글·투표는 로그인 필요 |
| 법적 안전성 | 뉴스·커뮤니티 메타데이터만 수집, 본문·AI 요약 영구 제외 |
| 관리자 개입 최소화 | AI 자동화 + 관리자 최종 승인 구조 |

## 4. 핵심 기능

### 4.1 이슈 관리 시스템

#### 이슈 카테고리 (6개)

연예 / 스포츠 / 정치 / 사회 / 경제 / 기술

#### 이슈 상태 (3단계 자동 전환)

| 상태 | 의미 |
|------|------|
| 점화(Ignited) | 막 시작되어 확산 중 |
| 논란중(Debating) | 양측 의견 대립 중 |
| 종결(Closed) | 사과문 발표 또는 결론 도출 |

#### 이슈 등록 기준

- 최근 1시간 내 동일 이슈 후보로 묶인 건수 5건 이상 + 고유 출처 2곳 이상 → 대기 등록
- 화력 15점 미만: 자동 반려
- 화력 30점 이상 + 허용 카테고리(사회/기술/스포츠): 자동 승인
- 8건 이상 + 6시간 무대응: 자동 승인
- 나머지: 관리자 수동 승인/반려

#### 화력 지수 계산

```
화력 지수 = newsCredibility × (0.3 + 0.7 × community_amp)
```

- 출력 범위: 0~100 (정수)
- 등급: 높음(70+) / 보통(30~69) / 낮음(~29)
- 자동 반영 (관리자 승인 불필요)

### 4.2 자동 수집 파이프라인

| 출처 | 수집 주기 | 수집 항목 |
|------|-----------|-----------|
| 네이버 뉴스 API | 30분마다 | 제목, 링크, 출처, 날짜만 (요약문 제외) |
| 더쿠 | 3분마다 | 제목, URL, 조회수, 댓글수, 작성시간 (본문 제외) |
| 네이트판 | 3분마다 | 제목, URL, 조회수, 댓글수, 작성시간 (본문 제외) |

### 4.3 AI 처리 시스템

**AI 모델**: Groq Llama 3.1 8B Instant (무료, 다중 키 순환)

| 기능 | 설명 |
|------|------|
| 카테고리 자동 분류 | 키워드 기반 분류 신뢰도 낮을 때 AI 재분류 |
| 4단계 중복 체크 | 제목 → 공통 키워드 → 반대어/숫자 필터 → AI 정밀 비교 (신뢰도 80% 이상) |
| 커뮤니티 급증 검증 | 밈·드립·장난 글 자동 필터링 (신뢰도 70% 이상) |
| 뉴스-이슈 연결 검증 | 복합 키워드 필터링 후 관련도 70% 이상만 연결 |
| 토론 주제 생성 | 이슈별 토론 주제 자동 생성 (관리자 승인 필수) |
| 투표 생성 | 이슈별 투표 주제·선택지 자동 생성 (관리자 승인 필수) |

**Rate Limit 대응**: Supabase 기반 키 차단 상태 공유, 429 에러 시 다음 키로 자동 전환

### 4.4 타임라인 시스템

- 4단계 시각화: 발단 → 전개 → 파생 → 진정
- 각 포인트에 출처 링크 연결
- 시간순 정렬
- AI 자동 생성 + 관리자 수동 편집 지원

### 4.5 사용자 참여 기능

#### 감정 표현 (7종)

좋아요 / 싫어요 / 화나요 / 팝콘각 / 응원 / 애도 / 사이다

- 이슈당 1인 1회, 변경·취소 가능

#### 댓글

- 베스트 댓글 상단 노출 (score = 좋아요 - 싫어요 상위)
- 정렬: 최신순 / 좋아요순 / 싫어요순
- 대댓글 지원
- 세이프티봇 자동 필터링

#### 투표 시스템

- 이슈당 복수 투표 (시점별 여론 변화 추적)
- 선택지 2~6개 지원
- 실시간 득표 비율 그래프
- 자동 마감 3가지 조건: 날짜 기준 / 참여자 수 달성 / 이슈 종결 연동 (3일 유예)
- AI 자동 생성 + 관리자 승인 구조

### 4.6 커뮤니티 (토론 주제)

- AI 토론 주제 자동 생성 (Groq 기반)
- 관리자 승인 후 노출
- 이슈별 토론 주제 진입 가능 ("이 이슈의 커뮤니티" 링크)
- 전체 목록 `/community` 메뉴에서 검색·필터 가능
- 댓글 참여 (로그인 필요)

### 4.7 인증 시스템

- 구글 / 네이버 / 카카오 오픈ID 로그인
- 비로그인: 이슈·타임라인 조회만 가능
- 로그인: 감정·댓글·투표·토론 참여 가능
- 관리자: ADMIN_EMAILS 환경변수 기반 권한 관리

### 4.8 세이프티봇

- 금칙어 목록 기반 자동 필터링 (단어 경계 매칭)
- 매칭 시 검토 대기 상태로 전환
- 안내 문구: "등록되었습니다. 내용 검토 후 공개되거나 삭제될 수 있습니다."
- 관리자 일 1회 이상 배치 검토 (또는 10건 이상 시)

### 4.9 관리자 시스템

| 페이지 | 경로 | 주요 기능 |
|--------|------|-----------|
| 대시보드 | /admin | 통계 요약, API 사용량 |
| 이슈 관리 | /admin/issues | 대기 목록, 승인/반려, 숨김 처리 |
| 수집 데이터 | /admin/collections | 뉴스/커뮤니티 현황, 이슈 연결 |
| 투표 관리 | /admin/votes | AI 생성, 승인/반려, 수동 마감 |
| 토론 주제 | /admin/discussions | AI 생성, 승인/반려 |
| 세이프티 | /admin/safety | 금칙어 관리, 검토 대기 처리 |
| 로그 | /admin/logs | 관리자 작업 기록 |

## 5. 화면 구성 (메뉴 구조)

```
[로고/홈]  [연예] [스포츠] [정치] [사회] [기술]  [커뮤니티]  [검색]  [로그인]
    |         └─ 이슈 목록 (카테고리별, 검색/필터/정렬)
    |               └─ 이슈 상세
    |                     ├─ 화력 지수 + 상태 배지
    |                     ├─ 타임라인 (발단→전개→파생→진정)
    |                     ├─ 출처 링크 (뉴스 + 커뮤니티)
    |                     ├─ 감정 표현 7종
    |                     ├─ 댓글 (베스트 + 리스트)
    |                     ├─ 투표 (복수, 실시간 그래프)
    |                     └─ "이 이슈의 커뮤니티" 링크
    ├─ 커뮤니티 목록 (토론 주제, 검색/필터)
    |       └─ 토론 주제 상세 + 댓글
    └─ 글로벌 검색 (이슈 + 토론 주제 통합 검색)
```

### 검색·필터 구분

| 구분 | 범위 | 기능 |
|------|------|------|
| 글로벌 검색 | 전체 서비스 | 이슈 + 커뮤니티 통합 검색 |
| 이슈 페이지 내 | 이슈 목록 | 키워드 검색, 상태별 필터, 정렬 |
| 커뮤니티 페이지 내 | 토론 주제 목록 | 키워드 검색, 이슈 연결 필터 |

### 6.2 시스템 아키텍처

```
[수집 레이어]
  네이버 뉴스 API (30분) ──► news_data 테이블
  더쿠/네이트판 스크래핑 (3분) ──► community_data 테이블

[처리 레이어]  
  이슈 후보 자동 생성
    └─ AI 카테고리 분류 (Groq)
    └─ AI 중복 체크 (4단계)
    └─ 화력 지수 계산
    └─ 관리자 승인 / 자동 승인

[서비스 레이어]
  이슈 상세 + 타임라인 + 출처 링크
  감정 표현 + 댓글 + 투표
  커뮤니티 토론 주제
  관리자 대시보드
```

## 7. 데이터베이스 스키마

### 핵심 테이블

| 테이블 | 역할 |
|--------|------|
| issues | 이슈 기본 정보 (상태, 카테고리, 화력지수, 승인상태, 병합) |
| timeline_points | 발단/전개/파생/진정 타임라인 포인트 |
| votes | 투표 (복수, 자동마감 옵션) |
| vote_choices | 투표 선택지 |
| user_votes | 사용자 투표 기록 (중복 방지) |
| discussion_topics | AI 생성 토론 주제 |
| comments | 댓글 (이슈 + 토론 주제, 대댓글 지원) |
| reactions | 감정 표현 7종 |
| news_data | 뉴스 수집 데이터 (이슈 연결) |
| community_data | 커뮤니티 메타데이터 (이슈 연결) |
| safety_rules | 금칙어·설정 |
| admin_logs | 관리자 작업 로그 |
| ai_key_status | AI 키 차단 상태 관리 (서버리스 인스턴스 간 공유) |

### 이슈 상태 흐름

```
[수집] ──► [이슈 후보 생성] ──► 대기(approval_status)
                                   ├─ 자동 승인 (화력 30+ + 허용 카테고리)
                                   ├─ 관리자 승인
                                   └─ 자동 반려 (화력 15 미만)

[승인된 이슈] ──► 점화 ──► 논란중 ──► 종결
              (화력 증가)  (지속)    (화력 감소/결론)
```

## 10. 향후 로드맵

### 단기 (즉시 처리)

- 투표 자동종료 DB 마이그레이션 실행 후 코드 주석 해제
- 구글·카카오 OAuth 키 설정 완료

### 중기 (1주 내)

- 투표 결과 통계 대시보드
- 여론 변화 추세 그래프
- 이슈 목록 화력순 정렬 UI 추가

### 장기 (확장 단계)

- 개인화 추천 이슈 (유저 조회·참여 이슈 분석)
- 이슈 업데이트 실시간 알림
- 숏폼 자동 생성·배포 (인스타 릴스, 유튜브 쇼츠, 틱톡)
- 모바일 앱 (React Native)
- 메타(페이스북) 로그인
