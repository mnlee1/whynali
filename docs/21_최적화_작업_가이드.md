### Phase 2: Connection Pooler 설정 (소요: 30분)

- [ ] **2-1. 서버 클라이언트 파일 업데이트**
    - 파일: `lib/supabase/server.ts`
    - ✅ 이미 업데이트됨 (Connection Pooler 사용 코드 적용)

- [ ] **2-2. 환경변수 추가 (선택)**
    - 파일: `.env.local`
    - 추가할 내용:
    ```bash
    # Supabase Connection Pooler 사용 (기본값: true)
    USE_SUPABASE_POOLER=true
    ```
    - 참고: 기본값이 true이므로 생략 가능

- [ ] **2-3. 로컬 테스트**
    ```bash
    npm run dev
    ```
    - 브라우저에서 http://localhost:3000 접속
    - 이슈 목록 페이지 확인
    - 개발자 도구 콘솔에서 확인:
        - ✅ "Supabase Admin Client initialized with Connection Pooler"
        - ❌ "initialized WITHOUT Connection Pooler" (이 메시지 나오면 안 됨)

- [ ] **2-4. Vercel 환경변수 설정 (배포 시)**
    - Vercel 대시보드 접속
    - 프로젝트 > Settings > Environment Variables
    - 추가: `USE_SUPABASE_POOLER` = `true`
    - 재배포

**예상 효과:**
- DB 연결 수: 250개 → 20-50개 (5배 감소)
- 연결 생성 시간: 100ms → 10ms (10배 향상)
- 동시접속자 1,000명 안정적 처리

    GROQ_API_KEY=gsk_your_key_1,gsk_your_key_2,gsk_your_key_3,...[발급받은 키들을 콤마로 구분하여 추가]

## ❗ 문제 발생 시

### 1. DB 연결 에러
```
Error: remaining connection slots reserved
```
**해결:**
- Connection Pooler 활성화 확인
- `USE_SUPABASE_POOLER=true` 설정 확인
- Vercel 환경변수 재배포

### 2. Groq API Rate Limit
```
Error: 429 Too Many Requests
```
**해결:**
- 키 개수 확인 (최소 10개)
- 키 순환 로직 확인 (`lib/ai/groq-provider.ts`)
- 키 추가 발급

### 3. 페이지 로딩 느림
**확인 사항:**
- DB 인덱스 생성 확인
- ISR 캐싱 설정 확인 (revalidate: 900)
- Vercel Edge Network 활성화 확인
