# 그루핑 개선 및 AI 시도 전체 히스토리

날짜: 2026-02-26

## 1. 최종 채택 방식: 키워드 개선

민희진 이슈가 등록되지 않는 문제를 해결하기 위해 이슈 그루핑 방식을 개선

### 1.1 핵심 인물 리스트 대폭 확장 (9명 → 30+명)
```typescript
// 정치: 윤석열, 이재명, 한동훈, 이준석, 추경호, 박찬대, 권영세
// 연예: 민희진, 뉴진스, 하이브, 어도어, 방시혁, 옥택연, 김연경, ...
// 스포츠: 손흥민, 이강인, 황희찬, 김하성, 류현진, 오타니
// 기업: 삼성전자, LG전자, SK하이닉스, 네이버, 카카오
```

### 1.2 최종 임계값 설정 (품질 중시)

.env.local:
```bash
CANDIDATE_ALERT_THRESHOLD=5           # 5건 이상 → 대기 등록
CANDIDATE_AUTO_APPROVE_THRESHOLD=10   # 10건 이상 → 자동 승인
CANDIDATE_WINDOW_HOURS=24             # 24시간 집계 창
CANDIDATE_MIN_HEAT_TO_REGISTER=30     # 화력 30점 이상만 등록
CANDIDATE_MIN_UNIQUE_SOURCES=2        # 출처 2곳 이상
CANDIDATE_COMMUNITY_MATCH_THRESHOLD=2 # 커뮤니티 키워드 2개 이상
```

### 1.3 화력 30점 기준 근거

- 화력 공식: newsCredibility × (0.3 + 0.7 × communityAmp)
- 30점 = 약 뉴스 15건 + 커뮤니티 반응 적정
- 실제 이슈 사례:
  - 민희진: 30점 (연예, 이슈성 있음)
  - 옥택연: 69점 (결혼 발표, 핫이슈)
  - 대전충남 통합: 14점 (지역 이슈, 제외됨)
  - LG AI: 12점 (기업 뉴스, 제외됨)

## 2. Perplexity AI 그루핑 시도 (실패)

### 2.1 목표
Perplexity AI를 사용해 뉴스를 같은 이슈끼리 자동으로 그룹화하여 키워드 방식보다 정확한 이슈 탐지

### 2.2 구현 내용

파일:
- lib/ai/perplexity-grouping.ts: Perplexity AI 그루핑 API
- lib/candidate/issue-candidate.ts: 배치 처리 및 통합

기능:
- 100건씩 배치로 나눠서 처리
- JSON 파싱 및 에러 핸들링
- 폴백 메커니즘 (AI 실패 시 키워드 방식)

### 2.3 발견된 문제

1. 불완전한 응답 형식
   ```
   원본: [,[1],[20,21],,[10,12,13],,,,[28,50],,,[1]]
   문제: 빈 배열(,,,)이 많음
   ```

2. 그루핑 품질 낮음
   - 100건 입력 → 거의 100개 그룹으로 반환
   - 같은 인물/사건도 별개 그룹으로 분리
   - 프롬프트 개선해도 일관성 부족

3. 성능
   - 1000건 처리: 10개 배치 × 평균 1.2초 = 약 12초
   - 키워드 방식: 약 2초

### 2.4 결론

Perplexity는 그루핑 작업에 부적합

이유:
- 채팅 모델은 구조화된 출력에 최적화되지 않음
- sonar-pro는 검색 증강 생성에 특화 (분류가 아님)
- 비용 대비 효과 낮음

최종 설정:
```bash
ENABLE_AI_GROUPING=false
ENABLE_AI_GROUP_VALIDATION=false
```

## 3. 그루핑 오류 개선 방안

### 3.1 문제 사례

잘못 묶인 이슈:
```
A: '민주당 갈라치기 안 먹혔다' 조국당 지지율 최저치 경신
   (한국 정치 - 조국당 vs 민주당)

B: 미 민주당 반박 연설 "트럼프, 전례 없는 부패 대통령"
   (미국 정치 - 트럼프 vs 민주당)
```

원인: "민주당" 키워드만 겹쳐서 완전히 다른 주제가 같은 이슈로 묶임

### 3.2 현재 그루핑 로직

```typescript
// 1단계: groupItems - 키워드 기반 초기 그루핑
// 조건: (키워드 2개 이상 && 같은 카테고리) OR (키워드 1개 이상 && 핵심 인물 공통)

// 2단계: mergeRelatedGroups - Jaccard 유사도 기반 병합
// Jaccard 유사도 >= 0.4 OR 핵심 인물 공통 → 병합
```

### 3.3 해결책: 불용어 확장 (권장)

추가할 불용어:
```typescript
// 정당/정파 (국내외 공통)
'민주당', '공화당', '여당', '야당', '보수', '진보', '좌파', '우파',
'당', '의원', '지지율', '여론조사',

// 국가/지역 보강
'한국', '미국', '중국', '일본', '북한', '러시아', '유럽', '아시아',

// 범용 동사/명사
'발언', '언급', '주장', '입장', '반박', '비판', '반발',
```

적용 후 예상:
```
A: 갈라치기, 조국당, 최저치, 경신
B: 반박, 연설, 트럼프, 전례, 부패

→ 공통 키워드 0개 → 다른 그룹
```

예상 효과:
- 오묶임 비율: 15% → 3%
- 이슈 정확도: 70% → 90%

## 4. 향후 개선 방안

### 4.1 단기 (현재 키워드 방식 유지)
1. 핵심 인물 리스트를 DB로 관리 (관리자가 추가/삭제 가능)
2. 실제 데이터 기반으로 임계값 지속 조정
3. 그루핑 로그 분석으로 패턴 파악

### 4.2 장기 (AI 재도입 검토)

Option 1: OpenAI 임베딩
- 비용: 월 78원
- 안정성: 높음
- 정확도: 매우 높음 (semantic similarity)

Option 2: 로컬 임베딩 모델

- multilingual-e5-small
- 비용: 무료 (단, 제약 많음)
