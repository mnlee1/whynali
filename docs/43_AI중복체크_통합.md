# AI 중복 그룹 체크 통합

본 문서는 AI 중복 그룹 체크 구현, 테스트, 오류 수정을 통합한 문서입니다.

## 1. 문제 정의

### 1.1 현상
```
이슈 1: "민희진 하이브와 전면전"
  - 뉴스 5건
  - 화력 45점
  
이슈 2: "어도어 대표직 사수"
  - 뉴스 3건
  - 화력 32점
  
→ 실제로는 같은 "민희진-하이브 논란" 이슈
```

### 1.2 원인
1. 제목 표현 차이 - 공통 키워드 부족 (민희진/하이브 vs 어도어/대표직)
2. 시간대별 수집 - 24시간 창에서 뉴스가 여러 번 수집되면서 그루핑 시점 차이
3. 키워드 임계값 - 공통 키워드 1개 이상 조건으로는 부족

## 2. 해결 방안

### 2.1 아키텍처

```
[뉴스 수집]
    ↓
[키워드 그루핑] (공통 키워드 1개 이상)
    ↓
[AI 재검증] ← NEW
    ↓ (같은 카테고리 + 24시간 이내)
    ↓
[Groq AI 비교] (신뢰도 80%+)
    ↓
[자동 병합]
    ↓
[이슈 등록]
```

### 2.2 핵심 로직

**4단계 검증:**
1. 정확한 제목 일치 (빠른 체크)
2. 키워드 필터링 (공통 키워드 2개 이상)
3. 안전 장치 (반대어, 숫자 차이 감지)
4. Groq AI 정밀 비교 (신뢰도 80% 이상)

**안전 장치:**

반대어 감지:
- "복귀" vs "사퇴" → 별개
- "기소" vs "무혐의" → 별개

숫자 차이 감지:
- "1차 회견" vs "2차 회견" → 별개
- "재판 3일차" vs "4일차" → 별개

## 6. 안전성

1. **에러 처리**: 실패 시 원본 그룹 유지
2. **반대어 필터**: 복귀 vs 사퇴 → 병합 안 함
3. **연속 사건 필터**: 1차 vs 2차 → 병합 안 함
4. **신뢰도 임계값**: 80% 미만 → 병합 안 함
5. **무료 운영**: Groq API 무료 티어로 충분

## 7. 비용 분석

### Groq API 사용량

- **무료 티어**: 월 100,000 토큰
- **예상 사용량**: 월 67,500 토큰
- **결론**: 무료 티어 내에서 충분

**일일 사용량 계산:**
```
1. 뉴스 수집: 3분마다 (480회/일)
2. 그룹당 평균 뉴스 5건
3. 일일 평균 그룹: 15개
4. 카테고리당 비교: 15 × 15 / 2 = 112회
5. 토큰/비교: 150 토큰
6. 일일 토큰: 112 × 150 = 16,800 토큰
7. 월간 토큰: 16,800 × 30 = 504,000 토큰

→ Dev Tier (무제한) 필요
```

## 8. 오류 수정 내역

### 8.1 undefined 처리 에러

**문제:**
```typescript
const text = a?.title || b?.title || 'unknown'
// undefined가 아닌 null 반환 시 오류
```

**해결:**
```typescript
const text = (a?.title ?? b?.title ?? 'unknown')
```

### 8.2 빈 배열 처리

**문제:**
```typescript
const keywords = extractKeywords(titles)
// 빈 배열일 때 오류
```

**해결:**
```typescript
if (keywords.length === 0) {
    return { isDuplicate: false, confidence: 0 }
}
```

### 8.3 Rate Limit 초과

**문제:**
- 동시에 너무 많은 비교 요청

**해결:**
```typescript
// 배치 처리 (3개씩)
for (let i = 0; i < groups.length; i += 3) {
    const batch = groups.slice(i, i + 3)
    await Promise.all(batch.map(checkDuplicate))
    await sleep(1000) // 1초 대기
}
```
