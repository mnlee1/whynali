1. 서비스의 배경
    1. 한국 사용자들은 연예·스포츠·정치·사회·기술 등 거의 모든 분야의 사건/이슈에 관심이 높고, 발생 직후 빠르게 정보를 탐색하며 반응(댓글·공감 등)을 남기는 문화가 강함
    2. 하지만 이슈 정보는 뉴스/커뮤니티/SNS로 분산돼 있어, “지금 무슨 일이 벌어졌고 왜 커졌는지”를 한 번에 파악하기 어려움
    3. 이슈의 진행 과정(발단→전개→파생→진정)을 시계열로 정리해주는 서비스에 대한 잠재 수요가 존재함
2. 서비스의 목적
    1. 국내에서 발생하는 다양한 분야의 이슈를 한 곳에 모아, 사용자가 빠르게 맥락과 핵심을 이해하도록 돕는 것이 목적
    2. 이슈의 흐름과 여론 변화를 시간축으로 정리해 “현재 상황”과 “왜 이렇게 되었는지”를 동시에 제공하는 것을 목표
    로 함
    3. 정보 탐색 비용을 줄이고, 사용자가 손쉽게 반응/피드백(댓글, 리액션 등)을 남기도록 만들어 재방문을 유도3.
3. 서비스의 방향
    1. 단순 기사 모음이 아니라, 이벤트 단위로 묶어 타임라인과 이슈 포인트를 제공하는 ‘이슈 내비게이션’형 서비스로 설계
    2. 해당 서비스에만 접속하면 한국에서 현재-과거에 발생한 다양한 카테고리(연예, 스포츠, 정치, 사회 등)의 이슈를 모두 확인 및 파악 가능
    3. 하나의 이슈에 대해서도 변화 되는 여론이나 각 사건의 대상자들의 입장등으로 인한 상황 변화를 타임 라인별로 쉽게 파악 가능
    4. 이슈의 주목도가 떨어지면 종료(Closed) 처리하고, 종료된 이슈도 과거 히스토리로 검색/열람 가능하게 운영
    5. 접근성을 위해 웹/앱 동시 제공을 염두에 두고, 로그인은 필요 기능(댓글 등)에만 선택적으로 적용하는 방향이 적합
4. 주요 기능 및 유저 어필 포인트
    1. 1시간 단위(또는 더 짧은 주기)로 이슈를 모니터링해 신규 사건을 탐지하고, 관련 기사/콘텐츠를 자동으로 수집·정리해 제공
    2. AI를 통해 이슈의 내용을 요약, 정리, 구성 등을 진행
    3. 이슈별로 타이틀, 상세 설명, 핵심 쟁점, 파생 사건을 타임라인 형태로 제공해 한눈에 이해 할 수 있게 만듦.
    4. 이슈 관련 기사/콘텐츠의 출처 링크를 모아 보여주어 신뢰성과 탐색 편의성을 동시에 확보
    5. 좋아요/싫어요, 댓글 등 간단한 반응 수단을 제공해 참여 장벽을 낮추고, 여론 변화(반응 추이)를 확인하는 재미를 제공
    6. 커뮤니티 기능
        1. 특정 이슈에 대해 토론 할 수 있는 장을 마련 (철학적 관점에서 접근이면 좋을듯 ^^)
        2. 댓글로 자기 의견들을 작성
        3. 관리자가 토론 주제를 선정해서 글을 올리면 유저들이 참여 가능
        4. 다음의 커뮤니티 기능과 유사하지만 투표만 있는 것이 아니고 글을 쓸 수 있음
        5. 유저 참여 한 댓글, 커뮤니티 글에 세이프티봇 기능 필요 검토
            1. 봇이 내용을 자동으로 검토해서 삭제 가능
5. 이슈 수집 방법
    1. 실시간 뉴스 조회
        1. 네이버 뉴스 API 활용
        2. 네이버 뉴스 카테고리를 활용하여 이슈 카테고리와 일치 가능할지 검토 필요
            1. 적절하지 않다면 이슈에 대해 카테고라이징 필요
    2. 실시간 커뮤니티 사이트 게시글 수집
        1. 더쿠, 펨코, 엠팍, 클리앙, 루리웹, 뽐뿌, 보배드림, 인스티즈, 네이트판 등
        2. 뉴스는 1시간 단위도 가능하지만 커뮤니티 사이트의 경우 글의 리젠 속도가 워낙 빨라서 1~5분 단위 수집 필요
6. 로그인/회원가입
    1. 모든 로그인 은 오픈ID로 처리
        1. 구글, 네이버, 카카오, (메타는 필요할지 고민 됨)
7. 유저 참여 방법 (로그인 필요)
    1. 감정 표현
        1. 좋아요, 싫어요, 화나요, 팝콘각/흥미진진, 애도/응원, 사이다/통쾌 등
    2. 댓글
        1. 댓글의 좋아요, 싫어요 도 필요함
        2. 댓글의 순서도 유튜브 처럼 반응이 좋은 댓글을 상위에 배치함
    3. 투표 기능
        1. 논란이 되는 이슈 (ex. 누구의 잘못인가?) 에 대해 실시간 투표 기능을 제공하여 참여를 유도
        2. 투표 결과는 즉시 그래프로 보여주어 ‘여론’을 확인 가능
        3. 한 이슈에 하나의 투표가 아닌 시점/상황에 따라 여론이 변할 수 있어서 복수개의 투표 가능
8. 타임라인 시각화 (사건의 기승전결)
    1. 사용자가 궁금해 하는 것은 “그래서 누가 잘못했는데?” 또는 “이게 어떻게 시작된거야?”
    2. 단순히 최신순 나열이 아닌 사건의 LifeCycle 을 시각화 해야 함
    3. 특정 이슈 클릭 시 위>아래 or 좌>우 시간 순으로 발단>전개>절정(현재)>결말 구조의 타임라인을 보여 줘야 함
    4. 각 타임라인의 포인트 마다 기사 링크만 주는 것이 아니라 AI가 요약한 3줄 핵심을 카드 형태로 보여줘야 함
        1. ex) 2025.12.28 10:00 사건 발생 > 14:00 사과문 발표 > 2026.01.01 10:00 반대측 반박
    5. 상태 표시기
        1. 이슈의 현재 상태를 직관적으로 표시함 (아래는 예시)
            1. 🔥점화 (Ignited):막 시작되어 확산 중
            2. ⚡논란 중 (Debating):양측 의견 대립 중
            3. 🏁종결 (Closed):사과문 발표 또는 결론 도출
    6. 정보의 경량화
        1. 3줄 요약
        2. 이슈 상세 페이지의 최 상단에는 AI 가 요약한 3줄요약을 먼저 표시 하고 상세 내용이 하단에 들어가야 함
9. 개인화 모델
    1. 유저가 조회(클릭) 및 참여한 이슈를 분석하여 메인 화면에 추천 이슈를 먼저 보여줌 (별도 탭이나 영역으로 구성할지 아닐지는 고민이 필요)
    2. 알림 전략
        1. 유저가 관심있어 하는 이슈에 특정 이벤트 발생 시 알람을 보냄
            1. ex) 조진웅 사건에 조진웅씨가 배우 은퇴를 선언했습니다
10. 숏폼 자동 생성 및 릴리스
    1. 인스타 릴스, 유튜브 쇼츠, 틱톡 영상 최소 3곳에 자동으로 등록함
    2. 등록한 숏폼을 통해 본 서비스로 유입 시킴
    3. 3개 플랫폼에 본 서비스 전용 채널을 생성
    4. 하루에 수십개의 숏폼을 AI 툴을 활용해 자동 생성 될 예정
    5. 이슈 생성 시 & 이슈의 특정 전환 시점 등이 발생 시 자동 생성
11. 관리자 기능에 대해
    1. 특별히 관리자 기능에 많은 리소스를 투입 할 필요는 없어 보임
    2. 다만 이슈가 될 만한 포인트에 사람이 개입 하는 정도가 좋을 듯
    3. 실시간으로 수집 되는 이슈를 등록 할지 말지 결정은 사람이 필요 하다고 판단됨
    4. AI를 활용한 가공 된 콘텐츠의 릴리스 직전 목록을 관리자에게 알람을 통해 알려주고 승인 버튼 클릭으로 릴리스 되게 하는 형태 정도면 좋을 듯
