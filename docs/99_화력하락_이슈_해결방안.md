# 화력 하락 이슈 문제 해결 방안

> 발견일: 2026-03-11
> 문제: 승인된 이슈의 화력이 15점 미만으로 하락

## 문제 요약

### 현상
- 승인된 이슈 4건이 화력 15점 미만 (최저 0점)
- 모두 등록 시에는 26-30점으로 자동 승인 기준 충족
- 이후 화력이 급격히 하락 (-18점 ~ -30점)

### 원인
1. **수집 데이터 정리 크론잡**
   - `cleanup-unlinked` 크론잡이 오래된 데이터 삭제
   - 이슈와 연결된 뉴스/커뮤니티 데이터 제거
   
2. **화력 재계산**
   - 연결된 데이터가 없으면 화력 0점
   - `recalculate-heat` 크론잡이 주기적으로 재계산
   
3. **승인 취소 없음**
   - 한번 승인된 이슈는 화력 하락해도 승인 상태 유지

## 해결 방안

### 방안 1: 승인 시 화력 스냅샷 보존 (권장)

**개념:**
- 승인 시점의 화력을 `approval_heat_index`에 저장 (이미 구현됨)
- 화력 표시 시 승인 시 화력도 함께 표시
- 저화력 이슈로 표시되지 않게 함

**장점:**
- 이미 구현된 필드 활용
- 승인 시점 화력 기록 유지
- 추가 개발 최소화

**구현:**
```typescript
// 화력 표시 로직 수정
const displayHeat = issue.approval_status === '승인' && issue.approval_heat_index
    ? Math.max(issue.heat_index, issue.approval_heat_index)
    : issue.heat_index
```

### 방안 2: 승인된 이슈 데이터 보호

**개념:**
- 승인된 이슈와 연결된 데이터는 삭제하지 않음
- `cleanup-unlinked` 크론잡에서 필터링

**장점:**
- 화력 유지 가능
- 원본 데이터 보존

**단점:**
- 데이터베이스 용량 증가
- 불필요한 데이터 축적

**구현:**
```sql
-- cleanup-unlinked 수정
DELETE FROM news_data 
WHERE issue_id IS NULL 
   OR issue_id IN (
       SELECT id FROM issues 
       WHERE approval_status != '승인'
   )
```

### 방안 3: 종결된 이슈 화력 동결

**개념:**
- 종결 상태로 전환 시 화력 고정
- 더 이상 재계산하지 않음

**장점:**
- 종결 시점 화력 보존
- 자연스러운 처리

**구현:**
```typescript
// recalculate-heat 크론잡 수정
if (issue.status === '종결') {
    continue // 화력 재계산 건너뜀
}
```

### 방안 4: 저화력 이슈 자동 반려

**개념:**
- 화력 10점 미만으로 떨어지면 자동 반려
- 단, 종결 상태는 유지

**장점:**
- 자동 정리
- 품질 유지

**단점:**
- 이미 승인된 이슈를 취소하는 것은 부적절
- 사용자에게 노출되었던 이슈가 사라짐

## 권장 조치 (단계별)

### 즉시 실행 (수동 처리)

```sql
-- 승인된 저화력 이슈를 종결 상태로 전환
UPDATE issues
SET status = '종결'
WHERE approval_status = '승인'
AND heat_index < 15
AND status != '종결';
```

**대상 이슈:**
- 무신사 스포츠 팬 스토어 (0점)
- 바디프랜드 헬스케어 (1점)
- AI 딥페이크 탐지 (1점)
- 걸그룹 프로듀서 마약 (8점)
