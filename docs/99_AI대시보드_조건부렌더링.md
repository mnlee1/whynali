# AI 대시보드 조건부 렌더링

## 개요

Perplexity AI를 활성화했을 때 자동으로 대시보드에 표시되도록 조건부 렌더링을 구현했습니다.

## 동작 방식

### 1. 백엔드 로직

`lib/api-usage-tracker.ts`의 `getAllApiCostsSummary()` 함수:

```typescript
// Perplexity 데이터가 있으면 상세 정보 포함, 없으면 null
perplexity: perplexityCallsMonthly > 0 ? {
    today: perplexityCostToday,
    monthly: perplexityCostMonthly,
    calls: { ... },
    tokens: { ... },
    successes: perplexitySuccesses,
    failures: perplexityFailures,
} : null
```

**조건:**
- `perplexityCallsMonthly > 0`: 이번 달에 호출이 1회 이상 있으면 데이터 반환
- 호출이 없으면 `null` 반환

### 2. 프론트엔드 렌더링

`app/admin/page.tsx`:

```tsx
{/* Perplexity AI (사용 시에만 표시) */}
{apiCosts.perplexity && (
    <div className="p-4 bg-purple-50 ...">
        {/* Perplexity 상세 정보 */}
    </div>
)}
```

**조건:**
- `apiCosts.perplexity`가 `null`이 아니면 렌더링
- `null`이면 완전히 숨김 (DOM에 존재하지 않음)

### 3. 총 비용 표시

```tsx
<p className="text-xs text-gray-500 mt-1">
    네이버 API + Groq AI{apiCosts.perplexity ? ' + Perplexity AI' : ''}
</p>
```

**조건:**
- Perplexity 사용 중: "네이버 API + Groq AI + Perplexity AI"
- Perplexity 미사용: "네이버 API + Groq AI"

## 활성화 방법

### 1. 환경변수 설정

`.env.local`:
```bash
ENABLE_AI_GROUPING=true
ENABLE_AI_GROUP_VALIDATION=true
```

### 2. 자동 감지

- Perplexity API를 호출하면 자동으로 DB에 기록됨
- 다음 날부터 대시보드에 Perplexity 섹션이 표시됨
- 별도 재배포 불필요

### 3. 비활성화

환경변수를 `false`로 설정하면:
- 새로운 호출은 중단됨
- 하지만 **이미 기록된 데이터는 남아있음**
- 대시보드에는 계속 표시됨 (월 호출 횟수 > 0)

**완전히 숨기려면:**
- 다음 달까지 대기 (자동으로 0회 → null)
- 또는 DB에서 직접 삭제

## 장점

1. **자동 감지**: 설정만 바꾸면 자동으로 표시/숨김
2. **불필요한 정보 없음**: 사용하지 않는 API는 화면에서 제거
3. **확장 가능**: 새로운 AI 추가 시 동일한 패턴 사용
4. **정확한 비용**: 실제 사용량 기반 표시

## 주의사항

- Perplexity 섹션은 **이번 달 호출이 1회 이상**일 때만 표시
- 환경변수를 끄더라도 이미 기록된 데이터는 남음
- 완전히 숨기려면 월이 바뀌길 기다리거나 DB에서 직접 삭제 필요
