2. **중복 제거**: 같은 글이 랭킹에 오래 머물러 신규 건수 적음
3. **정상 동작**: 네이트판의 특성상 예상되는 수준

## 개선 방안 (선택)

### Option 1: 전체 게시판 추가

**추가 가능한 게시판:**
- `/talk` - 전체 토크
- `/entertainment` - 연예
- `/sports` - 스포츠

**장점:**
- 수집량 3-5배 증가
- 더 다양한 주제 수집

**단점:**
- 서버 부하 증가
- 저품질 글 포함 가능성

**구현 난이도:** 쉬움 (1시간)

### Option 2: 현재 상태 유지 (권장)

**이유:**
- 랭킹 = 인기글 = 이슈화 가능성 높음
- 품질 높은 데이터만 수집
- 서버 부하 최소화

**현재 수집량:**
- 하루 112건 × 7일 = 784건/주
- 충분한 데이터량

### Option 3: 수집 주기 조정

**현재:** 3분마다
**변경:** 5-10분마다

**효과:**
- 서버 부하 감소
- 수집량 큰 변화 없음 (중복 제거로 인해)

## 권장 사항

### 현재 상태 유지 (권장)

**이유:**
1. 네이트판 랭킹 = 인기글 = 품질 높음
2. 이슈화 가능성 높은 글만 수집
3. 서버 부하 최소화
4. 하루 112건 = 충분한 데이터량

### 모니터링 지표

```sql
-- 주간 수집량 확인
SELECT 
    source_site,
    COUNT(*) as count,
    MIN(created_at) as oldest,
    MAX(created_at) as newest
FROM community_data
WHERE created_at > NOW() - INTERVAL '7 days'
GROUP BY source_site;

-- 평균 반응 확인
SELECT 
    source_site,
    AVG(view_count) as avg_views,
    AVG(comment_count) as avg_comments
FROM community_data
WHERE created_at > NOW() - INTERVAL '7 days'
GROUP BY source_site;
```
