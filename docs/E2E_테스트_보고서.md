# 왜난리 프로젝트 E2E 테스트 보고서

**테스트 일시**: 2025-03-05  
**대상 URL**: http://localhost:3000  
**테스트 환경**: Playwright E2E + 수동 체크리스트

## 1. 테스트 환경 구성

### 1.1 생성된 파일

| 파일 | 설명 |
|------|------|
| `playwright.config.ts` | Playwright 설정 (baseURL: localhost:3000) |
| `e2e/01-homepage.spec.ts` | 홈페이지 & 이슈 목록 테스트 |
| `e2e/02-category.spec.ts` | 카테고리 필터링 테스트 |
| `e2e/03-issue-detail.spec.ts` | 이슈 상세 페이지 테스트 |
| `e2e/04-vote.spec.ts` | 투표 기능 테스트 |
| `e2e/05-comments.spec.ts` | 댓글 기능 테스트 |
| `e2e/06-community.spec.ts` | 커뮤니티 테스트 |
| `e2e/07-search.spec.ts` | 검색 기능 테스트 |
| `e2e/08-navigation.spec.ts` | 네비게이션 테스트 |

### 1.2 테스트 실행 방법

```bash
# 1. Playwright 브라우저 설치 (최초 1회)
cd whynali && npx playwright install chromium

# 2. 개발 서버 실행 (다른 터미널)
npm run dev

# 3. E2E 테스트 실행
npx playwright test

# 4. HTML 리포트 확인
npx playwright show-report
```

## 3. API 기반 검증 (curl)

서버가 실행 중일 때 아래 명령으로 기본 동작을 확인할 수 있습니다.

```bash
# 홈페이지 HTML 로드
curl -s -o /dev/null -w "%{http_code}" http://localhost:3000
# 기대값: 200

# 이슈 API
curl -s "http://localhost:3000/api/issues?limit=5" | head -c 500

# 검색 페이지
curl -s -o /dev/null -w "%{http_code}" "http://localhost:3000/search?q=이슈"
# 기대값: 200

# 커뮤니티 페이지
curl -s -o /dev/null -w "%{http_code}" http://localhost:3000/community
# 기대값: 200
```

## 4. 알려진 이슈 및 권장사항

### 4.1 Playwright 실행 환경

- **현상**: Cursor 샌드박스에서 `chrome-headless-shell-mac-arm64` 실행 파일을 찾지 못함
- **해결**: 터미널에서 직접 `npx playwright install` 후 `npx playwright test` 실행

### 4.2 화력 분석 표시

- 이슈 상세 페이지에는 **화력 지수(숫자)**가 직접 노출되지 않음
- **상태 배지**(점화/논란중/종결)가 화력 기반 상태를 나타냄
- 관리자 페이지(`/admin/issues`)에서만 화력 수치 확인 가능

### 4.3 투표·감정·댓글 로그인

- **투표**: 로그인 필요 (`userId` 없으면 투표 불가)
- **감정 표현**: 로그인 필요
- **댓글 작성**: 로그인 필요 (조회는 비로그인 가능)
