# 트랙 A 테스트 빠른 시작

새로운 트랙 A 아키텍처를 테스트하는 가장 빠른 방법입니다.

## 1단계: 데이터 확인

현재 데이터 상태를 확인합니다.

```bash
npx tsx scripts/test-track-a-check-data.ts
```

## 2단계: 이슈 리셋 (선택)

깨끗한 상태에서 테스트하고 싶다면 모든 이슈를 삭제합니다.

```bash
CONFIRM_RESET=yes npx tsx scripts/test-track-a-reset.ts
```

⚠️ **주의:** 모든 이슈가 삭제됩니다!

## 3단계: 트랙 A 실행

로컬 서버가 실행 중이어야 합니다.

```bash
# 터미널 1: 서버 실행
npm run dev

# 터미널 2: 트랙 A 테스트
npx tsx scripts/test-track-a-manual.ts
```

## 4단계: 결과 확인

Supabase 대시보드에서 확인:
- issues 테이블에서 `source_track = 'track_a'` 이슈 확인
- `approval_status = '대기'` 상태 확인
- 연결된 뉴스/커뮤니티 데이터 확인

또는 관리자 페이지:
```
http://localhost:3000/admin/issues
```

## 문제 해결

### "최근 커뮤니티 데이터가 없습니다"

```bash
# .env.local에서 시간 창 늘리기
COMMUNITY_BURST_WINDOW_MINUTES=60
```

### "급증 키워드가 없습니다"

```bash
# .env.local에서 임계값 낮추기
COMMUNITY_BURST_THRESHOLD=5
```

### "인증 실패"

```bash
# .env.local에서 CRON_SECRET 확인
CRON_SECRET=your-secret-here
```
