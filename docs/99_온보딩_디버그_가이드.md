### 마이그레이션 확인 쿼리

```sql
-- 1. terms_agreed_at 컬럼 존재 확인
SELECT column_name, data_type, is_nullable 
FROM information_schema.columns 
WHERE table_name = 'users' 
AND column_name IN ('terms_agreed_at', 'marketing_agreed');

-- 2. 트리거 함수 존재 확인
SELECT routine_name 
FROM information_schema.routines 
WHERE routine_name = 'handle_new_user';

-- 3. 트리거 존재 확인
SELECT trigger_name 
FROM information_schema.triggers 
WHERE trigger_name = 'on_auth_user_created';
```

## 2. 기존 유저 데이터 확인

### 현재 로그인 유저 상태 확인

로그인 후 브라우저에서 접속:
```
http://localhost:3000/api/debug/check-user
```

**결과 예시:**
```json
{
  "authenticated": true,
  "authUser": {
    "id": "uuid",
    "email": "test@gmail.com",
    "provider": "google"
  },
  "dbUser": {
    "id": "uuid",
    "provider": "구글",
    "display_name": null,
    "terms_agreed_at": null,    // ← NULL이어야 온보딩 필요
    "marketing_agreed": false,
    "created_at": "2026-03-16T..."
  },
  "needsOnboarding": true
}
```

### 문제 케이스

**케이스 1: terms_agreed_at에 값이 있음**
```json
"terms_agreed_at": "2026-03-16T10:00:00Z"  // ← 이미 온보딩 완료
```
→ 기존 유저이므로 온보딩 스킵 (정상)

**케이스 2: dbUser가 null**
```json
"dbUser": null,
"dbError": "No rows found"
```
→ users 테이블에 레코드 없음 → 트리거가 작동하지 않음

**케이스 3: terms_agreed_at 컬럼이 없음**
```json
"dbError": "column \"terms_agreed_at\" does not exist"
```
→ 마이그레이션 미실행

## 4. 강제 온보딩 테스트

### 기존 유저를 신규 유저로 변경

```sql
-- 방법 1: auth.users에서 이메일로 ID 찾기
SELECT id FROM auth.users WHERE email = 'xxmemexx@naver.com';

-- 위에서 얻은 ID로 public.users 업데이트
UPDATE users 
SET terms_agreed_at = NULL, display_name = NULL 
WHERE id = '여기에-위에서-얻은-UUID-붙여넣기';

-- 방법 2: 한번에 실행 (서브쿼리)
UPDATE users 
SET terms_agreed_at = NULL, display_name = NULL 
WHERE id = (SELECT id FROM auth.users WHERE email = 'xxmemexx@naver.com');
```

이후 재로그인하면 온보딩 페이지로 이동해야 함

## 7. 빠른 확인 SQL

```sql
-- 모든 유저의 온보딩 상태 확인 (이메일 포함)
SELECT 
    u.id,
    au.email,
    u.provider,
    u.display_name,
    u.terms_agreed_at,
    CASE 
        WHEN u.terms_agreed_at IS NULL THEN '온보딩 필요'
        ELSE '온보딩 완료'
    END as status,
    u.created_at
FROM users u
LEFT JOIN auth.users au ON u.id = au.id
ORDER BY u.created_at DESC
LIMIT 10;
```
