# 관리자 대시보드 24시간 통계

관리자가 시스템의 수집 → 이슈 생성 → 연결 프로세스를 24시간 단위로 모니터링할 수 있는 기능입니다.

## 주요 기능

### 1. 3단계 프로세스 모니터링

**1단계: 데이터 수집**
- 뉴스 수집 건수 (출처별 상세)
- 커뮤니티 수집 건수 (사이트별 상세)
- 총 수집 건수

**2단계: 이슈 생성**
- 총 생성 건수
- 승인 대기 / 승인 / 반려 / 병합 건수
- 승인률 계산

**3단계: 이슈 연결**
- 뉴스 연결 건수 및 연결률
- 커뮤니티 연결 건수 및 연결률
- 미연결 건수

### 2. 자동 경고 시스템

시스템이 자동으로 문제를 감지하고 경고를 표시합니다:

**치명 (Critical) 경고:**
- 뉴스 수집이 24시간 동안 10건 미만
- 수집 데이터는 많지만 이슈가 생성되지 않음

**일반 (Warning) 경고:**
- 커뮤니티 수집이 24시간 동안 50건 미만
- 승인 대기 이슈가 10건 이상 쌓임

### 3. 시각적 표시

각 단계별로 색상으로 구분:
- 1단계 수집: 파란색
- 2단계 이슈 생성: 보라색
- 3단계 연결: 초록색

경고는 배경색으로 구분:
- 치명 경고: 빨간색 배경
- 일반 경고: 노란색 배경

## 사용 방법

### 관리자 대시보드에서 확인

1. `/admin` 페이지 접속
2. "최근 24시간 요약" 섹션 확인
3. 경고가 있으면 빨간색/노란색 배너로 표시됨
4. 각 단계별 상세 수치 확인

### 문제 발생 시 대응

**경고: 뉴스 수집이 24시간 동안 10건 미만**
→ GitHub Actions 크론잡 확인 (`cron-collect-news.yml`)

**경고: 이슈가 생성되지 않음**
→ `/api/cron/auto-create-issue` 크론잡 확인

**경고: 승인 대기 이슈가 10건 이상**
→ `/admin/issues` 페이지에서 이슈 승인 처리

**경고: 연결률이 낮음**
→ `/api/cron/auto-link` 크론잡 확인

## 주의사항

- 24시간 데이터이므로 새벽 시간대에는 수치가 낮을 수 있음
- 경고 임계값은 상황에 따라 조정 가능 (API 코드 수정)
- 실시간 데이터가 아니므로 "새로고침" 버튼으로 최신 데이터 갱신 필요
