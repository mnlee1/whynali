# 법적검토 통합

본 문서는 커뮤니티 수집 및 서비스 노출에 대한 법적 검토를 통합한 문서입니다.

## 1. 커뮤니티 수집 법적 검토

### 1.1 최종 결론

**수집 대상**
- 더쿠 (theqoo.net)
- 네이트판 (pann.nate.com)

**수집 범위**
- 제목, URL, 조회수, 댓글수, 작성시간 (메타데이터만)
- 본문, 닉네임, 기타 창작 콘텐츠 (수집 안 함)

**법적 판단**
- 약관 위반 아님: 수집 대상 2곳은 약관 전문 검토 후 명시적 수집 금지 조항 없음
- 저작권 침해 아님: 메타데이터는 사실 정보 (저작권 보호 대상 아님)
- DB권 침해 아님: 판례상 공개된 사실 정보 수집 합법
- 제목 표시: 링크와 함께 표시하여 공정 이용 범위 내

### 1.2 법적 이슈 있는 사이트 (수집 제외)

| 사이트 | 이슈 사항 | 비고 |
|--------|----------|------|
| 클리앙 | 약관 제13조: 무단 수집·복제·배포 금지 | 명시적 금지 |
| 인스티즈 | 이용규칙: 구성 요소 추출 금지 | 명시적 금지 |
| 보배드림 | 약관 제11조 ③: 사전 승낙 없이 복제·제공 금지 | 명시적 금지 |
| 펨코 | robots.txt: Disallow: /* (전체 차단) | 기술적 차단 |
| 엠팍 | robots.txt: Disallow: /* (전체 차단) | 기술적 차단 |
| 루리웹 | 약관: 사전 승낙 없이 복제·제3자 제공 금지 / robots.txt 미확인 | 그레이존 (보류) |
| 뽐뿌 | 약관: 무단 복제·제3자 제공 금지 / robots.txt 게시판 경로 허용 | 그레이존 (보류) |

### 1.3 메타데이터 수집 법적 안전성 근거

| 항목 | 성격 | 저작권 보호 | 법적 근거 |
|-----|------|------------|----------|
| 제목 | 창작성 있으면 저작물 가능 | 그레이존 | 링크와 함께 표시 시 합법 (대법원 2012다76829) |
| 조회수 | 사실 정보 | 대상 아님 | 저작권법 제2조 |
| 댓글수 | 사실 정보 | 대상 아님 | 저작권법 제2조 |
| 작성시간 | 사실 정보 | 대상 아님 | 저작권법 제2조 |
| URL | 사실 정보 | 대상 아님 | 저작권법 제2조 |

**판례 근거**
- 대법원 2009다6735: 단순한 사실적 정보는 저작권법의 보호 대상이 아니다
- 대법원 2008da55354 (잡코리아 vs 사람인): 공개된 단순 사실 정보 수집은 데이터베이스권 침해가 아니다
- 대법원 2012다76829 (네이버 뉴스 제목 사건): 뉴스 제목을 링크와 함께 표시하는 것은 공정한 이용으로 인정
- 대법원 2016da271580: 이미 공개된 정보를 수집하여 통계 목적으로 사용하는 것은 합법

## 2. 서비스 노출 법적 검토

### 2.1 기본 원칙

**허용되는 것:**
- 이슈 주제명
- 출처 링크 (원문 URL)
- 제목 (링크와 함께 표시)
- 메타데이터 기반 분석 (화력 지수, 급상승 등)
- 사용자 생성 콘텐츠 (댓글, 투표)

**금지되는 것:**
- 게시글/기사 본문
- 본문 기반 AI 요약문
- 본문 재가공 텍스트

### 2.2 AI 요약문 법적 검토

**본문 기반 AI 요약**
- 위법 가능성 큼
- 침해되는 권리: 복제권 (본문 수집·저장), 2차적 저작물 작성권 (AI 요약 생성), 배포권·공중송신권 (요약문 서비스 노출)
- 판례: 네이버 뉴스스탠드 사건 (2019가합573789) "뉴스 요약문도 저작물, 무단 사용 시 침해"

**메타데이터 기반 AI 설명**
- 상대적으로 안전
- 근거: 조회수·댓글수·링크는 사실 정보 (저작권 보호 대상 아님)
- AI 출력이 사실 나열이면 우리 쪽 창작물
- 본문 이용 없으므로 복제권·2차적 저작물 침해 없음

### 2.3 커뮤니티 기능(UGC) 법적 검토

**UGC 범위:**
- 이슈 상세 댓글: 사용자 댓글, 저장 시 세이프티봇 적용
- 커뮤니티(토론) 댓글: 토론 주제 상세 화면에서만 작성 가능, 저장 시 세이프티봇 적용

**정보통신망법상 의무:**
- 세이프티봇: 댓글 저장 시 금칙어 목록 매칭, 매칭 시 자동 비공개·검토 대기
- 신고 절차 필수: 권리자·이용자 신고 접수 방법·처리 기한·삭제 기준 수립
- 삭제 기준: 욕설·비방·혐오·저작권 침해·불법정보·권리자 신고에 의한 삭제 등 문서화

**저작권:**
- 이슈 댓글·토론 댓글: 이용자 글이 타인 저작물 무단 복제한 경우 작성자 1차 책임
- 플랫폼: 권리자 신고 접수 후 해당 글 삭제·접근차단 절차를 두면 손해배상 책임 감면
- 토론 주제(AI 생성): 이슈 제목·상태·카테고리·화력 등 메타데이터만 AI 입력, 본문·본문 요약 미사용

### 2.4 체크리스트

**수집 단계**
- 더쿠·네이트판만 수집 (클리앙 등 금지 사이트 제외)
- 메타데이터만 수집 (제목, URL, 조회수, 댓글수, 작성시간)
- 본문 수집 안 함
- 닉네임 수집 안 함
- 네이버 뉴스 API: 요약문(description) 미저장, 제목·링크·출처·날짜만 저장

**관리자 화면**
- 수집된 메타데이터만 표시
- 본문 표시 안 함
- 금지 사이트 데이터 표시 안 함

**서비스(사용자) 화면**
- 출처: 제목+링크만 표시
- AI요약: 메타데이터 기반 사실 서술만
- 핵심 정보: 본문 요약·추출 안 함
- 본문 텍스트 노출 안 함
- 네이버 API 데이터: 그대로 표시 (재가공 안 함)

**커뮤니티(UGC) 운영**
- 이슈 댓글·토론 댓글 모두 저장 시 세이프티봇 적용
- 토론 주제 AI 생성 시 이슈 메타데이터만 입력, 본문·본문 요약 미사용
- 권리자·이용자 신고 접수 창구·처리 절차·삭제 기준 오픈 전 수립
- 이용약관에 게시물 저작권·사용 허락·금지 행위·삭제 권한·권리자 신고 안내 반영

## 3. 최종 권고사항

### 3.1 법적으로 완전히 안전한 설계

```
수집: 제목, URL, 조회수, 댓글수, 작성시간만
    ↓
관리자 화면: 메타데이터 확인
    ↓
AI 처리: 수치 데이터만 입력 → 사실 서술 생성
    ↓
서비스 노출:
- 이슈명 + "관련 뉴스 N건, 커뮤니티 M건"
- 출처 링크 (제목+URL)
- 화력 지수 (메타데이터 기반)
- 투표·댓글 (사용자 생성)
```

### 3.2 피해야 할 것

- 본문 수집·저장·표시
- 본문 기반 AI 요약·재가공
- 네이버 API 요약문 재가공
- 금지 사이트(클리앙 등) 데이터 수집
- 출처 표시 없이 콘텐츠 노출
