# 투표 관리 페이지 오류 해결 가이드

## 문제

투표 관리 관리자 페이지에서 "투표 조회 실패" 오류가 발생합니다.

## 원인

`votes` 테이블에 `approval_status` 컬럼이 없어서 발생하는 오류입니다.
데이터베이스 마이그레이션이 적용되지 않았습니다.

## 해결 방법

### 1단계: Supabase 대시보드 접속

Supabase 프로젝트 대시보드의 SQL Editor로 이동합니다:
`https://supabase.com/dashboard/project/YOUR_PROJECT_ID/sql/new`

### 2단계: 마이그레이션 SQL 실행

다음 SQL을 SQL Editor에 붙여넣고 실행(Run)합니다:

```sql
-- 1. 필드 추가
ALTER TABLE votes ADD COLUMN IF NOT EXISTS approval_status TEXT DEFAULT '대기';

-- 2. 체크 제약 조건
ALTER TABLE votes DROP CONSTRAINT IF EXISTS votes_approval_status_check;
ALTER TABLE votes ADD CONSTRAINT votes_approval_status_check 
    CHECK (approval_status IN ('대기', '승인', '반려'));

-- 3. 기존 데이터 마이그레이션
UPDATE votes 
SET approval_status = CASE 
    WHEN phase = '대기' THEN '대기'
    ELSE '승인'
END
WHERE approval_status IS NULL OR approval_status = '대기';

-- 4. 인덱스 추가
CREATE INDEX IF NOT EXISTS idx_votes_approval_status ON votes(approval_status);

-- 5. 코멘트
COMMENT ON COLUMN votes.approval_status IS '관리자 검토 상태. 대기=검토 전, 승인=사용자 노출, 반려=거부됨(삭제 전)';
```

### 3단계: 확인

SQL 실행 후 투표 관리 페이지를 새로고침합니다.
"투표 조회 실패" 오류가 사라지고 투표 목록이 정상적으로 표시됩니다.

## 기술 세부사항

### approval_status 필드

- `대기`: 관리자 검토 전
- `승인`: 사용자에게 노출됨
- `반려`: 거부됨 (삭제되지 않고 보관)

### 기존 데이터 처리

- `phase='대기'`인 투표 → `approval_status='대기'`
- `phase='진행중'` 또는 `'마감'`인 투표 → `approval_status='승인'`

### 마이그레이션 파일 위치

`supabase/migrations/20260305134500_add_approval_status_to_votes.sql`

## 예방 조치

향후 새로운 마이그레이션 파일이 생성되면 자동으로 Supabase에 적용되도록 CI/CD 파이프라인을 설정하는 것을 권장합니다.
